PROFINET

Overview

This release adds PROFINET IO Device functionality to Soft dPAC, complementing its existing IO Controller support. With IO Device support, Soft dPAC can now operate as a subordinate device to other PROFINET controllers. This enables integration into higher-level control systems and facilitates data exchange between controllers without custom interfaces. The update includes an IO Device Configurator, GSDML file generation, runtime enhancements, and shared device support.

The implementation follows the latest PROFINET standards defined by PI (PROFIBUS & PROFINET International) for compatibility with major industrial controllers and engineering tools; however, this feature has not yet been officially certified by a PI test lab. Certification is planned for a future release.

This functionnality requires an Application Addons license . Refer  to the license chapter.

New Features

  • Soft dPAC can operate as a PROFINET IO Controller or as both an IO Controller and IO Device.

  • IO Device Configuration Editor: Configure network parameters, define input/output submodules, and generate/export GSDML files.

  • Cyclic and Acyclic IO Communication: Supports data exchange using Real-Time (RT) channels.

  • Shared Device: Multiple controllers can connect to the same Soft dPAC simultaneously to control different output submodules and access different input submodules.

  • Shared Input: One or more input submodules can be shared across multiple controllers, allowing each to read the same input data independently.

Known Limitations

To learn more about PROFINET IO limitations, refer to PROFINET Limitations section in the User Guide: User Manual.

Device Compatibility

PROFINET is currently compatible with EcoStruxure Automation Expert - Softdpac (Linux).

Capabilities

  • Maximum 5 connections with PROFINET IO Controllers.

  • Data item type limited to an array of bytes.

  • Maximum size of a single data item: 1024 bytes.

  • Total input data and status (IOXS): 1440 bytes.

  • Total output data and status (IOXS): 1440 bytes.
