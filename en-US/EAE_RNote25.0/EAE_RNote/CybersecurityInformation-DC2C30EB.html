General Cybersecurity Information

In recent years, the growing number of networked machines and production plants has seen a corresponding increase in the potential for cyber threats, such as unauthorized access, data breaches, and operational disruptions. You must, therefore, consider all possible cybersecurity measures to help protect assets and systems against such threats.

To help keep your Schneider Electric products secure and protected, it is in your best interest to implement the cybersecurity best practices as described in the Cybersecurity Best Practices document.

Compliance

Our system conforms to the California SB-327 Information Privacy: Connected Devices bill by requiring the user to generate a new means of authentication before granting initial access. This includes the Modicon M580 Distributed PAC, Altivar dPAC Module VW3A3530D, Modicon M251 Distributed PAC, and Modicon M262 Distributed PAC. Industry vulnerabilities are regularly identified and mitigated in order to minimize cybersecurity risks.

Improved Security Robustness

This release of EcoStruxure Automation Expert includes several cybersecurity fixes addressing vulnerabilities and improving security robustness.

Vulnerability Mitigation

There is an identified vulnerability on EcoStruxure Automation Expert - Buildtime on versions <25.0, with the CWE - CWE-356: Product UI does not Warn User of UnsafeAction.

NOTE: For detailed mitigation measures, refer to the EcoStruxure Automation Expert - User Manual > Handle EcoStruxure Automation Expert Solution.
