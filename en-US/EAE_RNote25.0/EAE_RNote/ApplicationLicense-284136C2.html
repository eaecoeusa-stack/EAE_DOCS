Application License

Overview

In addition to the Engineering license that is required to create Automation Expert applications, it will be required to add an Application License  for the operation and maintenance. 

Application License Types

Application licenses are based on IO Points and are available in two different types as,  

Expandable Application Licenses: valid for all type of projects and all type of controllers, designed typically for plant type of projects where:  

  • The number of IO’s may evolve in time with production line evolutions or others. 

  • Special capabilities are needed, like System Management, High-Availability, or High-Availability Disruption Avoidance for Advanced System Management (subscription-based only). 

Compact Application Licenses: for competitiveness in small projects for machine, discrete and hybrid process, where:  

  • Number of IO’s is not going to evolve with time. 

  • There is no need for special capabilities like system management beyond device health and security monitoring, high availability or high availability default avoidance Advanced System Management.

The Expandable and Compact application licenses are independent of the number of controllers used in the solution. 

Application licenses are required for each solution that is created with EcoStruxure Automation Expert. Contact your local Schneider Electric sales team if you need help to understand which Application licenses are needed. 

Starting from EcoStruxure Automation Expert V 24.1, Application licenses must also be activated to avoid a license non-compliance notice. These can be activated in the same way as Engineering and Add-on licenses. 

Application Addons

In addition to Application license, a solution may need some addons when a specific HMI or protocol are used by the application.  

The add-ons that are available with EcoStruxure Automation Expert: 

  • Automation Expert HMI Runtime - Panel (ST6) is required to deploy HMI and Archives runtimes on an Harmony ST6 HMI range.

  • Automation Expert HMI Runtime - Operator (iPC) is required to deploy HMI and Archives runtimes on a PC-Type HMI (Window 10/Linux).

  • Communication Protocol OPC UA Client is required to allow application to connect over OPC UA to multiple servers and exchange data with them, including read, write, and monitor data rights.

  • Communication Protocol Profinet RT IO-Controller is required to allow application to operate as a Profinet IO-Controller, allowing to exchange real time process data with multiple Profinet IO-Devices.  

  • Communication Protocol EtherCAT Main Device Client is required to allow the controller to function as an EtherCAT main device, managing and communicating with multiple EtherCAT sub devices, allowing to exchange data and precise synchronization with field devices. Reference Description.
