Multi-User

You can now collaborate with up to five colleagues simultaneously on the same project, using EcoStruxure Automation Expert Platform.

Two configurations are supported:

Configuration N°1:

  • One engineering workstation hosting at the same time EcoStruxure Automation Expert Platform and EcoStruxure Automation Expert.

  • Up to 4 engineering workstation hosting EcoStruxure Automation Expert.

Configuration N°2:

  • One server hosting EcoStruxure Automation Expert Platform.

  • Up to 5 engineering workstation hosting EcoStruxure Automation Expert.

This initial phase of the Multi-User Environment helps prevent merge conflicts, particularly when design outcomes must be shared with colleagues. To achieve this, a reservation mechanism is implemented based on a zone concept: when a user reserves a zone, it becomes read-only for others.

Zones are hierarchical:

  • A zone can contain children zones.

  • A zone can have dependencies.

In such case, When a user reserves a zone, all its dependent zones are automatically reserved as well.

Devices themselves function as zones. If a user is interacting with a device (logging in, deploying, monitoring, etc.), others cannot access it simultaneously. The reservation lock is managed by EcoStruxure Automation Expert Platform, requiring a live connection from each engineering workstation to EcoStruxure Automation Expert Platform.

New Features

UI Improvements

It is now possible to filter the zone of interest in the Reservation List view:

  • by zone identifier

  • by reservation owner

  • by zone status

Granularity of reservation has been improved in order to avoid a whole Distributed Control Application reservation : Named Style, library development.

Standalone Mode vs. Multi-User Environment

EcoStruxure Automation Expert currently supports two distinct modes of operation:

  • Standalone Mode: Launched via desktop shortcut

  • Multi-User Environment: Launched from the EcoStruxure Automation Expert Platform 

These two modes are mutually exclusive and not yet compatible with each other.

As a result, the following actions are temporarily not supported:

  • Opening or importing a project created in Standalone Mode into the Multi-User Environment

  • Using the Archive or Unarchive functions in the Multi-User Environment

  • Converting a solution created in Standalone Mode into a Distributed Control Application in the Multi-User Environment

  • Opening a Distributed Control Application created in the Multi-User Environment using Standalone Mode

Known Limitations

Multiple Sessions with the Same User

Logging in with the same user credentials from multiple engineering workstations simultaneously in the same EcoStruxure Automation Expert Solution is not recommended and may lead to data conflicts or unexpected behavior in a Multi-User environment. To ensure system integrity and optimal performance, each user should access the system from a single device at a time.
