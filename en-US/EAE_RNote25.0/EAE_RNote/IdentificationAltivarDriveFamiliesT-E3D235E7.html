Altivar Distributed PAC- Module VW3A3530D

NOTE: For more information about Altivar Distributed PAC-VW3A3530D, refer to the Altivar Distributed PAC Module VW3A3530D - User Guide.

New Features

Graphic Display Terminal

The automation engineer is now able to control the access to the ATV dPAC HMI displayed on the ATV Graphic Display Terminal, using a PIN code.

Several improvements are now available for the ATV dPAC Display:

  • Online change is now supported

  • Sub-CATs variables can be added to the screens

  • Up to 4 events can be supported for each variable displayed

Multi-Resource support – ATV dPAC

As for Soft dPAC and M262 dPAC you can now create multiple resources, up to three, and configure them with different execution priorities.

You can then isolate and prioritize bus masters and assign the time-critical parts of the application to higher prioritized resources. This can:

  • Reduce the frequency of occurrence and the magnitude of jitter.

  • Reduce bus cycle messages sent to EcoRT logs.

Altivar SoftStarter ATS490 integration in Ecostruxure Automation Expert

The Altivar Soft Starter ATS490 can be controlled by dPACs via the following protocols, using the hardware CATs from the SE.FieldDevice library.

  • Modbus TCP

  • Modbus Serial

  • EtherNet/IP

Controlling ATS430 Soft Starters via Modbus Serial

The Altivar Soft Starter ATS430 can be controlled by dPACs supporting the Modbus Serial protocol using the ATS4x0MBSerial hardware CATs from the SE.FieldDevice library.

Designed for industrial standard machines ranging from 4 to 400 kW, the ATS430 offers high durability, energy efficiency, and simplified integration. It delivers motor control and operation while helping reduce the overall cost of equipment ownership.

Net IO TLS Extensions Support

The NET IO function block has been updated to include an additional input parameter named SNI, which allows users to specify the target domain name. This enables the server to present the correct SSL/TLS certificate during the handshake process, establishing a secure connection.

It will facilitate seamless communication with commonly used servers, supporting IT/OT convergence and unlocking new integration possibilities.

It will also reinforce cybersecurity by adopting industry-standard encryption protocols (SSL/TLS) to enable secure data exchange.

The user experience will be simplified by enabling the use of a single IP address for multiple secure services, thereby reducing configuration complexity and improving manageability.

Known Limitations

NOTE: To learn more about Altivar Distributed PAC limitations introduced in a previous release version and still active today , please refer to the following link :Altivar Distributed PAC Module VW3A3530D - User Guide. For new limitations, please see the following sections.

Run Command During Drive Blocked State

If a stop command of type Transition ( 3–wires or 2–wires TRN), triggered by drive inputs (including, but not limited to digital inputs, STO inputs, STOP input or CMI stop), occurs while the drive is already in NST state, then any new run command occurring during this blocked phase will become active as soon as the stop command is removed.

Additional Delay in ComStatus Parameter of Hardware CATs

When there is a communication interruption, the ComStatus variable will change from True to False, but this change will occur after a delay. This delay is influenced by the value you set for the ComStatus Timeout parameter, plus an additional delay of 3 to 4 seconds.
