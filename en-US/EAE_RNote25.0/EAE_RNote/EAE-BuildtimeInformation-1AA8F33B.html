New Features

Open Adapter

Open Adapter is a flexible integration framework designed to simplify the connection and data exchange between diverse solutions. It enables seamless interoperability through straightforward configuration, significantly reducing the complexity of system integration.

Key Benefits:

  • Custom Adapter Design: Users can create their own adapters tailored to specific devices or systems. This promotes reusability and consistency, especially for frequently connected devices.

  • Proven-in-Use Reliability: Leveraging previously validated adapters for high reliability and minimizes integration errors.

  • Reduced Integration Risk and Cost: Simplified configuration and reusable components help lower the risk and cost associated with system integration.

  • Enhanced Engineering Efficiency: Streamlined integration processes lead to faster development cycles and improved productivity.

  • Scalability: Ideal for medium to large-scale projects, Open Adapter supports scalable architectures that grow with project needs.

Parallel System Views

This release introduces a significant enhancement for application and library developers, as well as commissioning engineers.

It is now possible to open and view multiple system editor windows in parallel, enabling seamless cross-verification of application (APPs), layers, and library logic without switching tabs or losing context.

Previously, opening a new APP or layer would overwrite the existing system tab, making comparisons cumbersome and time-consuming. With this update, editors can be opened side-by-side and even moved to extended monitors for improved visibility and workspace flexibility.

Key Benefits:

  • Improved Cross-Verification Workflow: Side-by-side viewing minimizes the need for constant switching or reloading.

  • Enhanced Debugging Experience: A simplified and productive environment accelerates troubleshooting and supports more accurate engineering decisions.

Licensing

Beside an Engineering license that is required to create Automation Expert applications for the operation and maintenance of the application also Application licenses are required.

With V25.0, new Application licenses are offered that are based on IO Points and are available in two different types:

  • Expandable Application Licenses:

    These licenses are valid for all type of projects and all type of controllers. They are designed typically for plant type of projects, where:

    • the number of IO’s may evolve in time with production line evolutions or others

    • special capabilities are needed, like System Management, High-Availability, or Advanced System Management (subscription-based only)

  • Compact Application Licenses:

    These licenses are designed for small projects for machine, discrete and hybrid process, where:

    • the number of IO’s is not going to evolve with time

    • there is no need for special capabilities like system management beyond device health and security monitoring, High-Availability or Advanced System Management

The Expandable and Compact Application licenses are independent of the type and number of controllers used in the solution. All previously offered Application licenses are compatible with V25.0.

Export Deploy Data

In the Deploy and Diagnostic action list of commands, the option Export Deploy Data is now available. This option provides a deployment Artifact of the selected device and is available in Advanced mode only.

IP Address Management in Deploy and Diagnostic

The IP address of the device are synced in the Deploy and Diagnostic. No need of compilation for the address to be updated. In case of address or port conflict , an info icon is highlighted in the Deploy and Diagnostic.

Copy/Paste and Import/Export for Library Objects

EcoStruxure Automation Expert environment now supports copy/paste and import/export operations for library objects. These capabilities help reduce manual effort when customizing and reusing application libraries across different projects.

Key Features

Simplified Library Customization

Users can quickly adapt SE Application Libraries to meet specific project or customer needs.

Support for Standardization

This feature helps maintain consistency in text, translations, and color usage with minimal configuration.

Bulk Creation and Update of Constant Variables

This feature allows users to create or update multiple constant variables at once, eliminating the need to perform these actions individually.

Benefits for the User:

  • Time-saving: Reduces repetitive tasks, streamlining the engineering workflow.

  • Error reduction: Minimizes the risk of manual errors by enabling bulk operations.

  • Improved usability: Provides a clean and efficient interface for a smoother user experience.

Step Timeout

The release will provide, on top of the existing Procedural Automation functionalities, Step and Transition time which that will help an operator to track and understand the time based transition and will also provide Application developers an option to make time based transitions.

In Procedural CAT, under a Step, the user has now the possibility to set the parameters of the Timeout : 

  • EnWaitTimer, StepWaitTime

If the Timeout is reached then the followed left most transition is executed.

Buildtime Performance Improvements

Editor Loading User interface hang issues have been resolved, and performance has been improved through dynamic loading for the following editors: Offline Parameterization, OPC UA Server, ASP Attribute Mapping, and Asset Mapping editors.
