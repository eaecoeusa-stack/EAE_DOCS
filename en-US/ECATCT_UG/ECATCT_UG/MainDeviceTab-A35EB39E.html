MainDevice Tab

The MainDevice tab is available in the Device Editor pad when selecting in the Project Editor a Main-device.

In this tab, you can configure the name of the Main-device and its cycle time. You can also configure there the Main-device to a control system.

  • General

    • Unit Name; Name of the Main-device device.

    • Cycle Time; Interval in microseconds in which all EtherCAT commands will be sent from the Main-device. You can choose between the following values: 125, 250, 500, 1000, 2000 and 4000.

    • Source MAC address; MAC address of the connected system.

      NOTE: This parameter is filled during bus scan.

  • Sub-Devices connected to local system

    • Link Layer; A link layer (also called data link layer) is the second layer in the OSI networking model and handles communication between directly connected devices on the same network segment.

    • Network Adapter; Network adapter which is connected to the control system.

  • Sub-Devices Connected to remote system

    • Protocol; A protocol is a set of rules and standards that define how devices or systems communicate and exchange data with each other (HTTP/HTTPS, TCP, PPP).

    • IP Address; An IP address is a unique numerical identifier assigned to each device on a network that allows it to be located and communicate with other devices over the internet or local networks.

    • Port; A port is a numerical identifier that allows a computer to distinguish between different applications or services running on the same device when receiving network traffic.

    • Main-Device-Instance; This is the EtherCAT Main-device that controls the Sub-devices connected to it.

  • Sub-Devices Simulated (SiL); A simulated (not physically connected) Sub-device that behaves like a real Sub-device.

  • Sub-Devices captured

    • Capture File; A file that contains the captured traffic from the Sub-devices.
