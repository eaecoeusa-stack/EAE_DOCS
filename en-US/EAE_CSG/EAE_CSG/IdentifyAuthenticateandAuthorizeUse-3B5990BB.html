Identify, Authenticate and Authorize User Account

In EcoStruxure Automation Expert, user account security is provided by the Microsoft Windows Active Directory Domain Services (ADDS) built-in mechanisms. It provides centralized access management of users, groups and resources from a server defined as a Domain Controller. These group policies govern the access control and security posture of the users and computers that are part of the Control System.

The access control of Control System user accounts incorporate the two security design principles:

  • Least Privilege

  • Separation of Duties

Access control to the user accounts is provided by fine a set of role-based group policies. The following user groups should be added during system installation:

  • Plant Administrators

  • Plant Engineers

  • Plant Operators

  • Plant View Only Users

EcoStruxure Automation Expert dPAC controllers support access control with limited privilege management, it:

  • Provides user authorization: The user accounts created in the EcoStruxure Automation Expert solution should be considered as Plant Administrators or Plant Engineers. They are given access to EcoStruxure Automation Expert workstation and solution files therein unlike Plant Operator user accounts, which should not have access to these.

    NOTE: To know more about user account management, refer to User Account Management topic.

  • Provides user authentication: Users are authenticated by the EcoStruxure Automation Expert dPAC device with a credential provided by the user at the EcoStruxure Automation Expert - Buildtime or EcoStruxure Automation Expert - HMI. Only Plant Administrators or Plant Engineers can access devices via EcoStruxure Automation Expert - HMI.

    NOTE: EcoStruxure Automation Expert - HMI is not supported in a IEC 62443-3-3 system as detailed in the Unsupported Configurations topic.

  • Does not provide definition of roles with permissions and assignment to users: Each user defined in a dPAC controller access control has the same level of privilege for actions upon the dPAC from the EcoStruxure Automation Expert - HMI or OPC UA client. Thus, the access control on the dPAC controllers are not to be used for separating the level of privilege assigned to each user.

This schema makes the comparison between the EcoStruxure Automation Expert dPAC controllers access control and the full Role Based Access Control (RBAC) provided by AVEVA OMI. As mentioned above, EcoStruxure Automation Expert dPAC controllers only check for authorized and authenticated user account. Two authorized users then have the same level of privileges. Only the full RBAC on AVEVA OMI gives access to different level of privileges in-between authorized users.

Thus, you must use the AVEVA OMI as the operator interface to meet IEC62443-3-3 requirements user access management.

NOTE: To know more about the AVEVA OMI policy configuration, refer to the AVEVA OMI Policy Configuration topic.
