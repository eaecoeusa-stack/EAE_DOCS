Configure Users and Roles and Allow Permission in IDE

Creating Users

  1. Launch the previously configured galaxy with user Administrator.

  2. From the ribbon, select Galaxy > Configure > Security. The Security page appears.

  3. Click on the Users tab.

  4. Right click and select Add.

  5. Give the name as Operator1 and click on Save.

  6. Do the steps 4 and 5 for the users: Maintenance1 and Engineer1.

Creating Roles

  1. Select Roles tab.

  2. Right click and select Add.

  3. Create the role Operator.

  4. Perform the steps 2 and 3 for the roles: Maintenance and Engineer.

Assigning Roles to Users

Next let’s assign the users to the roles:

  1. Open the Users tab, all users in the Galaxy are listed here.

  2. Select the user in the Authorized Users available area.

  3. Select a role in the Associated Roles for <user name> area.

    Every user is assigned to the Default role. Add other roles that users need to perform their job functions.

  4. Provide each user with a password.

    NOTE: Ensure to set new passwords for the Administrator and Default user accounts.

  5. As it is Galaxy Authentication mode, click Change password. The Change password dialog box appears.

  6. In the same way, provide password for other users also.

Let’s give permissions for the roles in the Roles tab by selecting the check boxes for the options to give for each role.

Here are the permissions for the created roles:

User Role Permissions
Operator1 Operator

General permissions:

  • Can Start the IDE

  • Importing and Exporting

  • Framework Configuration:

    • Can modify the Security Model

  • Deployment Permissions

  • Graphic Management Permissions

  • OCMC Permissions:

    • Can start the OCMC

Operational permissions:

  • Can modify Operate attributes

  • Can modify Tune attributes

Maintenance1 Maintenance

General permissions:

  • Can start the IDE

  • Importing and Exporting

  • Framework Configuration:

    • Can modify the Security Model

  • Deployment Permissions:

    • Can deploy and undeploy area objects

    • Can deploy and undeploy application object

  • Graphic Management Permissions:

    • Can deploy and undeploy view application

  • OCMC Permissions:

    • Can start the OCMC

    • Can start and stop engine and platform

Operational permissions:

  • Can modify Tune attributes

Engineer1 Engineer

General permissions:

  • Can start the IDE

  • Importing and Exporting

  • General Configuration

  • Device Integration Objects

    • Can create, modify and delete device integration object instances

  • Application Configuration

  • Framework Configuration

    • Can modify the Security Model

  • User Configuration

  • Deployment Permissions

  • Graphic Management Permissions

  • OCMC Permissions

Operational permissions:

  • Can modify Configure attributes

  • Can modify Operate attributes

NOTE: In AVEVA System Platform, if the user with one of these roles tries to perform operations without having the permission, the user then gets notified with a message saying that this role doesn’t have the permission to do the operation.
