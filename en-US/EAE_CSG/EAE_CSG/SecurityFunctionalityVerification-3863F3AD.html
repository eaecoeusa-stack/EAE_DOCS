Security Functionality Verification

The following functionalities should be tested at Factory Acceptance Test or System Acceptance Test:

  • Basic user access control tests:

    • Accessing the AVEVA OMI with invalid user credentials.

  • Application data encryption:

    • Where application data encryption is applied the DATA_CRYPTO function block should be tested against the FIPS-140-2 test known answer tests. A test application can be requested via Schneider Electric customer support.

  • Determinism of control:

    • I/O fallback behavior during network storm events.

  • Audit logging:

    • Verifying that the SIEM (Security Information and Event Management) logs messages when Security configuration is applied to the dPAC devices.

      NOTE: Find a list of all Syslog events available in the Syslog Message Triggering Events topic from the EcoStruxure Automation Expert - User Manual.

  • Secure defaults:

    • That the dPAC devices only respond on ports quoted in dPAC Platforms and Services topic.

    • That the control network devices are only accessible from other network level on the ports permitted in the firewall configuration.

  • Back and restore:

    • That the control system can be backed up and then restored.
