User Accounts Management

An advantage of a Control System is its ability to create and manage user accounts from central Domain Controllers.

As a security best practice, we recommend to create different user accounts depending on the users roles and responsibilities.

NOTE: As part of hardening the practice, the Administrator and Guest accounts should be disabled on all operator workstations and servers.

User account properties are set through group policy settings. The following password policy settings are recommended:

  • Expiration period - none

  • Lock account duration - 240 s

  • Account lockout threshold - 5

  • Inactive period - 15 min

  • Reset account lockout counter - none

The domain policy governs the password policy. By default, all user accounts across the domain will have the same password applied on them. The following domain policy settings are recommended:

  • Enforce password history - 3 password cannot be re-used

  • Maximum password age - 365 days

  • Minimum password age - none

  • Minimum password length - 12

  • Password complexity requirements - y

It is advisable to follow best practice depending on the trade-off of availability versus DoS tolerance by applying the guidance provided at Reset account lockout counter after - Windows 10 | Microsoft Learn.

It is also recommended to remove guest and admin local users from workstations.

EcoStruxure Automation Expert dPAC Device User Accounts

Before an application can be deployed to a dPAC, user accounts must be added to the EcoStruxure Automation Expert solution. Note that in this version of EcoStruxure Automation Expert all users have the same level of privilege. For more information refer to the EcoStruxure Automation Expert - User Manual topic User Management.

Edge I/O Device User Accounts

Configure the user accounts added to the Edge I/O devices to limit the level of privilege. The following settings are recommended:

  • Minimum Inactivity Period (min) = 15 min

  • Password Attempts Timer (min) = 240 s

  • Maximum Login Attempts = 5

For more information, refer to the topic Edge I/O Distributed IO Configuration.

ASRock iEP-5000G iPC and Linux Host User Accounts

User accounts control access to the device. If these accounts are not properly managed, attackers can gain unauthorized access, change configurations, or disrupt operations. Therefore, securing these accounts is important to maintaining device integrity.

ASRock iEP-5000G iPCs provided by Schneider Electric will come configured according to the following policy. Linux hosts should be configured according to the following policies:

  • The device uses local accounts and there is no integration with external directory services.

  • Default Accounts:

    • eeadmin is the only authorized account for SSH access.

    • The root account is disabled by default.

  • SSH Access:

    • SSH is disabled by default. Enable it only when required for setup or maintenance.

    • When SSH is enabled, only one SSH session is allowed at a time. This should be disabled after setup or maintenance.

  • Password Management:

    • Change default password during initial setup.

    • Save passwords securely. If you forget the new password, the device must be re-imaged.

      • To save passwords they must be either physically secured or secured by using a password manager such as Bitwarden1 to store a Note password.

    • Remove Unused Accounts: Delete or disable any accounts that are not required for operation.

The UEFI/BIOS Supervisor and User accounts should be protected by a strong password. Note that it is not normally possible to enforce a password policy in the UEFI/BIOS
