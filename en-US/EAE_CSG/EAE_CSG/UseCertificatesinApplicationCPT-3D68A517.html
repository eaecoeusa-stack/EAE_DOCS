Use Certificates in an EcoStruxure Automation Expert Application

Following are function blocks using certificates:

  • MQTT_CONNECTION

  • NETIO

  • DATA_CRYPTO

MQTT_CONNECTION

The CA certificate used to authenticate the connection and the certificate that the dPAC presents are specified as filename paths in the CACert and data inputs.

NETIO

The certificate used for both encryption and authentication is specified as the CERT_FILE data input. This certificate file is a PEM file containing both the SSL certificate and the private key.

NOTE: Soft dPAC and ATV dPAC controllers support the use of TCP with SSL encryption. Other devices - for example, M251d, M262d, and M580d controllers - support TCP but do not support SSL encryption.

DATA_CRYPTO

Certificates can be validated using the DATA_CRYPTO function block.

  • KEY: The Public Key certificate used to verify the certificate

  • SD: Certificate text to verify in base64 format

  • SIGNATURE: The expected signature in base64 format
