AVEVA Industrial Graphic Workflow Using Galaxy User Having Operator Role

Procedure

  1. Ensure to have installed EcoStruxure Automation Expert - Buildtime with the AVEVA Industrial Graphics Editor feature selected.

  2. Launch EcoStruxure Automation Expert and create a new solution.

  3. Configure a Galaxy using Operator1 credentials.

  4. Create a CAT.

  5. Open the Interface editor of its HMI CAT.

  6. Add new input and output variables and connect them with new input and output events.

  7. In Offline Parametrization editor, make the variables persist and in the OPCUA Server editor, make them exposed. Save the HMI CAT.

  8. Open the Function Blocks Network editor of the CAT and add there the HMI CAT.

  9. Right click on the HMI CAT in the Solution Explorer pad and select Add AVEVA Industrial Graphic.

  10. Create the CAT graphics there by dragging and dropping any tool. Save the graphic.

  11. Open the ASP Asset Mapping editor of the CAT and tick the AutoMap checkbox.

  12. In the Solution Explorer, right click on the Application node (the parent node of the previously created CAT) and select Sync Automapped AVEVA SP Templates.

As the Operator user has the Graphic Management permissions, all the templates and graphics will be created once the sync is completed.

NOTE: As the operator role doesn’t have permission to create, modify and delete graphic objects, the operator cannot create graphics manually in ASP.

Similar to Operator role, the different actions performed in this procedure are allowed to the users depending on the permissions allocated to the role they selected (Engineer or Maintenance).
