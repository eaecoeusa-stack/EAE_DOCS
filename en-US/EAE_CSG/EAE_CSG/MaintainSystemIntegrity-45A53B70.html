Maintain System Integrity

Maintain your system integrity by helping protect the integrity of the following elements:

  • Software package integrity

  • Switches integrity

  • Operations integrity

Software Package Integrity

The integrity and authenticity of software is essential to a trustworthy operation.

All EcoStruxure Automation Expert software components are digitally signed by certificate(s) issued by a third-party trusted certificate authority to verify integrity and authenticity. This indicates they have not been tampered with, from the moment they were built to the point they are executed.

Microsoft provides the ability to view the certificate by right-clicking the file in question and viewing the certificate details to validate any component that may be in question.

NOTE: For more information about how to verify the software package integrity, refer to the Signature Verification topic from the Soft dPAC - User Guide for Soft dPAC devices.

Switches Integrity

The Network Budget Calculation is required for an adequate network design. The calculation is based on port VLAN assignments. There are many ways of deploying VLANs across a network.

NOTE: You should understand these methods before configuring and deploying the network.

Within each switch is a CPU that manages signaling and processing of data with a finite capacity, as any computer.

An unconstrained network application might consume the full capacity of a switch. Therefore those constraints must be identified and mitigated.

At the same time, the bandwidth needs of each application must be established. Most often this is through testing to identify:

  • Average traffic

  • Peak traffic

This helps you to know how much bandwidth is needed to support each application and then take the design decision of how much bandwidth needs to be allocated to fulfill the communications requirement of that application.
