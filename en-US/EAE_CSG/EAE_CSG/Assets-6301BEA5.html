Assets

The following assets in the system are considered critical:

  • Cybersecurity configuration: usernames and password hashes.

  • Device configuration: the configuration of controller and device parameters that influences how the application is executed.

  • Control application: the control program and device configuration executed by the controllers.

  • Controller firmware: the device firmware that can be updated via the EADM tool.

  • Controller resources: the CPU and memory required to execute the application.

  • Control process state inputs / actuator outputs: process input and output data accuracy, plus the actuator outputs behaving in a deterministic way even during network storm events.

  • Operator inputs: the actions the operator can take, according to their level of privilege, to influence the process.

  • Audit log: log of events relating to cybersecurity.

The architecture presented in this guide helps to protect these assets according to the requirements of IEC 62443-3-3.

For each application, you must consider if any data within the application is considered as security critical and apply applicable counter measures.
