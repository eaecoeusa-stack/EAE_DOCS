Security Audit Logging on Syslog

Currently EcoStruxure Automation Expert devices support Syslog using both modes TCP and TLS (secured communication).

For a list of Syslog triggering events, refer to the EcoStruxure Automation Expert - User Manual topic Syslog Message Triggering Events.

Using the Graylog Syslog Server

EcoStruxure Automation Expert does not provide a Syslog server. For the system to provide audit logging a Syslog server with the IP address of the Graylog/SIEM must be defined in the Physical devices view and Syslog client must be configured on all dPAC devices.

To configure the Syslog server:

  1. In the Physical devices view add a server with IP address corresponding to the Graylog service.

  2. Right click and select Manage Services > Syslog Server "Activate Service".

  3. The Syslog settings can then be adjusted in the properties under the SYSLOG Server section.

Mutual authentication between the dPAC devices and the Graylog Syslog server is not possible. So the TLS connections between the devices and Syslog server must be added to the firewall access control list. If the security audit log is considered as critical and the network path cannot be trusted with the firewall in place, an IPSEC tunnel could be created between the control network and the Graylog server.

NOTE: For more information about Graylog, refer to Account Activity Log topic.

Securing the Perimeter

It is strongly recommended to secure the perimeter of the engineering workstations with EcoStruxure Automation Expert using Syslog in it and their connection to COTD.
NOTE: For more information about COTD, refer to the Account Activity Log topic.

For COTD to receive EcoStruxure Automation Expert log data, perform these two configurations:

  • On COTD, configure an input listener with Syslog using TCP.

    NOTE: Refer to CAP documentation on how to configure unsecured TCP and UDP as inputs.

  • Configure on EcoStruxure Automation Expert using Syslog the IP address and port to connect the engineering workstation to COTD Syslog input listener.

    NOTE: Refer to Configure SYSLOG topic of the EcoStruxure Automation Expert - User Manual.

Audit Logging Practices

The audit logs should be checked regularly to identify any unexpected security events. The following audit logs must be referred to for a complete view of security related events:

  • OT dashboard and SIEM Graylog

  • Windows Event Viewer

  • Aveva Alarms and Events subsystem

    NOTE: Security-audit (operator change) events Alarms and Events subsystem refer to the Types of Events chapter in the "AVEVA™ System Platform IDE User Guide" [ Publication ID: 922676 ]

  • EcoStruxure Device Maintenance logs

The following must also be considered for effective audit logging.

The Domain Server (Server Hosting the "Active directory Service") the Graylog server, the AVEVA Server and any workstation used to view the logs should be time synchronized so that all timestamps correlate.

Edge I/O devices also provide a log that includes security audit events, which should be checked regularly. For more information refer to the Modicon Edge I/O Configurator and Web Interface - User Guide topic Logs Section (under Island Management Main Page).

NOTE: Syslog is not yet supported by these Edge I/O devices, even if it is configured using Cybersecurity Admin Expert. Refer to the Modicon Edge I/O Configurator and Web Interface - User Guide topic Security Settings Configuration with Cybersecurity Admin Expert Software for more information.
