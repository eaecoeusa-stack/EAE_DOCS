EcoStruxure Automation Expert Known Limitations

OPC UA Client and Server Services

  • OPC UA Client for EcoStruxure Automation Expert - Soft dPAC is turned off by default.

    NOTE: Turn on and configure the service, if required for use.

  • OPC UA Server is turned on by default for all dPAC devices.

    NOTE: Turn off the service if the use is not required.

Power Meters

The power meters, available in the device catalog of the Physical devices editor of EcoStruxure Automation Expert, does not support the following services:

  • Discovery service

  • Locate service

  • Get Identity service

  • IP settings configuration

  • Web server configuration

Asset Tag Names Display

When the Display Alias Name Toggle Button checkbox is selected, the asset tag names configured in the CAT instances are displayed, if the CAT_HMI Symbol is configured with Asset Tag Label or Asset Tag Text. 

Asset Link for AVEVA OMI #166496

In the ASP Attribute mapping editor of the AVEVA System Platform (ASP), after the data from the ASP attributes that are automatically mapped to CAT attributes is saved, automatic mapping of ASP attributes cannot be performed again. The automatic mapping of attributes: 

  • Modifies the ASP Template for the mapped CAT, SubCAT and SubApp 

  • Modifies the relation type in ASP Asset Mapping editor 

  • Renames the SubCATs in the Solution Explorer 

EtherNet/IP

Although dPACs support up to 128 EtherNet/IP devices, managing more than 32 EtherNet/IP devices per dPAC significantly reduces the performance of EcoStruxure Automation Expert - Buildtime. 

Ethernet/IP Import and Export ,

It is not possible to export the Ethernet/IP devices contained in your hardware CAT configuration (HWConfig). 

If you import a HWConfig file, exported from an earlier version of EcoStruxure Automation Expert, the Ethernet/IP devices contained in the file, are imported with the rest of the configuration. 

Forced Values and Blocked Events

Forcing values or blocking events is possible from both EcoStruxure Automation Expert and EcoStruxure Automation Expert - HMI. When values are forced or events are blocked, they remain in the devices even after the user logs out, closes EcoStruxure Automation Expert or EcoStruxure Automation Expert - HMI, or if the communication between EcoStruxure Automation Expert or EcoStruxure Automation Expert - HMI and the devices is lost. As well, if some values are forced in EcoStruxure Automation Expert, they apply immediately to a rebooted device. 

Generic Function Blocks Displayed as Obsolete

Generic function blocks might be displayed as obsolete sometimes, even though the library contains the correct reference to the function block type. However, these function blocks are deployed correctly. 

Harmony EcoStruxure Automation Expert - HMI ST6 Discovery

The module is shown as not present and the discovery does not display all the EcoStruxure Automation Expert - HMI ST6 modules despite a correct configuration.  

Hyperlinks in EcoStruxure Automation Expert Main Help

Clicking on shortcuts to help in the ribbon bar or hitting F1 key will open the local help. This documentation is installed locally on your computer. It contains hyperlinks such as HowTo videos or external links that might not open when you click on them. 

If this occurs: 

  • right-click the link, select copy path, then open your favorite Internet browser and paste the link into the address bar to access the videos or resources. 

  • Or open the  same document on  https://product-help.se.com/EcoStruxure Automation Expert/24.0 

POU and FUNCTION

POU objects are no longer supported. If you open a project created with earlier versions of EcoStruxure Automation Expert that includes defined POU objects, you cannot view, edit or remove them nor create new ones. 

Function objects supports only Structure Text language. CFC and LD languages are no longer supported. If you open a project created with earlier versions of EcoStruxure Automation Expert that includes defined Function objects using CFC or LD languages, you cannot view, edit or remove them nor create new ones. 

Renaming the System Node

Starting from EcoStruxure Automation Expert V23.0, it is not possible to rename the System node. 

Special Characters in Path

EcoStruxure Automation Expert is not able to open or save files related to the solution if the full path of the file (folder path or file name) contains complex characters. 

The following characters are supported by EcoStruxure Automation Expert: 

a...z A...Z 0...9 à â ä é è ê ï î ô ù û ç – _ and space. 

Wrong Value for defaultsUsed output of PERSISTENCE_FB

Default value read instead of persisted value in a particular case: 

  • The defaultsUsed output flag will not change until default values are first persisted 

  • When EcoStruxure Automation Expert - Soft dPAC (including "local test" context) is started for the first time, any write action followed by a read action will still read the default values. 

  • After a firmware update 

  • Whenever persistent data is manually deleted 

A subsequent restart (including REBOOT DEVICE, RESTART RUNTIME and power cycle) is required at least once to persist the default values, and to have the proper behavior when reading after a write action. 

Remapping Device with Reliable Cross Communication

If you need Remapping a logical device to a physical device with a different IP address, requires to run the following commands on every device connected to the logical device thought Cross Communication: 

  • Compile for Online change 

  • Deploy→Online change 

EcoStruxure Architecture Builder

The latest EcoStruxure Architecture Builder (architecture designs tool for specific users) streamlines the hardware specification for EcoStruxure Automation Expert project with an updated configuration stepper that:  

  • Generate the architecture with the right dPAC and amount based on the number of IOs, communication protocol and application requirements (like High Availability)   

  • Include M262d, Essential Edge, ASRock and BMECRD0100 as part of the available hardware  

  • Configure High Availability SoftdPAC architectures
