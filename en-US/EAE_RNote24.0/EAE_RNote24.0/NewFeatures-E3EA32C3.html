MQTT client

MQTT Client functionality in EcoStruxure Automation Expert 24.0 is compatible with MQTT Version 3.1.1. 

For EcoStruxure Automation Expert 24.0, MQTT Client functionality is available with both EcoStruxure Automation Expert - Soft dPAC (Windows) and docker containerized EcoStruxure Automation Expert - Soft dPAC (Linux). 

Note: MQTT Client functionality is not available on EcoStruXure Automation Expert - High Availability Soft dPAC. 

For the MQTT Client functionality we provide MQTT_CONNECTION, MQTT_PUBLISH, MQTT_SUBSCRIBE function blocks to establish secure communication with the broker, publish and receive data from the broker. These function blocks are available in the Runtime.Base library. 

While utilizing the MQTT_SUBSCRIBE function block, please note that certain performance limitations have been observed. Please be careful to not exceed CPU load beyond 70%. This will result in disconnection from the broker. 

Please note -> As per MQTT standard, there is a tradeoff whilst utilizing QoS = 2 over QoS = 1 OR QoS = 0. CPU consumption will be drastically more (increases quadratically) for QoS = 2 in comparison to QoS = 0 OR QoS = 1.We recommend usage of QoS = 0. QoS = 2 should be used only, when necessary, i.e., when the data associated with the topic must not be lost. 

CPU consumption will also be determined based on the number of MQTT_SUBSCRIBE function blocks associated with a CONNECTION as configured in the MQTT_CONNECTION function block.  

Number of MQTT_SUSBCRIBE function blocks linked to one CONNECTION has an influence on CPU load so one solution to lower CPU load is to distribute MQTT_SUBSCRIBEfunction blocks among multiple connections. 

Prior to utilizing the MQTT Client functionality for your project, please consult with EcoStruXure Automation Expert Customer Support team to better understand suitability of the MQTT features.
