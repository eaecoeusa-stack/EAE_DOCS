Hardening Guideline for Containers on Soft dPAC

The purpose of this guideline is to explain the security concerns associated with container technologies and make practical recommendations for addressing those vulnerabilities when specifying, implementing, and maintaining containers. The use of container-specific operating systems (for example, Alpine OS) is more than required to reduce surface attack due to their construction, instead of generic ones (for example, Red Hat, or Ubuntu).

Architectural Point of View

If you modify or add containers, try to group them by functionality or security levels as a measure to containment.

  • Stay minimal: Remove any components that are not needed, or any outdated functionalities.

  • Only use trusted images: Use images that have been previously scanned and found to be trustworthy.

  • Harden the host: Use scripts to configure hosts properly, based on state-of-the-art cybersecurity best practices.

  • Remove privileges: Running privileged containers is a security risk because it creates an opportunity for a malicious user to take control of the host system:

    1. The root access must be limited to designated security member of the company.

    2. The docker members must be limited to designated people when installing and uninstalling the Soft dPAC solution.

  • Respect the CISO rules of the company.

Monitoring Point of View

Software solutions can be used to surveil the environment for suspicious activity.

There are several types of Intrusion prevention systems (IPSs), each with a different purpose:

  • Network intrusion prevention system (NIPS): This type of IPS is installed only at strategic points to monitor all network traffic and proactively scan for threats.

  • Host intrusion prevention system (HIPS): In contrast to a NIPS, a HIPS is installed on an endpoint (such as a PC) and looks at inbound and outbound traffic from that machine only. It works best in combination with a NIPS, as it serves as a last line of defense for threats that have made it past the NIPS.

  • Network behavior analysis (NBA): This system analyzes network traffic to detect unusual traffic flows, such as DDoS (Distributed Denial of Service) attacks.
    Here are some best practices for setting up an NBA system:

    1. Analyze your options carefully before picking an NBA offering: There are many different NBA solutions available, and it’s important to choose one that meets your specific needs. Consider factors such as the size of your network, the types of the used devices, and the required level of security.

    2. Do not rely on NBA alone: Use other security tools in addition to NBA, such as firewalls and antivirus software, to provide comprehensive protection for your network.

    3. Ensure live testing before full-scale adoption: Before deploying an NBA system across your entire network, test it in a limited environment to ensure that it works as expected and does not cause any disruption.

    4. Fine-tune NBA to minimize false positive: NBA systems can generate a large number of alerts, many of which might be false positive. It is important to fine-tune your NBA system to minimize false positive and ensure that you only receive alerts for genuine security threats.

    5. Leverage NBA for security and non-security monitoring: NBA systems can be used for more than just security monitoring. They can also be used to monitor network performance, identify bandwidth hogs, and optimize network traffic.

    6. Web application firewalls (WAF): A web application firewall is a security tool that monitors, filters and blocks HTTP traffic to and from a web application.  

    1. Choose the right WAF for your needs: There are many different WAF solutions available, and it is important to choose one that meets your specific needs. Consider factors such as the size of your network, the types of the used devices, and the required level of security.

    2. Configure your WAF properly: Proper configuration is essential to ensure that your WAF is effective. Configure your WAF to monitor all traffic that passes through it, and set up rules to block malicious traffic.

    3. Keep your WAF up-to-date: New threats are constantly emerging. Keep your WAF up-to-date with the latest security patches and updates.

    4. Monitor your WAF regularly: Regular monitoring is essential to ensure that your WAF is working properly and blocking malicious traffic. Review your WAF logs regularly to identify any suspicious activity.

    5. Integrate your WAF with other security tools: WAFs are an important part of a comprehensive security strategy, but they should not be relied on as the sole security tool. Use other security tools in addition to the WAF, such as firewalls and antivirus software, to provide comprehensive protection for your network.
