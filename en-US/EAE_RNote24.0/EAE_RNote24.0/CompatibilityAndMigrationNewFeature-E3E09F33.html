EcoStruxure Automation Expert - HMI Compatibility

The faceplates of the following libraries are compatible with EcoStruxure Automation Expert - HMI, hosted on an HMIST6 and a computer running Windows, assuming their size is larger than 10 In: 

  • SE.ModbusGateway 

  • Standard.IoModbus 

  • Standard.IoModbusGateway 

  • SE.IoTMx 

  • SE.IoATV 

  • SE.DPAC

  • SE.AppBase 

  • SE.AppCommonProcess 

  • SE.AppConveying 

  • SE.AppSequence

  • SE.FieldDevice 

Performance improvement

Communication protocol has been enhanced by: 

  • allowing to send subscription of more events in one message 

  • allowing to send subscription replies along with existing data from the queue 

  • allowing to send unsubscriptions of more events in one message allowing to send unsubscriptions replies along with existing data from the queue 

This will speed up the subscriptions and lower the round-trip rate. 

Secured communication

Communication between EcoStruxure Automation Expert - Buildtime and EcoStruxure Automation Expert - HMI is now encrypted leveraging CTI23 format. 

Known Limitations

Canvas Names

If you use the prefix of any existing library in a canvas name (for example, SE, or SE.MyCanvas), this results in a build error in the solution.

The word HMI cannot be used as a canvas name.  

EcoStruxure Automation Expert - HMI Service

EcoStruxure Automation Expert - HMI deployment service does not start immediately after installing the setup. 

VTQ

EcoStruxure Automation Expert - HMI function blocks support all VTQ data types through the HMI interface. 

For offline parametrization, Time and Quality always goes to fallback with the following default values: 

  • Time = 0 

  • Quality = OK (0) 

You can archive only the Value of VTQ variables. Time and Quality are not archived.
