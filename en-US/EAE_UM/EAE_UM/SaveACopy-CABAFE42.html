Save a Copy

The Save a Copy command allows the user to save a copy of the current solution on the engineering workstation. It is located in the File menu.

The Save a Copy command can be deactivated for the following reasons:

  • There is no open solution.

  • A build or compile is ongoing.

  • The current solution is not yet saved.

  • A device is in running state.

To store a copy of the solution, the following points must be checked:

  • The folder in which the copy of the solution is stored must be empty.

  • The folder cannot be the folder or a child folder of the current solutions folder.

  • There must be enough space on the hard disk of the engineering workstation.

Once these conditions are met, follow these steps to store a copy of the solution on the engineer's workstation:

  1. Click on the Save a Copy command to display the Save a copy of the solution dialog box.

  2. Following the conditions above, create a new folder in the location of your choice.

  3. Open the created folder.

    NOTE: If the conditions mentioned above are not met, a pop-up containing an error message appears on the screen.

  4. Click on save or press the enter button to copy the solution into the created folder.

When the solution is copied and saved, the following components are deleted from the created copy of the solution:

  • Snapshots

  • The project GUID is renewed

  • Certificates

  • User management

NOTE: The copied solution does not contain any generated binaries or files.
