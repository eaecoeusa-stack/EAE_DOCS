Mapping Conflicts

This dialog automatically opens when a conflict is detected during the configuration of function blocks, for example, by attempting to map them twice or a attempting to apply an existing function block name.

The dialog presents two parts:

  • The upper part lists the function blocks causing the conflict. When you click in the column Resolve conflict, a list appears from which you can select the action to be performed. If you select Rename, a new name is proposed in the column directly to the right. You can overwrite this name.

  • The lower part displays the cause of the mapping conflict.

Click OK to close the dialog and execute the selected action.

Click Cancel to close the dialog without an action.
