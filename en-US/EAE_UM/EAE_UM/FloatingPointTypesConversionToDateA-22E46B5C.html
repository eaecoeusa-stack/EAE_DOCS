3. Floating Point Types Conversion to Date and Time Types

All functions described in this topic are conforming to IEC 61131-3 third edition, unless specified otherwise.

 CAUTION
RISK OF DATA LOSS
When using floating point types (REAL or LREAL) conversion to Date or Time types (DT or LDT), make sure that the rounding of values has not unwanted effects on final calculations or process driving.
Failure to follow these instructions can result in injury or equipment damage.

Refer to the example provided for function REAL_TO_DT.

REAL_TO_DT, LREAL_TO_DT

Synopsis 

TO_DT(IN: REAL): DT
REAL_TO_DT(IN: REAL): DT
TO_DATE_AND_TIME(IN: REAL): DT
REAL_TO_DATE_AND_TIME(IN: REAL): DT
 
TO_DT(IN: LREAL): DT
LREAL_TO_DT(IN: LREAL): DT
TO_DATE_AND_TIME(IN: LREAL): DT
LREAL_TO_DATE_AND_TIME(IN: LREAL): DT

Description

Converts input parameter IN of type REAL or LREAL holding seconds since 1970-01-01-00:00:00 (local time zone) to a DATE_AND_TIME value holding milliseconds since 1970-01-01-00:00:00 (UTC).

Errors

ERR_DT_CONVERSION: A date or time conversion error is reported if the resulting date and time value is not in the range defined for data type DATE_AND_TIME.

ERR_RT_OVERFLOW: An overflow error is reported if the value of parameter IN converted to milliseconds exceeds the limits of a 64-bit signed integer value.

Example 

VAR
  R1 : LREAL;
  D1 : DT;
END_VAR
R1 := DT_TO_LREAL(DT#2021-5-12-13:15:35);
D1 := REAL_TO_DT(TO_REAL(R1));
IF D1 = DATE#2021-5-12-13:15:44 THEN
  (* the result differs from the original value by nine (9) seconds *) ;
END_IF;

REAL_TO_DT_UTC, LREAL_TO_DT_UTC

These functions are an extension to IEC 61131–3 third edition.

Synopsis 

TO_DT_UTC(IN: REAL): DT
REAL_TO_DT_UTC(IN: REAL): DT
 
TO_DT_UTC(IN: LREAL): DT
LREAL_TO_DT_UTC(IN: LREAL): DT

Description

Converts input parameter IN of type REAL or LREAL holding seconds since 1970-01-01-00:00:00 (UTC) to a DATE_AND_TIME value holding milliseconds since 1970-01-01-00:00:00 (UTC).

If a controllers time zone is UTC, these functions work the same way as the (L)REAL_TO_DT family.

Errors

ERR_DT_CONVERSION: A date or time conversion error is reported if the resulting date and time value is not in the range defined for data type DATE_AND_TIME.

ERR_RT_OVERFLOW: An overflow error is reported if the value of parameter IN converted to milliseconds exceeds the limits of a 64-bit signed integer value.

Example 

VAR
  DT1 : DT;
END_VAR
DT1 := LREAL_TO_DT_UTC(LREAL#1620825335.0);
IF DT1 = DT_UTC#2021-5-12-13:15:35 THEN
  (* correct *) ;
END_IF;

REAL_TO_LDT, LREAL_TO_LDT

Synopsis 

TO_LDT(IN: REAL): LDT
REAL_TO_LDT(IN: REAL): LDT
TO_LDATE_AND_TIME(IN: REAL): LDT
REAL_TO_LDATE_AND_TIME(IN: REAL): LDT
 
TO_LDT(IN: LREAL): LDT
LREAL_TO_LDT(IN: LREAL): LDT
TO_LDATE_AND_TIME(IN: LREAL): LDT
LREAL_TO_LDATE_AND_TIME(IN: LREAL): LDT

Description

Converts input parameter IN of type REAL or LREAL holding seconds since 1970-01-01-00:00:00 (local time zone) to an LDATE_AND_TIME value holding nanoseconds since 1970-01-01-00:00:00 (UTC).

Errors

ERR_DT_CONVERSION: A date or time conversion error is reported if the resulting date and time value is not in the range defined for data type DATE_AND_TIME.

ERR_RT_OVERFLOW: An overflow error is reported if the value of parameter IN converted to nanoseconds exceeds the limits of a 64-bit signed integer value.

Example 

IF LREAL_TO_LDT(1620825335.123456789 = DATE#2021-5-12-13:15:35.123456716 THEN
  (* correct (note the roundoff error due to LREAL having only approximately 16 significant digits compared to 18 for a signed 64-bit integer *) ;
END_IF;

REAL_TO_LTIME, LREAL_TO_LTIME

Synopsis 

TO_LTIME(IN: REAL): LTIME
REAL_TO_LTIME(IN: REAL): LTIME
 
TO_LTIME(IN: LREAL): LTIME
LREAL_TO_LTIME(IN: LREAL): LTIME

Description

Converts input parameter IN of type REAL or LREAL holding seconds to a LTIME value holding nanoseconds.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the real value does not fit into a 64-bit signed integer. The value returned will be the minimum or maximum signed 64-bit value depending on whether the input value is too small or too large.

Example 

IF TO_LTIME(REAL#3141.5927) = LTIME#3141.5927s THEN
  (* conversion succeeded *) ;
END_IF;

REAL_TO_TIME, LREAL_TO_TIME

Synopsis 

TO_TIME(IN: REAL): TIME
REAL_TO_TIME(IN: REAL): TIME
 
TO_TIME(IN: LREAL): TIME
LREAL_TO_TIME(IN: LREAL): TIME

Description

Converts input parameter IN of type REAL or LREAL holding milliseconds to a TIME value holding milliseconds.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the real value does not fit into a 32-bit signed integer. The value returned will be the minimum or maximum signed 32-bit value depending on whether the input value is too small or too large.

Example 

IF REAL_TO_TIME(3141.5927) = TIME#3.142s THEN
  (* conversion succeeded *) ;
END_IF;
