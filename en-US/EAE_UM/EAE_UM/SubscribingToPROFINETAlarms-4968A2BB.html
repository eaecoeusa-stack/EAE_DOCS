Subscribing to PROFINET Alarms

Use the PN_RECEIVE_ALARM function block to:

  • Create a single PROFINET alarm subscription, by setting INIT = 1 + QI = TRUE.

  • Remove a previously created alarm subscription, by setting INIT = 1 + QI = FALSE.

When you create an alarm subscription, that subscription remains in effect until it is removed.

To edit a previously created subscription, remove the original subscription, then create a new subscription.

Alarm Priorities

Each IO-Device maintains two PROFINET alarm queues: High Priority & Low Priority:

  • High Priority is for Process Alarms: triggered by events described in solution logic and reported by the program.

  • Low Priority is for all other alarms, including:

    • Diagnostic alarms (e.g., a field device becomes non-operational)

    • Maintenance alarms (e.g., a field device has been operating a period of time defined for scheduled maintenance)

    • IO-Device specific alarms (e.g., device manufacturer diagnostic alarms)

The IO-Device sends all queued High Priority alarms before it sends Low Priority alarms.

Filtering Alarms

You can configure the PN_RECEIVE_ALARM function block to receive:

  • ALL_ALARMS from the specified IO-Controller (BUS_ID), or

  • Only specified alarms, by filtering the alarms by alarm source. Filtering options include:

    • IO-Device (BUS_ID + STATION_NAME_FILTER)

    • Slot (BUS_ID + STATION_NAME_FILTER + SLOT_NO_FILTER)

    • Subslot (BUS_ID + STATION_NAME_FILTER + SLOT_NO_FILTER + SUBSLOT_NO_FILTER)

Configuring a PROFINET Alarm Subscription

  1. In the Function Blocks Network editor, add the PN_RECEIVE_ALARM function block from the library Standard.IoProfinet > Services.

  2. Configure QI to specify the requested action:

    • TRUE = create subscription

    • FALSE = remove subscription

  3. Configure the alarm subscription source:

    1. Specify the BUS_ID of the source PROFINET IO-Controller (bus master) of the type Standard.IoProfinet.PN_IOController.

    2. Specify the PROFINET filters that can be applied to the subscription:

      • STATION_NAME_FILTER:identifies the source IO-Device on the specified IO-Controller.

      • SLOT_NO_FILTER: identifies the Slot Number of the specified source IO-Device.

      • SUBSLOT_NO_FILTER: identifies the Subslot Number of the specified Slot Number.

    3. MODE : specifies the filters you specified in step 2.b. that will be applied to the subscription:

      • ALL_ALARMS

        : All alarms from all IO-Devices of the IO-Controller are sent.

      • ALARM_BY_DEVICE: Only alarms from the specified IO-Device are sent.

      • ALARM_BY_SLOT: Only alarms from the specified IO-Device Slot are sent.

      • ALARM_BY_SUBLOT: Only alarms from the specified IO-Device Subslot are sent.

  4. After the subscription action is completed, check the STATUS data output text value, which can display:

    • +Subscription OK: Adding subscription was processed and sent to PROFINET IO-Controller.

    • +Subscription Error: Adding subscription was not processed. Refer the Applications Design Guidelines topic Runtime Device and Cyclic Logs for more information.

    • -Subscription OK: Removing subscription was processed and sent to PROFINET IO-Controller.

    • -Subscription Error: Removing subscription was not processed. Refer the Applications Design Guidelines topic Runtime Device and Cyclic Logs for more information.

    • Notification OK: Alarm notification was received from the PROFINET IO-Device

    • Notification Error: Alarm notification contains one or more detected errors.

      NOTE: The Standard.IoProfinet library help for the PN_RECEIVE_ALARM function block includes a link to PROFINET detected error codes.

  5. Check the ALARM_TYPE output.

    NOTE: The Standard.IoProfinet library help for the PN_RECEIVE_ALARM function block includes a link to PROFINET alarm types.
