Unarchive Solution

The unarchive mechanism is available in EcoStruxure Automation Expert to open any archive file previously created following the archive mechanism. This mechanism will unarchive the file by following these two steps:

  1. Create a new solution folder where the unarchived solution will be stored.

  2. Then open the unarchived solution in EcoStruxure Automation Expert.

The unarchived solution will be identical to the original solution before archiving.

NOTE: In some cases, for example when using EcoStruxure Automation Expert - Buildtime version 24.1 to unarchive a solution created with EcoStruxure Automation Expert - Buildtime version 23.1, the links between the logical devices and physical devices are lost. In this case, you will need to add back these links after unarchiving.

To unarchive a previously archived file, follow these steps:

  1. Go to the File menu.

  2. Select Open and then Unarchive.

  3. A dialog box opens. There navigate into the folders to find the archive file to unarchive (the file extension is *.sln.zip).

  4. Select it and click on Open.

  5. A new dialog box opens. Select or create a new folder where the unarchive solution will be stored.

    NOTE: The default folder for archived solutions is: Documents\Schneider Electric\EcoStruxureAutomationExpertProjects

  6. Once created or selected, click on OK.

    NOTE: If the solution already exists in the selected directory, a dialog opens and asks to erase the old content of the selected directory or not.

At this point, depending on different cases, specific steps needs to be followed:

  • The engineering workstation unarchiving the solution has already installed all the libraries referenced in the solution at their specific version:

    The solution unarchives completely.

  • The engineering workstation unarchiving the solution doesn’t have installed all the libraries referenced in the solution at their specific version and the referenced libraries missing were included during the archive mechanism:

    1. During the solution unarchiving process, the dialog box Unarchive project library references appears and lists all the libraries referenced in the archived solution.

      The dialog box gives a status for all the libraries referenced in the library, either Installed or Not Intalled in the current engineering workstation.

    2. For the ones not installed, click on Install in the Action section.

    3. Complete the installation process by going through the Install Library dialog box for each not installed library.

    4. When all the libraries have the Installed status, click on Unarchive.

    5. The solution unarchive process is completed.

  • The engineering workstation unarchiving the solution doesn’t have installed all the libraries referenced in the solution at their specific version and the referenced libraries missing were not included during the archive mechanism:

    1. The solution unarchives.

    2. At the end of the unarchiving mechanism, a pop up appears listing all the libraries missing.

    3. Click OK to complete the unarchive mechanism. The solution is then unarchived and open but it cannot be built as the libraries missing are not referenced.

    4. To be able to build the solution, either:

      • Install manually the missing libraries and add them manually too as detailed in the topics Installed Libraries and Define Project Library References.

      • Reference in the solution other version of the missing libraries and all the libraries referencing to them as detailed in the topic Define Project Library References.

        NOTE: Choosing a different version of library can lead to changes into the library functionalities.

After following all these steps (the general steps then the specifics ones) then the solution is unarchived, it can build and is at the same state as the original solution just before archiving.

NOTE: Check the log file of the unarchive in the Output pad.
