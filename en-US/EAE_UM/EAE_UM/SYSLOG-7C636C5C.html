SYSLOG

EcoStruxure Automation Expert - Buildtime version 23.1 and later support the message logging protocol SYSLOG.

Use the SYSLOG service to capture an audit trail of cybersecurity-related system event messages, and store these messages on a third-party read-only SYSLOG server.

The SYSLOG protocol is standardized according to RFC 5424. SYSLOG messages incorporate the following features:

  • Message Structure: Defines the content and order of the data contained in each event message.

  • Facility Codes: Describe the kind or source of the SYSLOG message.

  • Message Severity Codes: Describe level of urgency assigned to the SYSLOG message.

Refer to the Appendix for a list of events that trigger SYSLOG messages.

SYSLOG plus TCP and SYSLOG plus TLS

EcoStruxure Automation Expert supports both encrypted and unencrypted SYSLOG messaging, depending on the device.

  • SYSLOG (version 23.1 and later) supports unencrypted communications and is sent over TCP.

    NOTE: For Soft dPAC and for HA Soft dPAC controllers using unencrypted SYSLOG communications, select the server device in the Physical Devices editor, then in the Properties pad set the Syslog Server > Logical port (unsecured) setting to 601.

  • SYSLOG sent over Transport Layer Security - TLS - (version 24.1 and later) supports encrypted communications.

    NOTE: For Soft dPAC and for HA Soft dPAC controllers using encrypted SYSLOG communications, select the server device in the Physical Devices editor, then in the Properties pad set the Syslog Server > Logical port (secured) setting to 6514 (the default).
