Home Menu

The Home menu presents a ribbon with groups of commands for:

  • clipboard

  • editing

  • verifying application consistency

  • building the HMI

  • Symbolic Links

  • zooming

  • printing

  • importing EDS files

The Home menu is displayed by default when you open EcoStruxure Automation Expert - Buildtime.

Clipboard Area

Command

Description

Paste

The content of the clipboard is inserted at the current position

Cut

The selection is cut out and placed in the clipboard

Copy

The selection is copied and placed in the clipboard

Editing Area

Command

Description

Select All

Selects the entire content of the active document.

Search

Opens the Search and Replace dialog box.

Verify Area

Command

Description

Verify

Tests the consistency of changes made to the solution since the most recent execution of the Verify or Verify all action. It does not impact the creation of the binary. For example, it does not apply to Online Change edits.

Clean

Deletes all binary files created by previous Verify or Verify all actions. After executing the Clean action, execute a Verify action to recheck your entire project. The Verify action can take a long time to execute.

Verify all

Executes in sequence the Clean action followed by the Verify action.

Build HMI Area

Command

Description

Build

Incremental build that creates binaries for non-IEC (hardware configurations, ATV HMI, AssetLink data, etc.) and HMI. Execute the Build action after you have made a modification to HMI symbols (for example, a symbol integrated into a CAT) to be able to perform a drag & drop onto a Canvas.

Clean

Deletes binaries, containing HMI symbols and hardware configurations, that were generated by previous Build or Rebuild actions. After executing a Clean action, you need to execute a Build action to again generate the binaries. The Build action can take a long time to execute.

Rebuild

Cleans the build binary files; then builds the HMI and hardware configurations defined for the entire solution.

NOTE: Windows Generated Detected Errors When Building a Solution:

  • If a detected error indicates the file path is too long, use a shorter file name and specify a location for the project that is higher in the file hierarchy. Refer to step 3 in the topic Create New Solution.

  • If the Message Log displays detected errors indicating the build includes files from an untrusted source, you need to perform the following steps for each file referenced in the log:

    1. Right click on the file in Windows Explorer and select Properties.

    2. At the bottom of the General tab, in the Security area, click Unblock.

    3. Click OK.

Devices Area

Command

Description

Import EDS

Imports EDS (electronic data sheet) files, used to generate device DTMs. The DTMs can then be associated to an EtherNet/IP compatible device in the configuration. EDS files are usually furnished by the device provider.

To import EDS files:

  1. In EcoStruxure Automation Expert, on the far-right of the Home menu ribbon, click the Import EDS button.
    Result: The EDS Addition wizard opens.

  2. Click Next.

  3. Either

    • select Add File(s) > Browse, then select the file and click Open,
      or

    • select Add all the EDS from the Directory and select the directory where the files are located.

  4. Click Next > Next > Finish.
