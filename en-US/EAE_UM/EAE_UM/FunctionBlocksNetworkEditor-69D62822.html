Function Blocks Network Editor

Depending on the access method used, one of the following function block editors is displayed:

  • The Application editor

  • The Device Editor

  • The Resource Editor

Double-click on an object (device, resource, function block, and so on) to display the next lower level and open the view of the selected object, if possible:

  • A double-click on a device opens the Device Editor.

  • A double-click on a resource within the device editor opens the Resource Editor.

  • Within the Resource Editor, a double click of a  function block instance opens that function block.

Depending on the project and the kind of function block, it may be possible to click into deeper levels of the function block.

Double-click on the editor background to display the next higher level (e.g. a double-click on the background within the Resource Editor opens the Device Editor and shows the resources created within this device).

The view is zoom-able if you scroll with the mouse-wheel within the editor, when the focus is on the editor. To pan the view, press the mouse-wheel and drag the view in the desired direction (or use the scroll bars).

Pressing the Shift-key activates a magnifier (see Editor Shortcuts).

Function Blocks Network Editor Toolbar

The toolbar includes the following tools:

Icon

Description

Search fields

The editor contains two search fields - in the form of drop down fields - on the upper border of the editor. These let you quickly search for selected items within an editor

Use the left drop down to select an element type (e.g. function block type or a Frame). The right drop down field gets updated with a list of each instance of this type. In this way you also gain an overview how many instances of the same type are implemented in the device, resource or application.

Navigator

The button next to the right drop down field activates the navigator. It is available if the current view does not show the entire content of the resource (because of zooming or an accordingly large project). For large projects it is a useful feature, because it shows the entire network and with a red frame the part which is currently viewed. The frame can be shifted to the desired position to show another part of the network.

Function Blocks Network Editor Actions

Depending on the open view (Application editor, Resource editor, Device editor) a right-click on the background of the editor opens different context menus. The menus are described in the respective editor descriptions.

NOTE: Depending on which editor and which object has been selected to open the context menu, some entries may be grayed out to show that they are not available, or are not displayed for some objects.

The only exception is the Application editor. The context menu of this editor contains an additional Mapping entry, which is described separately at Application editor.

A right-click on a function block within the Application editor, Device editor, or Resource editor opens a context menu which may contain the following entries:

Command

Description

Cut

Cuts the selected object(s) and copies it to clipboard.

Copy

Copies the selected object(s) to clipboard.

Delete

Deletes the selected object(s).

Hide ports (Ctrl+H)

Hides unused ports and hides the added watch points of the selected function block or blocks.

Comment

Opens a dialog to insert comments that are shown as tooltip for that object

Attributes

Opens the Attribute Editor of the selected function block or blocks. See also Attribute.

Add watch

Adds watch points on the selected function block or blocks for observing the events and data values during runtime. See Watch.

Remove watch

Removes the watch points of the selected function block or blocks. See Watch

Go to definition

Opens the definition of the selected function block in a separate tab.

Show in Solution Explorer

Shows the selected function block in the Solution Explorer by highlighting it in blue.

Find all references

Shows the references of the selected function block within the Search Results pad.

Focus on selection

Sets the focus on the selected function block or blocks by masking out the other function blocks.

Create Composite

This entry is used to create a new Composite function block that already contains the selected function blocks in the given arrangement. In addition, the function block is automatically inserted into the current system-view (e.g. Application or Device view) and thus replaces the selected function blocks. It is possible to Create Composite with all application CAT types implemented as Composite or SubApp. Refer to the Composite Function Block topic to see which function blocks can and cannot be combined to Composite.

Create CAT

This entry is used to create a new CAT function block in the Hardware or in the application that already contains the selected function blocks in the given arrangement. In addition, the function block is automatically inserted into the current system-view (e.g. Application or Device view) and thus replaces the selected function blocks. It is possible to Create CAT with some application CAT types implemented as Composite or SubApp.

Create SubApp

This entry is used to create a new SubApp function block that already contains the selected function blocks in the given arrangement. In addition, the function block is automatically inserted into the current system-view (e.g. Application) and thus replaces the selected function blocks. It is possible to Create SubApp with some application CAT types implemented as Composite or SubApp. Refer to the topic Sub-Application Function Block Editor to see which function blocks can and cannot be combined to SubApp.
