Step Timeout


In order to limit the time where a particular step is active (and avoid locked active state), the persistent tab of the procedural CAT shows all created steps, allowing for each of them to set a time duration.




Enable /disable time and time duration (measured in seconds) are modifiable in the persistence page of a procedural cat type for each step, and in the persistence page of an application/resource for each step of each instance of procedural cat.


Parameter

Value

Description

EnWaitTimer

TRUE / FALSE

Enable (TRUE) or disable (FALSE) the timer.

StepWaitTime

Seconds

When the EnWaitTimer is TRUE, the step will be active for a Maximum of x seconds. Even if the transitions conditions are not valid the graph will move to the next step.

When EnWaitTimer is FALSE, this parameter is ignored and no timer is used.

NOTE: As the Time Out is introduced as part of the Offline parametrization, if the user already deployed the Procedural CAT in the device and wants to change or add a new step time out, he needs to Clean Persistence Data and re-deploy again for the changes to be effective.

What is the next step after a timeout ? It is always the leftmost transition target.


Simple use case: a step followed by some transitions. The timeout destination is the step that follows the leftmost transition.


Special case: the last steps in the parallel branch: only the rightmost branch handles a timeout.
