User Management

To create and manage user accounts, select View > User Management.

At least one user account is required to configure and issue commands to devices.

Out-of-the-Box User Name and Password Combinations

For devices that support cybersecurity, new (out-of-the-box) devices and devices restored to factory (default) settings can be accessed only with a special user account provided by the device manufacturer.

For the M251d, M262d and M580d controllers, the default credentials are:

  • User name: installer

  • Password: Inst@ller1

This special user account must first be created and used in EcoStruxure Automation Expert to be able to access these devices. Thereafter, a different set of user profiles can be deployed to these devices to subsequently access them with different credentials

Managing User Profiles

Create and modify user profiles according to the requirements for user name and password.

NOTE: Any change to existing user credentials or removal of user accounts will not affect existing sessions that were opened using those credentials.. These sessions - for example, an active HMI to controller session - continue to be active thereby avoiding a break in the connection.
To apply new user credentials and the security they provide, first stop active sessions based on the credentials to be changed, then re-initiate the session using the new credentials.

Users: The user name can be from 4 to 64 characters long, and cannot:

  • Start or end with a space

  • Contain the characters “ [ ] ; : | = + * ? < > / \ ,

  • End with a period “.”

To create an account, double-click <new user>, input a valid user name, and validate with Enter.


Password/Change Password: Password length can range from 8 to 64 characters and needs to include one uppercase, one lowercase and one symbol character.

Valid symbols include “ ~ ! @ # $ % ^ * ( ) _ - + = { } [ ] | \ : ; ‘ . ? /


HMI Options (expand the section if not displayed):

  • Change Canvas: Allows the user to navigate out of the start canvas (if selected). Else, canvas topology buttons are hidden.

  • Open Faceplate: Allows to open faceplates (if checked. Else faceplate cannot be opened, but functions are still executed.

  • Set Values: In a range of 0-15, matching levels of accessibility defined in the libraries objects:

    • 0: the lowest level: no authorization.

    • n: only objects set to a level inferior or equal to n are accessible.

    • 15: the highest level with full authorization.

Refer to Security Level.

NOTE: In the HMI, the Login button is positioned in the upper–right part of the canvas. Refer to the topic Start Canvas.

Security Level in the HMI

You can only interact with HMI elements having a security level inferior or equal to your own. The security level of a user is set in the field Set Values of his user account.

A recommended best practice when creating libraries objects is to determine and set their appropriate security levels, based on an analysis of the potential impact they can have on the system operation.

The security level of a button or control element of the HMI is set in the application as follows.


For HMI symbols: set the SecurityLevel in the properties of the HMI symbols.

Default value is 15 (accessible to all users)

Refer to the topic Select Generic Symbol.


For all other HMI action buttons (such as Drawn Button, Two State Button, etc.): assign the SecurityLevel in the source code (C# program)

There are two ways to proceed:

  • a single SecurityLevel value

  • multiple SecurityLevel values

Assigning a Single Security Level to One or More HMI Controls

For CAT containing:

  • only one relevant button or action

  • or several control elements all requiring the same security level

Use the IsSetValuesSecure() method in a conditional statement to define the object behavior if the user has the appropriate security level to interact with the object (return value = TRUE), or not (FALSE)  

Double-click on the control to open its associated sourcecode in the editor.

Example:

This script illustrate the use of IsSetValuesSecure() method in a conditional statement, assuming the expected security level for this object is 10, as set in its properties. 

void DrawnButton1Click(object sender, EventArgs e){
{
if(IsSetValuesSecure())

    // FireEvent_CNF(); ...code
    System.Windows.Forms.MessageBox.Show("Button works, Level >= 10"); //for demonstration purposes
else
System.Windows.Forms.MessageBox.Show("Not authorized, Level < 10"); //for demonstration purposes
  }

Configuring User Access for Local HMI

If no user is configured, starting a local HMI in local test mode will trigger an error in the Message Log:

Security has been enabled. At least one user must be configured. Please go to User Management view to add user(s).
This happens because security is enabled by default in Settings > Simulation HMI , via the Security Active checkbox.
NOTE: This setting only applies when you Start Local HMI from Solution Explorer.

To prevent the error, you can either:

  • Add a user in User Management.

    NOTE: You can use this option for both Local HMI and simulated HMI operations.

    Or:

  • Disable security by unchecking the Security Active checkbox.

    NOTE: Use this option only for simulated HMI operations.

When using Start Local HMI from the Deploy and Diagnostic editor, the Force Anonymous Access property (found in the Properties tab) determines whether user authentication is needed:

  • True means that no user is required.

  • False means that a user must be configured to access the HMI application.

Deploy and Diagnostic Pad (Local Test)

During local testing in the Deploy and Diagnostic pad, users are not pre-configured. If security is enabled, clicking on the test function will trigger an error message. This message opens the User Management dialog, allowing you to add a user.

By following these configurations, you can ensure that the correct security settings are applied based on your deployment needs.

Assigning Different SecurityLevel Values to Multiple HMI Controls

For CATs containing several relevant buttons or actions which require a different security level.

Use the following method for each individual object: 

IsSetValuesSecure()
bool isOk = false;
isOk = ((NxtControl.Security.ISecuritySetValues) this).IsSetValuesSecure(level);

Where:

  • Each object is designated by "this"

  • ObjectSecurityLevel is the security level required for this object

  • ISecuritySetValues returns true when the current user has at least the same security level the value ObjectSecurityLevel

Example:

This script illustrates the control of security levels for two objects of a CAT (a Drawn Button assigned to level 8 and a Two State Button assigned to level 5). 

void DrawnButton1Click(object sender, EventArgs e)
 {
bool isOk = false;
isOk = ((NxtControl.Security.ISecuritySetValues) this).IsSetValuesSecure(8);
if(isOk)
 {
// code
System.Windows.Forms.MessageBox.Show("Level >= 8"); 
//for demonstration purposes
 }
else
 {
System.Windows.Forms.MessageBox.Show("Level < 8"); 
//for demonstration purposes
 }  
    // remaining code      
 }
void TwoStateButton1CheckedChanged(object sender, EventArgs e)
{
bool isOk = false;
  isOk = ((NxtControl.Security.ISecuritySetValues) this).IsSetValuesSecure(5);
  if(isOk)
{
// code
System.Windows.Forms.MessageBox.Show("Level >= 5"); 
//for demonstration purposes
}
else
{
System.Windows.Forms.MessageBox.Show("Level < 5"); 
//for demonstration purposes
 }
}

Example

This script sets the enabled/disabled property of controls according to the security level of the user. In order to update these properties when the user changes, eventhandler sDefaultUserChanged has to be activated within the properties of sDefault at Events at item Misc > UserChanged. 

public override void EndInit()
 {
 base.EndInit();
SetSecurity();
}

void SetSecurity()
{
drawnButton1.Enabled = ((NxtControl.Security.ISecuritySetValues) this).IsSetValuesSecure(8);
twoStateButton1.Enabled = ((NxtControl.Security.ISecuritySetValues) this).IsSetValuesSecure(5);        
}

void sDefaultUserChanged(object sender, EventArgs e)
 {
  SetSecurity();
}

Two other methods are available: IsSetValuesSecure() and IsOpenCanvasSecure().

Refer to the topic Settings for information about using code templates.

NOTE: Security levels set in CAT Types through their properties (inside libraries) only apply to instances at the time they are created inside a canvas:

  • subsequent modifications of the Type do not impact existing instances

  • existing instances retain their properties unchanged

To reapply a Type to a canvas, the existing instance must be deleted and recreated from the library. Properties of existing instances can be modified from the canvas to align with the value set at Type level.
