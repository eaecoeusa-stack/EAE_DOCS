2. Configure Connection to the AIS Server

To get data from AVEVA Engineering using AVEVA Integration Services (AIS) server, you need to:

  • Configure the AVEVA Client integration.

  • Configure the AVEVA project for use by the EcoStruxure Automation Expert Solution.

Configure the AVEVA Client integration

Step

Actions

1

  1. From AVEVA Product hub site, install the AVEVA Integration Client into the PC where EcoStruxure Automation Expert is installed

  2. Launch AVEVA Client integration

  3. In the Windows search field (located in the lower left of the PC’s task bar), type ‘Integration client’.

    Windows shows an icon for accessing the integration client application.

2

Click the icon to launch the integration client application.

3

  1. Click Set Service Location in the Service Settings section

  2. Enter the IP address of the machine on which AVEVA Integration Service is installed.

  3. Click Test Service to test the connection.

  4. Click Save to enable the Test Service.

  5. Click Save to save the server.

4

Optionally, verify that you have the required view and able to extract data as follows:

  1. Click Test Service in the Service Settings section.

  2. Click Get Available Products in the Available Products section.

  3. Select AVEVA Engineering from the select module list.

  4. Select the required project.

  5. If authentication is enabled for the project, enter the project username and click Validate User.

  6. Click Get tables .

    NOTE: You may need to wait a couple of minutes for a response.

  7. Select SEEngineeringView|DbView from the list next to the Get tables button.

  8. Click Get Data.

    NOTE: You may need to wait a couple of minutes to get a response.

Configure the AVEVA Project

Step

Action

1

Open EcoStruxure Automation Expert, then create a new solution or select an existing solution.

2

Right-click the Solution name in the explorer to open the context menu.

3

Select Configuration > Engineering Hub Server Configuration.

The Engineering Hub Server Configuration window opens.

4

Click AVEVA Project and select the appropriate AVEVA project from the list for extraction.

5

If authentication is enabled for the project, enter the Project User name that will be recognized by the Active Directory configuration.

6

Click Save to save the configuration.

Enter the Project User Name for Windows Active Directory Authentication

To find the Project User name to use for Windows Active Directory Authentication, follow these steps:

Step

Action

1

Open the AVEVA Integration Settings Editor.

2

In the Integration Settings Editor, select the desired version of AVEVA Engineering (AVEVA Engineering version...) and from the list of projects, select the AVEVA project.

If the Authentication column is set to User, this project applies Windows Active Directory authentication.

3

In AVEVA Engineering, select AD Configuration in the left menu. A list of user names shows.

4

Select the user who you want to edit the AVEVA project, then click Modify... to open the Active Directory User Modification dialog.

The Authenticated Users section displays a list of authorized names for the user.

5

From the Name column, copy the role name of the selected user.

6

In the Engineering Hub Server Configuration dialog, in the Project User field, enter the role name for the selected user (which you copied in step 5).

Result: If the user is logged into Windows Active Directory, AVEVA will authenticate the user, who can proceed to configure the AVEVA project.
