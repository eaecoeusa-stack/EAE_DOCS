Status

The Status column provides icons that display the following information for each device:

  • the existence of forced variables.

  • a comparison between the deployed application and the boot project.

  • a comparison between the application in EcoStruxure Automation Expert - Buildtime and the deployed application.

  • the status of watch variables.

Forced Variables

EcoStruxure Automation Expert - Buildtime notifies you of the presence or absence of forced variables in the deployed application by means of the following icons in the Status column:

Icon

Indicates...

One or more forced variables are active in the deployed application.

No forced variables are active in the deployed application.

Forced variable status is unknown. Log in to update the status.

EcoStruxure Automation Expert - Buildtime detects situations where potential risks may arise while using forced variables and triggers dialog boxes to characterize the risks and ask for user acknowledgement. For more information refer to the topic Forced Variables Impact on Action Commands.

Boot Project

Inside a device, a boot project can be the same as or different from the loaded application. For example, if the loaded application has been edited and compiled using the Compile for Online change command, the boot project and the loaded application will be different.

EcoStruxure Automation Expert - Buildtime notifies you of the comparative state of the boot project and the loaded application by means of the following icons in the Status column:

Icon

Indicates...

The boot project is up-to-date with the deployed application.

The boot project is different from the deployed application.

The boot project does not exist.

Log in to update the status.

You need to consider this boot project information because - after a firmware restart or a device power cycle - the device will execute the boot project program which is potentially different from the program previously executed, thereby directly impacting operations. For more information refer to the topic Boot Project Impact on Action Commands.

Deployed Application

EcoStruxure Automation Expert - Buildtime notifies you of the status of the deployed application by means of the following icons in the Status column:

Icon

Indicates...

The control logic deployed to the device is the same as the application control logic in EcoStruxure Automation Expert - Buildtime.

The control logic deployed to the device is different from the control logic in EcoStruxure Automation Expert - Buildtime. Perform a Deploy or Online Change, or undo the latest changes.

EcoStruxure Automation Expert - Buildtime does not know what control logic is deployed to the device. Login to the device to get a status update.

Watch Status

EcoStruxure Automation Expert - Buildtime notifies you of the status of watch items in the application by means of the following icons in the Status column:

Icon

Indicates...

The device is online and current configuration in EcoStruxure Automation Expert - Buildtime matches the deployed application with watched variables.

The device is online and current configuration in EcoStruxure Automation Expert - Buildtime does not match the deployed application with watched variables.

Watch subscription from buildtime to dPAC did not succeed.

The device is offline and the status of watched variables is unknown.

Login Status

EcoStruxure Automation Expert - Buildtime notifies you of the status of the login status of the device by means of the following icons in the Status column:

Icon

Indicates...

The device is online.

The device is logging in.

The device didn’t succeed to login.

The device is offline.

Device State

Icon

Indicates...

The device is Running.

The device is Stopped.
