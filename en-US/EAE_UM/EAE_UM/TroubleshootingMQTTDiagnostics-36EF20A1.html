Troubleshooting MQTT Diagnostics

MQTT Detected Error Messages

When deploying a configuration to the HA Soft dPAC, if the MQTT broker is properly configured in the Physical Devices editor and the MQTT broker service is reachable from the HA controller, then deployment does not generate MQTT-related detected error messages. However, in some cases the following messages can be displayed:

Message

Cause

Actions

MQTT Broker has not been configured in the physical topology of your system. Diagnostic capabilities will be limited without a valid MQTT broker installed and configured on your system.

An MQTT broker has not been defined for a server or workstation in the Physical Devices editor.

You can:

  • Stop deployment and:

    • Activate the MQTT service for a server or workstation (Manage Services > MQTT broker > Activate service)

    • Configure the MQTT service for the server or workstation in the Properties tab (Services > MQTT Broker).

  • Continue with deployment without the MQTT service.

MQTT Broker connection could not be established. Diagnostic capabilities could be limited.

An MQTT broker has been defined for a server or workstation but is not reachable. For example:

  • The MQTT broker service may not be enabled or running.

  • The IP address of the MQTT broker may be unreachable, incorrect or set to the default of 0.0.0.0.

  • The MQTT broker is configured with an incorrect port number.

You can:

  • Expand the detail section of the message for more information.

  • Stop deployment and resolve the source of the detected error.

  • Continue with deployment without the MQTT service.
