PROFINET Module and Submodule Diagnostics

Use the PN_GET_DIAG function block to provide a summary of the status of modules and submodules located on a specified slot. For any specified slot, the PN_GET_DIAG function block can return the following information:

  • Module is OK

  • Module is not present

  • Wrong module is present

  • Correct module is present, but one or more submodules are not present

  • Correct module and a wrong submodule are present

  • Module is in an undefined state

  • Diagnostics are not available

To configure the PN_GET_DIAG function block:

  1. In the Function Blocks Network editor, add the PN_GET_DIAG function block from the library Standard.IoProfinet > Services.

  2. Configure the data inputs by specifying the target slot to examine:

    • REQ links to the event that triggers the diagnostic operation.

    • BUS_ID identifies the PROFINET IO-Controller - bus master - (which you added when creating your PROFINET network) of the type Standard.IoProfinet.PN_IOController that is that target of the diagnostic operation.

    • STATION_NAME is the Station Name of the IO-Device (which you previously added to the PROFINET network) that is the target of the diagnostic operation.

    • SLOT_NO is the module slot number of that is the target of the diagnostic operation.

  3. The operation executes when:

    • REQ = 1, and

    • QI = TRUE

    After execution, the data outputs display the result of the write operation.

  4. After the operation concludes, check the STATUS data output text value, which can display:

    • OK: the diagnostic operation was successful.

    • Error: the specified channel, device or slot is invalid.

    If the STATUS response is Error, check both the Device and Cyclic Logs and the detected error code lists.

    NOTE: The Standard.IoProfinet library help for the PN_GET_DIAG function block includes a link to PROFINET detected error codes.

  5. Check the DIAG_STATE output text value. This is a DWORD data type containing:

    • a high word, which indicates the module diagnostic state, and

    • a low word, which indicates the submodule diagnostic state

    NOTE: The DIAG_STATE value is meaningful only if the STATUS output value is 0.

    Module state values (high word) can include:

    Value

    Description

    0x0000

    No Module

    0x0001

    Wrong Module

    0x0002

    Proper Module, but submodule difference exists

    0xFEFE

    Diagnosis not available

    0xFFFF

    Module OK

    Others

    Undefined state
