For the MQTT client functionality, the following function blocks are provided in the Runtime.Base library:

  • MQTT_CONNECTION

  • MQTT_PUBLISH

  • MQTT_SUBSCRIBE

Use these function blocks to establish secure or unsecure communication with the broker, and to publish and receive data from the broker.

NOTE: For secure communications, you need to use either:

  • a username/password combination, or

  • a client certificate

Performance Cost

The performance and resource cost of QoS = 1 - and especially QoS = 2 - is higher in comparison to QoS = 0:

  • Use QoS = 0 for data that is regularly updated.

  • Use QoS = 2 only for critical MQTT communications.

Compatibility of MQTT Client with MQTT Broker Versions

The MQTT client function blocks have been verified with Eclipse Mosquitto, v2.0.15.

NOTE: Mosquitto Broker v2.0.16 and later may exhibit loss of data during normal operation. Refer to Issue 3056 on the Mosquitto GitHub site for details.

The following Mosquitto broker configuration parameters and values are installed on Windows for Mosquitto by default:

  • max_queued_messages = 1000

  • max_inflight_messages = 20

These settings are usually sufficient for most MQTT communication systems and, if necessary, can be modified as follows:

  • max_queued_messages = 2 x (Number of topics published to broker per second)

  • max_inflight_messages = 1.5 x (Number of topics published to broker per second)

Values greater than 30000 for either of these parameters are not recommended.
