Asset Link Bulk Creation for the Hierarchy View

In large projects, you can speed up the development of an AVEVA hierarchy view and the assignment of instances therein by representing the hierarchy in Excel™.

In addition, this allows you to reuse a hierarchy view across several Solutions.

Prerequisites

  1. Configure and connect the Galaxy as described in 1. Configuring Galaxy Details in AVEVA SP Configuration; this task is not shown in the schema. The optional template prefix can be used in the step 3 hereunder.

  2. Start the Hierarchy view ( icon) as described in Creating an AVEVA Model Plant View; the instances whose Deployment state is Deployed show in grey in the hierarchy view.

  3. Map the template and attributes ( editor) in Online mode as described in 3. ASP Template Mapping of Library CATs or project CATs and 5. Assigning an ASP Template to CAT instances (the CAT instances must be assigned to an application).

Defining the Excel headers in EcoStruxure Automation Expert and Export

These operations are optional: you may also start from an empty Hierarchy view.

HEADERS FOR PLANT MODEL

  1. At the view hierarchy view, right-click and select Define Plant Model; this opens a water fall editor displaying opposite the default generic names and the new generic names on ten levels.

  2. Enter a new name describing the first level, for instance “Plant” (without quotes); special characters and spaces are not allowed.

  3. Repeat the step 2, for as many levels as desired, for example “Area”, ”Unit”, “Process”, etc

  4. Click Save; the new generic names will be the table headers in Excel. If the button is greyed out, check the names validity.

EXPORT

  1. In the Hierarchy view pad, click the icon.

  2. Name the destination Excel file (extension .xlsx) after the Galaxy name and click OK

    .

  3. If it is relevant, open the file and check it out.

The sheet name is identical to the view name, a table with headers has been created as follows:

A B ... J K L M
Header 1 Header 2 Header 10 ASP Area Template Instance Name ASP Deploy Status
xxx xxx xxx $...... CATxxx Deployed or NotDeployed

The K column receives the template name of Area object instances of the Assets as they are now in the AVEVA System Platform (Properties list); there is one row by asset.

The L column receives the names of the assets as they are now in the AVEVA System Platform. Each row describes the hierarchy of the asset using a bread crumbs trail across the A ... J columns.

The M column receives the Deploy statuses as they are now in the AVEVA System Platform. The instances that are deployed appear greyed in the Hierarchy view. This column is in Read-only mode.

NOTE:

An instance will be directly attached to the root if, in its row, only the Instance Name column is filled.

Developing a hierarchy in an Excel file

You can add asset instances, change area templates and create new areas. This is prepared in an Excel file as follows:

  1. Open the exported Excel file or create a table in a new Excel file; in the latter case, enter generic names describing roughly the levels (plant, unit, area, ...) in the headers row; anyway, these names will not show in the tree-view.

  2. Enter the names of plant, unit, area, ... as a bread crumbs trail across the successive columns A ... J; they will represent the hierarchy of nodes in the tree-view; do not use invalid characters such as *, &, $ or _; duplicated names are allowed as far as they are not in the same column and share the same ancestors.

  3. In existing rows whose Deploy status is NotDeployed (M column), you can change the area template name (K column); in the new rows you must assign an area template (unless the instance is directly under the root); in both cases, pick a template name in an area library ($aSEAreaRoot, $aSEArea, ...).

  4. In the L column enter CAT instance names existing in the Solution that must be assigned to the view.

  5. Close the file.

NOTE:

  • An instance can be directly attached to the root; for this purpose, in its row, fill only the Instance Name column.

  • The order of the nodes will reflect the order of the rows.

  • Comments are allowed after the M column.

Importing the data to EcoStruxure Automation Expert

  1. Make sure that the Excel file to import is closed.

  2. In the Hierarchy view pad, click the icon; a tab with a clickable tree shows.

  3. Click Browse, select the Excel file to import and click Open; the updated preview shows an outline with check boxes; anomalies are listed out in the Output pad, invalid names show in italics and Deployed items show in italics and greyed.

  4. Drill down and check areas to be created. If at least one selection name is valid, the Create Areas button gets enabled.

  5. Click Create Areas; the Confirm creation of Area objects in Galaxy window pops up.

  6. If you deny the right to change the pre-existing area templates, untick the Overwrite area templates check box.

  7. Click Yes; this adds the ticked areas to the hierarchy view.

  8. To check the area templates, compile, open the IDE as described in 10. Connecting to a Galaxy and Launching the ASP IDE and check out the Properties of the changed and new areas.

  9. Click Import; this populates the hierarchy view with newly added CAT instances.
    NOTE: Sub-CAT with the Aggregation value of the Relation Type field
    do NOT appear (regardless of the File > Settings > General > Retain CAT hierarchy while assigning to Hierarchy View value).

  10. To check the new instances in the IDE, open the Model — Tagname tab; the name between brackets after the asset names depends on the server configuration (refer to 1. Configuring Galaxy Details in AVEVA SP Configuration):

NOTE: As new CAT instances are assigned to the view, a ● mark is added before their names in the Solution explorer as long as the view is active.
