Troubleshooting a PROFINET Configuration

Unable to load PROFINET IO Configuration

If your solution includes a PROFINET IO-Controller that was created with an older version of the Standard.IoProfinet library, you may see the following message when you right-click on the IO-Controller in the Hardware Configuration editor and select Configure:

"Unable to load PROFINET IO Configuration for "PN_IO-Controller".

If this occurs:

  1. When asked if you want to upgrade, click Yes. The upgrade will fail.

  2. In the Hardware Configuration editor, right-click on the IO-Controller and select Upgrade Hardware CATs.

  3. After the upgrade, click OK.

You can now proceed to open the PROFINET IO Configurator.

On Communication Loss IO-Device Inputs Reset to 0 Value

For EcoStruxure Automation Expert version 24.1 and later, if communication between a PROFINET IO-Controller and an IO-Device is lost, IO-Device inputs revert to 0.

For EcoStruxure Automation Expert versions earlier than 24.1, if communication is lost, IO-Device inputs maintain their current value.
