Data Type

Use the Data Type functionality to create your own custom data types. EcoStruxure Automation Expert automatically includes your custom data types in the lists of data types available to be assigned to variables and other objects throughout your solution.

Custom data types can be created as any of the following types:

  • Enumeration: consists of specific, pre-defined values.

  • Subrange: includes values within a value range.

  • Array: includes values within a defined array structure.

  • Structure: a collection of elements of different data types in a defined order.

Context Menu Commands

Use the Data Type node’s context menu to create a new data type (New Item...), create a New Folder for data type items, Rename or Remove items, Paste copied items, and Export and Import data type items.

Use a data type item’s context menu to Open, Rename, Export, Copy, and Remove data type items, and to Find all references of the item in your solution.

Creating a New Data Type

To create a data type:

  1. Right click on the Solution > Data Type node and select New Item...

  2. In the File Wizard:

    • Select a Type.

    • Input (or accept the default) Name.

    • Click Finish.

      A new tab is created for the new data type.

  3. In the new data type tab, complete the definition of the new data type:

    • For Enumeration types: type in the list of valid values.

    • For Subrange types: enter the numeric range of values.

      NOTE: For both Enumeration and Subrange data types, the values can be one of the following types: INT, UINT, SINT, USINT, DINT, UDINT, LINT, ULINT, BYTE, WORD, DWORD, LWORD

    • For Array types: enter the elements that define the array structure, and the element data type.

      NOTE: Both single-dimensional and multi-dimensional array types are supported, limited only by function block memory.

    • For Structure types: define each element of the structure by specifying its name and type.

    NOTE: When you have properly defined a data type, the statement and data type text appears in red.

  4. Save your edits.

Use of Custom Data Type

Once a custom data type has been created, this type can be assign to a variable of a function block. This action is done in the:

  • Generic Interface Editor dialog: To set up the type of the variable ports of a function block (like COMPARE, PERSISTENCE, and so on).

  • Interface editor

    : To define the type of the variable of a custom function block (CAT or composite).

For both of this case, the custom data type will be added into the list of type available. Select it to assign it to the variable.

When assigned, the variable will then follow the structure set up during the creation of the custom data type.

There are some limitations on the use of custom data type made of structure of structure:

  • The maximum nesting is 2 levels of custom data type.

  • The maximum size of custom data type is 10.000 INT.
