Documentation Editor

The function blocks created in a project as well as the system, applications, resources, and so on, can be supplemented with custom documentation, to make the properties and details of the element evident for other users. Documentation can be adapted within the project to explain to a user, for example, the appropriate element within a library and how to use it (via right-click on the  <project node> > Documentation > Generate/show).

To display the Documentation editor:

  1. Double-click a node in the Solution Explorer (System, CAT > Hardware, CAT > Application, SubApp, Composite, Basic, Service) to open an editor dialog.

  2. Click the Documentation icon from the vertical toolbar on the left.

The Documentation editor presents three areas:

  1. The Author area

  2. The Abstract area

  3. The Description area

In the Author area, the name of the author of the project or a company name and an e-mail address can be entered. This can also be done for each function block individually, if there are different authors in one project. The information is provided in the documentation on the respective page.

The Abstract and Description areas allow you to create documentation. The toolbar contains icons for creating the documentation:

Tool Icon

Description

 Bold

Display the selected text as Bold. If the text is already formatted as bold, the icon is highlighted in blue. The formatting of selected text can be canceled by pressing this button again, or by pressing the Delete Formatting button .

NOTE: The keyboard shortcut Ctrl-B can also be used.

 Italic

Display the selected text as Italics. If the text is already formatted as italic, the icon is highlighted in blue. The formatting of selected text can be canceled by pressing this button again or by pressing the Delete Formatting button .

NOTE: The keyboard shortcut Ctrl-I can also be used.

 Underline

Display the selected text as Underlined. If the text is already formatted as underlined, the icon is highlighted in blue. The formatting of a selected text can be canceled by pressing this button again or by pressing the Delete Formatting button .

NOTE: The keyboard shortcut Ctrl-U can also be used.

 Ordered List

Create a numbered list.

 Unordered List

Create a bulleted list.

 Keyword

The selected text is formatted as a keyword. The text is then highlighted in blue. The button is only available if a text or character is selected. The formatting of selected text can be canceled by pressing this button again or by pressing the Delete Formatting button .

 Code

The selected text is formatted as a code snippet. The button is only available if text is selected. To remove the formatting of a selected text, use the Delete Formatting button when the corresponding text is selected.

 Delete Formatting

The formatting of the selected formatted text is removed.

 Image

Insert an image. A dialog opens to select the appropriate image. If the dialog is still empty or the corresponding image is missing, you can open the file explorer with the button  at Browse: at the lower left edge of the dialog to select the desired image. The image is then automatically inserted in the dialog in General tab. The image in the solution is copied to the folder .../IEC61499 /Images.

When editing the documentation of a CAT, the additional tab Specific for CAT is also available in the dialog. In this tab, you can insert images that were created as described below in Creating Images of HMI Symbols or Faceplates. These images are also located in the above-mentioned folder, but a separate subfolder is automatically created for each CAT.

 Table

Insert a table. The Table Properties dialog opens, where you can set the properties of the table.

 Hyperlink

Insert a hyperlink. The Create a Link dialog opens, where you can define the type of the link and its properties. In the URL type list (upper left), select the type of link: http, https, file and xref. The availability of the other fields in the dialog is adapted according to the selection.

http: You can enter the corresponding Internet address directly into the field next to it, or select it from the drop-down list if there do already exist some in the project. Alternatively, you can use the button  at the right side to open the File Explorer and navigate to the corresponding file. In the text field, you can specify the alternative text, which will be displayed in the finished documentation instead of the address.

https: Same as http, but for https addresses.

file: You can also create a link to a file (for example a text file), which is then opened by clicking on the link in the finished documentation. In this case, enter the desired title of the link in the text field at first, then enter the path of the file, or click on the button , in order to navigate to the desired file. In this case, the file is automatically copied to the folder .../IEC61499/Images and the link to this copy is created then.

xref: You can create a link to the documentation of another function block within the project. If you click on the link, you are then switched to the documentation of the linked block. Examples of valid links (assuming that the listed blocks actually exist in the project):

- Basic1 → creates a link to the documentation of the function block named Basic1

- CAT1.CAT1_HMI.sDefault → creates a link to the documentation of the HMI_Symbol sDefault of CAT1

- CAT3.CAT3_HMI.Faceplate1 → creates a link to the documentation of the Faceplate Faceplate1 of CAT3

NOTE: at xref you cannot define an alternative display text. The respective name of the linked element will be used as the link title. Add a proper additional link description, if necessary.
 

Warning, Caution, Important

Insert an information box into the documentation to highlight text within the documentation. The box gets a corresponding image, embedded on the left edge, depending on which of the categories was chosen.

Note, Tip

Insert a note or tip information box into the documentation.
