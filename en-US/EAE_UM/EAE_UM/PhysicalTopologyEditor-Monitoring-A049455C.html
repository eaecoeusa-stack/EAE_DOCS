Monitoring the Physical Architecture

The Start monitoring command, available on the toolbar of the Physical devices editor, is used for system diagnostic of the configured devices.

When you click Start monitoring, the status of the monitored device is displayed via icons:

  • a network status icon at the top left corner of the device.

  • a security status icon at the top right corner of the device.

NOTE: Status may take up to 5s to be updated.

Devices Health Status

During their usage within the project, devices can display the following network status:

Icon

Description

Status

Not available

The service is currently being started or is not available to establish a connection. 

Not reachable

The diagnostic service detected a connection error, for example, the device is not present or not reachable.

Device Different

The connected device is different from the one in the project.

Out of Order

The connected device is invalid or not operational.

Network error

For high availability systems, this can be caused by one of the following:

  • Cable to control network broken or missing.

  • Cable to dedicated high availability interlink broken or missing.

  • NIC non-operational.

Not fully functional

The device is not fully operational. 

No Icon

Good

The diagnostic service detected that the device is present and a connection can be established.

Security Status

During their usage within the project, devices can display the following security status:

  • Not Supporting Security

  • Security Not Set

  • Secured with Custom User mode or with Anonymous mode

  • Outdated Security with Custom User mode or Anonymous mode

  • Secured in Another Solution with Custom User mode or Anonymous mode

  • Not Available

Each of these status can transit accordingly to the following diagram:

  1. Set security in Custom User mode thanks to the Configure Security command.

  2. Set security in Anonymous mode thanks to the Configure Security command.

  3. Security of the device has been set in Custom User mode in another solution.

  4. Security of the device has been set in Anonymous mode in another solution.

  5. Reset the security in Custom User mode thanks to the Reset Security command.

  6. Reset the security without credentials thanks to the Reset Security command.

  7. Security of the device has been reset in Custom User mode in another solution.

  8. Security of the device has been reset in Anonymous mode in another solution.

  9. The certification of the device has expired or the user credentials have been modified or the Authentication Mode parameter switched to Anonymous.

  10. The certification of the device has expired or the Authentication Mode parameter switched to Custom Users.

  11. Set security in Anonymous mode thanks to the Configure Security command after the Authentication Mode parameter got switched to Anonymous.

  12. Set security in Custom User mode thanks to the Configure Security command after the Authentication Mode parameter got switched to Custom Users.

  13. The firmware has been downgraded and the feature is not supported anymore.

  14. The firmware has been upgraded and the feature is supported.
