[Compiler] Error Handling Solution

With the enhanced Error Handling solution, you can manage programmatically errors in ST standard functions without stopping the runtime, making available continuous operation and online changes.

The solution provides the ability to structure code in a way that enables applications to run uninterrupted, even when dysfunction occurs. Standard Functions that execute in parallel can handle errors without affecting performance.

The solution is conformed to the IEC 61131-3 standards, making it portable across different environments, including IEC 61499 muti-resource setups.



In the following example, the SQRT function is called with a negative value, which would result in an error. Instead of stopping the runtime, the ENO sucess parameter is set to FALSE, allowing the application to continue running and handle the error programmatically.
