Configure DCOM

Once both installation are done, DCOM needs to be set up in both server and client PC. This set up allows the Archestra IDE in the client PC to access the remote Galaxy repository.

This step is allowing EcoStruxure Automation Expert to map the CAT attributes and to synchronize them in the Galaxy environment.

To configure the DCOM:

  1. Make sure that:

    • The Windows user of both client and server PC have administrator rights.

    • Both client and server PC are logged in a Windows account with the same name and password (the ones created during the installation).

    If the client and server PC don’t have the same user and password:

    1. Press Windows.

    2. Search Change Network Account application.

    3. In the Change Network Account window opening, modify the User Name and Password so it matches in both client and server PC.

  2. In both client and server go to the user Properties:

    1. Press Windows + X.

    2. Click on Computer Management.

    3. Expand the folder Local Users and Groups.

    4. Click on Users.

    5. In the list of Users, right click on the Admin user and select Properties.

    6. In the Properties window opening, click on the tab Member Of.

  3. Once the ribbon Member Of opened in both PC, check that they both have the following groups:

    • aaAdministrators

    • aaConfigTools

    • aaInTouchRWUsers

    • aaInTouchUsers

    • aaPowerUsers

    • aaReplicationUsers

    • Administrators

    • ASBSolution

  4. Open in both client and server PC the hosts file in the following folder: C:\Windows\System32\drivers\etc.

  5. Add at the bottom of: both hosts files a line with the IP address and the hostname of the client and server

    • Client hosts file: The IP address of this PC and hostname shown in the Change Network Account window.

    • Server hosts file: The IP address of this PC and hostname shown in the Change Network Account window.

    In both case, it has to follow this template: 'IP address' + 'Space' + 'Hostname', e.g. 192.168.245.166 DESKTOP-EM4K5K8.

  6. Enable DCOM in both client and server PC by:

    1. Opening the Component Services application.

    2. Expand the folders Component Services and Computers.

    3. Right click on My Computer and select Properties.

    4. In the Properties window opening, go to the Default Properties tab.

    5. Select the check box Enable Distributed COM on this computer.

  7. Enable network access in both client and server PC by:

    1. Opening the Local Security Policy application.

    2. Expand the folder Local Policies and select Security Options.

    3. In the list of policies, find Network access. Let Everyone permissions apply to anonymous users and Enable its Security Setting.
