3. ASP Template Mapping of Library CATs or project CATs

To associate a CAT instance to a piece of equipment (asset), you have to map its CAT to an AVEVA System Platform (ASP) asset template.

The CAT to ASP Template mapping editor works in either online or offline modes. In either mode, you need to perform mapping for both templates and attributes. This topic describes:

  • Online Mode

    • Prerequisites

    • Template Mapping

    • Choices and Options in the ASP Template Mapping Editor

    • Attribute Mapping

  • Offline Mode

    • Prerequisites

    • Template Mapping

    • Attribute Mapping

    All these steps are similar for a composite CAT and a SubApp.

ONLINE MODE

Prerequisites – Online Mode

Before configuring a CAT – ASP template in online mode, configure the Galaxy you plan to use.

Template Mapping – Online Mode

Follow these steps:

  1. Double-click the desired CAT name (or right-click it and select Open) in the Solution Explorer pad; this opens the related CAT tab.

  2. Select the ASP Asset Mapping editor .

The ASP Asset Mapping editor displays information in the following columns:

  • CAT: Displays the possible hierarchical structure of the parent CAT and its SubCATs.

  • Relation Type: Defines the inheritance conditions from the parent CAT and its direct SubCATs. Values include:

    • Aggregation (default): If a SubCAT is set to Aggregation, all its own SubCATs are also set to Aggregation, and the ASP Template field is disabled for the SubCATs of any level. In other words, the settings for the CAT apply to all SubCATs.

    • Hierarchy: The settings for each Sub-CAT can be independently set and the ASP Template field is enabled for the SubCATs of the first level.

    If a SubCAT from the CAT defined here is added as a CAT instance into an AVEVA plant model view, see the Hierarchy View Pad topic, then:

    • The Relation Type in the sub-CAT can only be Hierarchy. This value cannot be modified unless all the CAT instances of this CAT are unassigned as described in Unassigning application instance(s) from an AVEVA hierarchy view.

    • A message is generated in the Output pad to inform the user about this restriction when trying to switch the Relation Type to Aggregation.

  • ASP Template: Presents the list of ASP templates from the configured Galaxy. Select the ASP template(s) to assign to the CAT.

    NOTE:

    • You can select several templates; all show highlighted in the drop-down and, in the box, separated by commas; the first listed template is the default template: you can specify any listed template as the default for this item by selecting SetAsDefault. Only one template can be set as the default template.

    • Click Clear Selection to clear the ASP template mapping. This option is not available for the SubCAT level ASP template mapping.

  • On the top level, you can derive a template on the fly from a base template. For this purpose, tick

    AutoMap; as a result, the ASP Template column is autofilled with <template prefix>_<CAT name> and a comment reads “This template does not exist...”; in the new Base Template column, select a template that is in the Solution; by default, the name $UserDefined is used.

    NOTE:

    • You can mix regular templates and an automapped template, though it is not a common practice.

    • After the ASP Template association is completed for a CAT in online mode, if the EcoStruxure Automation Expert Solution is configured with a different Galaxy, then the ASP Asset Mapping editor shows the rows highlighted in the red color, indicating that the mapped ASP Template name is not available in the current Galaxy.

    • If no template or attribute is listed in the ASP Template lists or they are not in sync with the list in the configured Galaxy, then in the Solution Explorer, right-click the project node and select Configurations > Sync AVEVA SP templates. This fetches templates from the AVEVA System Platform (normally it is done in the background).

After completing the mapping process, click Save or Save All in the quick access toolbar. The edited files are saved to the Solution folder.

Choices and Options in the ASP Template Mapping Editor – Online Mode

The selection of template(s) propagates to the second section, which displays all the Choices and Options along with their values configured in the Galaxy for the selected ASP templates.

Follow these steps to view and modify the Choices and Options for the ASP Template:

  1. Open the ASP Asset Mapping editor of the CAT.

  2. To also map the ASP Template for sub-CAT, change the relation from Aggregation to Hierarchy for the required SubCATs.

  3. Map the ASP Templates in the ASP Asset Mapping section.

  4. To view and modify the Choices and Options for the mapped ASP Templates, in ASP Template mapping section select the CAT/SubCAT: all the mapped ASP templates will be listed in the ASP Template column.

  5. Select a mapped ASP template in the list, to see its associated Choices and Options.

  6. Modify the choices and options for that ASP template, as needed.

  7. Select other ASP Templates from the list in the ASP Template column to view and modify their associated choices and options.

  8. Save your ASP Template mapping edits, including the Choices and Options details.

Attribute Mapping – Online Mode

Follow these steps:

  1. Double-click the CAT name in the Solution Explorer pad to open the CAT editor.

  2. Open the ASP Attribute Mapping editor .

Use this editor to map CAT attributes to attributes of the templates selected in the ASP Asset Mapping editor.

NOTE: If you did not select templates in the ASP Asset Mapping editor, no template attributes can be mapped to CAT attributes in the ASP Attribute Mapping editor.

The ASP Attribute Mapping editor displays information in the following columns:

  • Attribute: Displays the hierarchical structure of the parent CAT, its SubCATs, and the CAT/SubCAT attributes (which are selected in the Offline Parameterization and OPCUA Server editors). Expand the tree structure to display the child attributes.

  • ASP Template: This column presents a list of the ASP templates selected in the ASP Template column of the ASP Asset Mapping editor.

    NOTE:

    • If the default template was selected in the ASP Asset Mapping editor, that template is automatically pre-selected.

    • You can select a template from the list to use in place of the default template.

    • When you select a template, for which the Relation Type was set to Aggregation in the ASP Asset Mapping editor, all related attributes become available for selection in the ASP Attribute list.

    • You can map attributes at either the parent CAT level, or SubCAT levels. When mapping is performed at the parent CAT level, the selection is visible in the SubCAT level.

  • ASP Attribute: Use this column to map ASP attributes to CAT attributes by making selections from the list.

    • You can select one or more attributes for a single attribute.

    • Click Clear Selection to delete selections, and start over.

    • If the selected ASP Template

      includes attributes that match the CAT attributes, those attributes are automatically assigned (unless template mapping has been previously performed).
  • ASP IO Source: This column indicates the attribute source. Values can include: Input, Output, or InputOutput.

    • When you select an ASP Attribute, its IO reference value is automatically displayed in the ASP IO Source column for that attribute. If more than one attribute is selected in the ASP Attribute column, the Input value is set to ASP IO Source.

    • If the selected ASP Attribute has the IsOutput differ setting selected in the configured Galaxy, the ASP IO Source field presents a selection list containing three possible values: Input, Output, or InputOutput.

  • State Alarm: This column is visible only when AutoMap is selected in the CAT ASP Asset Mapping editor; a check box is available for the selected attributes of boolean type. After the Sync AutoMapped ASP Templates process, the ASP attributes will feature the State alarm option inside the AVEVA System Platform.

    NOTE: The State Alarm check box is available only to attributes of Sub-CATs whose Relation Type is set to Aggregation in the ASP Asset Mapping editor.
NOTE:

  • If you perform attribute mapping for a CAT while connected to an initial Galaxy, then connect to a different Galaxy, the ASP Attribute Mapping editor highlights the affected rows and displays a message indicating the selected templates do not exist in the currently configured Galaxy.

  • User extended attributes is not supported.

After completing the mapping, click Save or Save All in the quick access ribbon. The edited files are saved to the Solution folder.

OFFLINE MODE

Specifics

If EcoStruxure Automation Expert is offline, it is still possible to perform CAT - ASP template mapping. This section describes the tasks that uniquely apply to offline mapping. In every other respect, apply the descriptions for online template and attribute mapping.

In offline mode:

  • The Galaxy configuration is not present.

  • Although a connection to the Galaxy is not possible, the application will indicate it is in online mode.

  • Apply these rules to the creation of template names:

    • Valid characters include alphanumeric and special characters $, #, and _

    • The second character in a template name cannot be $

Template Mapping – Offline Mode

Follow these steps:

  1. Double-click the CAT name in the Solution Explorer pad to open the CAT editor.

  2. Open the ASP Asset Mapping editor .

  3. In the ASP Template column, in offline mode a text box appears (instead of the list that appears in online mode).

  4. In the ASP Template column text box, type in the desired template names. Separate each template name with a comma (,).

  5. If you enter an invalid name, a message box will indicate the error.

  6. To correct an invalid template name, do not use the backspace key. Instead, de-select the erroneous template name in the drop-down list and Save.

  7. Close and re-open the CAT to confirm the erroneous template name has been removed.

Attribute Mapping – Offline Mode

Follow these steps:

  1. Double-click the CAT name in the Solution Explorer pad to open the CAT editor.

  2. Click the ASP Attribute Mapping editor .

  3. In the ASP Attribute column, in offline mode a text box appears (instead of the list that appears in online mode).

  4. Type in the desired attribute names. Separate each attribute name with a comma (,).

  5. In the ASP IO Source column, select a value: None, Input, Output, or InputOutput.

  6. If you enter an invalid name, a message box will indicate the error.

  7. To correct an invalid attribute name, do not use the backspace key. Instead, de-select the erroneous attribute name in the drop-down list and Save.

  8. Close and re-open the CAT to confirm the erroneous attribute name has been removed.
