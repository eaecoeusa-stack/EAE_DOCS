State Management diagram editor

Open your State Manager CAT and select the State Management (diagram editor) pane.

An example of a State Management diagram description in XML language is provided in Appendix. You will find also the XSD code required to validate the description.

The diagram is described by nesting the following tags on three levels:

  1. StateManagerGraph (root node).

  2. StateGroup: a group of states which lead to one transition (for example Stop and Abort); this delineates operations and makes the diagram easier to read.

  3. State and Transition: the basic elements of the diagram.

Some attributes are mandatory whereas others may be missing. The Comment attributes are allowed for all the tags, but optional. A parser ensures these rules are not broken.

The editor uses the Col and Line attributes of each State to position the state ovals inside cells in a grid relatively to the upper left corner. The editor creates the connections between the states based on the transitions.

A group is represented by a dashed rectangle that encloses the cells of the states in the group. A Transition and a State outside a group are attached directly to the StateManagerGraph. The source of such a transition is the group key.
