4. Date and Time Types Conversion to Character Types

Conversions to or from STRING data type are not defined in IEC 61131-3 third edition. The base requirements are as follows:

  • Conversion functions …_TO_STRING can produce a textual representation of the source data type similar to literals of that type.

  • Conversion functions STRING_TO_... can convert the generated string back to the original data type (although other input formats may be possible too).

DATE_TO_STRING, LDATE_TO_STRING

Synopsis 

TO_STRING(IN: DATE): STRING
DATE_TO_STRING(IN: DATE): STRING
 
TO_STRING(IN: LDATE): STRING
LDATE_TO_STRING(IN: LDATE): STRING

Description

Converts parameter IN of type DATE or LDATE to a STRING value of maximal length 10 characters.

The resulting format is ‘yyyy-mm-dd’ and does not include a type prefix.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the value in parameter IN is not a valid date value.

ERR_RT_TRUNCATION: A truncation error is reported if the resulting string is assigned to a variable which is shorter than the required length.

Example 

IF CONCAT('D#', TO_STRING(DATE#2021-12-02)) = 'D#2021-12-02' THEN
  (* correct *) ;
END_IF;

DT_TO_STRING, LDT_TO_STRING

Synopsis 

TO_STRING(IN: DT): STRING
DT_TO_STRING(IN: DT): STRING
DATE_AND_TIME_TO_STRING(IN: DT): STRING
 
TO_STRING(IN: LDT): STRING
LDT_TO_STRING(IN: LDT): STRING
LDATE_AND_TIME_TO_STRING(IN: LDT): STRING

Description

Converts parameter IN of type DATE_AND_TIME or LDATE_AND_TIME to a local date and time value first which is then converted to a STRING value of maximal length 23 or 29 characters respectively.

The resulting format is ‘yyyy-mm-dd-hh:mm:ss.mmmuuunnn’ (local time) and does not include a type prefix. Micro- and nanoseconds are missing for short date and time values.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the value in parameter IN is not a valid date value.

ERR_RT_TRUNCATION: A truncation error is reported if the resulting string is assigned to a variable which is shorter than the required length.

Example 

IF TO_STRING(DT#2021-12-2-0:0:0) = STRING#'2021-12-02-00:00:00.00' THEN
  (* correct *) ;
END_IF;
(* There is no conversion which returns a given date and time value to its external representation in UTC. You can use the following code to do that: *) ;
VAR_TEMP
  dt1: DT := DT#2021-12-03-12:00:00;
END_VAR
IF CONCAT(DATE_TO_STRING(DT_TO_DATE_UTC(dt1)),'-',
  TOD_TO_STRING(DT_TO_TOD_UTC(dt1))) = '2021-12-03-11:00:00.000' THEN
  (* code probably execute in CET-1 *) ;
END_IF;

TIME_TO_STRING, LTIME_TO_STRING

Synopsis 

TO_STRING(IN: TIME): STRING
TIME_TO_STRING(IN: TIME): STRING
 
TO_STRING(IN: LTIME): STRING
LTIME_TO_STRING(IN: LTIME): STRING

Description

Converts parameter IN of type TIME or LTIME to a STRING value of maximal length 18 or 32 characters respectively.

The resulting format is ‘DDdHHhMMmSSsMMMmsUUUusNNNns’ and does not include a type prefix. All components evaluating to zero will not show up in the result string.

Errors

ERR_RT_TRUNCATION: A truncation error is reported if the resulting string is assigned to a variable which is shorter than the required length.

Example 

IF TO_STRING(T#20d0h12s5ms) = '20d12s005ms' THEN
  (* correct *) ;
END_IF;

TOD_TO_STRING, LTOD_TO_STRING

Synopsis 

TO_STRING(IN: TOD): STRING
TOD_TO_STRING(IN: TOD): STRING
 
TO_STRING(IN: LTOD): STRING
LTOD_TO_STRING(IN: LTOD): STRING

Description

Converts parameter IN of type TIME_OF_DAY or LTIME_OF_DAY to a STRING value of maximal length 12 or 18 characters respectively.

The resulting format is ‘hh:mm:ss.mmmuuunnn’ and does not include a type prefix. Micro- and nanoseconds are missing for short time of day values.

Errors

ERR_RT_TRUNCATION: A truncation error is reported if the resulting string is assigned to a variable which is shorter than the required length.

Example 

IF TO_STRING(LTOD#12:0:0) = '12:00:00.000000000' THEN
  (* correct *) ;
END_IF;
