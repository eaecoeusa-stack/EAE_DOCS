HMI - Graphics

The folder Graphics, within the Solution Explorer, contains those graphical parts of the program, which are designated for repeated use in various parts of the project. This makes it easier to maintain the project, and uses only a fraction of the storage space that would be used if they graphics were created or copied to each project location. Moreover, custom dialogs can be created to use them on the HMI, or custom classes in C# to use them for programming.

The whole .Net world can be used here to build static or dynamic graphics and symbols, which can be used in CAT function blocks and Canvases.

The graphics include several coding and styling elements:

  • static graphic elements and controls (circle, rectangle, labels, lines, bitmaps, ....)

  • dynamic elements of the graphics container

  • whole .Net world for customization

NOTE: Graphics do not have direct data access to the variables of logic. Use CAT symbols for these. Certainly, graphic files can be dragged into CAT symbols or faceplates via drag and drop.

If you use one graphic at several points in a project you can modify them all at once, independently where they are localized in the project, by modifying the graphic itself with the use of the Graphic Shapes Editor (double-click the graphic file in the Graphics folder). To refresh the graphic in each place where it is used, deploy the project and reopen the respective tabs (Canvas, CAT Symbol, etc.).

Graphics Folder Context Menu

Right-click on the main node Graphics folder, or on any of its sub-folders, to access these commands:

Command

Description

New Graphic

Opens a wizard to create a new graphic file. By means of the wizard the name, width and height of the graphic can be defined. Thereafter the editor opens and you can edit the file.

New Dialog

Opens a wizard to create a new dialog. By means of the wizard the name, width and height of the dialog can be defined. Thereafter the editor opens and you can edit the file.

New Customer Class

Opens a wizard to create a new customer class for the use in C#. By means of the wizard the name of the customer class can be defined. Thereafter the editor opens and you can edit the C# file. Refer to the example below.

New Folder

Serves to create a new folder to structure the files into subdirectories in order to keep them organized.

Export

Exports the elements of folder Graphics, that means any elements which are currently located directly in the parent folder Graphics, as well as in any existing sub-folders, but without structuring the exported files. They are stored together into the folder you have chosen. If you want to export only one element, use the right-click menu directly on the element, as described below, to export the file separately.

Import

Opens the standard dialog to import archived graphic files (*.graphic.zip). Imported files are located at parent folder Graphics at first, no matter which sub-folder was selected for import. Move the file to the desired sub-folder when it is imported.

Remove

Removes the graphic folder. It is not available with the main node Graphics. If you want to delete a sub-folder, move its elements to another folder at first. The sub-folder can be removed when it is completely empty.

Help

Opens the context help.

When creating a new graphic, dialog or customer class (with the commands New Graphic, New Dialog and New Customer Class) the name filled has to follow few rules:

  • The name cannot begin as the libraries referenced inside the project (e.g. it cannot start with 'SE.'). If doing it, it will create conflicts with a library name, compilation errors will be detected.

  • The name cannot be composed of keywords like HMI, Graphics, Dialogs, SupportClasses, etc.

Graphics File Context Menu

Right-click on a file within the folder Graphics to access these commands:

Command

Description

Open

Opens a wizard to create a new graphic file. By means of the wizard the name, width and height of the graphic can be defined. Thereafter the editor opens and you can edit the file.

New Dialog

Opens a wizard to create a new dialog. By means of the wizard the name, width and height of the dialog can be defined. Thereafter the editor opens and you can edit the file.

Export

Exports the selected file in form of an archive file (*.graphics.zip) and stores it at the chosen storage location.

Remove

Deletes the selected file.

Help

Not available for graphic files by now.

Adding a Customer Class to Services Initialized at Runtime Startup

You can add a custom service class to your solution, and include it in your runtime on startup.

To do this:

  1. In the Solution Explorer, right click on the Graphics node and select New Customer Class.

  2. In the Customer Class Definition dialog, enter a Name for the service class and click Finish. In the following example the name ServiceExample is used.

  3. In the Customer class tab, enter code to create the new Class. In this example, the following code creates the ServiceExample class: 

    using System;
using System.Drawing;
using NxtControl.GuiFramework;

namespace HMI.Main.SupportClasses
{
	/// <summary>
	/// Description of ServiceExample.
	/// </summary>
	public partial class ServiceExample : IExampleService
	{
		public ServiceExample()
		{
		}
		public string Name => "ServiceExampleText";
	}
	public interface IExampleService
		{
		class Name {get;}
		}
}

  4. In the Solution Explorer, in the Canvases node, either create a new canvas or double-click to select an existing canvas. In this example, a new canvas – Canvas1 – is created.

  5. Click the Source Code icon in the vertical toolbar on the left.

  6. Add code that initializes the custom service and includes it in the canvas at startup: 

    using System;
using System.Drawing;

using NxtControl.GuiFramework;


namespace HMI.Main.Canvases
{
	/// <summary>
	/// Description of Canvas1.
	/// </summary>
   
    
	public partial class Canvas1 : NxtControl.GuiFramework.HMICanvas
	{
		public Canvas1()
		{
			//
			// The InitializeComponent() call is required for Windows Forms designer support.
			//
			InitializeComponent();
		}
	}
}
