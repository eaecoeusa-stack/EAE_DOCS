6. Viewing and Updating the Mapping

After the data extraction and auto-mapping are completed, EcoStruxure Automation Expert automatically moves to the next stage: Review CAT Mapping.

In this stage is listed all the extracted engineering data in a clickable tree-view.

For each engineering instance the auto-mapped CAT shows in the CAT column; if none was found, the value is Select CAT Type.

NOTE:

  • The prefix is a space name which can be either Main or <Library name>.

  • A function block of a library may be auto-mapped on the Intermediate level.

Use the search field to filter the extracted data. When a row matches filter criteria, then its complete parent hierarchy is also displayed in the tree-view.

To review the options of a node, click > on the right-most column. This shows in read-only mode:

  • an Attributes tab that displays the optionally entered Name and Value (the value if any comes from the Offline Parameterization Editor). The InstrumentCategory is pre-filled with Field by default. The others come from plain extraction.

  • a Relations tab that displays a close-up of the containment relations and the matching CAT (auto-mapped or manually mapped).

Click X on the right-hand to exit.

Manual mapping with CATs or SubCATs

After the auto-mapping is completed, you can change an auto-mapping value or select one when none was found as follows:

  1. Double-click in the CAT column the cell of the desired engineering instance.

  2. A drop-down lists out a list of CATs or a list of SubCATs:

    • If there is no CAT assigned in the parent engineering instances hierarchy, the list of CATs in the Solution are displayed even if a CAT is assigned to engineering instances in the child hierarchy.

    • List of CATs displayed does not contain any Device CATs as the bulk engineering does not support them.

    • List of CATs displayed does not contain the list of CATs from libraries that are deselected in Library Rules Configuration.

    • If the nearest parent has been assigned a CAT, the list of SubCATs excludes the following SubCATs:

      • SubCATs which are selected in the child hierarchy.

      • Complete hierarchy of SubCATs which are selected in the different engineering instance hierarchy. This is due to the restriction if a SubCAT is assigned in an engineering instance hierarchy, then children of the SubCAT cannot be assigned to different engineering instance hierarchy.

      NOTE: This ensures that only child SubCATs be assigned to child engineering instances when a SubCAT is assigned to parent engineering instance.

  3. Select one of these elements to assign a CAT to the engineering instance and click Save.

When a CAT or SubCAT is selected, a few actions are automatically performed:

  • All the assignments of the child engineering instances are cleared.

  • The row appears in bold font to indicate that it has been manually assigned, except if the type selected is the same as the one selected by the auto-mapping. This action will also clear the assignments of child engineering instances.

Re-applying CAT Mapping Rules

Auto-mapping can be invoked anytime on the imported engineering instances to re-evaluate mapping rules and update the CAT assignments of engineering instances.

To do so, follow these steps:

  1. Click the icon.

  2. Choose to reset or not the manual mapping (even if it was saved).
