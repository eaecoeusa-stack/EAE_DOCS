Device

A device provides an infrastructure (a process interface and a communication interface) for a Resource. It supports one or more resources. Typically a device represents hardware, like a PLC, frequency converter, intelligent terminals etc., but it could also be merely a runtime on an ipc. (See also IEC 61499 documentation for Devices)

In one solution, there can be created as many devices as are needed.

This image divides into:

  • the Logical Device editor, showing the devices which are created in the project, and

  • the Device editor, graphically depicting the resources of the respective device.

To learn more about how to create a device, watch How to Create a New Device Instance and Distribute the Application.

Devices Node Context Menu

Right-click on Devices node in Solution Explorer to access the following actions:

Command

Description

New Device

Adds a new device in the Logical Devices editor.

New Device CAT
Adds devices of the type Device CAT.
NOTE: Requires the existence of a CAT with device identification.
New Folder

Creates folders inside the Devices node to organize the devices.

Help

Opens the Tool Help window that displays the topic Devices.

Device Context Menu

Right-click on a device under Devices nodes in Solution Explorer to access the following actions:

Command

Description

Remove

Use the delete key or right-click on the device to remove the device created under Devices node.

Rename

Rename a device inside the Define new name dialog box.

Open

Displays the device in the System editor.

Move to Root

Moves a device back to the Devices node from a folder.

NOTE: This command is enabled only when user selects a device located under a folder.

NOTE: This command is enabled only when there are multiple folders.

Adding New Device

Right-click on Devices node in Solution Explorer under Devicesnode to access the following actions:

  1. Go to Solution Explorer pad.

  2. Expand System editor.

  3. Right-click on Devices editor.

  4. Click on New Device.

  5. Enter a device name inside the New Device dialog box.

  6. In the Count field, enter the number of devices to create.

  7. Select the Type of devices.

  8. Click OK.

    Result: The new device appears after Devices node in the Logical Devices editor and in the Deploy and Diagnostic .

Adding Device CAT

Right-click on Devices node in Solution Explorer to access the following actions:

NOTE: Ensure that the device EcoRT_0 is created by default and appears under the Devices node in both the Solution Explorer and under Logical Devices in the System Editor.

  1. Go to Solution Explorer pad.

  2. Expand System editor.

  3. Right-click on Devices editor.

  4. Click on New Device Cat.

  5. Inside the Devices dialog box, tick one or multiple checkbox based on the number of CAT instances created from Device CAT.

  6. Click on Create Devices.

    Result: A new Device Cat is created and appears under Devices in Solution Explorer, in the Logical Devices editor and in the Deploy and Diagnostic pad.

Adding Folder

Right-click on Devices node in Solution Explorer to access the following actions:

  1. Go to Solution Explorer pad.

  2. Expand System editor.

  3. Right-click on Devices editor.

  4. Click on New Folder.

  5. Enter a folder name inside the Add New Folder dialog box.

  6. Click OK.

    Result: The folder appears inside the Devices node and in Logical Devices editor.

New Resource Context Menu

Right-click on any device, select New resource, click on Runtime.Management.EMB_RES_ECO under Devices nodes in Solution Explorer to create new resources and access following actions :

Command Description
Open

Displays the function block inside the Function Blocks Network in the System editor.

Remove

Deletes the device resource.

Rename

Rename a device resource inside the Define new name dialog box.

Comment Opens the Comment dialog box for the user to add comment for the resource.
Execution Priority
Displays a drop-down list with three priority options for the user to select: 

  • Normal

  • High

  • Highest

For more information, refer to the Resource chapter.
