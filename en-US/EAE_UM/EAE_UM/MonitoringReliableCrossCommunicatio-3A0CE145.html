Monitoring Reliable Cross Communication

When Reliable Cross Communication is enabled, EcoStruxure Automation Expert can collect statistics and monitor usage of system resources. These collected values are published to MIBs and can be accessed by setting the path and index inputs of the MIBGET function block.

Reliable Cross Communication supports communication between solutions.

NOTE: To distinguish between devices that have the same name but are part of different solutions, use the format ProjectName ! DeviceName when configuring the MIBGET function block.

Reliable Cross Communication Values

For a description of the Reliable Cross Communication values that can be monitored, refer to the topic Monitoring Reliable Cross Communication MIBs.

Except for the OutgoingMessageQueue resource - which is expressed in bytes - available system data is collected in freelist data structures that are lists of entries.

Using the resource PartnerInformation as an example, when Reliable Cross Communication protocol starts it reserves a fixed number of PartnerInformation freelist resources (this number is indicated by the path Network.CrossCommunication.ResourceUtilization.Size with Index 1).

  • When a partner connects, entries are taken from this freelist (increasing the MIB value ResourceUtilization.Used).

  • When a partner disconnects, the removed entries are released back to the freelist (decreasing the MIB value ResourceUtilization.Used).

This process also applies to CommunicationProcessor, NetworkPeerInformation, SocketEvent, HandshakePacket freelist structures.

Note that multiple MemoryPool freelists are supported. Each list varies in entry size (in bytes) as indicated in the name. For example, MemoryPool64 can include a list of entries, each comprising 64 bytes. The Network.CrossCommunication.ResourceUtilization.Size value indicates the number of reserved entries for this freelist MemoryPool64.

Use this information to see the size of the data being tracked.

NOTE: The name (ResourceUtilization.Name) of the resource MemoryPool1500 can change according to the larger of the EcoStruxure Automation Expert configurations Communication.ReliableCrossComm.Network.ReceivePacketSize and Communication.ReliableCrossComm.Network.SendPacketSize.

UdateCounter gives an idea of how much resource usage has changed during the last period. So for MemoryPools a high UpdateCounter value indicates the extent to which the communication is active, and a MemoryPool update count reflects how big is the data being trafficked.

MIBGET Function Block Example

For example, in the following MIBGET block:

  • REQ

    triggers block execution if QI is set to TRUE.

  • INDEX12 = 6 indicates the pin is linked to VALUE12: the OutgoingMessageQueue system resource.

  • PATH13 = Size indicates the capacity of the resource and is linked to VALUE13, in this case 262144 bytes.

  • PATH14 = Used indicates the max used during the last period and is linked to VALUE14, here 0 bytes.

  • PATH15 = MaxUsed indicates maximum utilization since the start of the project and is linked to VALUE15, here 256 bytes.

  • PATH16 = UpdateCounter indicates measure of utilization change during the last period and is linked to VALUE16, here 3 bytes.

Online Change Notification

When a device is connected and when a modification is made to a Cross Communication channel, a dialog box opens with the following message: “The last attempted modification of the project may lead to application behavior degradation after you are executing an Online Change”.

You can then confirm or cancel the modification.

If you select Confirm modification the modification will be made. An online change or a full deployment is required.

If you select Cancel the modification will be reverted to original settings, no online change or full deployment is necessary.
