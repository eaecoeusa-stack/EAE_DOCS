6. Numeric Functions of Time and Duration Data Types

All functions described in this topic are conforming to IEC 61131-3 third edition.

ADD, ADD_DT_TIME, ADD_LDT_LTIME

Synopsis 

ADD(IN1: DT, IN2: TIME): DT
ADD_DT(IN1: DT, IN2: TIME): DT
ADD_DT_TIME(IN1: DT, IN2: TIME): DT
ADD_DATE_AND_TIME_TIME(IN1: DT, IN2: TIME): DT
IN1 + IN2
 
ADD(IN1: LDT, IN2: LTIME): LDT
ADD_LDT(IN1: LDT, IN2: LTIME): LDT
ADD_LDT_LTIME(IN1: LDT, IN2: LTIME): LDT
ADD_LDATE_AND_TIME_LTIME(IN1: LDT, IN2: LTIME): LDT
IN1 + IN2

Description

Adds parameter IN2 of type TIME or LTIME to parameter IN1 of type DT or LDT respectively and returns the resulting date and time value of the same type as parameter IN1.

Errors

No errors are detected during the add operation. Overflow conditions are ignored.

Example 

VAR_TEMP
  someDT : LDT := LDT#2021-12-03-12:00:00;
END_VAR
IF someDT + LDT#10d < LDT#2022-01-01-0:0:0 THEN
  someDT := ADD_LDT_LTIME(IN2 := LTIME#10d,IN1 := someDT);
END_IF;

ADD, ADD_TIME, ADD_LTIME

Synopsis 

ADD(IN1: TIME, …): TIME
ADD_TIME(IN1: TIME, …): TIME
IN1 + … + INn
 
ADD(IN1: LTIME, …): LTIME
ADD_LTIME(IN1: LTIME, …): LTIME
IN1 + … + INn

Description

Adds all parameters INi of type TIME or LTIME and returns the resulting time value of the same type as the parameters.

Errors

No errors are detected during the add operation. Overflow conditions are ignored.

Example 

IF ADD_TIME(T#95s, T#5m10s, T#-10s) = T#6m35s THEN
  (* correct *) ;
END_IF;

ADD, ADD_TOD_TIME, ADD_LTOD_LTIME

Synopsis 

ADD(IN1: TOD, IN2: TIME): TOD
ADD_TOD(IN1: TOD, IN2: TIME): TOD
ADD_TOD_TIME(IN1: TOD, IN2: TIME): TOD
 
ADD(IN1: LTOD, IN2: LTIME): LTOD
ADD_TOD(IN1: LTOD, IN2: LTIME): LTOD
ADD_TOD_TIME(IN1: LTOD, IN2: LTIME): LTOD

Description

Adds parameter IN2 of type TIME or LTIME to parameter IN1 of type TOD or LDOT respectively and returns the resulting time of day value of the same type as parameter IN1. The function provides the resulting value within the range defined by the resulting data type [00:00:00..24:00:00) with the upper limit not included.

Errors

No errors are detected during the add operation. Overflow conditions are ignored.

Example 

VAR_TEMP
  currTOD : TOD := TOD#12:49:30;
END_VAR
IF ADD(currTOD, T#1d15h) = TOD#3:49:30 AND
ADD_TOD_TIME(IN2 := TIME#-9h, IN1 := currTOD) = TOD#3:49:30 THEN
  (* correct *) ;
END_IF;

DIV, DIV_TIME, DIV_LTIME

Synopsis 

DIV(IN1: TIME, IN2: ANY_NUM): TIME
DIV_TIME(IN1: TIME, IN2: ANY_NUM): TIME
DIV_TIME_ANY_NUM(IN1: TIME, IN2: ANY_NUM): TIME
IN1 / IN2
 
DIV(IN1: LTIME, IN2: ANY_NUM): LTIME
DIV_TIME(IN1: LTIME, IN2: ANY_NUM): LTIME
DIV_TIME_ANY_NUM(IN1: LTIME, IN2: ANY_NUM): LTIME
IN1 / IN2

Description

Divides parameter IN1 of type TIME or LTIME by parameter IN2 of type ANY_INT and returns the resulting time value of the same type as parameter IN1. All numeric data types are supported as a divisor except LINT and ULINT.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the result of floating-point division does not fit into the result data type.

Error handling of integer divisions is platform dependent.

Example 

IF DIV_TIME(T#1h, 4) = T#15m THEN
  (* correct *) ;
END_IF;

MUL, MUL_TIME, MUL_LTIME

Synopsis 

MUL(IN1: TIME, IN2: ANY_NUM): TIME
MUL_TIME(IN1: TIME, IN2: ANY_NUM): TIME
MUL_TIME_ANY_NUM(IN1: TIME, IN2: ANY_NUM): TIME
IN1 * IN2
 
MUL(IN1: LTIME, IN2: ANY_NUM): LTIME
MUL_TIME(IN1: LTIME, IN2: ANY_NUM): LTIME
MUL_TIME_ANY_NUM(IN1: LTIME, IN2: ANY_NUM): LTIME
IN1 * IN 2

Description

Multiplies parameter IN1 of type TIME or LTIME by parameter IN2 of type ANY_INT and returns the resulting time value of the same type as parameter IN1. All numeric data types are supported as a multiplicator except LINT and ULINT.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the result of a floating-point multiplication does not fit into the result data type.

Integer overflow conditions are not detected and reported.

Example 

IF MUL_TIME(T#15m, 4) = T#1h THEN
  (* correct *) ;
END_IF;

SUB, SUB_DATE_DATE, SUB_LDATE_LDATE

Synopsis 

SUB(IN1: DATE, IN2: DATE): TIME
SUB_DATE(IN1: DATE, IN2: DATE): TIME
SUB_DATE_DATE(IN1: DATE, IN2: DATE): TIME
IN1 – IN2
 
SUB(IN1: LDATE, IN2: LDATE): LTIME
SUB_LDATE(IN1: LDATE, IN2: LDATE): LTIME
SUB_LDATE_LDATE(IN1: LDATE, IN2: LDATE): LTIME
IN1 – IN2

Description

Subtracts parameter IN2 of type DATE or LDATE from parameter IN1 of the same type as parameter IN1 and returns the time difference between the two date values.

Errors

ERR_RT_OVERFLOW: An overflow error is reported if time difference between the two date values does not fit into the result data type.

Example 

IF SUB_DATE(IN2 := D#2021-12-3, IN1 := D#2021-12-2) = T#-1d THEN
  (* correct *) ;
END_IF;

SUB, SUB_DT_DT, SUB_LDT_LDT

Synopsis 

SUB(IN1: DT, IN2: DT): TIME
SUB_DATE(IN1: DT, IN2: DT): TIME
SUB_DATE_DATE(IN1: DT, IN2: DT): TIME
IN1 – IN2
 
SUB(IN1: LDT, IN2: LDT): LTIME
SUB_DATE(IN1: LDT, IN2: LDT): LTIME
SUB_DATE_DATE(IN1: LDT, IN2: LDT): LTIME
IN1 – IN2

Description

Subtracts parameter IN2 of type DATE_AND_TIME or LDATE_AND_TIME from parameter IN1 of the same type as parameter IN1 and returns the time difference between the two date and time values.

Errors

ERR_RT_OVERFLOW: An overflow error is reported if time difference between the two date values does not fit into the result data type.

Example 

IF (LDT#2021-12-2-12:35:55.12345 - LDT#2021-12-3-12:45:55.12345) = T#-1d10m THEN
  (* correct *) ;
END_IF;

SUB, SUB_DT_TIME, SUB_LDT_LTIME

Synopsis 

SUB(IN1: DT, IN2: TIME): DT
SUB_DT(IN1: DT, IN2: TIME): DT
SUB_DT_TIME(IN1: DT, IN2: TIME): DT
SUB_DATE_AND_TIME(IN1: DT, IN2: TIME): DT
SUB_DATE_AND_TIME_TIME(IN1: DT, IN2: TIME): DT
IN1 – IN2
 
SUB(IN1: LDT, IN2: LTIME): LDT
SUB_LDT(IN1: LDT, IN2: LTIME): LDT
SUB_LDT_LTIME(IN1: LDT, IN2: LTIME): LDT
SUB_LDATE_AND_TIME(IN1: LDT, IN2: LTIME): LDT
SUB_LDATE_AND_TIME_LTIME(IN1: LDT, IN2: LTIME): LDT
IN1 – IN2

Description

Subtracts parameter IN2 of type TIME or LTIME from parameter IN1 of type DATE_AND_TIME or LDATE_AND_TIME respectively and returns a new date and time value of the same type as parameter IN1.

Errors

No errors are detected during the operation. Overflow conditions are ignored.

Example 

IF (LDT#2021-12-2-12:35:55.12345 - T#-1d10m = LDT#2021-12-3-12:45:55.12345) THEN
  (* correct *) ;
END_IF;

SUB(IN1: LTIME, IN2: LTIME): LTIME

SUB, SUB_TOD_TIME, SUB_LTOD_LTIME

Synopsis 

SUB(IN1: TOD, IN2: TIME): TOD
SUB_TOD(IN1: TOD, IN2: TIME): TOD

  (* correct *) ;
