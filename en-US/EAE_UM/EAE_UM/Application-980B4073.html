Application

An application contains interconnected function blocks, including Basic function blocks, Composite function blocks and Subapplication function blocks, to provide a solution for your requirements. There can be more than one application to structure your solution. Applications can be distributed to run on more than one Resource. (See also IEC 61499 documentation for Applications)

The application is a parent layer and in some ways represents the basic networking plan. Every function block inserted in an application must subsequently be mapped (Mapping) to an available resource (within a device).

NOTE: The project cannot be loaded to a controller without mapping.

The application enables you to quickly change between devices, respectively also to change the allocation of some parts of the project on different devices (for example in a building, to allow different parts running on different controllers) by simply modifying the mapping. This leads to maximum independence during engineering, because the mapping can be made subsequently at any time.

Design your application so that any controller can drop out and rejoin at any time without causing harmful output. System integrity needs to be tested during commissioning, and continuously monitored during operations by means of dedicated function blocks in the application.

 WARNING
PARTIAL EXECUTION OF THE APPLICATION
Ensure that the application is designed to safely handle situations where a subset of devices are not running.
Failure to follow these instructions can result in death, serious injury, or equipment damage.
 WARNING
LOSS OF COMMUNICATION DATA
Do not use WIFI communication between controllers. Always use robust cable communication means between the controllers.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Layers

Layers serve to graphically subdivide the network within an application (examples: different floors of a building, or assignment to different devices). Connections within the network between elements of different layers are possible because they are the same network. A default layer named default is automatically generated when an application is created.

There are two different types of layers:

  • Default layer (): Layer at the top of the list that opens by default when you open the application.

  • Other layers (): All the others layers created by the user.

NOTE: Connections between different applications are not possible!
Right-click a layer to see the different elements within it:

Command

Description
