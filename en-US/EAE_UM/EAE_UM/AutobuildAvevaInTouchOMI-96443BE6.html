Asset Link for AVEVA OMI

Use Asset Link for AVEVA OMI to create application objects (AppObjects) in the AVEVA System Platform (ASP) from EcoStruxure Automation Expert CAT instances in a workflow. As part of this workflow, the Asset Link for AVEVA OMI feature: 

  • Supports integrated engineering workflow to configure and define ASP template mapping during CAT creation including SubCATs and corresponding attributes.

  • Enables creation of AppObjects with linked graphics in ASP in a local or remote Galaxy corresponding to CAT instances using customizable knowledge base.

  • Auto-configures OPC UA reference values from EcoStruxure Automation Expert CAT applications.

  • Auto-configures the AVEVA OI Gateway server instance by creating the OPC UA connection and OPC UA group connection nodes for each EcoRT OPC UA server and updating the connection information.

  • Enables deployment of the AppEngine and AppObjects with corresponding HMI Symbols into the ASP platform runtime and test Asset Link for EcoStruxure Automation Expert.

After completion of the workflow, HMI Symbols deployed to Asset Link for EcoStruxure Automation Expert can display OPC UA values read from EcoStruxure Automation Expert CAT instances deployed into one or more EcoRT devices and be used to dynamically overwrite these values.

Using Asset Link for AVEVA OMI to create a workflow involves several discrete tasks:

ASP Template Mapping of Library CATsor project CATs:

Necessary Tasks:

  • Custom Setup of EcoStruxure Automation Expert - Buildtime with Asset Link for AVEVA OMI

  • Configuring Galaxy Details in AVEVA SP Configuration to Connect with ArchestrA Galaxy

  • ASP Template Mapping of Library CATs or project CATs

    NOTE: Refer to the topic ASP Template Mapping for a description of this task.

Asset Link for AVEVA OMI:

Necessary Tasks:

  • Adding the SE.EAEPortal library to the Solution

  • Adding an AVEVA server device and configuring OPC UA Gateway

  • Assigning ASP Template to CAT instances

  • Assigning scan groups to ASP attributes

  • Compiling the AVEVA server device and configuring the OPC UA_EcoRT connection and DeviceGroup in the ArchestrA System Management Console

  • Syncing the Instances to the AVEVA System Platform

  • Connecting to a Galaxy and launching the ASP IDE

  • Creating Process Graphics in ArchestrA IDE based on the AppObject instances

Optional Tasks:

  • Testing Asset Link for EcoStruxure Automation Expert Locally

  • Archiving and Unarchiving the EcoStruxure Automation Expert Solution

Custom Setup of EcoStruxure Automation Expert - Buildtime

When installing EcoStruxure Automation Expert - Buildtime, in the Custom Setup dialog, select the feature Asset Link for AVEVA OMI.

This is a necessary selection for activation of the Asset Link for AVEVA OMI feature.

NOTE:

  • If the add-on license for Asset Link for AVEVA OMI is not activated, the context menu options and navigation toolbar items on CAT editors and the System editor that are related to Asset Link for AVEVA OMI remain disabled and inaccessible.

  • After activating or returning add-on license, if the solution is open, you need to close and reopen the solution.

  • For the procedure to activate or to return add-on license, refer to the Schneider Electric License Manager User Manual.
