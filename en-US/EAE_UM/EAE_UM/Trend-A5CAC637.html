Trend

Use the Trend graphic control to show the history of data values over time in both Windows and Linux HMI operating system types. How far back in time you can display, depends on whether an Archive database is installed. If an Archive database is installed and active, the trend uses it automatically when a point in past time is selected and data is available for this point in time. There is no extra parameterizing required, the connection to the database is sufficient. If there is no Archive database available, the parameterization of the Trend control determines which part of the immediate past can be displayed. In this case, the Trend serves to show the current process of the data points. For more information on how to create and configure an Archive database, refer to the EcoStruxure Automation Expert - Archive User Guide.

Creating Trend in Faceplates

Trend is used with faceplates, which can be opened by a link from the HMI. Select the Trend symbol  in a faceplate of a CAT_HMI from the toolbar and drag a rectangle in the faceplate in the desired size. Refer to EcoStruxure Automation Expert HMI. After that, you can drag and drop the desired Input or Output Variable(s) directly into the Trend faceplate.

In the Properties pad, select Misc > Series to display the TrendSeries Collection Editor window that serves to parameterize Trend Series.

You can also adjust the view and behavior of the Trend window in the Properties pad.

NOTE: In the Appearance section of the Properties pad, the visibility of legend and tool bar can be defined. The length of the displayed time span can be defined with the  TimeSpan item.

Next, the faceplate has to be predefined so that it can be opened by clicking the symbol in the HMI. Do this by clicking in the properties of the CAT_HMI symbol (sDefault in the previous graphic) on Misc > OpenFaceplates to open the OpenFaceplate Collection Editor and enter the desired faceplate name:

If Members is still empty a new entry can be added using the Add button. After that, choose the desired faceplate and the mouse action to open that faceplate from within the right part of the window. Drag the CAT_HMI Symbol from CAT Instances into the Canvas (after recompiling the solution).

Inserting Trend into a CAT_HMI

The Trend can also be inserted into a CAT_HMI Symbol directly, to embed it in the HMI.

If you want to show - within the bounds of possibility - a little bit of the past using the Trend, this can be implemented by parameterizing the attributes. Set the values Depth, Smoothing > Time and Smoothing > Value of the attribute Runtime.History.TrendingAndArchive.

The attribute TrendingAndArchive of CAT-HMI-function blocks or CATs has to be parameterized.

To parameterize the TrendingAndArchive attribute:

  1. Open the relevant HMI.fbt in the Interface editor.

  2. Click the ellipsis (...) button in the Attr- column to select the relevant variable.

  3. Click the ellipsis (...) button, and the Attributes VarDeclaration dialog opens:

In the  Attributes VarDeclaration dialog:

  1. In the attribute Runtime.History.TrendingAndArchive, click in the field in the Value column, an ellipsis (...) appears

  2. Click on the ellipsis (...). The Historization settings dialog opens.

The Historization settings dialog presents these settings:

Setting

Description

Activate Historization

Select this to enable archiving. If de-selected, the remaining settings in this dialog are disabled.

Size of the in-memory historization

Enter the number of data items to be stored in memory, from 0 to 20000. If 0 is entered, no historization will occur and the remaining settings in this dialog are disabled.

Historize new value based on

Select a basis for data capture and storage:

  • Value difference only: Captures only data items with a value change.

  • Time difference only: Captures only data items with a time change.

  • Specified value or time difference: Captures only data items that satisfy the minimum value or time criteria. Enter a value in either:

    • Time difference is more than: enter a minimum time change greater than 0 and less than the data type’s maximum value. See the section Data Type Supported Ranges, below.

    • Value difference is more than: enter a minimum value change.

      NOTE: For values of data types Boolean or String, the only setting is Different.

  • All incoming events: Captures all incoming events with changed value or time attributes.

Authorize archiving for this variable

Enables - but does not activate - archiving for this variable.

Activate archiving for this variable AND for all function block instances

Activates archiving. If this attribute is selected at the:

  • Variable type definition level, archiving will be performed for all instances of this HMI variable.

  • Variable instance level, archiving will be performed only for the selected variable instances. 

NOTE:  It is recommended that you leave this setting de-selected when creating the variable type, and select it only for specific instances.
