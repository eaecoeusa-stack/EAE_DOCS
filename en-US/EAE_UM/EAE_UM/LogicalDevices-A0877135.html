Logical Devices Editor

 Use the Logical Devices editor to manage logical devices in the project, including:

  • Change device type

  • Associate a physical to a logical device (see the topic Physical Devices Editor)

  • Configuring real and simulated device settings

  • Associating devices in simulation mode

  • Editing a Logical Device Configuration

  • Editing a logical device encrypted communication settings

  • Deploying device configurations

  • Restarting a device runtime

  • Testing a configuration in simulation mode

This topic also describes the menu bar and context menu of the Logical Devices editor.

Also see how to edit devices using the Device Editor.

Changing the Device Type

Changing the type of a logical device is necessary when the associated physical device type does not match. Click the Device Type column for the device whose type you want to change, then click the arrow icon ( ) to open a list of available device types.

When you change the device type, a Confirmation message appears, listing the potential impact of changing the device type and asking to confirm the change.

The potential impacts of device type change on connected devices include:

Potential Impact

Description

Loss of physical device association

The existing association with the connected physical device is automatically removed. The logical device must be associated with another physical device matching the changed Type.

Device logged out

The physical device is disconnected (logged out). 

Existing device properties dropped

Not all devices have the same properties. If the replacement device does not have some of the properties of the replaced device, those properties will be dropped.

Default device values changed

Not all devices have the same default settings for the same properties. If the replacement and replaced devices have different default settings for the same properties, the default settings of the replacement device will be applied.

Online change cannot be performed

Because the hardware is changing, the Online Change function is not available and a full deploy is necessary.

Function blocks not supported by new device

Not all devices support the same collection of function blocks. If a function block supported by the replaced device is not supported by the replacement device, you will need to redesign your project.

Associating a Physical Device to a Logical Device

Only physical devices configured with an IP address and network settings (see Physical Devices Editor) can be associated to logical devices. Follow these steps:

  • Create a physical devices network and configure the Ethernet interface(s) of physical devices with IP address settings and a logical network assignment.

  • When the physical device is a server or workstation, verify that the type of software the device is hosting (Soft dPAC, Soft HMI, Archive Database or Archive Link) is properly configured. Do this by selecting the device in the Physical Devices editor, then in the Properties pad (under Runtimes > <type> > Services > Deployment), configure the Interface\Endpoint and Logical Port settings of the device.

    NOTE: For M251 dPAC, M262 dPAC, M580 dPAC, and ATV dPAC device types, these settings are pre-configured.

  • In the Logical Devices editor:

  1. Set the Network Profile to Default.

  2. Click inside the Physical Device column for the device that you want to associate, then click the down arrow to open the list of available devices.

    NOTE: A physical device can be associated only to a single logical device.
NOTE:  The physical device details can be displayed by clicking the + button in the Info column cells, including:

  • Service Name

  • Interface

  • IP Address

  • Logical Port

Configuring Real and Simulated Device Settings

Set Network Profile in the Logical Devices editor to:

  • (Default) to configure settings for real devices.

  • (Local Test) to configure settings for simulated devices.

EcoStruxure Automation Expert - Buildtime saves both configurations. During operations in the Deploy and Diagnostic editor, use the Active Network Profile field to apply configuration settings for real or simulated devices.

Associating Devices in Simulation Mode

Simulation mode is activated by setting the Active Network Profile to (Local Test) in the Deploy and Diagnostic editor: this lets you run the application in the absence of actual physical devices. In this mode, simulated devices are automatically created and associated to logical devices. 

NOTE: When you operate in simulation mode, the device configuration settings set in the Logical Devices editor with Network Profile set to (Local Test) are applied.

When the simulation is started local addresses (127.0.0.1) are automatically used for dPAC simulated runtimes. But the configured addresses for other devices (for Modbus TCP, OPC UA...) are used as is and running the application may affect these physical devices also in Local Test (simulation) mode.

 WARNING
UNINTENDED EQUIPMENT OPERATION
When using the Local Test mode (simulation), make sure that the potential activation of communications with physical devices, addressed through the Hardware Configuration or anywhere in the application logic (i.e.: Modbus TCP, NETIO, OPC UA, etc...), will not cause unwanted effects to the operations.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

To run the application in simulated mode:

  1. Display the Deploy and Diagnostic editor.

  2. Set the Active Network Profile to (Local Test).

  3. Verify that the application builds successfully.

  4. Click Action > Runtime Simulation > Start.

Editing a Logical Device Encrypted Communication Settings

The following logical device configuration settings work together to turn on (True) and off (False) encrypted communication between EcoStruxure Automation Expert - Buildtime and the selected logical device:

  • Security Settings > Use Encryption: Enables the use of encrypted communication in EcoStruxure Automation Expert - Buildtime, and makes it possible to enable encrypted communication for the logical device.

  • Runtime Configuration > Communication > Endpoint > Web Socket > Encryption: Activates an encrypted connection to the selected logical device.

When both settings are set to:

  • True, encrypted communication is enabled and transmitted using the device’s Logical port (secured).

  • False, encrypted communication is enabled and transmitted using the device’s Logical port (unsecured).

When you toggle Security Settings > Use Encryption between True and False, the designated Logical Port number for the device’s Deployment HMI Watch service toggles between the Logical port (secured) and Logical port (unsecured) values. You can observe this value change in the Logical Devices editor by clicking the + button for the device in the Info column.

Both properties need to be configured to the same setting. By default, both properties are set to True. If you change either setting, a message will be displayed reminding you to also change the other setting.

NOTE: After changing these properties, the modified Device Configuration must be deployed and the target runtime must be restarted. In case these settings are changed and the modified Device Configuration is not deployed, the EcoStruxure Automation Expert - Buildtime software will not be able to communicate with the device.

Deploying a Device Configuration

  1. Select the device in the Deploy and Diagnostic editor.

  2. Click Action > Device Configuration > Deploy Device Configuration as described in the topic Other Action Menu Commands.

Restarting a Device Runtime

  1. Select the device in the Deploy and Diagnostic editor.

  2. Click Action > Device actions > Restart runtime as described in the topic Action Commands Triggering.

Testing a Configuration in Simulation Mode

When the simulation is started local addresses (127.0.0.1) are automatically used for dPAC simulated runtimes. But the configured addresses for other devices (for Modbus TCP, OPC UA...) are used as is and running the application may affect these physical devices also in Local Test (simulation) mode.

 WARNING
UNINTENDED EQUIPMENT OPERATION
When using the Local Test mode (simulation), make sure that the potential activation of communications with physical devices, addressed through the Hardware Configuration or anywhere in the application logic (i.e.: Modbus TCP, NETIO, OPC UA, etc...), will not cause unwanted effects to the operations.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

  1. Select the device in the Deploy and Diagnostic editor.

  2. Launch the simulation: Action > Runtime Simulation > Start.

  3. Deploy the device configuration: Action > Device Configuration > Deploy Device Configuration.

  4. Stop the simulation: Action > Runtime Simulation > Stop.

  5. Start the simulation: Action > Runtime Simulation > Start.
