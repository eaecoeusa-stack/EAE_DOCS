Assigning scan groups to ASP attributes

  1. In the System tab, select the application of interest through the bread crumbs trail or in the ASP Attribute Mapping editor.

  2. In the first case, open the ASP Attribute Mapping editor ; all the instances assigned to the application are listed out.

  3. In the scan group column each drop-down is pre-filled with the values as Default, Slow, Medium, or Fast.

  4. For each ASP attribute, select the scan group value that fits best. By default, the scan group value will be set to Default.

  5. To manage scan group assignments more efficiently, users can take advantage of the Import/Export functionality, which allows bulk updates using CSV or Excel files. This process includes validation and error handling to ensure data accuracy.

Exporting Scan Group Data

  1. In the ASP Attribute Mapping Editor, click the Export button located in the toolbar.

  2. Choose the preferred file format: CSV or XLSX.

  3. Once the export is complete, a success message will appear in the Output window.

    NOTE: Export works even if Microsoft Excel is not installed on the machine.

Modifying and Importing Scan Group Data

  1. Open the exported file and make the necessary updates in the ScanGroup column, using only the accepted values: default, slow, medium, or fast.

  2. Save the file once your changes are complete.

  3. Return to the ASP Attribute Mapping editor and click the Import button.

  4. Select the updated CSV or XLSX for upload.

  5. The system will validate the data:

    NOTE: If everything is correct, a message confirming successful import will appear, and the updated values will be applied. If errors are found, such as missing columns, invalid scan group values, or incorrect file format, detailed messages will guide you in resolving them.

  6. The newly imported scan group assignments will now be visible in the ASP Attribute Mapping editor, reflecting your latest updates.
