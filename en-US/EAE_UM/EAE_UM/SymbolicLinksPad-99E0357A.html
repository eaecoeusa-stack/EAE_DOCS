Symbolic Links Pad

The Symbolic Links pad lists the Symbolic Links contained in the project folder. To open the window click View > General pads > Symbolic Links in the main menu bar. By default, it is positioned in the upper right corner.

Symbolic Links, also known as symlinks, enable the interlinking of individual process variables in a project.

Import

Import Symbolic Links from a spreadsheet:

  1. Click the button.

  2. Browse and select a previously exported Symbolic Links spreadsheet file (XLS).

  3. Click Import.

  4. If errors occur, or if importing takes too long, click Cancel and edit the file as necessary (see the Rules set forth in the Export function description immediately below). Then restart the Import procedure.

Export

Export Symbolic Links to a spreadsheet:

  1. Select the Symbolic Links to export.

  2. Click the button.

  3. Browse to select the folder in which to save the spreadsheet file (XLS).

  4. Type the spreadsheet file name and click Save to start the export.

    Result: All selected application-level Symbolic Links and all physical I/Os from all selected application devices are exported to the specified spreadsheet file.

  5. Edit the exported file as necessary. For example, drag and drop a Symbolic Link to the row containing a device and I/O port to associate the Symbolic Link with that device and I/O port.

    Rules:

    • Output Symbolic Links must be associated with output ports and input Symbolic Links with inputs.

    • Each Symbolic Link can appear in one row only. However, physical inputs or outputs can be associated with multiple Symbolic Links, and the same Symbolic Link can be associated with multiple inputs or outputs.

    • All Symbolic Links of a function block must be associated to the same device.

Symbolic Link Name, Connection, and Type

Symbolic Link Name lists the existing Symbolic Links, depending on the currently used filter adjustment. These entries can be assigned directly to a hardware port within the HW Configurator.

Connected to lists the hardware ports, if the respective variable is already connected to a hardware port (also depending on the currently used filter adjustment).

Type. If the Show data types button is pressed, lists the data type of the Symbolic Links and, if the corresponding variable is connected to a hardware port, the data type of the hardware port.

Remove

Remove the Symbolic Link connection, that is, the connection to the hardware port. The  Connected to column is empty now and can be reconnected with a hardware port (see Symbolic Links, HW Configurator).

Add Watch

Set a watch point on the ports of the Symbolic Link connection. A sub menu opens to define where the watch point is set (see below). To enable this function, the project must be loaded on a controller and EcoStruxure Automation Expert - Buildtime logged in to the controllero so that a connection to the watch server exists. Login is performed in the Deploy and Diagnostic editor via Action > Login.

Further information about watch: Watch pad, Watch.
