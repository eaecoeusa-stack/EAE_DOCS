Solution Node

A solution includes:

  • External libraries: these consist of pre-existing libraries that were added to the solution when the project was created, or subsequently.

  • Custom libraries: you can create these libraries 

  • Distributed PAC Project: The project you create.  

The external libraries collection can include e.g. Runtime.Base or SE.Standard, according to requirements. These libraries contain various function blocks, including Basic Function block, Service Function block, Composite Function block and CAT. The hardware corresponding function blocks are among the latter.

Custom libraries in open source form (e.g. created in the solution) are in contrast to referenced, installed libraries because the folders are displayed in another color (and there is no version number added).

The node Distributed PAC Project shares the same name as the solution itself. Of course the function blocks mentioned above can be created within this node directly, however in that way they are only available in this project, they are open source and from within this project at any time accessible and alterable. If you want to use the function blocks in other projects anyway, use the Export (right-click on the function block or the parent node for the function blocks of the same type) and the Import option.

NOTE: Source libraries that are located in the project need to be manually referenced after the creation. When you instantiate a block from such a library, a query dialog opens asking whether you want to reference the library. No blocks can be instantiated from non-referenced libraries.

Unreferenced libraries are not compiled. Once you have referenced them, they are compiled. When you generate a binary library it is also compiled and any detected errors are displayed.

Refer to the Solution Explorer pad topic for more information about its features. 

Context menu

Rght-click on Solution - the upper-most node in the Solution Explorer — to open a context menu with the following items:

Command

Description

Documentation

  • Generate → This will create the documentation of the project, after that a dialog appears to ask whether or not you want to open the documentation

  • Show → existing documentation is opened (or generated and opened, if it has not yet been generated)

Archive

Creates a zip-file of the project, storage path is freely selectable

Rename Solution

Renames the solution and the project node (Distributed PAC Project)

Select HMI Compatibility

  • Multi-OS HMI: Supports HMI that uses both Microsoft Windows™ and non-Windows (for example, Linux) HMI operating systems.

  • Windows HMI: Supports HMI that uses only the Microsoft Windows HMI operating system.

See Changing the HMI Operating System for an Existing Solution.

Options

Opens the file Solution Options; here you can specify which directories you do not want to store with archive

Create Library

  • New → creates a new library within the project, the New Library Wizard opens; the input of Company Namespace name and Library name settings are required to create a new library (the library name is composed of them: 'Companyname.Libraryname'). The Description field is optional. Any remaining steps (declaration of company, author, license terms and definition of a library icon (- with size set only to 16 x16 pixels)) are optional.

  • New Base Library → same as 'New' but Base Libraries cannot contain CAT and UI-features as Graphics and Named Styles (Colors, Brushes, ..) are not available.

  • Add existing → adds an already existing library to the project, that means a library which already exists in the project but is not shown in the Solution Explorer. (e.g. Because it was removed by a right-click on the library node → Remove, or it was copied into the solution using windows explorer and therefore it is not shown in the Solution Explorer.) A window opens containing a list of the libraries currently available. Select the appropriate library and confirm with OK. If there are no libraries currently available, the window remains empty.

  • Unarchive → unzipping and adding of an archived library (*.lib.zip). This type of file can be created, by a click on entry Archive in the context menu, that opens after a right-click on an open source library (see further down on this side).

To learn more about how to create a library, watch How to Create a Library in EcoStruxure Automation Expert.

Working with Open Source Libraries

A right-click on an open source library, i.e. a custom library, opens a context menu:

Command

Description

Remove

Removes the library from the solution but without deleting it from file system. If you want to delete the library completely use Windows Explorer or the like (the simplest way: right-click on the main nodeSolution and click on the context menu entry Open Solution Directory in Windows Explorer), navigate to the directory of the current solution and delete the whole library folder. A corresponding message where to find the library will be displayed after removing the library.

NOTE: A removed library, that has not yet been deleted, can be integrated into the solution again, by a right-click on the main node Solution (<Project_name>) and then a click on the context menu entry Create Library/Add Existing. Select the desired library in the dialog window and confirm with OK.
Archive

Exports the library as an archive file to the selected target directory.

References

Opens dialog Project Library References that shows the current library references of this library and that allows you to delete existing references as well as create new one.

Change Library Reference Version

Opens dialog Project Library References which allows you to change a library reference from one library version to another version of the same library, provided that there are multiple versions of the same library installed on the machine.

Add HMI External Reference

Offers the possibility to reference external .net-files used by the HMI, to make them available in the project. In that way, ready-made program parts (e.g. for weather information, to determine the location, etc.) which have been self-made, or are freely available in the internet can be used in the project easily. Specify the path where you have stored the .dll file. With the next compile, it will be integrated in the HMI.

Remove HMI External Reference

Used to remove references to external .net-files that were inserted to be used in the HMI.

Import

Allows to import archived elements (*.Adapter.zip, *.Basic.zip, *.Composite.zip, ...). After selecting the element to be imported dialog Import will open.

Documentation

  • Project Information → opens the file Project.doc.xml, containing the information about author, company, description, etc. which has been typed in using the wizard while creating the library.

  • Generate → generates a documentation of the library and shows the whole documentation of the project and the included libraries if desired.

Options

  • General → opens a dialog to edit the general information of company, author and web site of the library creator

  • IEC61499 → opens tab IEC61499, containing settings for compiling, deploying and creating a watermark for function blocks used in this library

  • HMI → opens tab HMI ; here you can select various HMI symbol libraries, if multiple versions are installed on the machine.

For further information see Project Options

General HMI Properties

General HMI Properties > Open opens the file Properties.xml of the relevant library. Using this list you can create new data points for the HMI and adapt existing ones. The functionality is just the same as in the context menu of the project.

Image Storage

Opens a dialog to manage images used in the library (hardware images, etc.).

Create Installation Package

Creates an installation package for the library in order to pass it on to other project clients in a limited access read-only form, or in an accessible form, to install them on their machines. After choosing a target folder dialog Library Properties will open.

Open Library Directory in Windows Explorer

Uses windows explorer as current file manager to show the directory structure of the library.
