PROFINET Limitations

Supported Numbers of PROFINET Network Devices

As implemented in EcoStruxure Automation Expert, PROFINET supports real-time unsynchronized communication over networks that include up to the following maximum number of devices:

  • 1 PROFINET IO-Controller and IO-Device instance per Soft dPAC.

  • 1 port per PROFINET IO-Controller instance.

  • Up to 128 connected IO-Devices operating in real-time.

Supported Update Times

The minimum IO-Device Update Time depends on several factors, including:

  • the communication share set for PROFINET IO.

  • the number of IO-Devices.

  • the quantity of configured user data.

Each of the following Send Clock settings support IO-Device minimum Update Time settings equal to or greater than the settings indicated below, up to the maximum supported by the IO-Device:

Conformance Class A Subset Supported Features

EcoStruxure Automation Expert supports all mandatory CC-A features including the following:

  • Cyclic I/O

  • Acyclic communication

  • Standard Parameters (F-Parameters and Dynamic Parameters are not supported.)

  • Diagnostic Alarms

Supported GSDML Versions

GSDML versions 2.45 and earlier are supported. Versions later than version 2.45 are not supported.

Supported Data Types

PROFINET supports all data types.

NOTE:

  • E2, C4, Unipolar2.16, V2, L2 data types are represented as Octet string.

  • UnicodeString8, 61131_STRING, 61131_WSTRING data types are represented as VisibleString.

Hardware Configuration Import Limitations

Not all elements of a hardware configuration are included when importing a solution. The following elements need to be imported separately:

  • GSDML files

  • Hardware CATs

Data Size Limitations

  • Maximum I/O Data Item length: 1024 bytes.

    NOTE: If the DataItem Boolean attribute UseAsBits in the GSDML is set to True, the Maximum I/O Data Item lengthis limited to 8 bytes by default. The Maximum I/O Data Item length value can be configured using the MaxLengthForBitAccess property in the app.config file (located at: C:\Program Files\Schneider Electric\EcoStruxure Automation Expert - Buildtime 25.0\Studio\bin\Plugins\ProfinetConfiguration\ProfinetConfiguration.dll.config).

  • There is no limit to the maximum number of Symbolic links per IO-Device.

    NOTE: You can use the Network Configuration > Device > Cyclic Data settings in the PROFINET IO Configurator to select only the data items for which symbolic links need to be created.

  • I/O communication over a fieldbus supports devices based on hardware size. EcoStruxure Automation Expert - Buildtime displays a message when a busmaster is configured with a greater number of devices and registers than it can support.

Unsupported Features

PROFINET supports CC-A features, except for the following:

  • Topology Information (CC-A LLDP)

Other PROFINET features also are not supported, including System redundancy, Fast start-up, MRP, iPAR server, and so forth.
