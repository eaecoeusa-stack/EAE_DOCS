CAT

A Composite Automation Type (CAT) function block includes objects that can be used for the integration of HMI or SCADA functionality of vertically integrated automation solutions. CATs also contain the documentation describing their functionality, operating mode, and faceplates. This documentation is implicitly part of the project online help (Solution Help or Library help depending if the CAT is part of the project or part of a library).

A CAT represents the interface between solution logic and end user.

Each CAT can have multiple symbols, which can be used in the Canvas. Like symbols, a CAT can contain several faceplates (HMI Controller). Using the faceplates, you can view, control and parameterize your solutions in detail.

In EcoStruxure Automation Expert, several types of CAT is available, their description and the process to create them is all detailed in the topic Create New CATs.

Otherwise, to create a CAT, watch How to Create a CAT in EcoStruxure Automation Expert.

In the Solution Explorer, all the CAT objects, whatever their type, present three nested levels:

  • First (topmost) level: Interface

  • Second level: HMI

  • Third level: Symbols and Faceplates

Access to each of these levels by expanding the CAT object with the + sign in the Solution Explorer pad.

Top Level: Interface

This level contains the interface of the function block. This includes the ports which are used to connect the function block to other function blocks within the program. A description of the interface editor can be found at Function Block Interface Editor.

This level also contains the composite and its editor is available, to implement the inner structure of the function block. At Composite Function Block Editor you can find a description about how to use the editor.

Second Level: HMI

This level is the HMI part of the function block. It includes the interfaces needed for the connection to the HMI. In this function block new symbols or faceplates can be created.

The difference between Symbol and Faceplate is described below.

The HMI function block consists of an event and an data part, like any other function block. A double-click on the HMI function block opens the Interface Editor of this function block. Although the HMI function block does already exist in the tree when you create a new CAT, it has to be inserted in the Composite Editor of the CAT to be instanced and has to be properly connected.

The HMI function block represents the connection to the HMI. The interfaces of the HMI function block can be instanced directly in the Symbol Editor, in order to show the values of the variables of this interfaces in the HMI or to enable the user to set them (e.g. a label to show a value, a text box to edit a value via HMI, a button, etc.).

While connecting the HMI function block in the composite editor of the CAT, you can also expand the HMI function block here. When you draw a new connection to the CAT function block, a new port will automatically be added to the HMI function block, thereby creating a new port. But, if you make the connection to a pre-existing port, the automatically generated port will disappear immediately. On the other hand, if you use that port for the connection the With Type Editor opens. Use this editor to define either:

  • The events that have to be connected with the newly created data port (provided, that there are event ports already created at the HMI function block), or

  • The data ports that have to be connected with the new event.

Name and type (in case of a data port) of the new port will be carried over from the port of the connected function block. Use the Interface editor of the HMI function block to modify subsequently both name and type.

NOTE:

  • When adding a new data input on HMI function block, first ensure to have an event input available in the HMI function block to associate this data input with (other than INIT). Indeed, if a data input in the HMI function block is not associated with an event input, it won’t be able to connect with an output data in the Function Block Network editor of the CAT.

  • Design your CAT_HMI function block so that the INIT event must always be triggered first, i.e., before any other events in the same CAT_HMI function block are triggered. Otherwise, the triggering of a non-INIT CAT_HMI event can unintentionally impact the HMI display.

For CATs (or composites) additionally ports can also be created with a right-click on a port of a function block within the Composite tab and then selecting Add Event Output (or Add Event Input, Add Data Output, Add Data Input).

The same context menu also presents entries Go To, Routing Linker, Connections..., which are described in the topic Watch.

Remove: Remove the selected port.

Rename: Rename the selected port via a dialog box.

Third Level: Symbols & Faceplates

In this level, within the HMI function block, the symbols and faceplates are located. They are created by the use of a graphic editor and then shown in the HMI (see Symbol Editor and Faceplate Editor).

Symbols (or terminal view of Hardware CATs):

The view is created graphically in the symbol which is visible in the HMI after the respective CAT instance (from folder CAT Instances) has been dragged into a canvas. Symbols contain those graphical elements which are necessary to control and operate the functionality of the CAT as well as images which can, for example, form the background of the Symbol (this may be e.g. the terminal view of a hardware terminal).

Faceplates:

Faceplates can be linked with the symbol in such a way that the faceplate can be opened on the HMI, using a button or an action, to enable access to additional adjustment possibilities, trend watching, etc. Faceplates may also contain graphical elements for controlling and operating, however these are primarily functions that do not always have to be visible on the HMI – for example, for parameters that are not often modified or for functions that are available only after they are called explicitly. How to create the link between a faceplate and an element within the symbol is described in the Faceplate Editor topic.

The symbols and faceplates are built with several coding and styling elements:

  • static graphic elements and controls (circle, rectangle, labels, lines, bitmaps, ....).

  • elementary symbols with connections to the variable and event interface of the CAT.

  • elements of the Graphics container.

  • CAT symbols from the CAT in CAT structure.

  • the whole C# and .Net world for customization.

This deletes the symbol or faceplate completely. This function is only available for faceplates or if several symbols exist. At least one symbol needs to exist and therefore it cannot be deleted.

Duplicate

This option is used to duplicate a symbol or faceplate within the function block. Choose an appropriate name.
