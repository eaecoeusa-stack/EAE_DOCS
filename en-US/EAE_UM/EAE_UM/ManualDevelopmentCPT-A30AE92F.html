Manual Development

The manual development is a second way to configure the Modbus registers inside the Modbus communication previously created. This way is recommended when the automatic generation is not available or didn’t succeed, even after troubleshooting, or when the Modbus communication doesn’t require a lot of Modbus registers.

The manual development is then given by EcoStruxure Automation Expert to add and set manually Modbus registers. This configuration is done in the Hardware Configuration editor for each variable to transfer through Modbus.

Configuring the Modbus registers through this way leads the user to set manually all parameters of each Modbus register. That is why if there are a lot of variables to transfer, it may be worth to automatically generate the Modbus registers.

To manually develop the Modbus registers, follow the few next steps:

  1. Add Modbus Registers into the Hardware Configuration editor.

  2. Set the Modbus Registers parameters.

These two steps have to be followed for each variables transferred through Modbus.

NOTE: Before starting this procedure, make sure that the latest version of the SE.ModbusGateway library is referenced in the project. To learn more about this library and its function blocks, press F1 on it in the list of project references.
