Installed Plug-ins

The Installed Plug-ins dialog shows a list of all currently installed plug-ins of the host PC. Use this dialog to:

  • View the list of plug-ins.

  • View and edit settings for a selected plug-in.

Click the File menu, then in the Backstage area select Installed Plug-ins to display this dialog.

Features

Element

Description

Name

Name of the installed plug-in.

Version

Version number of the plug-in.

Author

The name of the developer of the plug-in.

Description

Statement of the function performed by the plug-in.

Settings

Opens a dialog that displays settings for the selected plug-in. You can edit and Save these default settings. Settings will vary depending on the plug-in. Select a setting to display its description at the bottom of the dialog. Some common settings include:  

NOTE: If the Settings button is disabled, no settings display for that plug-in is available.
Answer Timeout

Timeout in seconds after which EcoStruxure Automation Expert software will logout from the targeted device.

Connection Timeout

Timeout in seconds for the initial connection with the target device. If the device doesn't respond within specified time, the login will be aborted.

Http Timeout

Timeout in seconds for the application and device settings transfer to the target device or uploading log files. If the device does not respond within the specified time, the login will be aborted.

Long Command Timeout

Timeout in seconds for commands like Deploy and Online Change. If the device doesn't respond within specified time the command will be aborted.

Parallel Thread Count

Number of simultaneous commands that can be executed with all devices.

Short Command Timeout

Timeout in seconds for commands like Login, Logout, Run, Start and so forth. If the device doesn't respond within specified time the command will be aborted.

Upload Timeout

Time in seconds before upload timeout is spotted while uploading files to device.

Verbose mode

Enables detailed level of log inside the target devices.

Sequence of Plug-In Timeout Settings

The timer for each timeout setting operates independently from other timers.

For instance, the Long Command Timeout timer starts only if the Parse, Connect, and Send files actions are successful, and not when the a Deploy or Online Change action is initiated.

Thus, if it takes 5 seconds to parse, connect and send files + 60 seconds for the runtime to respond, the user will have waited 65 seconds - not 60 seconds - for the Long Command Timeout to expire.
