Solution Integrity

This section describes the recommended precautionary cybersecurity measures that end users should take when creating, opening, archiving, and unarchiving solution files.

EcoStruxure Automation Expert - Buildtime does not provide functionality to encrypt solution files saved on the end user’s local disk. In a Multi-User Environment or on an Active Directory domain-authenticated workstation, there is a possibility that another local or domain user on the engineering workstation (including a less-privileged user) could gain access to EcoStruxure Automation Expert solution files — including archived solutions — created by another user.

Below are the recommended practices for end users:

Creating Solution

To prevent unintended access to solution files on an engineering workstation where there are multiple local users or an Active Directory domain-authenticated workstation, there is a possibility that another local or domain user on the engineering workstation (including a less-privileged user) could gain access to EcoStruxure Automation Expert solution files — including archived solutions — created by another user.

Below are the recommended practices for end users:

C:\Users\<Your Username>\Documents\

This ensures that Windows directory access permissions automatically restrict other users on the same machine from accessing your solution files. By default, Buildtime prompts users to save solution files in user's home directory.

If you prefer to save solution files in another location (e.g., another drive such as D:\), we highly recommend protecting that directory by restricting access to unintended users using the Windows file system security controls:

Right-click the folder Properties > Security > Advanced > Edit permissions.

Opening a Solution

When opening a solution, it is the end user’s responsibility to make sure that solution files are trusted and has not been tampered by any unintended user. Please refer to the precautionary measures described above for creating solutions.

Archiving a Solution

When archiving a solution, the same Windows file system security principles apply as creating solution. Make sure that the directory where the archive ZIP file is saved is either within your user home directory or protected with access restrictions that allow only intended users to access it.

Opening Archived Solutions

When opening an archived solution, it is the end user’s responsibility to verify that the archive file is trusted and has not been modified by an unintended user. Please refer to the precautionary measures described above for creating archives.

It is also recommended to use a commercial encryption solution to protect solution and archive files stored in directories or on secondary drives that may be accessible by other users, in order to prevent unauthorized access.
