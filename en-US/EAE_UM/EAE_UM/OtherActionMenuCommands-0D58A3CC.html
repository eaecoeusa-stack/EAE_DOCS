Other Action Menu Commands

The following action commands, available in the Deploy and Diagnostic editor, can impact a device’s operating state:

  • Compile

  • Compile for Online Change

  • Delete Boot Project

  • Deploy Device Configuration

  • Login

  • Logout

  • Online Change

  • Set Running Project as Boot Project

Compile

Compiles the project, including both the application (program logic) and hardware configuration (I/O and communication) settings, for the selected device. The compiled project can be deployed via the Deploy > Deploy (full deploy) command.

Compile for Online change

Compiles only the changes made to the application (program logic) and does not include changes made to the hardware configuration (I/O and communication) settings. The compiled changes can be deployed via the Deploy > Online Change command.

Delete Boot Project

The Boot Project > Delete Boot Project command removes the boot project, including both application (program logic) and hardware configuration (I/O and communication) settings, that was stored in the device. The device operating state remains unchanged.

In the absence of a boot project:

  • Executing the Boot Project > Load Boot Project command has no effect.

  • Cycling power to the device places the device into the Clean state.

Use the Boot Project > Auto set deployed project as boot project command to create in the device a new boot project from the application existing in the device’s execution memory.

Deploy Device Configuration

The Deploy Device Configuration command is available for a connected device in any operating state and sends to the device the defined hardware configuration, which will only become effective after a subsequent Restart Runtime or Reboot Device operation. The device operating state remains unchanged.

Login

Establishes a connection with, and logs in to, the device. The device operating state remains unchanged, but the existence of the device becomes known to the buildtime.

NOTE: If forced variables exist in the project (in EcoStruxure Automation Expert - Buildtime) or the device, or both, you will be prompted to select which collection of forced variables will be kept after login.
NOTE: On Login, if you see one of the following Action Progress column messages for the selected device:

  • Connection Closed, or

  • Unable to connect: For device '<local device name>’, please check that IP address '<IP address>' is valid and the port '<port number>' is accessible. Cannot connect to device '<target device name>'. Please click to read related help section and review output pad for more information.

check the following possible causes:

Possible cause

Corrective action

The device IP Address may be erroneously configured.

Verify the correct IP Address and edit the device configuration.

The device Logical Port number may be erroneously configured.

Verify the correct Logical Port number and edit the device configuration

The Port number may be filtered as a result of firewall configuration.

Check your firewall configuration, and confirm the Logical Port number is not blocked.

The device was not commissioned as Secured.

Secure the device as a trusted device in the Physical devices editor

Logout

Closes the current device connection. The device operating state remains unchanged.

NOTE:

  • If forced variables exist in the project, the logout command triggers a confirmation dialog as by default the forced variables will remain active in the device after logout completion.

  • If the boot project stored in the device is different from the project executing in the device, a logout triggers a confirmation dialog giving an opportunity to resolve this discrepancy (executing the Boot Project > Auto set deployed project as boot project command).

Online Change

Applicable only to devices in the Running state, the Deploy > Online Change command deploys to the device only the application (program logic) changes that have been compiled via a Compile for Online Change command. Changes to the hardware configuration (I/O and communication) settings are not included. For more details on the Abort online change command, please refer to the help for the Hardware Configuration editor toolbar. The device operating state remains unchanged.

Set Running Project as Boot Project

Applicable only to devices in the Running or Stopped states, the Boot Project > Auto set deployed project as boot project command stores the project existing in the device execution memory as a copy-project stored in a non-volatile device memory. The device operating state remains unchanged.

The boot project includes both the application (program logic) including any online changes and the hardware configuration (I/O and communication) settings. This package is sufficient to restore a fully operational device, for example, after a power cycle.
