Table Properties

Use the Table Properties dialog to define properties for a table created when documenting function blocks. Access the dialog by clicking the Table icon  in the Documentation editor.

Configuring a Table

The Table Properties dialog includes the following settings and commands:

Command

Description

Caption Properties:

Caption

A text, used as table heading, can be specified here. The position and orientation of the heading can be defined by the two properties listed below.

Alignment

Defines the horizontal alignment of the heading. Left, Center or Right are available. Default is Center.

Location

Defines the vertical position of the heading. Top creates the heading above the table, Bottom creates the heading below the table. Default is Top.

Table Properties:

Border

Defines the line width of the outer table border.

Alignment

Defines the horizontal alignment of the table, where it is placed on the documentation page. Default is left.

Width

Defines the width of the table. You can choose between percent or pixel.

Percent:  Refers to the ratio of the width of the documentation page.

Pixel:  creates a table with the respective width.

Columns:

List of columns

Shows the currently predefined columns. The header text of the column and its width can be changed in the fields to the right, as soon as one of the columns is selected.

Text

The header text of the selected column can be changed here.

Width

The width of the selected column can be changed here (in %).

In this context, columns generated further to the left have priority. For example, if you define 2 columns, where the first column is 50% width, the second column can only be 50% of the table, even if it is created with 100%. If the total width of 100% has already been reached but additional columns are to be displayed, they are created with a minimum width.

Add

Click this button to create additional columns.

Remove

Click this button to remove the selected column. It is available only when a column of the list is selected. Only individual columns can be removed.

Rows:

Count

Here you define the number of rows at the time the table is created.

You can also adjust the number of rows in a table afterwards, by right-clicking in a field of the table and selecting from the context menu of the entry Table > Insert Row or Delete Row.

OK

Closes the dialog and confirms the changes.

Cancel

Closes the dialog without accepting any changes.
