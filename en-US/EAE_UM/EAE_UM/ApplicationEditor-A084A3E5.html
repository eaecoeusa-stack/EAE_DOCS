Application Editor

This function block editor opens if you double-click an application in the System node of the solution in the Solution Explorer pad.

Use this editor to add, modify, or delete function blocks and their properties and connections within an application.

 WARNING
LOSS OF EVENTS
Identify the events on which the safety of the application relies.
Always handle these events at application level, implementing algorithms (like  events reception acknowledgement and event re-send) to take account the eventuality that events could be lost between emission and reception.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Function blocks and elements added to an application have to be mapped to a resource with Mapping, or use the context menu of an function block or a selection of function blocks and select a resource of the menu item Mapping.

For a quick selection of items within an application, the editor contains two search fields as drop down fields on the upper border of the editor.

A description of the general functions of the toolbar can be found in the Function Blocks Network editor.

Application Editor Context Menu

A right-click on the background within the editor opens a context menu:

Command

Description

FB

Opens a sub menu to choose one of the available function blocks to insert it into the application.

Frame

Opens a sub menu with the following items:

Add Frame:  adds a colored frame, which can be used for structuring and grouping assemblies. The size of the frame can be varied by pulling one of the green points. Color, name, display text, etc. can be adjusted in the properties of the frame. In order to open the properties double-click the frame or right-click the frame and click on Frame > Properties in the context menu. Elements arranged within the frame are moved in common when the frame is moved. In this sense, a frame serves as a group element. (See also Editors.)

Add Comment:  adds a colored rectangle which can hold a multi-line comment. It serves to create a comment or a short note. The difference between frame and comment is, that a comment does not have group functions. The text of the comment can be modified after double-clicking the text, whereas the width of the rectangle increases with the length of the header line text. The text of the comment as well as the color, font, etc. can be modified within the properties, too. In order to open the properties, double-click the rectangle of the comment or right-click to select Comment > Properties in the context menu. (See also Editors.)

Go To Frame:  choose one of the existing frames from the sub menu to select it and bring the focus on it. Remark: this entry does only exist when frames have already been created.

Properties ...: opens the properties of the selected frame or comment. If no frame or comment is selected, or existent, the entry will remain grayed out.

Layers

Opens a sub menu to create additional layers and, if already existing, to switch between existing layers. In the lower left area of the editor a button is also displayed for each existing layer to switch between the layers. A short description of the layers can be found below.

Mapping

To use this feature a device CAT needs to be available in the project and be used in the application. It serves to map the CAT from within the application, whereas the CAT will not be mapped to an existing device but an own device will be created where the application is mapped to. A mapping dialog opens. The device CATs which are necessary for this can be taken from an appropriate library which is referenced in the project or you can create them by your own.

NOTE: in order to map elements from an application to a resource use the context menu which appears after a right-click on selected elements as it is described below.

Attributes

Opens dialog Attribute

Redraw connections

After moving the position of one or several connections between function blocks, click Redraw connections to return the connections to their default position, i.e., the position EcoStruxure Automation Expert automatically drew when you first created connections between function blocks.

Application Editor Function Block Context Menu

A right-click on one or several selected function blocks within the Application Editor opens a context menu. Some entries may vary, are missing or shown grayed out depending on the project and status of the project (deployed and logged in or not, etc.).

The entries of this context menu are described on page System Editor, with the exception of entry Mapping which can be found only in the context menu of the Application editor and is described below:

Command

Description

Mapping

In order to map individual parts or the entire content of an application to an existing device, select the appropriate elements and use entry Mapping from this context menu. A submenu opens with the following items:

  • List of existing project resources: resources to which the function block can be mapped.

  • Remove mapping: deletes an existing mapping relationship.

and offers the available options (the existing resources of the project).

NOTE: The Resource Editor offers a second option for mapping. Use the button Mapping as described at Mapping.
Map

Alternatively to the above mentioned entry Mapping, this entry is available when the selected element is a subapplication. Because a subapplication consists of several CATs, it offers the possibility to map individual parts to different resources. A dialog opens, which is discussed at Subapplication Mapping Settings in detail.

All other entries of this menu

see System Editor

Layer

The layers in the application provide a better overview. They represent an additional option for creating a structure within an application.

  • You can use it to visually separate certain areas within an application from each other. (Example: individual rooms or floors of a building.)

  • If elements are mapped from a layer, this division is also created automatically in the device.

  • In contrast to the use of several applications, however, the blocks between the layers within an application can also have connecting lines.
