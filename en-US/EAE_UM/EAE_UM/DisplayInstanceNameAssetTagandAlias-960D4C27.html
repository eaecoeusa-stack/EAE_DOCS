Display Instance Name, Asset Tag Name and HMI Alias Name

This topic explains how to display dynamically the different names of a CAT instance such as instance name, Asset Tag name and HMI Alias name on the HMI with the toggle button.

NOTE: To know the definition of these three names, refer to the topic Creating and Localizing different CAT and Sub-CAT Instance Names.

To display the different names, follow one of the procedure depending on your use case:

  • You developed the CAT used as CAT instance or it is coming from a custom library:

    1. Modify the symbol of that CAT.

    2. Set the CAT instance in the project.

    3. Add the toggle button.

    4. Add the symbol of the CAT in a canvas.

    5. Use the toggle button.

  • You use a CAT coming from a Schneider Electric library:

    1. Set the CAT instance in the project.

    2. Add the toggle button.

    3. Add the symbol of the CAT in a canvas.

    4. Use the toggle button.

    NOTE: Some CAT symbols of Schneider Electric libraries might not display the Asset Tag name, the HMI Alias name and the instance name once online. If the CAT symbol doesn’t, refer to the situation below.

  • The CAT used from the Schneider Electric library doesn’t display the different names once online:

    1. Unlock the symbol of the CAT from the Schneider Electric library.

    2. Modify the symbol of that CAT.

    3. Set the CAT instance in the project.

    4. Remove type cast errors, if there are some.

    5. Add the toggle button.

    6. Add the modified symbol of the CAT in a canvas.

    7. Use the toggle button.

Unlock the Symbol of a CAT

The purpose of this topic is to unlock the symbol of a CAT by copying its function blocks directly into the project.

If the symbol of your CAT is blocked, follow these steps:

  1. Open the Solution Explorer pad.

  2. Find the CAT in the Libraries folder.

  3. Open Application.

  4. Right click on the CAT and select Copy.

  5. Expand the CAT folder from your project.

  6. Right click on Application and select Paste.

  7. Name the CAT inside the Define new name dialog box.

  8. Click OK.

The copied CAT is then unlocked.

Modify the Symbol

The purpose of this section is to modify the symbol to add dynamic text to display the different names.

To modify the symbol, follow these steps:

  1. Open the Solution Explorer pad.

  2. Find the CAT with the symbol you want to modify.

  3. Expand the CAT and its CAT HMI nodes.

  4. Open the symbol that you want to modify.

    NOTE: If the CAT symbol is locked refer to the section Unlocking the Symbol of a CAT.

  5. Open the following drop down list from the graphic editor toolbar:

  6. Select on Asset Tag Text or Asset Tag Label and place it on the CAT symbol.

    The difference between these two elements are that with Asset Tag Label you can align the text, whereas with Asset Tag Text you can not, which can cause layout problems. This difference only impacts the aspect of the name and not the behavior of the Asset Tag displaying.

    NOTE: To know more about of Asset Tag Text and Asset Tag Label, refer to the topic Graphic Editor Toolbar.

    If you choose Asset Tag Label and want to align the text in the label, follow these steps:

    1. Click on the text of the Asset Tag Label.

    2. Open the Properties pad.

    3. Go to Appearance > TextAlignment.

    4. Click on the text TextAlignment property and click on its drop down list arrow.

    5. Pick the alignment of your choice.

The symbol is now modified, save the modifications. Repeat these steps for each symbol you want to modify.

Set the CAT Instance in the Project

Follow the next steps to create an instance of a CAT in your project and set its different names:

  1. Open the Function Blocks Network editor.

  2. If not already done, insert the wanted CAT in the editor.

  3. Right-click on the CAT instance, click on Mapping and then select a resource.

  4. Save and build the project.

    NOTE: If type cast errors appears when building, refer to the section Remove Type Cast Errors.

  5. Define the Asset Tag name.

  6. Define the HMI Alias.

The CAT instance is created and its different names are set.

Remove Type Cast Errors

If type cast errors appear when setting of the CAT instance, follow these steps:

  1. Open the Solution Explorer pad.

  2. Right click on the project node and select Options.

  3. Click on IEC61499.

  4. In the Compiling tab, untick the checkbox Report type cast problems in composite FB networks as detected error.

  5. Save the modifications and build.

The type cast errors are now considered as warnings, meaning that you can compile, deploy and run online the application to use the toggle button.

Add the Toggle Button

This section shows how to add the toggle button in the resolution depending on your situation:

  • You have not yet created a resolution:

    1. Open the Solution Explorer pad.

    2. Right click on Canvases in the project.

    3. Select New Resolution.

      RESULT: The Start Canvas dialog box appears.

    4. Define the size of your resolution.

    5. Tick the checkbox Display Alias Name Toggle Button.

    6. Click Finish.

  • You have already created a resolution:

    1. Open the Solution Explorer pad.

    2. Expand Canvases and right click on the resolution.

    3. Select Redefine Start Canvas with wizard and click Yes on the Redefine Start Canvas pop-up.

      RESULT: The Start Canvas dialog box appears.

    4. Tick the checkbox Display Alias Name Toggle Button.

    5. Click Finish.

The toggle button is now added to the canvas.

Use the Toggle Button

To use the toggle button, follow these steps:

  1. Open the Solution Explorer pad.

  2. Expand Canvases.

  3. Right click on the resolution and select Start Online HMI.

  4. Click on the toggle button:

  5. Select Instance name, Asset Tag and Alias Name.

You can now display the instance name, the Asset Tag name, and HMI Alias name with the toggle button. When selecting the names with the toggle button, the text dynamically changes and shows the names defined in the section Set the CAT Instance in the Project.
