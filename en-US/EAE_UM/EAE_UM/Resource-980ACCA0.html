Resource

An IEC61499 resource is a functional unit contained in a device that has independent control of its operation. A resource may be created, configured, parameterized, started up, deleted, and so forth, without affecting other resources.

The functions of a resource are to:

  • Accept data and/or events from the process and/or communication interfaces.

  • Process the data and/or events.

  • Return data and/or events to the process and/or communication interfaces, as specified by the applications utilizing the resource.

In EcoStruxure Automation Expert, function blocks can inserted in one or more resources or Mapped from an Application, and need to be properly connected.

If you open a resource it will be shown in the Resource editor.

Prioritizing a Resource

You can prioritize a Resource. During execution, function blocks mapped to a higher priority resource will not be interrupted by execution of function blocks mapped to a lower priority. 

For applications where I/O jitter may be a concern, dedicate a resource to the bus master (EtherNet/IP, Modbus client and server, TM3) with the highest priority. To avoid introducing additional jitter, configure a short process time for Symlinks linked to these I/O.​ Refer to the Applications Design Guidelines topic Reducing Jitter in an EcoStruxure Automation Expert Application for more information.

NOTE:

  • Resource prioritization is supported only on Modicon M262 Distributed PAC and EcoStruxure Automation Expert - Soft dPAC.

  • Resource prioritization is not designed to be a means of sequencing program execution, especially in a multi-core environment. In a multi-core environment, function blocks mapped to Resources with different priorities may be executed in parallel on distinct cores, depending on CPU load.

Using Symbolic Links (Symlinks) to Connect Multiple Resources

You can use Symlinks to connect multiple resources within the same device. For example, you can use Symlinks to connect multiple resources in a design where:

  • The application is mapped to one resource.

  • The hardware configuration is mapped to a different resource in the same device.

The Symlinks pad contains the Symlinks from every resource within the device.

This feature is supported for device libraries and EcoRT version 23.1 and later.
