Attributes

The Attributes dialog presents the attributes of the selected function block for viewing and editing.

Accessing Attributes

You can open this dialog in several ways:

  • In the Interface Editor for a function block, place the cursor in the Attr column, then click the ellipsis (...).

  • Right-click on the function block (or on any selected function block) and click Attributes

  • Right-click on the background of the Device Editor, the Resource Editor, the Application Editor, or the topmost level of the System Editor and in the context menu select Attributes.

  • When creating or editing a composite or subapplication function block, right-click on the background of the Composite Function Block or Communication Network editors and select  Attributes.

    Click the ellipsis in the Attributes row of the Properties pad when function block properties are displayed. 

Customizing the View

The order of the entries within the columns can be changed as follows:

  • a click on the column header executes a sorting of the list by the selected column (a secondary column is removed from the sorting)

  • two columns can be selected in common, to sort for e.g. Name AND Attribute at once: the secondary column can be defined with Ctrl+click.

  • the next Ctrl+click on the same column changes the sorting order.

  • the next Ctrl+click on another column changes the secondary column

Viewing Multiple or Nested Function Blocks

The Attributes dialog lists the attributes of both the selected function block and the contained blocks. The Attributes dialog can also display a combined view of more than one selected function block. Displaying the attributes of an entire application or resource enables you to efficiently engineer the attribute values.

Filtering Attributes

Open the Filter Attributes dialog by clicking the Filter icon at the top left of the Attributes dialog. Use this feature to filter the attribute list according to, for example, an element name or element type. That helps to keep track of a large number of attributes, even if you have displayed for example all attributes of an entire application or resource.

Using the Filter Attributes dialog, you can filter on the following attributes:

  • Attribute Name

  • Context / Is Empty

  • Value / Is Empty

NOTE: The filter is case sensitive.

It is also possible to define several expressions in one line, by separating them with a semicolon (;). (e.g. at Attribute Name: HMI.AlarmClass;HMI.AlarmText)

The fields Context and Value may contain several empty cells. Use the Is Empty check boxes to filter for attributes of that kind.

NOTE: The filter can also be used to show specific values of "sub-attributes". That means it is also possible to search e.g. within attribute TrendingAndArchiving in the AttributeCollection for specific values of depth, value, time, etc. For example:

  • All attributes with activated archiving: *Runtime.Archive.Id Value=1 *

  • All attributes with archive smoothing = 10: *Runtime.History.SmoothValue Value=10 *

To filter for precise content:

  1. Use Attribute types ....

  2. Open the superior attribute (in our example TrendingAndArchiving > AttributeCollection).

  3. Select the text and insert it with Ctrl+C and Ctrl+V into the Filter Attributes dialog.

It is necessary to use the * in front and after the expression to receive a correct result.

A right-click on an entry or a couple of selected entries (use Shift-click or Ctrl-click) within the Attributes dialog opens a context menu, the contents of which can vary depending on the selected attribute type, and can include the following items:

Command

Description

Default value

Sets back the attribute value to default value, which is that value that has been predefined for that attribute of the element (Device, CAT, ...).

Copy

Copies the value of the selected attribute to the clipboard. This value can be inserted into another attribute directly, or into a text file or Excel-chart (as CSV list).

NOTE:  In some Excel versions (e.g. German version) it may be necessary to use the 'Textconversion assistant' when copying the content of the clipboard to Excel-chart. Already inserted text, when it is inserted only in one column, can also be converted to the correct format by the use of this assistant (e.g. at Microsoft Excel 2010 → Data/Text to Columns). Choose option 'Delimited' and then set the check box at 'Comma'. Clear the other check boxes.

In this way the stored attributes can also be re-imported to the Attributes dialog. Verify that every attribute is complete, containing the label and every column.

Import

Inserts the value, stored in the clipboard, to the selected attribute(s). Attribute values which are not selected will not be changed, even if the values are stored in the clipboard.

Verify that the attributes, which have to be imported from Excel chart, are completely selected. That means the selection includes label as well as every column and row belonging to the attribute, and the selection is then copied to the clipboard.

Select all

Selects all attributes of the active tab.

Importing Attributes and their Values

Importing attribute values is also possible via drag & drop. That means, a series of attributes can be selected from a text file or excel-table and directly dragged into the attribute window. In this case, different from import via context menu, the values stored to clipboard are copied, no matter whether some attributes are selected.

Overparameterizing Attributes

Attributes can also be overparameterized. That means, when an attribute of a function block has already been set, this attribute setting is valid for all instances of this function block. If that attribute of one instance is subsequently provided with another value, the new setting is valid for only that one instance. All other instances retain the predefined value.

While creating your own CATs, you can parameterize individual data inputs - which are designated to be used with a constant - in such a way that they offer a list of constants for the user at runtime. In order to prepare a data input for that purpose, open the Attributes dialog and set the check mark in the checkbox for the attribute ParameterEditor.

You can open a dialog for parameterizing by clicking on the ellipsis in column Value of row ParameterEditor. In this dialog you can define how the parameter will be offered (for example, Masked Text Box, Multiline Text Box, Numeric Up Down, Combo Box, Custom).

For example, for a parameter defined to be offered in Combo Box style, the entries to be offered in the Combo Box can be defined freely.

For Combo Box, there are two options:

  • Define only the entries; the value which has to be passed is determined automatically (the topmost entry uses value 0, the second entry from above uses value 1, etc.)

  • Define the entries and assign each of them the value that has to be passed, (for example, textvalue1=0, then textvalue2=1, and so forth). Note that in this case there is no space between the text, the =, and the value.

Set the check mark at Enumeration so that only the defined entries are offered for the user. If the Enumeration check mark is not set, an additional text field will be offered to the user at runtime, which could be used to set an unintended value apart from the predefined entries.

NOTE: An example for the use of attributes can be found on page Symbol Editor at the description of the TextBox.
