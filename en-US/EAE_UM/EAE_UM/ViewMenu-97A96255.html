View Menu

The View menu presents a ribbon with groups of commands for opening the system file, general viewing, debugging, window layout, plus a command for closing all open tabs. View menu areas include:

  • Pad area, which includes:

    • System File

    • General pads

    • Debug pads

  • Window Layout area

  • Asset Tag button

  • Window area

  • Security Management area

Window Layout Area

In this area are available all the commands needed to control the window layouts. To know all the actions available with the layouts, check the topic Handle Custom Layout.

Here are the descriptions of all these commands:

Command

Description

Default

Accessible in the drop down list, it displays the default layout as the user defined it before creating the first custom layout.

Debug

Accessible in the drop down list, it displays the Solution Explorer pad, the Resource editor, and the Message Log pad.

Plain

Accessible in the drop down list, it displays only the Resource editor.

[...]

Accessible in the drop down list, it lists all the custom layouts one below the others. If no custom layout is defined, nothing is displayed.

It displays the custom layout selected.

Edit the custom layouts

Accessible in the drop down list, it opens Edit Custom Layout dialog box.

Reset the layout

Accessible in the drop down list, it resets the current selected layout to its original layout.

NOTE: This command is available with the predefined layouts (Default, Debug and Plain) and the custom ones.
Update the layout

Accessible in the drop down list, it updates the selected custom layout to the current layout. Next time this custom layout is selected, this newly saved layout will be displayed.

NOTE: This command is available only with the custom layouts. If a predefined layout is selected, this command is disabled.
Toggle Grid

Available directly in the View ribbon, it turns the grid ON and OFF for the Resource editor.

Show/Hide Print Layout

Available directly in the View ribbon, it turns the print layout ON and OFF.

Asset Tag button

The Asset Tag button allows to show and hide Asset Tag names of the CAT instances and their sub-CATs of the application.

Clicking on the button to switch between these two positions:

  • Green position: All the Asset Tag names defined are shown. The application instance naming then follows these standards depending on the pad or editor opened:

    1. On pads or editors listing the application instances by name: Only the Asset Tag names of the application instances are shown. If an application instance doesn’t have Asset Tag name, it will be displayed in italic with its instance name.

    2. On the other pads or editors: The application instance name and Asset Tag name inside parenthesis appear. If an application instance doesn’t have Asset Tag name, only the application instance name will appear.

  • White position: Only the application instance names are displayed, the Asset Tag names are hidden.

Thus, when the Green position is select, here are pads and editors following the first standard:

  • The CAT Instances sub-node of the Solution Explorer pad.

  • The Hierarchy View pad.

  • The OPC UA Server editor.

  • The Offline Parametrization editor.

  • The Watch Trend pad.

  • The ASP Asset Mapping editor.

    NOTE: This editor is available in the System editor if the Asset Link for AVEVA OMI option is selected during the installation of EcoStruxure Automation Expert.

Here are the pad and editors following the second standard:

  • The Function Block Network editor.

  • The Function Block Mapping editor.

  • The Symbolic Link editor.

    The Symbolic Link pad.

When an Asset Tag name is modifying, it will automatically update that modification in all these pads and editors.
