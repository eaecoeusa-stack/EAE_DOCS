History Trend Manager

You can use the History Trend Manager in two contexts, to perform the following tasks:

  • Create static configurations in EcoStruxure Automation Expert - Buildtime that can be applied to archived data and used as a data source by an HMI client. In this context, access the dialog in Solution Explorer  by selecting the project node, then right-click and select Configurations > History Trend.

  • Select existing configurations for an HMI client during runtime to apply to the history trend control displayed in a canvas during runtime. 

NOTE:

  • A locked icon indicates the history trend configuration is not editable - for example, a configuration created in buildtime that is displayed during runtime.

  • An unlocked icon indicates the configuration is editable - for example, a configuration created in buildtime accessed during buildtime, or a configuration created during runtime that is accessed during runtime. 

Managing History Trend Configurations

The History Trend Manager presents the following areas and tools:

Area

Description

Configurations

Contains a collection of folders and configurations, presented in nested navigation tree form. The following context menu commands are available.

When you right click on a folder, the following commands are available:

Command

Description

New

Opens the History Trend Configuration dialog, where you can create a new configuration.

New Folder

Opens the Add Folder dialog, where you can create and name a new child folder that will be nested beneath the selected parent folder. 

Rename

Opens the Name dialog, where you change the selected name.

Duplicate

Opens the Add Folder dialog, where you can add the name of the folder that will be a copy of the selected folder. 

NOTE: All child folders and configurations of the selected folder are also copied and nested beneath the new folder.

Delete

Deletes the selected folder.

NOTE:  You can only delete the lowest level nested folder.

When you right click on a configuration, the following commands are available:

Command

Description

Edit

Opens the History Trend Configuration dialog, where you can edit the properties of the selected configuration.

Rename

Opens the Name dialog, where you change the selected configuration name.

Duplicate

Immediately creates a copy of the selected configuration, assigning it a unique name by adding the next available integer as a suffix to the name of the selected configuration. The new configuration is placed beneath, but on the same nested level as the original configuration.

Delete

Deletes the selected configuration.

Description

The description of the selected configuration that was entered in the History Trend Configuration dialog.

Runtime Directory

Click the ellipsis (...) to open a the Browse For Folder dialog where you can navigate to and select the directory in which the configuration will be saved from HMI runtime. 

NOTE: If you leave this field empty, the configuration will not be saved.

Creating and Editing History Trend Configurations

When you select New to create a new configuration, or Edit  to edit an existing configuration, the History Trend Configuration dialog opens. 

The dialog presents the following features:

Setting

Description

Configuration

The name you assign to the configuration. 

Description

The short description of the configuration viewed from HMI runtime. 

Title

The label displayed by the History Trend control during runtime.

Time

Sets the time period - in relation to runtime - for displaying archived data.

Click the ellipsis (...) to open the Time Period dialog, where you can select one of the following settings:

  • All

  • Last Five Minutes

  • Last Ten Minutes

  • Last Hour

  • Last Two Hours

  • Last Day

  • Last Week

  • Last Month

  • This Hour

  • This Week

  • This Month

  • From Time: Displays additional settings where you can specify a start date and a time period.

  • From Time To Time: Displays additional settings where you can specify a start date and end date.

Series

Specifies the variables to include in the configuration. 

Click the ellipsis to open the TrendSeries Collection Editor dialog.

In the TrendSeries Collection Editor dialog, you can:

  • In the Members list, select variables to include in the configuration.

  • Configure display settings for the variable selected in the Members area.

You can configure the following properties for a variable that is a member of the configuration:

Parameter

Description

Color

The editable color of the line used to display values for the selected variable.

DataLimit

The maximum number of data points to display for the variable.

LegendFormat

The format for the display of the variable data value in the legend.

NOTE: The legend displays at runtime.

LegendName

The label that displays for the variable in the legend during runtime.

Maximum

Used in scaling, this setting is mapped to the maximum variable value.

Minimum

Used in scaling, this setting is mapped to the lowest variable value.

PenStyle

The line style: solid or dashes.

PenWidth

The line width in points.

ScaleName

The name of the value scale assigned to this variable.

NOTE: By default, Left and Right scales are available. In addition, you can create custom scales using the Value Scales settings.

ShowValueInLegend

Shows (True) or hides (False) the variable value in the legend during runtime.

Stairs

Displays the value line as a series of steps (True) or as a straight line (False).

Click Add  to open the Select Archive Path  dialog, where you can select - one at a time - variables to be added to the list of available variables in the Members area of the  TrendSeries Collection Editor dialog. Click Remove to delete a variable from the list.  

Value Scales

Specifies the default or custom scaling display to apply to the configuration.

Click the ellipsis (...) to open the TrendSeries Collection Editor.

The properties in this dialog define how the axis will be presented By default, this editor presents a left and right scale, indicating the location of the scale will be on the left or right side of the history trend display. Click Add to create additional scaling templates for left-side scales. 

This dialog contains the following properties:

Parameter

Description

AutoScale

Scales the axis automatically to the maximum and minimum range of scaled values (True). Or displays non-scaled data (False).

EndPosition

Percentage of the chart that is set as the scale's maximum value.

Format

The format for the display of the variable data value in the axis.

MasterScale

Determines the scale's position when more than one left scale is included. For example, when MasterScale is set as Left for CustomLeft, then CustomLeft scales are also aligned in the same place (Left).

Maximum

The axis value mapped to the variable maximum value.

NOTE:  0 indicates actual variable value is displayed.

Minimum

The axis value mapped to the variable minimum value.

NOTE:  0 indicates actual variable value is displayed.

SeriesMustBeInsideRange

When set to True, all series using this scale are shown in scale's range. Therefore, the Minimum and Maximum scale values are automatically set based on the series' Maximum and Minimum values.

ShowOnlyMinAndMax

Displays only the maximum and minimum values in the scale (y-axis).

ShowScrollArrows

Displays scroll arrows in the vertical axis when runtime values exceed the scale's Maximum and Minimum values - i.e., as is done with the arrows in the horizontal axis.

StartPosition

Percentage of the chart that is set as the scale's minimum value.

Visible

Indicates the axis, with its values and labels, is displayed (True) or hidden (False).
