Import Modbus Mapping File

The second step of the automatic generation of the Modbus registers is to import the Modbus mapping file previously created in EcoStruxure Automation Expert.

The import will be done in the Hardware Configuration editor, into the bus and Modbus devices previously set up:

  1. Right click on the Modbus device and select Import Modbus Gateway.

  2. In the Import Options dialog, first choose between:

    • Suffix variable name with device name: It will add the name of the Modbus device after the name of each generated Modbus register to be sure they all have an unique name.

    • Prefix variable name with device name: It will add the name of the Modbus device before the name of each generated Modbus register to be sure they all have an unique name.

    NOTE: The preference can be stored and skipped for the next imports. Find this preference in the Settings of EcoStruxure Automation Expert section Hardware Configuration > Modbus Gateway.
  3. Then select the Modbus mapping file where it is stored and open it.

  4. A confirmation dialog then appears asking for generating the associated symlinks. Indeed, each Modbus register variable is associated to a symbolic link. It creates the link between the hardware function block Modbus register and the application function block. Thus, when the process value is updated, it is transmitted to its connected symlink which transmits the new value to the hardware Modbus register to communicate it through Modbus.

    In the confirmation dialog, two options are available:

    • Yes: In addition to the creation of the Modbus registers, a new symbolic link will be added for each of them. The symlinks will be added to the resource network in the Function Block Network editor. These symlinks will be set up accordingly to their Modbus register characteristics and they will be associated to their Modbus register in the Hardware Configuration editor.

      NOTE: This option is recommended for new application or when symlinks don’t already exist.

    • No: Nothing else than the Modbus register will be created. The user will then have to manually associate symbolic links to the Modbus registers, as described in the topic Finalize Generation.

      NOTE: This option is recommended for existing application since symlinks already exist.

After answering this last question, the Modbus registers will be generated. Expand the Modbus device to discover them.

Few additional steps have to be done after this generation to make sure the Modbus registers correctly communicate. Check the topic Finalize Generation to see all these steps.
