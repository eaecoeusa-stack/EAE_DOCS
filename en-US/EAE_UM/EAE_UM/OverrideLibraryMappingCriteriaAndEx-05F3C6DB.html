Override library CATs mapping criteria and exclude library CATs from auto-mapping

The default criteria of the library SubCATs can be overridden in the following scenarios:

  • Engineering instance mapping criteria specified for library CATs is not meeting your project requirements in auto-mapping.

  • Library CATs which you want to use in auto-mapping have no engineering instance mapping criteria specified.

One or several libraries can also be excluded from the auto-mapping process. When a library is excluded, all its CATs are excluded from the process, which means:

  • There would not be bulk created CAT instances from the CATs of excluded library.

  • SubCATs in the created CAT instances, which belong to CATs of excluded library, would not be associated with any engineering instance. When a SubCAT is not associated with engineering instance, its offline parameter values will not be updated.

Step Action

1

In the Asset Link for Bulk Engineering tab, click the icon.

2

In the Library Rules Configuration tab, are hierarchically listed the library CATs. Drill down the CATs where rules need to be overridden.

NOTE: The SE.AppCommonProcess > SignalProcessing node is of major interest as it contains several default rules.

Proceed as in the Set the Mapping Criteria for the Engineering Instances topic.

NOTE: if one of the mapping criteria of a node is modified, then the * is appended to the name till the modification is saved.

3

If needed, uncheck a library in the left pane to exclude its CATs from the auto-mapping.

4

Click Save.

NOTE: If during the saving validation errors are found, a pop up will appear to inform you about it. Then go back to the tool tips to find all CATs with validation errors, they are displayed with the following icon: , then navigate through them to fix the errors.

5

If you want to revert to the default rule of a terminal node, right-click it and select Reset.

To revert the exclusion of a library, tick the check box of the library.
