Device actions

 WARNING
AUTOMATIC APPLICATION START AFTER DEVICE REBOOT
Ensure that the application can start safely after a voluntary or involuntary device reboot.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

The Action Menu Device actions item links to the following commands:

Command

Description

Run
(Single Run -
HA Partner)

Starts the execution of the application, after a full deploy of the application.

Refer to the Operating States chapter for information on how the Run command impacts device operating states.

NOTE: If both main and diagnostic interlinks are broken, and the other Partner is already running, this action would result in both Partners running concurrently.
 WARNING
concurrent running devices
Before sending a RUN command to a single HA Partner, ensure that the system will not result having the two partners concurrently running.
Failure to follow these instructions can result in death, serious injury, or equipment damage.
Run
(HA Pair)

Starts the execution of the application in two high availability paired controllers. . Refer to the topic describing the HA Pair level command Run in the EcoStruxure Automation Expert – HA Soft dPAC User Guide.

Stop
(Single Stop -
HA Partner)

Stops the execution of the running application.

Refer to the Operating States chapter for information on how the Stop command impacts device operating states.

Stop
(HA Pair)

Synchronizes the stop execution of the application running in both high availability paired controllers. Refer to the topic describing the HA Pair level command Stop in the EcoStruxure Automation Expert – HA Soft dPAC User Guide.

Single Restart Runtime
(HA Partner)

Stops the application and the firmware execution. Restarts the firmware and loads the Boot Project (if any). For an HA Partner running in Paired role, this command causes a switchover to the HA Partner Soft dPAC.

Refer to the topic describing the HA Partner Paired role in the EcoStruxure Automation Expert – HA Soft dPAC User Guide.

Refer to the Operating States chapter for information on how the Restart runtime command impacts device operating states.

Reboot Device

Re-starts the device and applies the stored boot project settings: The reboot action does not include any changes that were deployed to the device and not saved to the boot project. A boot project exists only if deployed settings are saved to a boot project. In the absence of a stored boot project, the application is lost when the device is re-started because there is no application to load into the device.

NOTE:  After a successful reboot, the behavior of the application depends on the attribute (set in the Logical Devices editor) Runtime ConfigurationBootBehavior on reboot, as follows:

  • Stop: the device enters Stopped state.

  • Run: the device enters Running state, if a boot project exists.

  • LastState: the device re-enters the state it was in before the reboot.

  • LastStateIgnoreErr: the device attempts to re-enter the state it was in before the reboot, even if that state was the detected error state or the halt state.

Refer to the Operating States chapter for information on how the Reboot Device command impacts device operating states.
