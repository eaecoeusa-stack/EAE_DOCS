** Comment on the default value for DT:

1. The first permissible value is DT#1970-01-01-00:00:00, however, when using this value as input value in the test container, consider the time zone of the test computer. The input in the test container is applied with local time. Example: When using UTC +1 the first theoretically permissible input value for an input in the test container thus would be DT#1970-01-01-01:00:00.

2. Furthermore, in the immediate border area of the value limit inaccuracies may occur, due to data type conversions depending on the platform used. It is therefore strongly recommended to select a later date also for test purposes, since the accuracy of the calculations in the immediate border area of the permissible value for data type DT is not certain. (In most cases any date later than DT#1970-01-02-00:00:00 should work correctly already.)

*** Comment on the length of STRING variables:

For structured text-based Basic function blocks, by default EcoStruxure Automation Expert Buildtime creates a variable that can contain up to a maximum of 15 characters. To be able to process a longer STRING variable, the length of the variable can be defined in square brackets (example: STRING[60]). See also the explanation for column Type in Function Block Interface Editor.

NOTE: The use of special characters in string variables is generally not recommended in EcoStruxure Automation Expert Buildtime and should be avoided. Special characters in constants are not allowed by the compiler and therefore lead to a detected error message during compilation.

Arrays

An array contains an indexed list of elements of the same data type. The array elements can be addressed with the field index.

The initial and final value are specified via the array definition.

The maximum array size is 65535 bytes.

Syntax: 

Field_1 : ARRAY[1..5] OF INT; 
Field_2 : ARRAY[1..5,0..7] OF INT;

Example for accessing array elements within algorithms: 

Field_1[1] := 15;
Field_1[2] := 4;
 Field_1[3] := 32;
 Field_1[4] := 9;
 Field_2[1,3] := 55;
 siElement := Field_1[5];
 siElement := Field_2[1,3];
