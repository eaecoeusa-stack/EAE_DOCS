Reliable Cross Communication

Reliable Cross Communication provides a greater degree of certainty – than standard cross-communication – to the transmission of events and data.

You can configure Reliable Cross Communication by performing the following tasks in Installing EcoStruxure Automation Expert – Buildtime:

  • Enable Reliable Cross Communication for all devices of a solution.

  • Configure Reliable Cross Communication default settings for all devices of a solution.

  • Configure Reliable Cross Communication for specific devices.

  • Override the Reliable Cross Communication default settings for specific communication channel and messages (bundles of events with variables).

NOTE: To reduce the memory, processing and bandwidth required by device communication:

  • Use the Send and Forget (AtMostOnce) QoS setting. The Resend a message if it is not acknowledged (ExactlyOnce) setting causes a message to be sent at least twice, and therefore requires at least twice as much memory, processing, and network bandwidth.

  • In the Properties pad, navigate to and if necessary edit the property Communication > Reliable CrossComm > Protocol > DisableReliableQoS setting to True. The default setting is True for the ATV, M251, M262, and M580 dPACs; False for the HA and non-HA (Linux and Windows) Soft dPACs.

Enabling Reliable Cross Communication

To enable Reliable Cross Communication for all solution devices:

  1. In the Solution Explorer, right click on the project node and select Options > IEC61499.

  2. Click the Communication tab.

  3. Select Enable Reliable cross Communication protocol for all devices.

NOTE: If you do not-select the Reliable Cross Communication option, standard cross-communication will be applied. The selection of cross-communication or Reliable Cross Communication is project-wide and becomes the default setting for every solution device.

Configuring Solution-Wide Reliable Cross Communication Default Settings

After enabling Reliable Cross Communication, you can configure solution-wide Reliable Cross Communication default settings in the Communication tab as follows:

  1. Select a Quality of Service (QoS) Level for Reliable Cross Communication:

    • Resend a message if it is not acknowledged: Applies the ExactlyOnce strategy. Sent messages that have not been acknowledged are retransmitted for a user defined period of time until they are acknowledged, or until a user defined time period has elapsed. Select this for sending events (with or without variables) that must be received, for example control commands.

    • Send and Forget: Applies the AtMostOnce (also called the Best Effort) strategy. A loss of sent messages is permitted. Select this for sending cyclic measurements that are frequently updated with new values.

    NOTE: The ExactlyOnce QoS level requires more resources than AtMostOnce, including RAM usage, CPU load and network load.

  2. Select a Max Buffer Time: The time the sender buffers unacknowledged messages so they remain available for retranmission. Applied if you selected Resend a message if it is not acknowledged.

  3. Select a Packet Rate: The minimum time that can elapse before re-transmitting a message. Applied if you selected Resend a message if it is not acknowledged.

Configuring Reliable Cross Communication for Specific Devices

By selecting a specific device in the Logical Devices editor, then navigating to Communication > Reliable CrossComm > Protocol in the Properties pad, you can edit properties that will:

  • disable Reliable Cross Communication for the selected device

  • optimize the transmission and processing of unfragmented SendAndForget messages for the selected device

  • cause queued messages to be transmitted as soon as possible.

NOTE: Only users assigned the Advanced user level can edit the following properties, which apply to messages when Reliable Cross Communication is enabled in the Communication tab.

DisableReliableQoS : Setting this property to:

  • True disables the reliable transmission mechanism (Resend a message if it is not acknowledged / Exactly Once) for a selected device and reduces resource usage (RAM, CPU and network load). Only the AtMostOnce QoS protocol is supported.

    NOTE: For the ATV dPAC and the M251 dPAC controllers, True is the default setting and only the AtMostOnce QoS level is supported. You can override this by setting the parameter to False.

  • False allows a selected solution-wide default QoS level: ExactlyOnce or AtMostOnce. More resources are used.

    NOTE: For the Soft dPAC and HA Soft dPAC controllers, False is the default setting.

SendAndForgetShortcut: Setting this property to:

  • True (default) optimizes the transmission and processing of unfragmented SendAndForget messages.

  • False increases the CPU load used to perform the transmission.

NOTE: This setting applies to messages when the QoS Level is set to Send and Forget in the Communication tab.

AsynchronousTransmission: Setting this property to:

  • True (default) causes queued messages to be transmitted as soon as possible and helps reduce system jitter.

  • False causes queued messages to be transmitted upon the expiration of the communication timer PeriodicTickMs setting.

    NOTE: This setting applies to messages of all QoS levels when Reliable cross Communication is enabled.

Overriding Reliable Cross Communication Settings for Specific Communication Channel and Messages (Bundles of Events with Variables)

You can override solution-wide Reliable Cross Communication settings set in the Communication tab.

To override the Max Buffer Time and Packet Rate settings for a specific communication channel:

  1. Open the Communications Network editor.

  2. Select a device communications channel.

  3. In the QoS Parameters column, click the ellipsis (...).

  4. In the dialog, you can edit the Max latency (i.e., the Max Buffer Time) and the Packet rate settings. Click OK to save your edits and close the dialog.

To override the QoS Level setting for a specific event:

  1. Click in the QoS column.

  2. Select a QoS Level value:

    • Resend a message if it is not acknowledged

    • Send and Forget

NOTE: Only users assigned the Advanced user level can override these settings.

Reliable Cross Communication impact on VTQ Variables

When using Reliable Cross Communication to send VTQ (value - time - quality) variables, a change of state in the connection generates an event modifying the quality of the VTQ variables:

  • Establishing a connection sets the quality to UncertainLastUsableValue (0x40900000 hex, 1,083,179,008 dec).

  • A disconnection sets the quality to BadCommunicationError (0x80050000 hex, 2,147,811,328 dec).
