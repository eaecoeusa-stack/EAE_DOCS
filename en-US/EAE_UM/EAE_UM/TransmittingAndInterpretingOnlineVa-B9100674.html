Transmitting and Interpreting Online Values

At this point, the configuration of the Modbus communication and Modbus registers are complete, let’s:

  • Transmit Online Values.

  • Interpret the Online VTQ Values.

Transmit Online Values

Start the online Modbus transmission of values as followed:

  1. Build the solution.

  2. Open the Deploy and Diagnostic editor.

  3. Deploy and run online the solution.

  4. Start the client or server that EcoStruxure Automation Expert is dialoguing with.

The current value transmitted is now transmitting the configured values through Modbus communication.

To check what value is transmitted:

  1. Open the Hardware Configuration editor.

  2. Expand the Modbus communication and a register.

  3. Right click on the variable of the register and select Watch.

The transmitted value then appears in the Watch column of the selected variable.

To exploit this transmitted value, search in the Function Block Network editor the symbolic link associated with this variable.

Interpret the Online VTQ Values

Unlike the other types of variable that only transmit value, the VTQ variables are transmitted three different types of values at once:

  • Value: Contains the value of the defined type.

  • Time: Contains the time of the last value transmitted.

  • Quality: Contains a value that indicates the current quality of the Modbus communication. Find below a table explaining what is the quality for each possible value.

Column description:

  • VTQ Value: Lists all the quality value possible.

  • VTQ Error: Gives the VTQ error associated with a VTQ quality value.

  • Modbus Standard Exceptions: Inform if the quality value is coming from a Modbus standard exception or if it is specific to EcoStruxure Automation Expert.

  • Modbus Error Code: It is the Modbus error code shown either in:

    • The Diagnostics and SlaveID fields of the Modbus TCP/RTU Coupler symbol of the function blocks MODBUSGENTCPS or MODBUSGENSERIALS.

      NOTE: The symbol only shows the last error code appearing of all the register devices communicating.

    • The error description appearing in the Device Log Viewer dialog box. Open this dialog box from the Deploy and Diagnostic editor, then select action Log file > Fetch Log Files.

  • Remarks: Gives short description of the quality error.
