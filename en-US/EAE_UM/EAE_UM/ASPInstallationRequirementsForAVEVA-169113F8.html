ASP Installation Requirements for AVEVA OMI

Supported Installation Environments

AVEVA OMI supports the following installation environments:

If both EcoStruxure Automation Expert and AVEVA System Platform (ASP) are installed on the same machine, AVEVA OMI installation requirements include:

  • EcoStruxure Automation Expert

  • ASP All-In-One Node setup

NOTE: Licenses for both EcoStruxure Automation Expert and ASP are activated.
EcoStruxure Automation Expert and ASP are installed on different machines, a DEV Client and a GR Node, AVEVA OMI installation requirements include:
  • On the DEV Client:

    • EcoStruxure Automation Expert

    • ASP - DevClient Setup

    • Reference GR Node details in the ASP Configurator window of the System Management Server

    NOTE: Licenses for both EcoStruxure Automation Expert and ASP are activated.
  • On the GR Node:

    • ASP - All-In-One node Setup

    NOTE: ASP license to be activated.

Operating System Minimum Requirements

A 64-bit Windows operating system is required for installing and running the latest patch of System Platform 2023 R2 SP1 and its component products. Some System Platform 2023 R2 SP1 components, such as the Application Server Galaxy Repository and the AVEVA Historian also require a 64-bit version of Microsoft SQL.

NOTE: To know the what is the latest patch of the AVEVA System Platform 2023 R2 SP1 supported with EcoStruxure Automation Expert, refer to the Software Compatibilities topic of the Compatibilities and Dependencies - User Guide.

Check the AVEVA GCS Technology Matrix Website for more information about supported Windows and SQL Server versions for the System Platform 2023 R2 SP1 component products you are installing.

The earliest versions of Windows that support latest patch of System Platform 2023 R2 SP1 are:

  • Long Term Servicing Channel

  • Windows 10 Enterprise, IoT Enterprise 2015 LTSB (1507)

  • Windows Server 2016 LTSC Standard and Datacenter

General Availability Channel:

  • Windows 10 21H2 Pro, Enterprise, and IoT Enterprise

See the GCS Technology Matrix for complete information about operating systems that you can use with the latest patch of System Platform 2023 R2 SP1 products.

System Sizing Guidelines

The following table provides guidelines for System Platform hardware configurations, based on application size. These guidelines are subject to the limitations of your Windows operating system, and if applicable, to the SQL Server edition (Express, Standard, or Enterprise). See the Technology Matrix on the AVEVA Knowledge and Support Center website for supported versions of Windows operating systems and SQL Server.

NOTE:

  • We recommend an HD display resolution for engineering tools such as the Integrated Development Environment (IDE).

  • A Windows Server operating system is required for large installations.

  • SQL Server Express is not supported for installations with more than 25,000 I/O per node (small installations only).

  • A 64-bit operating system is required, regardless of system size.

  • Pagefile.sys, the Windows paging file (also called the swap file or virtual memory file), must be enabled. The

    Windows default setting is enabled.

The following guidelines are provided for reference only. The system configuration required for your application will depend on multiple factors, including but not limited to the size and complexity of the application, and the features and components used.
