Editing a Logical Device Configuration

To edit a logical device configuration, several actions are available as listed below:

  • Access the properties

  • Find a specific property

  • Get property description

  • Modify one property

  • Visualize the custom properties

  • Revert a property to its default value

  • Impact of a property change

Access the properties

To access the properties of the logical device:

  1. Open the Logical Device editor, as described in the topic System Editor Overview.

  2. Select the logical device the need to be configured.

  3. Open the Properties pad, as described in the topic Properties Pad.

All the properties of the device are located in this pad, organized by category.

The properties available in this pad change depending on:

  • The device type.

  • The User Level, defined in the General Settings of EcoStruxure Automation Expert. If the User Level is:

    • Default, the most common properties are available.

    • Advanced, specific properties are added to the common ones.

Find a specific property

To find a specific property into the Properties pad:

  • Check manually the categories available in the Properties pad by expanding them and their sub categories.

  • Check the chapter Device Properties of SE.DPAC library documentation. All the properties for each device type are listed and described there.

  • Check the device documentation.

Get property description

To get the description and the use of a specific property:

  • Check the chapter Device Properties of SE.DPAC library documentation. All the properties for each device type are listed and described there.

  • Select the property and check at the bottom of the Properties pad to see its detailed explanation that includes:

    • A description of the setting.

    • The default value defined to the setting.

    • The unit of the setting.

    • The minimal value of the setting allowed.

    • The maximum value of the setting allowed.

    • The type of the setting.

    • A remark indicating when the setting takes effect.

Modify one property

To modify one specific property:

  1. Access to the properties

  2. Find the specific property

  3. Change the parameter value as wanted. The new value is included between the minimal and the maximal value specified in its description.

    NOTE: As described in the Visualize the Customized Properties section, when the property value is modified, it becomes highlighted by showing the value in bold. And as described in the Revert a Property to its Default Value section, if the value is set back to its initial value, its format is changed back to regular (the bold is removed).

  4. Check the impact of a property change.

Visualize the customized properties

A property is customized if:

  • The Network Profile is set to Default and its value is not equal to the default value specified in its description.

  • The Network Profile is set to Local Test and its value is not equal to the value of given to this property in the Default profile.

Thus, once a property value becomes customized, the property becomes highlighted by showing the value in bold.

In addition to this, all the parent categories of the property gets their values changed from DEFAULT to MODIFIED in bold.

Thus if a category has the value MODIFIED, it means that at least one of its properties is customized. Expand it and its MODIFIED sub categories to find all its customized properties.

Revert a property to its default value

To revert a property to its default value:

  1. Access to the properties

  2. Find the specific property

    NOTE: All its parent categories have their value set to MODIFIED.

  3. Then either:

    • If the Network Profile is set to Default, manually set the parameter value to default specified in its description.

    • If the Network Profile is set to Local Test, manually set the parameter value to the value given to this parameter in the Default profile.

    • Right click on the parameter value and select Reset.

The parameter value is then set back to default and its format is changed back to regular (the bold is removed). Thus, if its parent categories don’t contain modified parameters no more, then their value are changed back to DEFAULT.

Impact of a property change

When a parameter value is modified, perform deploy the device configuration to apply the modifications.

NOTE: For the parameters of the following categories, applying the modifications requires other actions:

  • Runtime Configuration: Restart the target runtime or reboot the device to apply the modifications.

  • OPCUA Stack Configuration: Restart the target runtime or reboot the device to apply the modifications.

  • NTP Server: Reboot the device to apply the modifications.
