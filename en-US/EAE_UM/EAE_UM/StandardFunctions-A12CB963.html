ABS

Computes the absolute value of the argument. 

(* ======================= *)
(* ANY_NUM := ABS(ANY_NUM) *)
(* ======================= *)
varABS := ABS(-1234); (* result varABS = 1234 *)
varABS := ABS(5678);  (* result varABS = 5678 *)

ACOS

Computes the arc cosine of the argument. 

(* ========================== *)
(* ANY_REAL := ACOS(ANY_REAL) *)
(* ========================== *)
varACOS := ACOS(1.0); (* result varACOS = 0.0 *)

ADD, ADD_DT_TIME, ADD_TIME, ADD_TOD_TIME

Adds the arguments. The number of arguments can be 1 or more. 

(* ========================================= *)
(* ANY_MAGNITUDE := ADD(ANY_MAGNITUDE, ....) *)
(* ========================================= *)
varADD := ADD(21,22,-23); (* result varADD = 20 *)

When adding time and/or date variables the number of arguments is two, as shown beneath. 

(* ============================== *)
(* TIME := ADD_TIME(TIME, TIME)   *)
(* DT := ADD_DT_TIME(DT, TIME)    *)
(* TOD := ADD_TOD_TIME(TOD, TIME) *)
(* ============================== *)
time1 := TIME#1m30s;
time2 := T#15s;
time3 := TIME#2.5m;
tod1  := TOD#18:30:25.500;
dt1   := DT#2011-05-23-16:20:55;
(* results are TRUE *)
out0  := (time1 + time2) = TIME#105s;
out1  := ADD(time1) = TIME#90s;
out2  := ADD(time1, time2) = TIME#105s;
out3  := ADD(time1, time2, time3) = TIME#255s;
out4  := ADD_TIME(time1, time2) = TIME#105s;
out10 := ADD_TOD_TIME(tod1, time3) = TOD#18:32:55.5;
out11 := ADD_TOD_TIME(tod1, T#12h) = TOD#6:30:25.5;	
out12 := ADD_TOD_TIME(tod1, T#-18h30m25.5s) = TOD#0:0:0;
out13 := ADD_TOD_TIME(tod1, T#-18h30m26.5s) = TOD#23:59:59;
out14 := ADD_TOD_TIME(tod1, T#-5d18h30m26.5s) = TOD#23:59:59;
out20 := ADD_DT_TIME(dt1, time1) = DT#2011-05-23-16:22:25;
out21 := ADD_DT_TIME(dt1, time2) = DT#2011-05-23-16:21:10;
out22 := ADD_DT_TIME(dt1, time3) = DT#2011-05-23-16:23:25;
out23 := ADD_DT_TIME(dt1, TIME#12h) = DT#2011-05-24-04:20:55;

AND

Returns a Boolean And linking of the arguments as the result of the function. The number of arguments can be 1 or more. 

(* ============================= *)
(* ANY_BIT := AND(ANY_BIT, ....) *)
(* ============================= *)
dint1 := 25;
bool1 := TRUE;
bool2 := TRUE;
word1 := 16#A5A5;
word2 := 16#6E6E;
(* results are TRUE *)
out0  := (AND(bool1));
out1  := (AND(bool1, bool2));
out2  := (AND(bool1, bool2, dint1 < 0));
out3  := (AND(bool1, bool2, dint1 < 0, dint1 = 25));
out4  := (AND(bool1, bool2, dint1 < 0, dint1 = 25, bool2));
out5  := (bool1 AND bool2);
out6  := (bool1 AND bool2 AND dint1 < 0);
out7  := (bool1 AND bool2 AND dint1 < 0 AND dint1 = 25);
out21 := (AND(word1) = word1);
out22 := (AND(word1, word2) = 16#2424);
out23 := (AND(word1, word2, 16#0F0F) = 16#0404);
out24 := (AND(word1, word2, 16#0F0F, NOT word1) = 16#0000);
out25 := (word1 AND word2) = 16#2424;
out26 := (word1 AND word2 AND 16#0F0F) = 16#0404;
out27 := (word1 AND word2 AND 16#0F0F AND NOT word1) = 16#0000;

real1 := 1.0;

ATAN

Computes the arc tangent of the argument. 

(* ========================== *)
(* ANY_REAL := ATAN(ANY_REAL) *)
(* ========================== *)
real2 := 1.0;
(* results are TRUE *)
out2 := ABS(ATAN(real2) + ATAN(-real2)) < 0.00001;

ATAN2

Computes the arc tangent by using two arguments. 

(* =========================== *)
(* ANY_REAL := ATAN2(ANY_REAL) *)
(* =========================== *)
real1 := REAL#4.0 * ATAN(REAL#1.0);
real2 := 1.0;
lreal1 := LREAL#4.0 * ATAN(LREAL#1.0);
lreal2 := 1.0; 
(* results are TRUE *)
out20 := ABS(ATAN2(real2, real2) - real1 / 4.0) < 0.0001;
out21 := ABS(ATAN2(lreal2, lreal2) - lreal1 / 4.0) < 1.0E-7;

CONCAT, CONCAT_DATE, CONCAT_DATE_TOD, CONCAT_DT, CONCAT_TOD

Combines the arguments to form one string. The number of arguments can be 1 or more.

Combines the arguments to DATE_AND_TIME, to TOD or to DATE. In this cases the number of arguments has to fit the type to be generated. 

(* ===================================== *)
(* ANY_STRING := CONCAT(ANY_STRING, ...) *)
(* ===================================== *)
str1 := 'abcdefghij';
(* results are TRUE *)
out3 := (CONCAT('AB', str1, 'YZ') = 'ABabcdefghijYZ');

(* ============================= *)
(* DT := CONCAT_DATE_TOD(D, TOD) *)
(* ============================= *)
date1 := D#2011-11-24;
tod1  := TOD#10:10:00;
dt1   := CONCAT_DATE_TOD(date1, tod1);
(* results are TRUE *)
out0  := dt1 = DATE_AND_TIME#2011-11-24-10:10:00.00000;

(* ======================================== *)
(* DATE := CONCAT_DATE(DINT, DINT, DINT)    *)
(* DATE := CONCAT_DATE(UDINT, UDINT, UDINT) *)
(* DATE := CONCAT_DATE(YEAR, MONTH, DAY)    *)
(* ======================================== *)
dintYear := 2011;
dintMonth := 11;
dintDay := 24;
date1 := DATE#2011-11-24;
(* results are TRUE *)
out1 := CONCAT_DATE(dintYear, dintMonth, dintDay) = date1;
out2 := CONCAT_DATE(MONTH := dintMonth, DAY := dintDay, YEAR := dintYear) = date1;
NOTE: If the arguments are passed in named form (MONTH := dintMonth, etc.), the order is not essential. If the arguments are passed in unnamed form (dintMonth, 11, etc.), they have to be passed in correct order. 
(* ======================================================================================================= *)
(* DT := CONCAT_DT(DINT, DINT, DINT, DINT, DINT, DINT, DINT <, DINT <, DINT)                             *)
(* DT := CONCAT_DT(UDINT, UDINT, UDINT, UDINT, UDINT, UDINT, UDINT <, UDINT <, UDINT)                    *)
(* DT := CONCAT_DT(YEAR, MONTH, DAY, HOUR, MINUTE, SECOND, MILLISECOND <, MICROSECOND <, NANOSECOND)     *)
(* DT := CONCAT_DT_UTC(DINT, DINT, DINT, DINT, DINT, DINT, DINT <, DINT <, DINT)                         *)
(* DT := CONCAT_DT_UTC(UDINT, UDINT, UDINT, UDINT, UDINT, UDINT, UDINT <, UDINT <, UDINT)                *)
(* DT := CONCAT_DT_UTC(YEAR, MONTH, DAY, HOUR, MINUTE, SECOND, MILLISECOND <, MICROSECOND <, NANOSECOND) *)
(* ======================================================================================================= *)
dintYear := 2011;
dintMonth := 11;
dintDay := 24;
dintHour := 10;
dintMinute := 20;
dintSecond := 30;
dintMillisecond := 111;
dt1 := DATE_AND_TIME#2011-11-24-10:20:30.111;
(* results are TRUE *)
out1 := CONCAT_DT(dintYear, dintMonth, dintDay, dintHour, dintMinute, dintSecond, dintMillisecond) = dt1;
out2 := CONCAT_DT(MONTH := dintMonth, DAY := dintDay, YEAR := dintYear, HOUR := dintHour, SECOND := dintSecond, MINUTE := dintMinute, MILLISECOND := dintMillisecond) = dt1; 

(* ================================================================================== *)
(* TOD := CONCAT_TOD(DINT, DINT, DINT, DINT <, DINT <, DINT)                         *)
(* TOD := CONCAT_TOD(UDINT, UDINT, UDINT, UDINT <, UDINT <, UDINT)                   *)
(* TOD := CONCAT_TOD(HOUR, MINUTE, SECOND, MILLISECOND <, MICROSECOND <, NANOSECOND) *)
(* ================================================================================== *)
dintHour := 10;
dintMinute := 20;
dintSecond := 30;
dintMillisecond := 111;
tod1 := TOD#10:20:30.111;
(* results are TRUE *)
out1 := CONCAT_TOD(dintHour, dintMinute, dintSecond, dintMillisecond) = tod1;
out2 := CONCAT_TOD(HOUR := dintHour, SECOND := dintSecond, MINUTE := dintMinute, MILLISECOND := dintMillisecond) = tod1;

(* =============================== *)

EXP

Natural exponential function, computes the result of the exponential function with base Euler's number e, while the argument serves as exponent. 

(* ========================= *)
(* ANY_REAL := EXP(ANY_REAL) *)
(* ========================= *)
real1 := 1.0;
lreal1 := 1.0;
real2 := EXP(real1 - 1.0);
(* results are TRUE *)
out0  := (ABS(real2 - 1.0) < 0.0001);

EXPT

Exponential Function, alternative to the ** operator.

The first argument contains the base, the second argument contains the exponent. 

(* =================================== *)
(* ANY_REAL := EXPT(ANY_REAL, ANY_NUM) *)
(* =================================== *)
real1 := 1.0;
(* results are TRUE *)
real2 := EXPT(REAL#10.0, real1 - 1.0);
out0  := (ABS(real2 - 1.0) < 0.0001);  (* results are TRUE *)
real2 := EXPT(REAL#10.0, real2);
out1  := (ABS(real2 - 10.0) < 0.0001); (* results are TRUE *)

FIND

Returns the character position of the first occurrence of IN2 in IN1. If IN2 is not found, the result = 0. 

(* ======================================= *)
(* ANY_INT := FIND(ANY_STRING, ANY_STRING) *)
(* ANY_INT := FIND(IN1, IN2)               *)
(* ======================================= *)
str1 := 'abcdefghij';
(* results are TRUE *)
out0 := (FIND(str1, 'abc') = 1);
out1 := (FIND(IN1 := str1, IN2 := 'j') = 10);
NOTE:

  • If the arguments are passed in named form (IN1 := str1, etc.), the order is not essential. If the arguments are passed in unnamed form (str1, 'abc'), they have to be passed in correct order.

  • Supported data types are DINT, LINT, ULINT, whereas the function is implemented to return datatype DINT. LINT and ULINT are supported by an implicit cast. If a smaller INT-type (UDINT, UINT, etc.) is used for output data type, this has to happen via explicit conversion. However, data loss has to be expected in doing so and you should be quite certain that the expected value is not bigger than the selected data type.

FROM_BIG_ENDIAN, FROM_LITTLE_ENDIAN

The endianess conversion function serves to influence the byte order of a variable.

Data values which are stored in big endian form are placed in the memory beginning with the leftmost byte first and the rightmost byte last.

Data values which are stored in little endian form are placed in the memory beginning with the rightmost byte first and the leftmost byte last.

The argument and the result shall be the same data type. 

(* ============================== *)
(* ANY := FROM_BIG_ENDIAN(ANY)    *)
(* ANY := FROM_LITTLE_ENDIAN(ANY) *)
(* ============================== *)
int1 := 16#A55A;
dint1 := 16#A500005A;
lint1 := 16#A5000000_0000005A;
(* results are true *)
out0  := (FROM_BIG_ENDIAN(int1) < FROM_LITTLE_ENDIAN(int1));
out1  := (FROM_BIG_ENDIAN(dint1) < FROM_LITTLE_ENDIAN(dint1));
out2  := (FROM_BIG_ENDIAN(lint1) < FROM_LITTLE_ENDIAN(lint1));

GE

Checks whether the arguments are equal or increasingly greater. The number of arguments can be 1 or more.

The function returns TRUE if the respective next argument is smaller than or equal to the preceding argument. 

(* =============================== *)
(* BOOL := GE(ANY_ELEMENTARY, ...) *)
(* =============================== *)
bool1 := TRUE;
sint1 := 55;
int1 := 277;
word1 := 16#A000;
dt1   := DT#2009-10-19-15:30:00   
(* results are TRUE *)
out0  := GE(bool1, TRUE, FALSE);
out1  := GE(sint1, 55);
out2  := GE(sint1 + 1, sint1, sint1, sint1 - 1);
out3  := GE(110, 2 * sint1, sint1);
out4  := GE(int1, 275, int1 - 7, -5;);
out14 := GE(word1, 16#A000);
out23 := GE(dt1, DT#2009-10-19-15:30:00);
out61 := GE('YY', 'YY', 'Y');

(* results are FALSE *)
out1  := GE(10, 9, 11, 8);

Checks whether the arguments are increasingly greater. The number of arguments can be 1 or more.

The function returns TRUE if the respective next argument is smaller than the preceding argument.

 out62 := GT('YY', 'Y');

INSERT

The string IN2 is inserted in String IN1 after the character at position P.

All values for P outside the range will create a record in the cyclic log, P<0 will be treated equally as P := 0, and P>LENGTH(IN1) will be treated equally
as p := LENGTH(IN1). 

(* ===================================================== *)
(* ANY_STRING := INSERT(ANY_STRING, ANY_STRING, ANY_INT) *)
(* ANY_STRING := INSERT(IN1, IN2, P)                     *)
(* ===================================================== *)
str1 := 'abcdefghij';
out0 := (INSERT(str1, 'MID', 6) = 'abcdefMIDghij');
out1 := (INSERT(str1, 'F',  1) = 'aFbcdefghij');
out2 := (INSERT(str1, 'F',  0) = 'Fabcdefghij');
out3 := (INSERT(str1, 'B', 10) = 'abcdefghijB');
out3 := (INSERT(IN1 := str1, IN2 := 'B', P := 10) = 'abcdefghijB');

LEFT

The function returns the number of L characters from the string IN, starting from the left. 

(* ======================================= *)
(* ANY_STRING := LEFT(ANY_STRING, ANY_INT) *)
(* ANY_STRING := LEFT(IN, L)               *)
(* ======================================= *)
str1 := 'abcdefghij';
sint1 := 3;
(* results are TRUE *)
out1 := (LEFT(str1,  sint1) = 'abc');
out2 := (LEFT(str1, 3) = 'abc');
out3 := (LEFT(str1,  1) = 'a');
out4 := (LEFT(str1, 0) = '');
out5 := (LEFT(str1,  10) = 'abcdefghij');
NOTE: If the arguments are passed in named form (IN := str1, etc.), the order is not essential. If the arguments are passed in unnamed form (str1, 'abc'), they have to be passed in correct order. Whereby the following applies:

L<0 is treated as error. / out3 := (LEFT(str1, -1) = '');

L>IN(length) is treated as error. / out4 := (LEFT(str1, 15) = 'abcdefghij');

LEN

Function Description

The function returns the length of the string in the argument.

Variant of Calling the Function

  • Argument types: STRING.

  • Return types: DINT.

Errors Conditions

  • None.

Examples 

str1 := 'a';
out0  := LEN('') = 0;
out1  := LEN('a') = 1;
out2  := LEN('Abc') = 3;
out3  := LEN('$$') = 1;
out4  := LEN('$41') = 1;
out5  := LEN(str1) = 1;

LIMIT

Limits the argument IN to at least the value in the argument MN and maximally to the value in the argument MX. 

(* ======================================================================= *)
(* ANY_ELEMENTARY := LIMIT(ANY_ELEMENTARY, ANY_ELEMENTARY, ANY_ELEMENTARY) *)
(* ANY_ELEMENTARY := LIMIT(MN, IN, MX)                                     *)
(* ======================================================================= *)
(* results are TRUE *)
out57 := LIMIT(MN := -5,IN := -6, MX := 7) = -5;
out57 := LIMIT(-5, -6, 7) = -5;
out54 := (LIMIT('abcdee',   'abcdfa',   'abcdef')   = 'abcdef');
NOTE: If the arguments are passed in named form (MN := -5, etc.), the order is not essential. If the arguments are passed in unnamed form (-5, -6, etc.), they have to be passed in correct order. Minimum is the first, the value to be checked is the second and maximum is the last argument. Incorrect usage will produce incorrect results.

Example: If MN and MX is used in changed order all values smaller than the value of the first argument are limited to the value of the first argument and all values bigger than the value of the last argument are limited to the value of the last argument. Definitely only this two values will be admitted. 

(* results are TRUE *)

out57 := LIMIT(7, -6, -5) = 7;


out57 := LIMIT(7, 0, -5) = 7;
out57 := LIMIT(7, 8, -5) = -5;

TO_BIG_ENDIAN, TO_LITTLE_ENDIAN

The endianess conversion function serves to influence the byte order of a variable.

Data values which are stored in big endian form are placed in the memory beginning with the leftmost byte first and the rightmost byte last.

Data values which are stored in little endian form are placed in the memory beginning with the rightmost byte first and the leftmost byte last.

The argument and the result shall be the same data type. 

(* ============================ *)
(* ANY := TO_BIG_ENDIAN(ANY)    *)
(* ANY := TO_LITTLE_ENDIAN(ANY) *)
(* ============================ *)
int1 := 16#A55A;
dint1 := 16#A500005A;
lint1 := 16#A5000000_0000005A;
(* results are true *)
out0  := (TO_BIG_ENDIAN(int1) < TO_LITTLE_ENDIAN(int1));
out1  := (TO_BIG_ENDIAN(dint1) < TO_LITTLE_ENDIAN(dint1));
out2  := (TO_BIG_ENDIAN(lint1) < TO_LITTLE_ENDIAN(lint1));

NOTE: the conversion from <ANY> REAL to DINT carries out an undistorted mathematical rounding of the value (according to IEC 61131) in the way as it is used, for example, for statistics. The above mentioned example, on the other hand, carries out a rounding where values in the range n.1 - n.4 are rounded down to the next integer number and values in the range n.5 - n.9 are always rounded up to the next integer.
