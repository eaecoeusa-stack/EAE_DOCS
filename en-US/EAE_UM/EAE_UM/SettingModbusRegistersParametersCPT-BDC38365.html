Setting Modbus Registers Parameters

The second step of the manual development is to set all parameters inside the Modbus registers added in the previous step in the Hardware Configuration editor.

Here is the list of all the parameters needing to be set inside a Modbus register:

Available by expanding the Modbus register
LinkName

In the cell has to be entered the symlinks that will create the link between the hardware function block Modbus register and the application function block.

NOTE: To know how to fill this cell, check the section Associate Symlinks to the Modbus Registers from the topic Finalize Generation. All the steps already have been described there.
Comment

In the Comment column of the Modbus register can be added a comment used to described the variable that will be added to the Modbus register.

MSB

It defines the reading order of the bits inside the byte, either from the least significant bit (LSB), when set to FALSE, either from the most significant bit (MSB), when set to TRUE. By default the MSB value is set to FALSE so the reading order is LSB.

swap

It defines the reading rule of the double and long registers. By default at FALSE, it is set at TRUE when the reading order of the bytes composing the register has to be inverted.

NOTE: This parameter is generally set to TRUE when the parameter MSB also has been set to TRUE.
High

This parameter defines which information the byte reads:

  • FALSE: With this default setting, the byte only receives the value of the 8 lowest bits of the WORD transmitted.

  • TRUE: With this setting, the byte only receives the value of the 8 highest bits of the WORD transmitted.

In both case, the value read is given through 2 hexadecimal digits.

reqReadLinkName

Available only for read Modbus registers, configure it only when ioevent of the Modbus register is set to requestread. Give to this parameter a symlink name from a SYMLINKMULTIVARSRC function block. This symlink function block has to:

  • Set as INT its value type.

  • Set its value to 1 and set its QI input to TRUE.

  • Once online, trigger its REQ input event every time you want to read the Modbus register value.

Available in the Modbus register Properties
addressautocalculation

Automatic register address calculation is enabled when this property is set to enable and the address property is set to 0. The auto-calculation function assigns sequential address values to registers in the order in which they appear in the Hardware Configuration editor, based on Register Type.

Register Type can be determined by the combination of Device Type and Data Type and can be: Coil bits (register range 1 - 9999), Discrete input bits (register range 10001 - 19999), Input (register range 30001 - 39999), or Holding (register range 40001 - 49999).

For more information refer to the topic How is data stored in Standard Modbus.

If the Hardware Configuration editor contains the following sequence of registers, the auto-calculated address values are as shown:

  • ModbusWReg1: a write register with datatype output (input register type) > address 0

  • ModbusRReg1: a read register with datatype input (holding register type) > address 0

  • ModbusRWReg1: a read/write register with datatype inout (holding register type) > address 1

  • ModbusWReg2: a write register with datatype inout (holding register type) > address 2

  • ModbusWReg3: a write register with datatype output (input register type) > address 1

  • ModbusWReg4: a write register with datatype output (input register type) > address 2

address

It defines the address number inside the primary table where the variable of the Modbus register will the communicated. This number has to be an integer greater than or equal to 0.

ioevent

It defines how the information is communicated:

  • cyclic: With this setting, selected by default, the value is sent on a cyclic basis, even if the value hasn’t changed.

  • requestwrite: With this setting, the value is sent on trigger event base, only when the value has changed.

    NOTE: It is recommended to choose requestwrite for writing Modbus registers. The only reason to choose cyclic for writing Modbus registers is to help prevent another device from overwriting the value.

  • requestread; available only for read Modbus registers, it enables the Modbus register to read IO value on event base instead of cyclically.

modbusfunction

It defines how the Modbus server should read the value sent by the writing Modbus client:

  • (Default): When this setting is chosen, it worth the same as chosen Write Multiple Registers (FC16). It means that the server will read as a whole the bytes of the Modbus register.

  • Write Single Registers (FC06): This setting means that the server will read individually each byte of the Modbus register. This setting induces a lot more requests to the Modbus server, which is not recommended. Thus, this setting is needed only if the device reading the Modbus client requests doesn’t allow the Write Multiple Registers (FC16) setting.

NOTE: This setting is only available with the writing Modbus Registers transferring register.
datatype

Used only for the reading Modbus register, it defines if the variable can only read value or if the variable can read and write value. It will define the primary table of the variable communicated:

  • 0x Discrete Input: It is a single bit read only variable. It will be in this state if in EcoStruxure Automation Expert the variable type is a coil and the datatype is set to inonly.

  • 1x Coil: It is a single read and write variable. It will be in this state in EcoStruxure Automation Expert if the variable type is a coil and the datatype is set to input.

  • 3x Input Register: It is a multiple bit read only variable. It will be in this state if in EcoStruxure Automation Expert the variable type is a register (single, double or long) and the datatype is set to inonly.

  • 4x Holding Register: It is a multiple bit read only variable. It will be in this state in EcoStruxure Automation Expert if the variable type is a register (single, double or long) and the datatype is set to input.

Set datatype to inonly or input accordingly to the primary table wanted.

When all these parameters are correctly set, the Modbus register will be ready for use. Set all parameters of each Modbus registers to have the global Modbus configuration ready for communication.

To know if the Modbus communication is usable, few actions can be done, like compiling the application, testing communication with Modbus server simulation application and so on.
