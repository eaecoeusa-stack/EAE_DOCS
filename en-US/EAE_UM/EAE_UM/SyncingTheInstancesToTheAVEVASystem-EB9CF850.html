9. Syncing the Instances to the AVEVA System Platform

The sync process reads information compiled by the AVEVA server device for CAT instances and then creates, updates, renames or deletes application objects in the AVEVA System Platform (ASP). The graphics in the OMI folder will also be created in ASP. For CAT instances and SubApp instances the application objects will be created with the hierarchy following the view created in the Hierarchy View pad.

To sync the AVEVA server device:

Step

Action

1

Confirm the following tasks have been completed:

  • ASP mapping is completed for the CATs.

  • The AVEVA server device has completed compilation.

  • Assigning the CAT instances into the AVEVA Plant Model view of the Galaxy.

2

In the Deploy & Diagnostic editor, select the AVEVA server device, and select Action > Sync to AVEVA System Platform > Sync ....

3a

NOTE
In large Solutions the syncing MAY take a lot of time
If you have only done changes on CATs, to reduce the scope of syncing, refer to the row below.
default.consequence.Note

If you select Sync Selection, a preview mimics the hierarchy view of the Plant Model. Drill down and tick the CATs instances of interest and click Sync. This creates the selected AppObjects.

NOTE: If you tick a parent, all the underlying instances get selected. On the contrary, a green — denotes an area whose only some items are ticked.

3b

If you select Sync Changes, the first time all the objects are loaded to the AVEVA System Platform, but the other times ONLY the changes (CAT name and position in the hierarchy, graphics, ...) in the AppObjects are loaded. The process is indicated in the Action Progress column by the Sync in progress with AVEVA System Platform statement.

4

On successful sync operation, the Action Progress cell reads:Sync Successful; read the details in the Output pad.

5

To check the results, open the IDE as described here. Open the Deployment tab: only the ticked instances show in the hierarchy.

All these steps are similar for CAT Composite instances and SubApp instances.

NOTE: Here ‘SubApp instances’ is referring to top instances. Sub instances as SubApp are not supported.
NOTE: AppObjects created in AVEVA System Platform for CAT instances or for SubApp instances align with AVEVA Plant Model view. System Objects - for example, Platform, AppEngine, ViewEngine, OPC Client and ViewApp - are also created as part of the Sync operation. For each EcoRT device with CAT instances mapped to it, an OPC client instance is created. The OPC UA references of the CAT and SubCAT instances are added to the scan group as Item References to the corresponding OPC client instance.

Scan groups are created in the OPC client based on the scan group selection in ASP Attribute Mapping editor and the corresponding attributes are assigned to scan group as per this assignment. Each scan group has a specific naming convention.

Additional scan groups are created when attribute limits are exceeded. This allows that instances are correctly distributed based on their required scan rates.
