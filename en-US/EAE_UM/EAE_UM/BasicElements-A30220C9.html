Basic Elements

 Basic elements of structured text include:

  • Expression and Assignment

  • Comments

  • Operators

  • Implicit Dataconversion

Expression and Assignment

Expressions contain operators and operands.

An assignment describes a variable in the left-hand side with the ":=" operator, with a result of a calculation on the right-hand side;

End an expression or an assignment with an ";".

Example: 

boolResult := NOT var1 AND var2 OR var3 XOR var4;
boolResult := var10  var20;
intResult := var10 + var20 - var30;
intResult := var10 * (var40 - var50) / var20;

Comments

Comments can be used within the ST code.

Comments start with the character string "(*" and end with "*)".

In IEC 61131 3rd ed. (feb 2013) it has been specified, that comments are also possible with the following syntax:

A line starting with "//" is a comment line.

A click on the icon  in the symbol bar of the $key:{StudioName} inserts the characters "//" in front of each selected line. By this, several lines together can be turned very comfortably into a comment. With a second click on the icon the characters "//" can be removed again. Here again, the feature works with the currently selected lines.

Example: 

(* This is a comment of one line *)(* 
  This is a multiline
  comment
  This is a multiline
  comment
*)
// This is a comment of one line
// or
// a multiline

// comment

Implicit Dataconversion

Structured text defines different data types. If different data types are used within an expression and an assignment to a data type of a higher value takes place, an implicit conversion takes place, that is without a function having to be specified.

A conversion to a lower-value data type must always occur explicitly, that is through the use of a function.

Example: 
(* implicit conversion *)
varWORD := 16#FFFF;
varDWORD := varWORD; (* success, results in varDWORD = 16#0000_FFFF*)
(* explicit conversion *)
varDWORD := 16#FFFF_FFFF;
var;WORD := varDWORD; (* produces error *)
varWORD := DWORD_TO_WORD(varDWORD); (* success but with data loss, varWORD = 16#FFFF *)
NOTE: Avoid the use of untyped literals, especially if NOT is involved due to possible change of signed bit after implicit up-cast.

For example:

word1 := word2 AND NOT_WORD(16#01); //OK

word1 := word2 AND NOT WORD#16#01; //OK

untyped literals may lead to a detected error like:

word1 := word2 AND NOT 16#01; //Value '18446744073709551614' is out of range for constants of type 'WORD'

The compiler emits a WARNING like: " NOT operation on untyped integer value 16#1, assuming 64 bit wordsize (LWORD) [...] "
