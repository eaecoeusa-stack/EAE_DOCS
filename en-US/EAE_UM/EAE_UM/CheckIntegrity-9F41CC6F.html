Check Integrity

The integrity of the solution is automatically checked when you open the solution. If the check detects integrity errors, a dialog opens asking you if you want to resolve them. You can select Yes to resolve them, or select No to close the dialog and address these later. 

To subsequently address solution integrity, right click on the Project node in the Solution Explorer and select Check Integrity in the context menu.

The types of detected errors revealed by the check integrity function include the following:

  • Unused CAT Instances

  • Missing Files

  • Instance Conflict

  • Missing Instances

  • Non-referenced Library Instances

  • Non-referenced CAT Instances

  • Deleted CAT Instances

Unused CAT Instances

A dialog displays asking if you want to delete CAT Instances which are not used in the project. Select Yes to open a dialog listing the presently unused CAT Instances. That means, only those CAT Instances are listed which are not used in either a function block in the project (application or resource) or for graphical purpose in a Canvas. To remove unused instance, select  the CAT Instances from the list which you do not need any longer and click Delete.

Missing Files

If files which are referenced in the project are missing, a dialog opens listing the missing files. When you right click on an item, a menu opens offering two options:

Options

Description

Copy File Path

Copies the entire file path to the clipboard. Paste it in windows explorer to open the relevant folder easily and then you can copy the missing file(s) to that folder.

Remove from Project

Removes any references of the missing files from the project. Use this option if you do not need the missing files any longer.

Instance Conflict

If there is an instance conflict found, e.g. because of a corrupt instance type or name, a detected error message appears describing the offending instance. In this case, select the respective instances and click on Repair to correct the detected error.

Missing Instances

If the integrity check reveals references to instances which do not exist in the project, a dialog is presented where you can either

  • create instances for the missing references

  • delete the instance references

To do this, select the relevant references in the center pane of the window and push them into that field providing the desired action by a click on the appropriate arrow button. Execute the prepared operations by a click on Repair.

if you are unsure what to do, close the dialog and check node CAT instances within the Solution Explorer, to see whether there are duplicate or incorrectly named instances, then open the relevant Canvases where the instances are used. Check whether the instances are used within the canvas and whether the TagName within the properties is correct (select the instance in the canvas and check item Tag within the properties). The dialog to repair the instances can be opened again at any time by a right-click on the project node and a click on Check Integrity within the context menu.

Non-referenced Library Instances

On opening a solution, the software checks for CAT instances coming from non-referenced libraries. If non-referenced library references are found, a message displays listing the unreferenced instances. Click OK to close the message, then either remove the reference or add the appropriate library. The cannot be compiled or deployed until this is completed.

Non-referenced CAT Instances

On opening a solution, a message opens giving an overview of CAT instances created directly on the CAT Instance node in the Solution Explorer but without any reference in the project (that is, with no instance in any network). It is possible to delete non-referenced CAT instances if you do not want to use them. To do this, move the unused CAT instances from the Found section to the Delete section, then click Delete.

Deleted CAT Instances

On saving, if you delete some CAT Instances from networks (CAT, Application, Resource, and so forth) coming from a non-referenced or missing installed library, you will be informed and asked if you also want to remove corresponding configurations from the system database (System.cfg file).
