Faceplate Editor

Use the Faceplate Editor to create the graphical view of faceplates that can be opened at runtime by an action, for example, clicking  on a button. Faceplates contain elements that do not have to be continuously visible on the HMI, for example: 

  • parameters to adjust user-specific values. 

  • a trend view that is used only occasionally.

  • an HMI that could contain an extension of the CAT_HMI Symbol which can be opened optionally (refer to the topic CAT).

How to create a new faceplate is described in the topic CAT. The faceplate has to be created first and the Symbol Editor can then be opened by a double-click on the faceplate in the Solution Explorer.

The Faceplate Editor has an identical structure as the Symbol Editor. But unlike in the Symbol Editor, the size of the background in the Faceplate Editor defines the size of the faceplate, which can be opened on the HMI at runtime. Furthermore, the faceplate cannot be dragged into the canvas directly, but has to be linked with an element of a symbol of the same CAT. By instancing the symbol in the canvas the faceplate is also available.

Integrating Faceplates

After the faceplates is created, switch back to the Symbol Editor of the relevant symbol within the CAT and select the desired element. This element will be used to open the faceplate, or create one.

In the Properties pad for the element (e.g. a rectangle, or button), click the ellipsis () at Misc > OpenFaceplates in the Properties pad to open the OpenFaceplate Collection Editor dialog, which you can use to select the desired faceplate and associate it with the action that will open it.

Open Faceplate Collection Editor

The OpenFaceplate Collection Editor presents the following elements:

Command

Description

Member

The existing links are listed here. If there is no existing entry use the button Add to create one.

Add

Creates a new link. It has to be parameterized accordingly in the field properties at the right side of the dialog. At the time of creating there is no mouse action linked with the faceplate.

Remove

Removes the selected link.

Faceplate..|.. properties

In the FaceplateType field, select the desired faceplate from drop down list.

In the MouseButtonType field, assign the appropriate mouse action.

The faceplate opens. The list of selectable faceplates remains empty as long as no faceplates exist in the CAT_HMI.

OK

Closes the window and saves any changes.

Cancel

Closes the window without saving any changes.

The description for the Symbol Editor also applies to the Faceplate Editor. In this way you can nest a faceplate within a faceplate, but the first and top-level instance to open a faceplate has to be a symbol.

As stated above, the size of the faceplate is defined by the background. However, the faceplate window can be parameterized in such a way that it can be resized at runtime by dragging the frame. In order to do so, select the background of the faceplate and choose the appropriate mode in the Properties pad at Appearance > FormBorderStyle. At the time the faceplate is opened it is displayed in the maximum size (= size of the background).

Furthermore, the appearance of the window frame around the faceplate can be defined in the Properties pad with Appearance > FaceplateFrame property: either standard (None), Rectangular or Rounded. These three options have a standard window mechanism allowing the user to close the faceplate whereas the fourth option, No Frame, doesn’t have. It has no window mechanism, hence the closing of the window needs to be handled with custom logic.

NOTE: All these four options are only available for the Windows version. For the Multi-OS version, the FaceplateFrame property is not supported.

Accessing Faceplates in C# Code

If you elect to access faceplates using C# code (instead of via the OpenFaceplate Collection Editor dialog) , you need to check if the user has been granted faceplate access privileges. If a user does not possess faceplate access privileges, the Standard object (used to access faceplates) will not be initialized and the following unhandled exception will occur:

“Exception thrown: System.NullReferenceException: Object reference not set to an instance of an object.”

To avoid this unhandled exception, add the following C# code to your project. It will perform a null check of the Standard object when using the IsOpenFaceplateSecure() method: 


void DevIconDevIcon_Click(object sender, EventArgs e)
{
     if (IsOpenFaceplateSecure(*)) {
         ProjectExample.SupportClasses.ProjectExample_Var.OpenFaceLateIconName = this.GetType().Name;

         if (Standard.Initialized == true)
             Standard.Activate();
         else {
             OnInitializeFaceplate(Standard);
             Standard.Show(this);
         }
      }
}
