Creating Asset Tag for CAT and Sub-CAT Instances

You can assign an Asset Tag to any hardware or application CAT or sub-CAT instance.

The Asset Tag you assign to the CAT or sub-CAT instance will be accessible:

  • In AVEVA Operations Management Interface (OMI).

  • EcoStruxure Automation Expert editors, including Symbolic Links, OPC UA Server, Offline Parameterization, and so forth.

  • In OPC UA tag browser, based on AVEVA OMI hierarchy.

  • As Asset Tag for HMI symbols, faceplates, alarm summaries, realtime time or historical trends.

NOTE: The new tag can be created in your choice of localized language by entering text in that target language.

Valid Asset Tag

A valid Asset Tag will:

  • Contain 32 characters or fewer.

  • Include only characters that are alphanumeric or the special characters ‘$’, ‘#’ and ‘_’.

  • Be unique, i.e., an Asset Tag can be used only once in a solution.

NOTE: If you enter invalid characters, more than 32 characters, or a duplicate Asset Tag:

  • An icon ( ) appears, and

  • The OK button is disabled.

How to Set an Asset Tag for a CAT or Sub-CAT Instance

In a solution that includes one or more application or hardware CAT or sub-CAT instances, follow these steps:

  1. In the Solution Explorer, navigate to (solution) > CAT Instances > (APP name) > Application / Hardware > (CAT name) or (CAT name > sub-CAT name).

  2. Right click on the CAT or sub-CAT, and select Set Asset Tag.

  3. In the Edit Asset Tag Name dialog, type in a valid Asset Tag.

  4. Click OK to save the new Asset Tag.
