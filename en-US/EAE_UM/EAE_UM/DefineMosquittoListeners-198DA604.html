Define Multiple Mosquitto Listeners

A single MQTT broker can support several MQTT client network applications that use MQTT messaging. These clients may require different security configurations to connect to - and be authenticated by - the broker. For example, some clients may support username and password authentication, while other clients may require TLS certificate-based authentication.

Mosquitto MQTT broker can define multiple Mosquitto listener interfaces, each interface having its own security configuration.

NOTE: You need to configure multiple Mosquitto listener interfaces only when using a previously installed Mosquitto broker with a different security configuration for your MQTT client network application. For a first-time Mosquitto broker installation with no other MQTT client connections, you do not need to create multiple Mosquitto listeners. The procedure defined in Configure Mosquitto is sufficient for EcoStruxure Automation Expert HMI Diagnostics.

To use multiple Mosquitto listeners, each with its own security configuration, follow these steps:

  1. In the Mosquitto installation folder, open the file mosquitto.conf.

  2. Above the lines described in the topic Configure Mosquitto, add the following line:

    • per_listener_settings true

    NOTE: This requires administrator rights.

  3. Save the configuration file and restart Mosquitto to apply the changes.

Adding this line lets you define multiple Mosquitto listener interfaces, each with its own security configuration.

For example, if you have Mosquitto MQTT broker security configured with the following user authentication:

  • listener 1883

  • allow_anonymous false

  • password_file c:\Program Files\mosquitto\passwordFile.txt

Adding the line per_listener_settings true to the file mosquitto.conf prevents a conflict with the EcoStruxure Automation Expert HMI Diagnostic configuration performed using the procedure set forth in Configure Mosquitto. With this line added, the listener interface configured for EcoStruxure Automation Expert Diagnostic will use its own security configuration and will not conflict with your Mosquitto listener interface security configuration for listener interface 1883.
