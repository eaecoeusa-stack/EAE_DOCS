Open Adapter

The Open Adapter is a functional component that acts as a bridge between similar or different technologies, enabling seamless communication. It serves as a plug-and-play alternative to OPC UA, without its typical limitations or prerequisites. This approach simplifies integration by removing dependencies and offering greater flexibility and automation.

NOTE: .Open Adapter is not supported for Inner/nested CATs scenarios.

Configuring Open Adapter Communication Between Two Solutions

The steps below guide you through configuring, exporting, importing, and validating the connection to enable reliable data exchange.

Creating Two Solutions

  • Create two solutions: Solution_1 and Solution_2.

  • In both solutions, right-click on each solution then go to Options > IEC 61499

    .

  • In the IEC 61499 pad, go to Communication and enable the checkbox Enable Reliable Cross Communication protocol for all devices.

NOTE: This activates the HA SRT protocol, that enables stable and reliable communication.

Configuring the First Solution

Start by creating an Open Adapter:

  1. Right-click on Adapter > New Item to display the File Wizard dialog box .

  2. From the Type drop-down list, select either Normal or Open Adapter.
    NOTE: The Adapter type can be changed in Solution Explorer pad by right-clicking the adapter and selecting Change to Normal Adapter or Change to Open Adapter from the drop-down menu.

  3. Give a Name to your adapter.

  4. Go to the properties tab of the adapter

    .

  5. Under Attribute, set Configuration.FB.OpenAdapter to True if configuring an Open Adapter, or to False if configuring a Normal Adapter.

Create the CAT and add the Adapter:

  1. Right-click on Application > New item to create a CAT.

  2. Drag and drop the Adapter into the CAT.

  3. Use one Open Adapter as Socket type (ADP1_S1) and the other as Plug type (ADP1_P1).

    Attention: Modifying Open Adapter interface elements will make the existing open adapter connections invalid.

Instantiate and Map the CAT:

Go to the system tab.

  1. Instantiate the CAT in your application and place it in the System tab.

  2. Right-click on CAT > Mapping > EcoRT_O_RESO to map it to the controller.

Check Open Adapter interfaces:

  • Open the Open Adapter Editor

  • You will see the interfaces automatically created for Solution_1.

  • The Plugs and Sockets names displayed in the Open Adapter Editor combines the EcoRT name, instance name and adapter name.
    NOTE: To check that the configuration settings are correct, right‑click the System editor, select Open Containing Folder, then open the obj directory and view the XML file using a Text Editor Software. It is essential that ProjectGUID is unique when solutions are communicating with each other.

Export the Configuration:

  1. Go to Solution Explorer and right-click on Solution_1, then click on Export.

    NOTE: The interface IDs in both solutions must match for the connection to work.

  2. Use the Export icon in the Open Adapters editor to save the interface connection configuration.

Configuring the Second Solution

Import the configuration:

  1. Open Solution_2.

  2. Right-click on Adapter > Import to select the file exported from Solution_1.

Create CAT and Use Imported Adapters:

  1. Create a new CAT in the Application.

  2. Drag and drop the imported adapters into it: Use one Open Adapter as Socket (ADP1_S2) and the other as Plug (ADP1_P2) then reverse it.

  3. Save the CAT.

Instantiate and Map the CAT:

  1. Open the System tab.

  2. Instantiate the CAT in your application and place it in the system tab.

  3. Right-click on CAT > Mapping > EcoRT_O_RESO to map it to the controller.

Import and Map the Interface:

  1. Open the Open Adapter Editor

    in Solution_2 and use the Import icon to import the configuration exported from Solution_1.

  2. Map the imported interface connections to their corresponding elements in Solution_2.

  3. Export the mapped interface configuration from Solution_2 to Solution_1 .

  4. Back in Solution_1, import the final configuration from Solution_2.

Result: This step automatically completes the interface mapping.

Deploy and Simulate:

  1. Go to the Deploy & Diagnostic tab.

  2. Deploy both solutions to both controllers.

  3. Open the System tab and add a watch to the Applications.

  4. Simulate data and verify that data flows correctly from Solution_1 to Solution_2.
