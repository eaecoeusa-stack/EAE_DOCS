Limiting Edits and Actions in the Hardware Configuration Editor

You can add program logic to CAT objects that controls the ability to edit, add, and export objects in the Hardware Configuration editor. Do this in the CAT > Hardware > <CAT name> > HWConfig > Configuration tab. For more information refer to the help topic for HWConfig.

.

The implementation of limiting edits and actions in the Hardware Configuration editor is optional, depending on your solution’s requirements.

Editor and Module Attributes

You can use the following editor and module CATConfiguation attributes in program logic. Editor attributes apply to the CATConfiguration object and apply to the modules defined in the Configuration tab. Module attributes apply only to the SubModules and Types for which they are made.

Editor attributes: By default, the following attributes are set to "true". They may not appear in program logic by default. To set an attribute to "false" you may need to manually type in both the attribute and its setting.

  • CanEditProperties: to permit ("true") or deny ("false") editing object properties in the Properties dialog opened from the Hardware Configuration editor.

  • CanAdd: to enable ("true") or disable ("false") the ability to Add descendant items to the configured CAT in the Hardware Configuration editor in any of the following ways:

    • from the context menu,

    • from the toolbar,

    • using drag and drop.

  • CanExport: to enable ("true") or disable ("false") the Export context menu command in the Hardware Configuration editor for BusMaster HW CATs.

Module attributes:

  • Max: the maximum number of modules permitted to be included in the CATConfiguration.

  • MaxOccurrence: the maximum number of modules of the specified type permitted to be included in the CATConfiguration.

  • SortOrder: an assigned integer that specifies the display sequence of modules by type.

  • Group: Module types can be assigned to a group. Both the MaxOccurrence and SortOrder attributes can be specified for the group. When a module Type is assigned to a group, the MaxOccurrence and SortOrder settings specified for that type are overridden by the same group settings.

Example 


<CATConfiguration CanEditProperties="true" CanExport="true" CanAdd="true">
   <SubModules Max="10">
      <SupportedTypes>
         <Type Name="ChannelComplexA" Namespace="Main" Group="Complex" />
         <Type Name="ChannelComplexB" Namespace="Main" Group="Complex" />
         <Type Name="ChannelComplexC" Namespace="Main" Group="Complex" />
         <Type Name="ChannelComplexD" Namespace="Main" Group="Complex" />
         <Type Name="ChannelSimple" Namespace="Main" MaxOccurrence="10" />
         </SupportedTypes>
      <Groups>
         <Group Name="Complex" MaxOccurrence="2" SortOrder="0"  />
      </Groups>
   </SubModules>
</CATConfiguration

Where:

  • Properties are editable (CanEditProperties = “true”).

  • The CAT configuration can be exported (CanExport = “true”).

  • New modules can be added (CanAdd = “true”), subject to the following limitations:

    • A Max of 2 of any Type module in the Complex group.

    • A Max of 10 of the Type = ChannelSimple.

    • A Max of 10 modules of all types.

  • Modules of the Complex group will be displayed first (SortOrder="0".
