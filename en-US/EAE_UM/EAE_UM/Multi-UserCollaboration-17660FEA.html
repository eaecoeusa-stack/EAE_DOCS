SVN Limitations

Limitations

Currently EcoStruxure Automation Expert does not provide explicit information about multiuser situations, therefore users must organize themselves outside of EcoStruxure Automation Expert to ensure their proper collaboration.

However, EcoStruxure Automation Expert provides some information you can use to detect or understand certain multiuser situations:

Situation

Tip

Screen Location

Login in to a dPAC to which another user is currently logged in

An Output message related to the Forcing capabilities is repeatedly triggered (“Unable to Connect: Too many connections of this role active”). Despite this message, user login to the dPAC is performed.

Output pad

Synchronization state between the local and dPAC applications

Application and boot project synchronization icons will show de-synchronization if another user modifies the dPAC application or boot project (and also if the local user modifies his local application).

Deploy and Diagnostic editor

Detection of ‘locked’ Forced values or events

If Forcing capabilities are locked for the local user, this indicates another user is already logged in and is currently Forcing values or events in the dPAC.

Watch pad

Checking the application versions as local, inside a dPAC, as SVN version

Detailed application identification data can be checked and compared between the local application, the application deployed in a dPAC, and the SVN application (if used), to understand the trail of modifications that have been performed.
