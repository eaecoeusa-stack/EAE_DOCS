Basic Function Block

A basic function block is a function block that provides basic functionality, like AND, OR, ADD, MUL, etc. The functionality of a basic function block is described in the ECC logic (Execution Control Chart/ECC according to IEC 61499) and in one or more algorithms (Basic Function block Editor, see also a detailed description of the functionality at Fundamental Functionality of the Basic Function Block). This type of function block cannot run distributed. It can run only on a single Resource. (See also IEC 61499 documentation for Basic Function Blocks)

To learn more about how to create a basic function block, watch How to Create a Basic Function Block in EcoStruxure Automation Expert.

For generating or deleting a basic function block, refer to the topic Elements.

For editing help refer to the topics Function Block Interface Editor and Basic Function block Editor.

The interfaces and associations between input variables (InternalVars) and event inputs (EventOutputs) — and the same on output side — are shown in the graphic view of the Basic Function block. The Basic Function block presents the following element groups:

  • EventInputs (interfaces)

  • EventOutputs (interfaces)

  • InputVars (variables)

  • OutputVars (variables)

  • InternalVars (variables))

  • Sockets (adapters)

  • Plugs (adapters)

Adapter Usage in Basic Function Blocks

Fixed Length Strings:

Although Adapters can be linked to dynamic STRING type variables, Basic Function Blocks do not support the dynamic STRING data type. When using an Adapter in a Basic Function Block, use a fixed length STRING type in place of a dynamic STRING type (e.g. STRING[30]).

Unintentionally Exposed Data:

A Basic function block (FB1, below) outputAdapter (plug) may hold in memory an updated data value, even if the event associated with the inputAdapter has not been triggered. As a result:

  • if the FB1 Basic function block outputAdapter (plug) is connected to a destination Basic function block (FB2, below) inputAdapter, and

  • if the FB2 event associated with the FB2 inputAdapter is triggered, then

FB2 may receive the data value from FB1 even though the data value was not intentionally exposed by FB1.
