Support for Multiple HMI Operating Systems

Beginning with version 21.2, EcoStruxure Automation Expert supports multiple HMI operating systems. When you create a new solution, you specify both the type of solution (with full or minimal libraries) and the targeted HMI operating system (Windows only, or multiplatform: Windows™ and Linux®).

Multiplatform solutions can be deployed on host machines running either Windows™ or Linux. Windows solutions can only be deployed on host machines running Windows. If you attempt to deploy a Windows solution to a host machine that is running Linux, a detected error message appears: Unable to deploy. Target platform is not compatible.

NOTE: HMI objects (symbols, faceplates, graphics) contained in a library specifically designed for a Windows platform, cannot be used in a multiplatform (Multi-OS HMI) solution. If you add a Windows HMI object to a multiplatform solution, you receive the detected error message: Usage of objects from "Windows HMI" Library is not allowed in "Multi-OS HMI".

However, HMI objects contained in a Multi-OS library can be used in a Windows platform solution.

To identify the platform of a specific library: in the Solution Explorer right click on a library and select Show Info; in the Info dialog check the Platform setting. (Or click the Info button in the Installed Libraries dialog.)

The HMI operating system you specify when creating a new solution can later be changed.

Creating a New Solution

To create a new solution:

Step

Description

1

Click File > New.

2

In the dialog, select a template that includes both the solution and HMI types:

  • EcoStruxure Automation Expert- Full Solution (Windows HMI): Creates a default solution with all Schneider Electric resources (platforms and application libraries).

  • EcoStruxure Automation Expert- Starter Kit (Windows HMI): Creates a default solution with minimal libraries.

  • EcoStruxure Automation Expert- Starter Kit (Multi-OS HMI): Creates a default solution for multiple operating systems with minimal libraries.

NOTE: If you select a Starter Kit template and create a basic solution, you can subsequently extend the solution by additional features.

3

Enter a solution Name, and specify a Location.

NOTE:

  • The default location is:
    Documents\Schneider Electric\EcoStruxureAutomationExpertProjects\<solution_name>

  • Microsoft Windows™ limits the file path length to a maximum of 247 characters. Because the names of files in your project can be much longer than the project name, create a path and project name that does not exceed this limit. For example, locate the project file in a folder at a higher level in the file hierarchy and use a shorter project file name.

4

Click Create to configure solution properties.

5

If you selected EcoStruxure Automation Expert- Starter Kit (Multi-OS HMI) in step 2, a wizard automatically opens where you can configure Start Canvas settings, including:

  • Resolution

  • Canvas Logo

6

Set other optional properties, or click Finish.

Optional properties can be set later, including:

(<project name> > Options > General)

  • Company Address

  • Author

  • Website

(Canvases > <resolution> > Redefine Start Canvas with wizard)

  • Resolution

  • Canvas Logo
