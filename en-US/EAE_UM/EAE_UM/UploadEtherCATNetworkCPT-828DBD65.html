Fine Tune EtherCAT Network

After having exported the ENI file, the file is automatically uploaded in EcoStruxure Automation Expert - Buildtime.

As a result, all the Sub-devices previously configured on the EtherCAT Configuration Tool are automatically generated by creating in your project:

  • One CAT for each Sub-device, located under the following node in the Solution Explorer pad: CAT > Hardware.

  • One CAT instance for each created CAT, located under the following node in the Solution Explorer pad: CAT Instances > Hardware.

  • A representation of every CAT instances under the EtherCAT bus in the Hardware Configuration editor.

Next, finalize the EtherCAT network by fine tuning it in the Hardware Configuration editor:

  1. Open the Symbolic Links pad and locate it next to the Hardware Configuration editor.

  2. Expand the EtherCAT bus and the Sub-devices.

  3. Drag and drop symbolic links into the Value cell of the input and outputs of the Sub-devices. This action links the parameters set on the SYMLINKMULTIVARDST and SYMLINKMULTIVARSRC function blocks (the source of the symbolic links) to the Sub-devices inputs and outputs.

    NOTE: Ensure that the parameter type in these two function blocks are matching the type requested in the Sub-devices inputs and outputs, in the Type column.
    NOTE: For more information about the SYMLINKMULTIVARDST function block, refer to the SYMLINKMULTIVARDST topic in the Runtime.Base - Library Guide.
    NOTE: For more information about the SYMLINKMULTIVARSRC function block, refer to the SYMLINKMULTIVARSRC topic in the Runtime.Base - Library Guide.

  4. Right click on the EtherCAT bus and select the Properties command.

    RESULT: The Properties dialog box opens.

  5. Set the properties according to your needs:

    • Bus ID; The name of the EtherCAT bus.

    • CpuIndex; The CPU core number isolated and used for the EtherCAT network. By default set at 3 on Schneider Electric iPCs.

    • BusCycleTimeUsec; : Process Cycle Time is the safe waiting time after receiving data before the next update, set to 80% of the Bus Cycle Time in microseconds(µs) .By default empty, set here the bus cycle time in microseconds (µs). If kept empty, the bus cycle time is automatically set to 120000 µs.

    • ENIFilename; The name of the ENI file used previously to automatically generate the Sub-devices of the EtherCAT network.

    • MaxBusSubDevices; The maximum number of Sub-devices allowed in this EtherCAT network.

    • LogFileprefix; The prefix assigned to every logs generated for this EtherCAT network.

    • PerfMeasLevel: performance; From 1 to 4, it defines the level of performance measure. By default at 1.

    • Verbose; From 0 (no log generated) to 5 (every log generated), it defines the level of detail given in the logs about this EtherCAT network.

    • NumLinkLayer; Link layer number. By default at 0.

    • Ethernet; Defines which Ethernet communication is used between:

      • Sockraw <AdapterName>; The default value, it is the standard Ethernet communication.

    • AdapterName; It defines the adapter name of the Ethernet interface that links your iPC with the Sub-devices.
      NOTE: Only available if you select Sockraw value for Ethernet property.

    • Dcmmode; Defines the distributed clocks used in this EtherCAT network between:

      • Off; Disables the distributed clock feature.

        NOTE: To select only if the Sub-devices connected do not support the distributed clock feature.

      • busshift; The default value, it synchronizes the Sub-devices to the Main-device timer.

      • maindeviceshift; Synchronizes the Main-device timer to the Sub-device used as distributed clock reference.

        NOTE: The feasibility of this mode depends on the hardware and software capabilities, as the reference clock controls the Main-device time.

      • linklayerrefclock; Uses the Link Layer clock controlling.

        NOTE: To select only if your iPC is compatible to link layer clock.

      • maindevicerefclock; Allows bus shifting without the reference clock controlling, which lowers CPU usage but demands high timer accuracy.

        NOTE: To select only if your iPC has a high level quality timer.

    • Ctloff; By default at False, set it to True to be able to disable and enable the distributed clock control for diagnostics.

    • DcmSyncToCycleStart; Select 1 to synchronize the distributed clock cycle start or select 0 to disable the synchronization.

  6. Click OK to validate the properties changes and save the solution.

This concludes the configuration of the EtherCAT network communication.
