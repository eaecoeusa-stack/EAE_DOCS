Define Project Library References

In a project, any installed library on the engineering workstation can be referenced.

NOTE: To find the list of all the installed libraries in an engineering workstation, check the topic here.

Referencing a library gives to the user access to all the predefined function blocks of the library.

The user then gets more options on how to develop its application without programing manually specific function blocks. Indeed, a predefined function block with the same functionalities may already exist in the referenced libraries.

Thus, here are all the actions available to define the referenced libraries in a project:

  • Open the Project Library Reference dialog box.

  • Reference a new library in the project.

  • Update to the latest version every referenced libraries.

  • Change version of a referenced library.

  • Remove a referenced library from the project.

NOTICE
IMPROPER CHANGE TO LIBRARY REFERENCES
When switching to another library version in a project, also re-deploy the project to the controller. Only then can changes on blocks of the referenced library become active on the controller.
In the case that changes only have been made in symbols or faceplates, it is sufficient to restart the HMI (but if the HMI block of a CAT is affected by the changes, also re-deploy the project).
If you do not know the exact extent of the changes to the libraries, it is recommended to re-deploy both the project and the HMI as well.
Failure to follow these instructions can result in equipment damage.
NOTE: Changing the library version may prevent making online changes to a device. Online changes are executed in the Deploy and Diagnostic editor by selecting a device, then selecting Action > Deploy > Online Change. In this case, only a full deploy (Action > Deploy > Deploy) can implement the change.

Open the Project Library Reference Dialog Box

To open the Project Library Reference dialog box, follow the next steps:

  1. Open the Solution Explorer pad.

  2. Right click on the project node.

  3. Select Project Library References.

The Project Library Reference dialog box opens.

NOTE: Find description of this dialog box in the topic here.

Reference a New Library in the Project

To reference a new library in the project, follow the next steps:

  1. Open the Project Library Reference dialog box.

  2. In the Installed / Source Libraries section, scroll to find the wanted library to reference.

  3. Once found, either:

    • Tick the checkbox to select the latest version available of this library.

    • Click on the + sign to show older versions of this specific library, then tick the checkbox of the wanted version of this library.

  4. The selected library version is added in the Project Referenced Libraries section. In this section, check that the wanted library version is selected.

  5. If the library selected is referencing a library that is also not referenced in the project, the dialog box Change library reference in the project opens. If selecting:

    • Yes: Both the wanted and the referencing libraries will be added in the Project Referenced Libraries section.

    • No: None of the wanted and the referencing libraries will be added in the Project Referenced Libraries section.

  6. Click Apply to add this library as referenced library in the project.

  7. A pop up appears informing that to take into account this change, the solution must be reopened. Click OK to acknowledge and proceed.

The solution reopen with the new referenced library in the project as expected.

NOTE: Several libraries can be added at the same time. Select all the ones wanted before clicking on Apply.

Update to the Latest Version Every Referenced Libraries

To update to the latest version every referenced libraries, follow the next steps:

  1. Open the Project Library Reference dialog box.

  2. Click on the Select Latest Library(s) button. It selects the latest library version for every referenced libraries.

  3. Check the column Selected Version of the Project Referenced Libraries section to see all the modifications.

  4. Click on Apply to update the library versions in the project.

  5. A pop up appears informing that to take into account this change, the solution must be reopened. Click OK to acknowledge and proceed.

The solution reopen with all the library referenced in the project updated to their latest version.

Change Version of a Referenced Library

To change the version of a referenced library, follow the next steps:

  1. Open the Project Library Reference dialog box.

  2. In the Installed / Source Libraries section, scroll to find the wanted library.

  3. Once found, either:

    • Tick the checkbox to select the latest version available of this library.

    • Click on the + sign to show older versions of this specific library, then tick the checkbox of the wanted version of this library.

  4. Depending on the following cases, few different actions could happen:

    • If the wanted version of the library is older and that library (A) is referenced by another library (B) in this project then:

      The dialog box Change library reference in the project opens. If selecting:

      • Yes: The library A version will be changed to the wanted one, but the library B will be removed from the project.

        NOTE: Removing a referenced library from a project may cause issue if function blocks of this library already have been used in the project.

      • No: Both the library A and B version will not be changed.

    • If the wanted version of the library is more recent and that the library (A) is referencing another library (B) in the project then:

      The dialog box Change library reference in the project opens. If selecting:

      • Yes: The library A will be changed to the wanted version and the library B will be changed to the version mentioned in the dialog box. Both version will be udpdated.

      • No: Both the library A and B version will not be changed.

    • If not in one of those cases then:

      Only this library will be changed to the wanted version.

  5. Check the column Selected Version of the Project Referenced Libraries section to see all the modifications.

  6. Click on Apply to change the library versions in the project.

  7. A pop up appears informing that to take into account this change, the solution must be reopened. Click OK to acknowledge and proceed.

The solution reopen with all the library versions modified.

NOTE: Several libraries’ version can be changed at the same time. Select all the ones wanted before clicking on Apply.

Remove a Referenced Library from the Project

To remove a referenced library from the project, follow the next steps:

  1. Open the Project Library Reference dialog box.

  2. Untick the checkbox of the library to remove either in the Project Referenced Libraries section or in the Installed / Source Libraries section.

  3. Once unticked, the library is automatically removed from the Installed / Source Libraries section.
    NOTE: Removing a referenced library from a project may cause issue if function blocks of this library already have been used in the project.

  4. Click on Apply to remove the library from the project.

  5. A pop up appears informing that to take into account this change, the solution must be reopened. Click OK to acknowledge and proceed.

The solution reopen with all the previous library referenced except the one removed.

NOTE: Several libraries can be removed at the same time. Select all the ones wanted before clicking on Apply.
