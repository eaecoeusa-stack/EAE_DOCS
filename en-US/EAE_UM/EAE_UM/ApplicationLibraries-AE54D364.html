Basic & Composite Function Blocks

CamelCase naming begins with lowercase letters.

CAT, Sub Application & User define datatypes/structures

CamelCase: Starts with an uppercase letter and continues with lowercase letters. This also applies to abbreviations and truncations. If the name contains the short name of the product (e.g., ATV), then it should be written in all capital letters.

AnalogInput

DiscreteOutput

SpMon

OosMode

SeqManager

Service Interface Function block

All Service Interface Function block should be written in all capital letters.

 

Adapter

Use "I" to start each name, and use CamelCase with upper-case letters at the start.

IPv

IAnalogValue

Graphical Items/Elements

CamelCase naming begins with lowercase letters.

 

HMI Interface (CAT)

Follow the CAT naming rule with "_HMI" as the suffix.

Analog_HMI if the CAT name is ANALOG.

HMI Interface (Instance)

Always IThis

 

Symbols (graphic)

Always use the "s" prefix and CamelCase, beginning with an uppercase letter.

sMotor

sValve

Faceplates

Always use the prefix "fp" and camel-case naming, beginning with an uppercase letter.

fpAnalogInput

Event Names

Use an underscore to separate all upper-case letters.

 

Regular input and output variable

CamelCase naming begins with lowercase letters.

PermitOn

FeedbackOn

Internal variable name in function

CamelCase naming begins with lowercase letters.

error

outMinActiveLast

Boolean variables

The name should explicitly state whether 0 or 1 is the active state.

1= Start

1= Stop

1= Activate

1= Interlocked

HMI Design guidelines

Use named styles to define specific themes and customize the graphics for segments and customers.

State Names

CamelCase naming begins with lowercase letters.

Algorithm

Always use the prefix "alg" and CamelCase, beginning with a lowercase letter. If the algorithm uses only one state, ensure that the state name is used when naming the algorithm.

Folder Names

Use CamelCase, starting with an uppercase letter. Use single words for folder names.

Instance Naming – Best Practices

  • Avoid using auto-generated names such as “FB1”, “FB2”, etc.

  • Use the names from Process and Instrumentation schemas when naming CAT instances.

  • For all other instances, use CamelCase starting with a lowercase letter.

  • Include the type name in the instance name whenever possible (e.g., AND, OR, basic FBs).
