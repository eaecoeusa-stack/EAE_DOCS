Multi-User Collaboration

EcoStruxure Automation Expert allows several users to connect to and interact with the same devices (dPACs), for the purpose of working collaboratively with the same solution, during both operating and maintenance phases.

This multiuser capability must be used with care. The actions of multiple users need to be coordinated toward a common, shared solution goal to avoid unsafe system behavior.

 WARNING
unexpected system behavior
Do not allow several users to simultaneously modify a system during operations or maintenance, without using proper work practices to organize and coordinate their work to ensure that modifications do not result in unsafe system behavior.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Risk Characterization

EcoStruxure Automation Expert misuses can occur when several users do not act collaboratively toward a common goal, but instead perform modifications that result in unexpected or unsafe system behavior.

For example: two users, unaware of each other, could perform conflicting application modifications or deployments that could result in unexpected application behavior.

The consequences of such misuse typically can include:

  • Unexpected complete or partial change of application behavior.

  • Unexpected complete or partial running or interruption of operations.

  • Lack of a user’s awareness of the trail of changes in the solution.

Preventive Measures

To help prevent such misuses, you need to organize the usage of your resources in a collaborative way, through adequate work practices, so that each user is aware of other users’ requirements and purposes, as well as each user’s interactions with the system.

All users should follow defined workflows oriented toward a common and shared goal for the solution.

Typical goals include:

  • Modifying the operating state of a device (Run, Stop…).

  • Modifying the application in the device (Online Change, Full Deploy).

  • Modifying the boot project of a device.

Typical best practices include:

  • Defining, planning and sharing system updates requirements.

  • Define users’ roles and responsibilities.

  • Granting adequate access right to each user.

  • Organizing work collaboration and communication between users.
