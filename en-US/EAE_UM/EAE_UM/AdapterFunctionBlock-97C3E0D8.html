Adapter Function Block

Adapter function blocks are aggregations of event and data connections between function blocks resulting in one single line connection. Adapter function blocks only consist of an interface, they do not have an implementation.

The Adapter function block splits into two views:

  • Socket - left-side input

  • Plug - right-side output

Adapter function blocks improve program clarity if function blocks are connected using many ports. Furthermore Adapter function blocks make programming easier and make it easier to change communication interfaces between function blocks.

NOTE: For EcoStruxure Automation Expert version 24.1 and later, you can use the Adapter function block to create connections between different solutions, using:

  • Reliable Cross Communication protocol (UDP connection).

  • WebSocket communication protocol (TCP connection).

Refer to the topic Elements for how to create and delete Adapter function blocks.

To learn more about how to create an adapter, watch How to Create an Adapter in EcoStruxure Automation Expert.

How to Edit Adapter Function Blocks

Use the Function Block Interface Editor and Sub Application Function Block Editor. The tab Service of the Adapter function block editor serves the same purpose as the Service Function Block Editor.

The interfaces and the associations between input variables (InputVars) and event inputs (EventInputs) — and the same on the output side — can be configured in the Adapter function block. The Adapter function block presents the following element groups:

  • EventInputs (interfaces)

  • EventOutputs (interfaces)

  • InputVars (variables)

  • OutputVars (variables)

When an Adapter function block has been created, the ports Sockets and Plugs can be edited in the Interface editor of Composite, CAT, Service and Subapplication function blocks. Socket- and Plug-ports are marked by two arrows (>>). The connection between adapter ports of function blocks (Sockets, Plugs) is shown as a thicker brown line, to mark it as an adapter connection line. By touching such a line with the mouse, the Adapter function block is displayed as tooltip. With that it can be quickly identified which connection lines are embedded in this one line.

For instance, the whole interface, which is necessary to control one sunshade unit (sunblind) respectively to read out its state, could be collected by an adapter to one single line. In that way additional units can be added very quick.
