Configuring a Soft dPAC Controller as I-Device

Procedure

After you add a Standard.IoProfinet.PN_IOController node to your Hardware Configuration, you can configure your Soft dPAC I-Device.

NOTE: As an I-Device, the Soft dPAC can be configured as an IO-Controller, an IO-Device, or both an IO-Device and an IO-Controller.

  1. In the Hardware Configuration editor, right click the Standard.IoProfinet.PN_IOController node and select Configure.

  2. Select a Role for the device:

    • IO Controller: Only an IO-Controller can be configured. The IO Device configuration option, available using the Advanced… command, is disabled.

    • IO Device: Only an IO-Device can be configured. The Advanced… command (for device configuration) is enabled; the Available Devices catalog (for IO-Controller configuration) is disabled.

    • IO Controller and IO Device: Both an IO-Device and an IO-Controller can be configured. Both the Advanced… command and the Available Devices catalog are enabled.

    NOTE: The following instructions describe how to configure a Soft dPAC as only an IO-Device or as both an IO-Controller and an IO-Device.

    For instructions on how to configure the Soft dPAC as only an IO-Controller, refer to the instructions for Configuring a PROFINET IO-Controller.

    NOTE: Changing a previously configured device Role can change other device settings. For more information, refer to the topic Changing the I-Device Role.

  3. To configure the Soft dPAC as IO-Device, click Advanced…. The Network Configuration > Advanced Configuration page opens. Use this page to configure IO-Device properties and IO data items.

  4. Configure the following properties:

    • Maximum Connections: Input the maximum number of Application Relation (AR) connections supported by the IO-Device. If the value is 2 or greater, the IO-Device will support Shared Device and Shared Input.

    • Allow parameter assignment of PROFINET interface by higher level controller: The PROFINET interface of an IO-Device can be assigned parameters only from the higher-level IO-Controller or from the IO-Device itself. If this item is:

      • Selected: Configuration of the PROFINET interface and its ports is performed by the higher-level IO-Controller.

      • De-selected (cleared): Configuration is performed at the IO-Device.

      NOTE: This option is enabled only when the selected role is IO Device. This option is disabled and cleared when the selected role is IO Controller and IO Device because, when the controller is running, parameterization from an external controller is not permitted.

    • Address Assignment: Indicates how the device IP address is set:

      • LOCAL: Assigned at the IO-Device.

      • DCP: Assigned using a DCP Set command from a remote controller.

        NOTE: DCP option is available only if Allow parameter assignment of PROFINET interface by higher level controller is selected.
      NOTE: When you select the option Allow parameter assignment of PROFINET interface by higher level controller or set Address Assignment to DCP, a cybersecurity message is displayed stating:
      You are enabling advanced features that may expose the device to increased cybersecurity risks. Ensure these changes are necessary and that your network is properly secured.
      Click OK to close the message, then take any necessary cybersecurity precautionary actions.
  5. Configure IO data items. For IO-Device configuration, at least one IO data item is required.
    NOTE: By default, the IO Data Configuration grid (at the bottom of the Advanced Configuration page) is populated with one data item.
    1. To add a data item, click Add DataItem. A new data item is added to the grid. Each data item presents the following attributes:

      • DataItem Name: Read Only assigned name of the data item.

      • DataItem Type: Select the IO type: Input or Output.

      • Direction: Read Only automatically assigned target direction of the IO data item, based on the selected DataItem Type.

      • DataItem Length: Input the data item length in bytes. Value range = 1 to 1024 bytes.

      • SymLink: The symbolic link name for this data item. The PROFINET configurator suggests a default symlink name, which you can accept or edit.

    2. To delete a data item, click the delete icon ( ) at the right of the data item.

    3. To save the new or edited IO-Device configuration, including the new data items:

      1. Click Update and Close in the Network Configuration main page.

      2. When prompted to Update and Close, select Yes to save and close the configuration.

      NOTE: Alternatively, if you select No the unsaved configuration remains open. Then you can click Cancel in the Network Configuration main page to close the configuration without saving your edits.
