Workflow from Stopped State

  1. The Run command executes only after a full deploy.

  2. After a Restart Runtime command, the state of the device is determined by the Runtime Configuration > Deploy Settings Auto start after deploy setting (true, false).

  3. After a Reboot Device command, the state of the device is determined by the Runtime Configuration > Boot > Behavior on reboot parameter setting: Run, Stop, LastState, or LastStateIgnoreErr.

NOTE: For devices that include a physical STOP switch, the device enters the Running state only if the switch is set to the OFF position. If the switch is set to ON, the device enters the Stopped state.

Workflow from Running State

  1. On execution of the Restart Runtime and Reboot Device commands, the state of the device is determined by the Runtime Configuration > Boot > Behavior on reboot parameter setting: Run, Stop, LastState, or LastStateIgnoreErr.
