Deploy

The Deploy menu item links to child commands that depend upon the selected object or device.

The Action Menu Deploy item links to the following commands:

Command

Description

Deploy
(Single Deploy -
HA Partner)

Sends the compiled application to selected device.

  • After a successful deployment, the behavior of the controller depends on the attribute (set in the Logical Devices editor of the System editor)  Runtime Configuration > Deploy Settings > Auto start after deploy. If this attribute is set to:

    • true: A non-HA controller immediately enters the Running state (unless a physical button on the device is used to prevent the start of the application). An HA controller performs a clean operation and enters the Clean state and Muted role.

      NOTE: Execute a Single Deploy on an HA controller only when the HA controller is in an Unpaired role and Clean state.

    • false: (default setting) The controller enters the Stopped state. To start the controller, select Device actions > Run.

  • For very large solutions or very slow devices, the time to complete deployment can be long, resulting in a timeout. In this case, execute a Clean then another Deploy command. To reduce the likelihood of a timeout, increase the values of the long and short timeout settings. To do this, make edits in two places:

    • Select File to open the backstage, then select Installed Plug-ins EcoRT Plugin, then click  Settings to access and edit the values for Long Command Timeout . Also:

    • In the Logical Devices editor, select a device, then in the Properties pad, navigate to Runtime Configuration > Communication > Deploy  then edit the setting for the TimeoutLong parameter.

Refer to the Operating States chapter for information on how the Deploy command impacts device operating states.

Deploy
(HA Pair)

The high availability Deploy command sends the compiled application to one or more pairs of high availability controllers. Refer to the topic describing the HA Pair level command Deploy in the EcoStruxure Automation Expert – HA Soft dPAC User Guide.

Deploy Archive Link Configuration

Sends to devices the configured Archive_Link object configuration by connecting EcoStruxure Automation Expert - Buildtime to the Archive_Database object previously created using the Create Project Database command. 

NOTE: Refer to the EcoStruxure Automation Expert - Archive User Guide.
Clean
(Single Clean - HA Partner)

Deletes both the application (program logic) and hardware configuration (I/O and communication) binary files created for the device. 

Refer to the Operating States chapter for information on how the Clean command impacts device operating states.

Clean
(HA Pair)

Deletes the application (program logic) and hardware configuration (I/O and communication) binary files for a pair of high availability controllers. Refer to the topic describing the HA Pair level command Clean in the EcoStruxure Automation Expert – HA Soft dPAC User Guide.

Online Change

Sends the compiled online changes to selected devices.

Refer to the topic Other Action Menu Commands for information on how the Online Change command impacts device operating states.

Single Online Change
(HA Partner)

Deploys the application’s online change snapshot to a selected high availability controller during runtime. Refer to the topic describing the HA Partner level command Single Online Change in the EcoStruxure Automation Expert – HA Soft dPAC User Guide.
