Operating States

The operating state of a dPAC indicates its current execution status that determines its current operating behavior and possible transitions to other operating states.

State Machine:

dPAC Operating States

A dPAC manages four operating states::

  • Clean: The execution memory of the controller is empty (there is no project nor hardware configuration deployed).

  • Running: The controller is executing the application.

  • Stopped: The controller is idle, not executing the application.

  • Error Halt: The controller is idle, not executing the application, and reporting an error detected while:

    • Checking the validity of the application when deployed or loaded (in Clean state), or

    • Executing the application (in Running State).

State Transitions

Transitions between states can be triggered:

  • Manually, by sending commands to the controller (for instance a Running controller goes to Stopped after a Stop command).

  • Automatically, under predefined conditions (for instance a Running controller goes to Error Halt when program execution errors are detected).

Operating state diagrams represent visually the logic allowing state transitions under specific conditions or when commands are triggered and are a convenient tool to understand the controller behavior.
