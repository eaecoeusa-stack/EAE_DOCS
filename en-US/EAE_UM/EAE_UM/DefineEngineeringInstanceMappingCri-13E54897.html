3. Set Mapping Criteria For Engineering Instances

You have created at least a CAT with instances, usually from the SE.AppCommonProcess library.

To help associate engineering instances to CATs or SubCATs , you must set mapping criteria on various levels from the CAT and launch a multi-faceted search.

To edit criteria for CATs and SubCATs:

Step

Action

1

In the Solution Explorer pad, drill down and double-click the CAT (or Open it).

2

In the Engineering Instance CAT Mapping editor >CAT Details pane, select the CAT or a Sub-CAT in the hierarchy.

3

In the Mapping Rules pane:

  1. Tick all the mapping rule types to apply to the CAT or SubCAT.

  2. Specify the priority of the ticked rules by dragging and dropping them in the list. The top-most rule is assigned first priority; the second rule is assigned next priority, and so forth.

4

Click a mapping rule to display its criteria form in the Criteria pane.
NOTE: To select a rule type, click the rule type (not the check box).

5

In the Criteria pane, set the criteria for the ticked rule as described in the section Mapping Rule Types.

6

Click the Save icon in the quick menu bar.

NOTE: If a rule was ticked, but no criteria value was entered, the step 6 will be denied.

Mapping rule types

The following rule types are supported:

Mapping Rule Type

Description

Engineering Instance Name

Name of the piece of equipment or instrument. To enforce this rule, you need to set criteria in one of the following ways:

  • Start with: the Engineering instance name must start with a particular string (for example 100)

  • Contains Match: the Engineering instance name must contain a particular string (for example ZSH )

  • Regex Match: the Engineering instance name must match a particular regular expression ((for example [0-9]*-[A-Z]*T-[0-9]*).

To have multiple values to the matching criteria, separate the values by pipe '|'. For example, to set Contains Match criteria that includes ZSH and ZIL enter the value as ZSH|ZIL

NOTE: Signal conditioning node: a list of ISA codes is proposed for selection.

Signal Count

Number of expected signals (number of IO signals required by this CAT).

To enter signal count rules:

  1. Enter a Type (AI, DI, AO, DO, ...). You can enter multiple signal types ( DI, SB).

    NOTE: Type setting is required.

  2. Enter an Exact value or Min threshold.

  3. Click Add to create the new rule; this populates the summary table.

  4. If needed, tick a row to delete in the summary table, and click Clear.

NOTE:

  • Total Signals is in read-only mode (sum of Exact and Min inputs).

  • The table is pre-filled based on CAT or SubCAT main properties.

Signal-Type

Signal type (AI, DI, AO, DO, ...).

In the Equals field, enter the type.

NOTE: The value of signalConditioning is pre-filled based on subCAT main properties.

Attribute

This rule is used ONLY WHEN NO other rule got a hit.

This rule lets you define different criteria for different attributes (for example Instrument type == Indicating Instrument, here “Instrument type” is an attribute and its expected value is “Indicating Instrument” and the operator is Equals (==)). Multiple attributes can be defined for this rule.

To enter attribute type rules:

  1. Enter in the Attribute field for example HMI.AlarmText.

  2. Select the Operator.

    NOTE: The Attribute and Operator settings are required.

  3. Enter in the Value field the value to hit for example SignalFailure.

  4. Click Add to create the new rule; this populates the summary table.

  5. Select AND if all rules should be satisfied for match or select OR if any rule should be satisfied for match.

  6. If needed, select a row to delete in the summary table, and click Clear.

NOTE: To increase the specificity, define more rules in attribute rule criteria. Higher specificity can eliminate ambiguity and result in desired match.
NOTE: If a rule type is ticked, but no criteria is set, an error will be raised on Build.

Criteria based on CAT HMI Output Variable Attributes

In addition and at the lowest level, you can specify the expected HMI output.

Attribute mapping can be edited in the Engineering Instance Attribute Mapping editor. This editor shows the hierarchy of the CAT with the output variables for each CAT and SubCAT.

Step

Action

1

In the Solution Explorer pad, drill down and double-click the CAT (or Open it).

2

In Engineering Instance Attribute Mapping editor , drill down to the lowest level.

3

In the Engineering Instance column, enter the required string (the field may be pre-filled with a list of synonymous strings).

4

Click the Save icon in the quick menu bar.
