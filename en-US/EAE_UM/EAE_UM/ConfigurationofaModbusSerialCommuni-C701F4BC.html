Configuration of a Modbus Serial Communication

The configuration of a Modbus serial communication requires to add and set up:

  • A bus, here using the protocol Modbus RTU.

  • A generic Modbus device.

  • And Modbus registers coming from the SE.ModbusGateway library.

All these elements have to be added into the Hardware Configuration editor:

  1. Open the Hardware Configuration editor.

  2. Add a new bus by clicking on the button Add new master HW-CAT:

  3. In the Add Bus dialog opening, select Standard.IoModbus.MODBUSSERIAL, name it and click on Add.

    NOTE: Clicking on Add doesn’t close the Add Bus dialog thus several buses can be added in the row. Click on Close to close it.

The bus is now added to the Hardware Configuration editor. It has several parameters available by:

  • Expanding the bus node. These parameters can remain by default.

  • Opening the Properties window. There are all the parameters defining the serial communication.

Check that all the parameters are correctly set for your Modbus serial communication.

Once done, a Modbus device shall be added to the bus:

  1. Select the bus line.

  2. Once selected a new button next to Add new master HW-CAT will appear: Add new HW-CAT. Click on it.

  3. The Add To MODBUS dialog appears with all the devices available to add to the bus. Select Standard.IoModbus.MODBUSGENSERIALS, choose a name and add it.

The Modbus device is then appearing under the bus.

Once added, set up the parameters stationid and responsetimeout in its Properties.

Finally, Modbus registers shall be added to the Modbus serial device.

As detailed in the chapter Modbus, the only way to do this action is to add manually the Modbus registers into the Modbus serial device. Thus, to know how to add these Modbus registers, follow the chapter Manual Development.

Once this action will be completed, the Modbus serial communication developed in EcoStruxure Automation Expert will be ready for transmitting information.
