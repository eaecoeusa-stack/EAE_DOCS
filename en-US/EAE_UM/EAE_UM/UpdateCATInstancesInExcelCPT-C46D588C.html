Update CAT Instances in Excel

Once the CAT Instances and their information have been exported into an Excel file, open this file to bulk create new CAT instances and to modify existing offline parameters.

Before to be able to do so, let’s discover how the Excel file is organized:

Index sheet

It lists all the CAT types selected during the Configure worksheet breakdown step of the CAT instances export. For each of these CAT type is given a link to move to the CAT type sheet.

CAT type sheet

For each of the CAT type sheet, it gathers the following information:

CAT Instance ID

It is the identity number of the CAT instance. Thus this cell cannot be updated by the user. During the import, EcoStruxure Automation Expert will use this cell to know if it is a:

  • Modification of an existing CAT instance: If the CAT Instance ID cell contains the ID of a CAT instance. Thus all modifications done will be applied to the CAT instance with the same ID in EcoStruxure Automation Expert.

  • Creation of a new CAT instance: If the CAT Instance ID cell is empty. Thus a new CAT instance will be created in EcoStruxure Automation Expert with all the information given in its line.

Application Name

It defines in which application the CAT instance is or will be located.

Layer Name

It defines in which layer of the application, the CAT instance is or will be located. The Layer Name will be blank when CAT instance is located in the default layer.

CAT Instance

It is the path to find the CAT instance in EcoStruxure Automation Expert. Thus it contains all the names of the function block from the CAT instance until the function block in the application. For example, if the function block exported is a sub-sub CAT instance, its CAT Instance value would be ‘CATInstanceName.SubCATInstanceName.Sub-SubCATInstanceName’, in practice it could be ‘DigitalOutput.signalConditioning.ChFailureAlarm’.

Comment

In this cell is shown the comment given to the CAT instance.

HMI Alias

This cells shows the HMI Alias name given to the CAT instance.

NOTE: To know more about HMI Alias name, check the topic Creating and Displaying HMI Alias Names.
Asset Tag

This cell shows the Asset Tag name given to the CAT instance.

NOTE: To know more about Asset Tag name, check the topic Creating Alias Names for CAT and Sub-CAT Instances.
The rest

In the column at the right of Asset Tag is located:

  • The offline parameters of the CAT instance.

  • The information and offline parameters of the sub CAT instances not selected to have their own sheet. To identify them, their name is composed of the sub CAT instance name and the name of the information or offline parameters.

    Example: ‘fbRun.ChFailureAlarm.Asset Tag’

Connection Details

In this sheet is listed all the connections available between the CAT instances defined in the CAT type sheets.

Each line in this sheet defines one connection and shows the following information:

  • AppName: It defines the application name in which the connection is done.

  • Source: It defines the full source of the connection following this standard: ‘[Function block name].[Event or data output name]’.

  • Destination: It defines the full destination of the connection following this standard: ‘[Function block name].[Event or data input name]’.

  • Connection Type: It defines the type of connection between:

    • AdapterConnection

    • DataConnection

    • EventConnection

  • Cross Reference: It defines if the connection is using cross reference (True) or not (False).

NOTE: This Excel only supports English terms.
NOTE: Each sheet and each of these columns are necessary. They can remain empty but should not be removed to avoid issue during the import.

Knowing this, a few actions are available:

  • Modify existing CAT instance: As the CAT Instance ID remains the same, anything else can be modified (even the application name, layer name and the CAT instance name), the CAT instance will always be recognized.

  • Modify the layer to another existing layer will move the CAT instance to this second layer. If the layer mentioned does not exist in the application, it will be created and the CAT instance will be assigned to it.

  • Modify or suppress connections: Existing connections received from the export can be modified by changing values in the line or deleted by removing the full line.

  • Create a new connection: The recommended practice is to copy paste an already existing line of a connection and to modify the values as wanted.

  • Create a new CAT instance: The recommended practice is to copy paste an already existing line of a CAT instance and to modify the values as wanted.

    If a CAT instance is created from scratch, the only parameter mandatory to fill is the CAT Instance. The other parameters will either take the value given to them, either take the default value if they remain empty. In the sheet of the sub CAT instances, mind the value given to the CAT Instance. It shall be the path from the application until the sub CAT instance. Example ‘DigitalOutput.signalConditioning.ChFailureAlarm’.

    One way to create new CAT instances and fill all the sheets without mistake could be to:

    1. Fill just the CAT Instance value of the new CAT instances.

    2. Import them to create them into EcoStruxure Automation Expert.

    3. Export the CAT instances just created with all its offline parameters.

    This way the new exported Excel file will be automatically filled with the default value and right names for each of the new CAT instances. Their values can then easily be modified.

NOTE: Check that the application name written is with the right spelling otherwise the updated CAT instance will be located into the folder Not Assigned to System Application of the CAT Instances section. More information about this folder will be given in the topic Import CAT Instances.

When all updates in the Excel file are done, save and close it. Its content is then ready to be imported into EcoStruxure Automation Expert.
