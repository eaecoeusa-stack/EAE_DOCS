Filtering Alarms in the EcoStruxure Automation Expert - HMI

There are two mechanisms for filtering alarm events in the EcoStruxure Automation Expert - HMI:

  • Toolbar icon commands

  • Context menu commands

The following commands are supported:

  • Filter Alarms (): opens the Alarm Filter dialog.

    For Online Alarm Type (i.e. active alarms):

    Select either:

    • Filter from list: select a function block instance name, to display only alarms for that function block; or

    • Filter from searched text: enter the text search criteria, using asterisk as a wildcard to represent any combination of characters.

      NOTE: for Online alarm types, each active alarm normally appears against a white background. A pink background indicates the connection between the controller and the HMI has been lost.

    For Archive Alarm Type (i.e. alarms stored in the archive server):

    Select either:

    • Filter from list: as described above, or

    • Interval Type: filter by specified historical time period

    NOTE: When filter is applied, the Local alarm filter applied icon () appears on the right side of the toolbar.

  • Restore alarm filter notification (): Applies the filter used at HMI startup. In, the Restore alarm filter notification dialog, select: Yes to apply the original filter, No to keep the current filter.

  • Clear alarm filter notification (): Turns off the alarm filter. In, the Clear alarm filter notification dialog, select: Yes to remove the alarm filter, No to keep the current filter.

  • Alarm details (context menu only): Select an alarm, then click this to show its details.
