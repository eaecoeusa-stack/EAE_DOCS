Configure SYSLOG

To implement SYSLOG, you need to:

  1. Install third-party SYSLOG server software on a network workstation or server.

    NOTE: Refer to the documentation provided by the third-party manufacturer of the SYSLOG server software for instructions on how to install the SYSLOG server.

  2. Then in the EcoStruxure Automation Expert - Buildtime Physical Devices editor:

    1. Enable the SYSLOG service.

    2. Configure the SYSLOG client in each device or HMI that supports the SYSLOG service.

Enable the SYSLOG Service

  1. In EcoStruxure Automation Expert - Buildtime, open the System > Physical Devices editor.

  2. From the Workstation & Servers group of the device catalog, drag and drop a workstation or server into the work area.

  3. Right click the workstation/server and select Manage Services > SYSLOG Server > Activate service. In the Properties pad, the Services > SYSLOG Service node is added

    .
    NOTE: You can add multiple workstations and servers and enable the SYSLOG service for each device.

  4. Expand the SYSLOG Server node, and complete the following SYSLOG server settings:

    NOTE: You will take these settings from the remote SYSLOG server configuration.

    • Interface\Endpoint :Select Any or a specific NIC.

    • Security mode: Select one of the following:

      • Secured: Accepts only Transport Layer Security (TLS) transmissions.

      • Unsecured: Accepts only Transmission Control Protocol (TCP) transmissions.

      • Secured & unsecured: Accepts both TLS and TCP transmissions.

      NOTE: Once selected, you cannot de-select Secured or Unsecured if a SYSLOG client is configured to use this mode.

    • Logical port: select a port on the server host for secured and/or unsecured messaging.

Configure a Controller SYSLOG Client

  1. From the Distributed automation controllers > Modicon dPAC group of the device catalog, add a controller that supports a SYSLOG client.

  2. Click the controller (the smaller rectangle) to display its Properties.

  3. Expand the Services > SYSLOG Client node, and complete the following settings:

    • Enabled: Select True to enable the SYSLOG service for this controller.

    • SYSLOG server: Select a configured SYSLOG server from the list.

    • Security mode: This field is read-only, and is set to the Security mode (Secured, Unsecured, or Secured & unsecured) that you selected when enabling the SYSLOG service.

Configure an HMI Runtime SYSLOG Client

  1. In the Physical Device editor device catalog, navigate to Harmony > Harmony operating Panel, and add a Harmony ST6.

  2. Open the Solution Explorer navigate to and right click on [Solution Name] > System > Devices, then click New Device. The New Device dialog opens.

  3. Select a device of Type SE.Standard:HMI_NET, enter a Name and Count, click OK.

  4. In the Logical Devices, select the HMI_NET device.

  5. In the Properties pad, under Syslog Server, configure the connection as follows:

    • Set Enabled to True.

    • Enter the IP Address of the SYSLOG server.

    • Enter a Port number: either the secured or the unsecured logical port assigned to the SYSLOG server in the Physical Devices editor.

    • Select a Mode: TCP for unsecured / TLS for secured.

Deploy the SYSLOG Configuration from the Server to a Client

  1. Confirm that the controller is reachable.

  2. Confirm that the controller is secured.

  3. If the controller is not secured, secure the controller.

  4. Right click on a reachable, secured controller for configuration as a SYSLOG client, then select Configure device settings.

  5. Click on the Services tab. A row showing the status of the SYSLOG service is displayed. SYSLOG service status can be:

    • Up-to-date: SYSLOG server and client have the same settings. The APPLY command is disabled.

    • Out-of-date: SYSLOG server and client have different settings. The APPLY command is enabled.

      NOTE: The SYSLOG service will change to the Out-of-date state from the Up-to-date state when:

      • The IP address or logical port of the SYSLOG server is changed.

      • The SYSLOG service has been disabled in the EcoStruxure Automation Expert Physical Devices editor.

    • Not supported: Controller firmware needs to be updated.

    • Unknown: The controller is no longer secured.

  6. If the SYSLOG service status is Out-of-date, click Apply to send the URL address and logical port settings from the SYSLOG server to the controller:
