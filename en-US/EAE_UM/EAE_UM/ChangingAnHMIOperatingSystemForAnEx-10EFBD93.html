Modify HMI Operating System

The HMI operating system type – Windows or Linux – is specified when you create a new solution. You can also change the specified HMI operating system type for an existing solution.

To change an HMI operating system type, you need to complete the following tasks:

  • Specify a new HMI operating system type

  • Remove external references and dependencies related to the old HMI operating system type

  • Create a new canvas resolution

  • Configure a hardware physical HMI device to serve as the HMI

  • Map the hardware physical HMI device to a logical HMI device

  • Configure the new canvas to act as the start-up canvas for the logical HMI device

  • Save, Compile and Deploy

Specify a New HMI Operating System Type

  1. Open an existing solution.

  2. In the Solution Explorer, right click the Solution node and click Select HMI Compatibility. Two HMI operating system options are displayed:

    • Multi-OS HMI

    • Windows HMI

    A check mark indicates the current selection, which is disabled.

  3. Click the enabled option to change the HMI operating system type.

Remove External References and Dependencies

  1. In the Solution Explorer, right click the dPAC node and select Remove HMI External Reference. The Remove External SearchResultMatch dialog opens.

  2. Select any external dependencies and references that do not apply to the new HMI operating system type, then click OK.

Create a New Resolution

  1. To create a new resolution, right-click the Canvases node and select New Resolution.

    The Start Canvas dialog opens.

  2. Complete the settings for the Resolution, then click Next.

    NOTE: The Resize Behavior setting does not apply to the Multi-OS HMI operating system type.

  3. Complete the settings for the Canvas Logo, then click Finish

Configure a Hardware Physical HMI Device

  1. In the System editor, drag and drop the HMI device into the Physical Devices editor.

  2. Select the device, and in the Properties pad, configure its Name and IP Address.

  3. Create a wiring link in the Physical Devices editor connecting the HMI device to the Soft_dPAC controller.

    NOTE: This step assumes the Soft_dPAC controller has already been created and properly configured.

Map the Physical HMI Device to a Logical HMI Device

  1. In the Logical Devices editor, select the logical HMI device.

  2. In the Physical Device column, map the logical HMI device to the physical HMI device.

Configure the New Canvas to be the Start-Up Canvas

  1. In the Logical Devices editor, select the logical HMI device.

  2. In the Properties pad, click the Startup Canvas field to open the CanvasTopologyStartConfigEditor.

  3. Set the new canvas as the Startup Canvas, and configure any other settings as needed.

  4. Click OK.

Save, Compile and Deploy

  1. In the main toolbar, click Save All.

  2. In the Deploy and Diagnostic editor,

    1. Click Action > Compile.

    2. Click Action > Deploy.
