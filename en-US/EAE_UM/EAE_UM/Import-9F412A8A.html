Import

Use this command to import archived elements into the project or library, depending on where the command is executed.

To open the dialog:

  • In the Solution Explorer right-click on the solution node or a node of a library which has been created within this solution (that means no installed library) and then click on context menu entry Import. The Open dialog displays.

  • In the Open dialog, navigate to and select the file to import (it will have a .zip extension), then click Open. The Import dialog displays.

  • In the Import dialog, select the Import Type and click OK to complete the import process.

The dialog will also open after a right-click on a FB node within the solution or a library that has been created in the solution (e.g. on node Basic, Service, CAT or Composite , etc.) and then a click on Import in the respective context menu. But in this case only the archived files of the respective type of the selected folder will be listed.

Importing Archived Elements

Right-click in the Name column on the solution node or a library node to list the archive files of the types of the selected folder. Depending on the selected type, the element will then be imported into the relevant node automatically.

The Import dialog includes the following fields and commands:

Command

Description

Name

Shows the function block or element which has been selected for the import.

Import Type

Click on the existing entry and an arrow appears on the right edge of the field. Click on that button to opens a menu that contains various entries depending on the requirements, including:

  • Nothing → The dialog will be closed without causing any effect in the solution after clicking OK (same as clicking Cancel).

  • Overwrite → An element with the same name already exists and will be overwritten now by the imported element.

  • Overwrite Detail → Overwrites only a part of the element. (e.g. one faceplate of a CAT; Basic function blocks, for example, can only be overwritten completely and therefore this functionality is not available for Basic FBs.)

  • Add → This is selected automatically when importing an element with a name that is not already used in the solution. But Add can also be used to import an element with the same name as an element already in the solution. In this case a dialog opens to edit the new name for the element to be imported.

  • Add Detail → Use this to add only some parts of the archive file. (e.g. one CAT from a .zip-file containing a couple of CATs.)

Reason

Read only field automatically populated by EcoStruxure Automation Expert - Buildtime indicating the reason for a suggested selection of Import Type. See note, below.  

OK

Closes the dialog and performs the selected action.

Cancel

Closes the dialog without any action.

NOTE:  When importing blocks, the internal GUID of the block to be imported is compared with the existing blocks in the project. If this number matches, an Overwrite action is proposed by default. This means that - even with different block names - an Overwrite action can be suggested in the case where the module was initially exported, but the name of the same block in the project was subsequently modified. In this case, the block name in the project will be left unchanged by this import action. The column Reason presents the explanation, in this case, that ' Same GUID found'.
