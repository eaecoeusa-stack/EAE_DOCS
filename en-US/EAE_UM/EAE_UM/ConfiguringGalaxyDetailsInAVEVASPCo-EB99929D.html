1. Configuring Galaxy Details in AVEVA SP Configuration

In the Solution Explorer pad, right-click the project node. From the context menu, select Configurations > AVEVA SP Configuration to launch the System Platform Configuration dialog. In this dialog, you can:

  • In the Galaxy tab, create a new Galaxy or select an existing Galaxy.

  • In the Object Naming tab, specify a 2-character Template Prefix to be added to Galaxy System Objects names and area/application object names. The use is described in AVEVA System Platform Template Mapping of Library CATs and Solution CATs.

    NOTE: The Template Prefix is used in conjunction with the Automap feature. (Refer to Template Mapping – Online Mode.) It is optional.
    NOTE: When you click on icon, it opens this current topic from the EcoStruxure Automation Expert - User Manual.

A Galaxy server may host a Galaxy with several System Objects sets. That is why in the dialog you may have to select the System Objects individually through drop-downs. Various situations can be encountered:

To select an existing Galaxy:

Step

Action

1

Enter the Galaxy Server name. Click anywhere to validate the server name.

2

In the Galaxy tab, tick Select Galaxy and System Objects then select a Galaxy from the list.

NOTE: By default, the Galaxy’s on the local server are listed. Change the Galaxy Server to list the Galaxy’s on a remote server.
NOTE:

  1. Galaxies list can be fetched either by selecting the Select Galaxy dropdown list or by selecting the (refresh icon ) beside the Galaxies list.

  2. After selecting the Galaxy from the list, the associated Authentication mode is displayed against the Authentication Mode.

3

Enter the Username and Password for the selected Galaxy.

NOTE:

  • If the Authentication Mode of Galaxy is other than None (No Security Enabled Galaxy), then provide a username and password to proceed.

  • Since you are selecting an existing Galaxy, Galaxy Type (cab file name) is in read-only mode.

4

Click Validate.

If the validation succeeds, the Object Naming tab shows the current prefixes.

5

If you are using an existing galaxy that does not include the latest OPC client template ($aSEOPCClient), then import the following files into the Galaxy

  • The “OpcUaClientConfigurator.aaSLIB” script library file from the path “<EAE Installation Path>\ASP Libraries\Common Process Library\Installation Files\Script Function Libraries“.

  • The “ApplicationTemplates.aaPKG” file from the path “<EAE Installation Path>\ASP Libraries\Common Process Library\Installation Files\Application Templates”. This file contains the new OPC client template.

6

If any error occurs during selection or validation of the Galaxy connection, a brief information is shown in the Error section of the window with an icon. Hover your mouse on the error to see the complete error message.

For corrective action, a Need help ? link is available beside the error. Click on it to open the Troubleshooting Tipsof the EcoStruxure Automation Expert - User Manual.

To create a new Galaxy:

Step

Action

1

In the Galaxy tab, tick New Galaxy with System Objects, then enter a name for the new Galaxy.

NOTE: By default, the new Galaxy is created on the local server. To create a Galaxy on a remote server, change the Galaxy Server and verify that the CFLBlankGalaxy.cab file is placed in the BackupGalaxies folder of the remote Galaxy server machine.

The Galaxy Type list displays a single, read-only .cab file type: CFLBlankGalaxy.cab.

NOTE: The new Galaxy is accessed with the default user name and password of the AVEVA System Platform, which are applied automatically.

2

The Authentication Mode area offers three options (Galaxy, OS User Based, OS Group Based) for authenticating the newly-created Galaxy. When you set Authentication Mode to OS User Based or OS Group Based, you can access the IDE and log in using the default user name and password, then add the OS user / group to the users list.

NOTE: When Authentication Mode is set to OS Group Based, the user needs to add a group in IDE whose s/he is a member, and provide application object creation and deployment permission to this group.

3

Click the Next button to access the Object Naming tab.

To use a Galaxy Cabinet file other than the default one provided with EcoStruxure Automation Expert, follow these steps:

Step

Action

1

Place the cabinet file in the ASP path: C:\Program Files (x86)\ArchestrA\Framework\Bin\BackupGalaxies

2

Launch ASP IDE and create galaxy selecting the cab file copied in above step.

3

In EcoStruxure Automation Expert, go to AVEVA SP configuration and in Choose existing section, select the galaxy created in above step from the drop-down list, validate it and save the configuration.

4

Template synchronization is started automatically and all ASP templates and attributes available in the configured galaxy are synchronized with EcoStruxure Automation Expert.

Follow the next steps in the Object Naming tab to specify object name prefixes:

Step

Action

1

Specify the prefixes in the System Objects area. By default the value is AB (which stands for AutoBuild), modify the prefixes following these rules:

  • The maximum length of the prefix is two characters.

  • The first character must be alphabetic (A...Z).which

  • The second character must be alphanumeric (A...Z or 0...9).

If NO prefix is specified, the prefixes will take the value: EAE_OMI_.

2

Click the Next button to open the Application Objects sub-tab.

3

Define the AppObject prefix. The default value is AB.To modify the prefixes, follow the same rules as in the previous step.

If NO prefix is specified, no prefix will be assigned to the application objects.

3

Define the AppObject Naming source:

  • CAT Instance: With this default value, the name of an application object will be the one of the CAT instance.

  • Asset Tag: The name of the application object will be the Asset Tag. If the latter is missing, the former will take over in the AVEVA System Platform.

  • HMI Alias: The name of the application object will be the same as the HMI alias. If the latter is missing, the CAT Instance name will take over in the AVEVA System Platform.

For details on how to:

  • Configure an Asset Tag, see the topic Creating Asset Tag for CAT and SubCAT Instances.

  • Configure an HMI alias, see the topic Creating and Displaying Alias Names for CAT and SubCAT Instances.

4

Define the Template prefix. If the Galaxy is shared by multiple EcoStruxure Automation Expert solutions, it is recommended that you assign a different prefix for each Solution to avoid naming conflicts.

5

Click the Next button to open the Communication Objects sub-tab.

6

Define the OPC Client prefix. The default value is AB, which stands for AutoBuild. To modify the prefixes, follow the same rules as in the previous step.

Result with AL: AL_OPCClient_<EcoRTDeviceName>

7

Click Save to save the Galaxy configuration file (AvevaSPconfig).

8

Since the configuration is completed, you may want to disconnect the Galaxy: click Disconnect in the Galaxy tab and close the dialog. As a result, the cached information and the relations are deleted.
