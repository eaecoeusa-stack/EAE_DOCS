Copy/Paste & Export/Import for Library Elements 

This feature allows duplication and customization of elements such as CAT, SubApp, Composite, Basic, Adapter, and Graphics within or across solutions.

Prerequisites:

  • Ensure required libraries are installed and referenced.

  • Save and build the solution before starting.

  • Confirm the destination solution has access to the required source libraries

Supported Elements:

  • CAT

  • SubApp

  • Composite

  • Basic

  • Adapter

  • Graphics

  • Symbols, Named Styles, and $Key Text (for translations)

Copy/Paste Process (Same Solution)

Copying CAT or Other Function Blocks

  1. Right click

    on the CAT (or SubApp, Composite, Basic, Adapter) you want to copy.

  2. Select Copy.

  3. In the destination custom library, Right click and select Paste.

  4. If the required source library is not referenced, a dialog appears:

    • Click Yes to add the required reference.

    • A Define New Name dialog appears. Enter a new name, and the solution will be rebuilt. The element can now be modified.

    • Click No if you do not want to add the reference. The element will be pasted, but a warning icon appears due to the missing reference.

      NOTE: Only one CAT can be pasted at a time.

Copying Symbols, Styles and $Key Text

All associated Styles and $Key Text (translation data) within CAT elements are automatically duplicated when you Right click → Copy and Right click → Paste.

These can be edited after pasting.

Copying Graphics:

  1. Right click

    on the Graphic you want to copy.

  2. Select Copy.

  3. In the destination library or graphic, Right click and select Paste.

  4. The graphic and its associated C# code are duplicated.

    NOTE: If required libraries are not referenced, an unknown symbol icon will be shown.

Export/Import Process (Across Solutions)

Export

  1. Right click

    on the element (CAT, SubApp, Composite, Basic, Adapter, or Graphic).

  2. Select Export…

  3. In the Browse for Folder dialog, select the target location and click OK.

Import

  1. In the target solution, Right click on the Application node.

  2. Select Import… and browse to the exported element.

    Result: An Import dialog appears:

    • Name: Displays the original name.

    • New Name: Enter a new name if needed.

    • Action Type

    • Add: Creates a new element.

    • Overwrite: Replaces an existing element.

    • Do Nothing: Skips the import.

  3. 4. Click OK.

Sub-Elements

If the element contains a SubApp or sub-CAT, a warning dialog appears:

  • Click Yes to add required references.

  • Click No to skip.

Important Notes

  • Export/Import allows duplication across different solutions.

  • All associated data (offline parameters, OPC UA variables, persistent data, HMI properties, HMI faceplates, symbols, styles, $Key Text, HMI graphics, ASp engineering mapping files) are duplicated. 

  • Copy/Paste works only within the same solution.
