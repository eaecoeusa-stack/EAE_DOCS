Online Change

EcoStruxure Automation Expert intends to offer the capability to make application updates and to deploy them into devices, while they are running, without requiring the overall system to stop.

The online change features are currently still under development and have not yet been fully tested nor validated. Therefore, it is recommended that you use the online change feature only for design and commissioning phases, and not in operations.

NOTE: Use online change features for application design, tests, and commissioning purposes only. Do not use online change features in operational conditions.

Online Change with Operating Application

The following events may occur while performing online change during operations:

  • I/O may not go to fallback state as intended.

  • Hardware and fieldbus scanning cycles may timeout.

Careful examination, understanding, and design of the application is necessary to prevent such event from arising, or to resolve them. For more information, refer to the document Applications Design Guidelines.

Performing an Online Change

Online changes refer to the modifications made on an application, which can be deployed while the dPAC is in Running state.

Online changes can be deployed only after a previous full Deploy or Deploy for Online Change action has been successfully performed.

NOTE: The application is not available in dPACs. Instead, you need to have the existing solution opened in EcoStruxure Automation Expert on your PC.

Not all modifications are compatible with online change deployment:

  • If online change is possible, the Next step status in the Deploy and Diagnostic tab is Online Change deployment possible.

  • If online change is not possible, a full deployment to the target is needed and the Next step status in the Deploy and Diagnostic tab is Deploy necessary.

NOTE: The project state is available only when logged in to the dPAC.

To apply online changes on a device, perform the following two actions in the Deploy and Diagnostic Editor:

  • Compile for Online change

  • Deploy > Online change

 WARNING
UNINTENDED EQUIPMENT OPERATION
Before deploying Online Changes, ensure that the modifications of the running application will not have adverse effects on the operations.
Failure to follow these instructions can result in death, serious injury, or equipment damage.
NOTE: Deploying the changes in simulation mode can be useful for testing the application before deploying the changes to the physical devices.

Online change deployment operations are not instantaneous, and may temporarily reduce the performance of dPAC execution.

The modifications of the application listed below are not compatible with Online Change, as they will require a full deployment if performed (therefore stopping the execution of the application):

  • Any editing of the application source files, i.e., modifying the application manually and thereby invalidating the consistency.

  • Renaming an instance of a deployed object. All data of the re-named object - including inputs, outputs and ECC - are reset to their initial values and states. To continue to use the re-named object, you need to re-initialize it.

NOTICE
ONLINE CHANGE OF UDT TYPE DOES NOT UPDATE GENERIC FUNCTION BLOCKS
Do not attempt to update the type definition of a UDT in the following circumstances:
  • The UDT is assigned to a variable for a generic function block, for example DS_SELECTX
  • A data connection exists for the UDT variable.
  • The generic function block is instantiated inside a composite function block.
In this case, the following detected error message is displayed in the Deploy and Diagnostic window: "Creating FBType Failed to add type template to type library".
Failure to follow these instructions can result in equipment damage.
NOTE: Online change is always applied to all resources present in a device (the device on which online change is deployed), even if the application modification impacts only one of these resources.
NOTICE
UNSUCCESSFUL ONLINE CHANGE DEPLOYMENT
In case the deployed Online Changes have conflicts, the device resource will reject the deployed changes and notice of the rejection will be displayed in the Deploy and Diagnostic window. In this case you must perform a full deployment.
Failure to follow these instructions can result in equipment damage.

Boot Project for Online Change

The Set Active Project as Boot Project property, which is set to false by default, is located in the Properties tab of the Logical Devices editor. When this property is set to True, the user does not need to manually execute the Set Active Project as Boot Project command when performing an online change in the Deploy and Diagnostic tab.

If the Set Active Project as Boot Project property is set to False in the Properties tab of the Logical Devices editor, the active project will not boot automatically after online change is complete. To boot the project, the user can set the active project as the boot project using the device drop-down context menu in the Deploy and Diagnostic tab. This method avoids redeploying the project. Alternatively, you can navigate to the Deploy and Diagnostic tab and use the Deploy To Run Assistant dialog box, where the Set Active Project as Boot Project checkbox will appear checked. However, using the Deploy To Run Assistant will redeploy the project.

Online Change Limitations

Hardware Configuration Online Changes, except Symlinks and the Modbus client Hardware Configuration Online Changes Supported Actions, described above, require a deploy.

Runtime configuration Online Changes require a Restart Runtime or a Reboot Device command.

Updating a CAT library, requires a full deploy of the devices using HW CATs of the updated library.

Online Change Recommendations During Library Development

To perform a successful Online Change of already referenced libraries, follow these recommendations when you develop your libraries:

  • Changes, not allowed for any existing element in a library:

    • Renaming or removing function block types

    • Renaming or removing ports from the interface of function block types

    • Modifying or removing SymLinks names

    • Modifying or removing adapters

    • Modifying or removing ports of adapters

    • Reducing the range of a subrange

    • Reducing an array size

    • Renaming or removing UDTs

    • Renaming or removing members of a STRUCT

    • Renaming or removing enumeration items

  • Allowed changes:

    • Adding new ports

    • Adding new SymLinks

    • Adding new function blocks

    • Modifying the algorithm and/or ECC for Basic function blocks

    • Modifying the data type of a port, only if the existing value can be converted to the new type. Otherwise, the value of the variable is lost after an Online Change and no message is displayed.

NOTE:

  • Always test Online Change of the library for the modified function blocks before releasing the library.

  • In case, a function block or ports of a function block need to be renamed or removed, the function block must not be modified. The existing function block must be marked as obsolete instead. Then a new function block with a new name must be added. The new function block can then be used to replace the existing function block in a solution.

Actions Not Supported by Online Change

EcoStruxure Automation Expert displays a message indicating Project modification is incompatible with Online Change if you attempt to include the following edits in an online change action:

  • Overwriting a type that is mapped to a device.

Additionally, EcoStruxure Automation Expert displays a message indicating Cannot convert variable <variable name> of type <original data type> to type <target data type> when one of the following data types is renamed and mapped:

  • IEC 61131 enumeration types

  • IEC 61131 subrange types

  • IEC 61131 alias types

  • IEC 61131 structure types

  • IEC 61131 function types

  • IEC 61131 function block types

  • IEC 61131 reference types

  • IEC 61499 adapter types

  • IEC 61499 function block types

  • VTQ data types
