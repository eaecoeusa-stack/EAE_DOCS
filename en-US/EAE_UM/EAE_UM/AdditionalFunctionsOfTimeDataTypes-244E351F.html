7. Additional Functions of Time Data Types

All functions described in this topic are conforming to IEC 61131-3 third edition, unless specified otherwise.

CONCAT_DATE, CONCAT_LDATE

Synopsis 

CONCAT_DATE(YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT): DATE
 
CONCAT_LDATE(YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT): LDATE

Description

Creates a date from parameters YEAR, MONTH and DAY. All parameters must be of the same type DINT or UDINT or must be implicitly convertible to one of these types.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the date cannot be converted to an internal representation because of operating system limitations.

ERR_RT_TYPERANGE: A type range error is reported if any of the parameters passed to this function are out of range.

The function IS_VALID can be applied to the date value returned to check if the conversion succeeded.

Example 

VAR_TEMP
  d1 : DATE;
END_VAR
d1 := CONCAT(DAY := 5, MONTH := 12, YEAR := 2021);
IF IS_VALID(d1) THEN
  (* 2021-12-5 is a correct date *) ;
END_IF;

CONCAT_DATE_TOD, CONCAT_LDATE_LTOD

Synopsis 

CONCAT(DATE: DATE, TOD: TOD): DT
CONCAT_DATE_TOD(DATE: DATE, TOD: TOD): DT
 
CONCAT(DATE: LDATE, TOD: LTOD): LDT
CONCAT_LDATE_LTOD(DATE: LDATE, TOD:  LTOD): LDT

Description

Creates a date and time value from a local date and time value given by parameters DATE of type DATE or LDATE and TOD of type TOD or LTOD respectively. The resulting local date and time value is converted to UTC time zone and returned.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the input parameter DATE is not a valid date.

The function IS_VALID can be applied to the date value returned to check if the conversion succeeded.

Example 

VAR_TEMP
  d1 : LDATE := LD#2021-12-05;
  dt1 : LDT;
END_VAR
dt1 := CONCAT(DATE := d1, TOD := LTOD#12:00:00.012345678);
IF IS_VALID(dt1) AND
dt1 = LDT#2021-12-05-12:00:00.012345678 THEN
  (* correct *) ;
END_IF;

CONCAT_DT, CONCAT_LDT

Synopsis 

CONCAT_DT(YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT, HOUR: ANY_INT, MINUTE: ANY_INT, SECOND: ANY_INT, MILLISECOND: ANY_INT): DT
 
CONCAT_LDT(YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT, HOUR: ANY_INT, MINUTE: ANY_INT, SECOND: ANY_INT, MILLISECOND: ANY_INT, MICROSECOND: ANY_INT, NANOSECOND: ANY_INT): LDT

Description

Creates a local date and time value from parameters YEAR, MONTH, DAY, HOUR, MINUTE, SECOND, MILLISECOND, MICROSECOND and NANOSECOND which is then converted to UTC time zone and returned. Parameters MICROSECOND and NANOSECOND are optional.

All parameters must be of the same type DINT or UDINT or must be implicitly convertible to one of these types. The function may also be typed to describe the type of all input parameters (e.g. CONCAT_DT_UINT).

Errors

ERR_RT_CONVERSION: A conversion error is reported if the date cannot be converted to an internal representation because of operating system limitations.

ERR_RT_TYPERANGE: A type range error is reported if any of the parameters passed to this function are out of range.

The function IS_VALID can be applied to the date value returned to check if the conversion succeeded.

Example 

IF CONCAT_DT(2021,12, 5, 10, 10, 0, 100) = DT#2021-12-05-10:10:00.100 THEN
  (* correct *) ;
END_IF;

DAY_OF_WEEK

Synopsis 

DAY_OF_WEEK(IN: DATE): UINT
DAY_OF_WEEK_DATE(IN: DATE): UINT
 
DAY_OF_WEEK(IN: LDATE): UINT
DAY_OF_WEEK_LDATE(IN: LDATE): UINT

Description

Returns the day in the week of the date passed in parameter IN of type DATE or LDATE. The function returns 0 for Sunday, 1 for Monday, …, 6 for Saturday.

Errors

ERR_RT_DTCONVERSION: A data and time conversion error is reported if the date passed in parameter IN is invalid. The value returned in case of an error is larger than 6.

Example 

IF DAY_OF_WEEK(DATE#2021-12-5) = 0 THEN
  (* correct *) ;
END_IF;

DAY_OF_YEAR(IN: DATE): UINT
DAY_OF_YEAR_DATE(IN: DATE): UINT
 
DAY_OF_YEAR(IN: LDATE): UINT
DAY_OF_YEAR_LDATE(IN: LDATE): UINT

Description

Returns the day in the year of the date passed in parameter IN of type DATE or LDATE. The function returns 1 for January 1, 2 for January 2, … .

Errors

Example 

IF DAY_OF_WEEK(DATE#2021-2-1) = 32 THEN
  (* correct *) ;

SPLIT_DATE, SPLIT_LDATE

Synopsis 

SPLIT_DATE(IN: DATE, YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT)
 
SPLIT_LDATE(IN: DDATE, YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT)

Description

Splits the date value in parameter IN of type DATE or LDATE into its components YEAR, MONTH and DAY.

All result parameters must be of the same type either DINT or UDINT. Implicit conversion rules are not applied. If named parameter lists are used all result parameters are optional and can be used in any combination.

Errors

ERR_RT_DTCONVERSION: A data and time conversion error is reported if the date value passed in parameter IN is invalid.

Example 

VAR
  d1: DATE := DT#2021-12-5;
  y, m, d: DINT
END_VAR
SPLIT_DT(dt1, y, m, d);
IF (y = 2021 AND m = 12 AND d = 5 THEN
  (* correct *) ;
END_IF;

SPLIT_DT, SPLIT_LDT

Synopsis 

SPLIT_DT(IN: DT, YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT, HOUR: ANY_INT, MINUTE: ANY_INT, SECOND: ANY_INT, MILLISECOND: ANY_INT)
 
SPLIT_LDT(IN: LDT, YEAR: ANY_INT, MONTH: ANY_INT, DAY: ANY_INT, HOUR: ANY_INT, MINUTE: ANY_INT, SECOND: ANY_INT, MILLISECOND: ANY_INT, MICROSECOND: ANY_INT, NANOSECOND: ANY_INT)

Description

Converts the date and time value in parameter IN to local time and splits the resulting value into its components YEAR, MONTH, DAY, HOUR, MINUTE, SECOND, MILLISECOND, MICROSECOND and NANOSECOND. Parameters MICROSECOND and NANOSECOND are optional.

All result parameters must be of the same type either DINT or UDINT. Implicit conversion rules are not applied. If named parameter lists are used all result parameters are optional and can be used in any combination.

Errors

ERR_RT_DTCONVERSION: A data and time conversion error is reported if the date and time value passed in parameter IN is invalid.

Example 

VAR
  dt1: DT := DT#2021-11-25-10:03:30.123;
  y, m, d, h, mi, s, ms : DINT
END_VAR
SPLIT_DT(dt1, y, m, d, h, mi, s, ms);
SPLIT_DT(IN := dt1, DAY => d, MONTH => m, YEAR => y);

SPLIT_TOD(IN: TOD,HOUR: ANY_INT, MINUTE: ANY_INT, SECOND: ANY_INT, MILLISECOND: ANY_INT)
 
SPLIT_LTOD(IN: LTOD,HOUR: ANY_INT, MINUTE: ANY_INT, SECOND: ANY_INT, MILLISECOND: ANY_INT, MICROSECOND: ANY_INT, NANOSECOND: ANY_INT)

Errors

None

Example 

VAR
  S, m, ms: DINT

IF m = 10 AND s = 35 AND ms = 123 THEN
  (* correct *) ;
END_IF;
