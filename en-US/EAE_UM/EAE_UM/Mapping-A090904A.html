Function Blocks Mapping

Mapping is the assignment of parts of an application to different devices, or to different resources within a single device if the device supports the operation of several resources.

Open the Function Blocks Mapping editor by opening the System editor and clicking on the button Function Blocks Mapping.

The mapping view consists of two areas:

  • Application area

  • Resource area

Mapping can be performed graphically or via the context menu. Connection line styles indicate mapping relationships, and function block icons indicate mapping status.

Application Area

The application area shows the available applications of a solution, it consists of the upper part of the editor. An application is selected using the breadcrumbs trail at the top of the editor. Select an application to show the function block network of this application. You can switch to the communication network of the selected application using the Communication Network editor (this feature is available only in advanced mode which can be selected in the Backstage by clicking File from main menu and then Settings > General > UI).

Resource Area

The resource area shows the current network of the selected resource in the selected device. It is located in the lower part of the editor. The breadcrumbs trail shows the name of the device and the resource for the mapping action.

Graphical Mapping

You can adapt existing mapping by selecting function blocks the add them to the resource via drag and drop.

To perform graphical mapping:

Step

Description

1

Select an application tab in the application area.

2

Select a single function block or a set of function blocks.

3

Drag the selected function blocks from the application area to the resource area and drop it somewhere in the resource area.

NOTE: If function blocks are grouped by frames, this frame can also be used for the drag and drop action. The frame is mapped together with all containing function blocks.

Mapping Via Context Menu

To perform context menu mapping:

Step

Description

1

Go to the network view of the application.

2

Select a single function block or a set of function blocks.

3

Right click on one of the selected function blocks to get the context menu, select Mapping.

4

The sub context menu entries list shows the available resources of the solution.

5

Select one of the available resources.

6

The selected function blocks are assigned to this resource now.

Connection Styles

  • Connections between function blocks mapped to the same resource have solid style.

  • Connections between function blocks mapped to different resources have dashed style.

  • Connections between mapped and not mapped function blocks have dash-dotted style.

Function Block Icons

  • Function blocks that are not mapped have the normal function block icon.

  • Mapped function blocks have an icon overlay to indicate the mapped status.
