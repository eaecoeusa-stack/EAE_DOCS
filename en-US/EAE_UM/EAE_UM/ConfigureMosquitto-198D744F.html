Configure Mosquitto

After the Mosquitto broker is installed, few configurations need to be done.

First, you need to verify that Mosquitto start running its service automatically at each reboot of the engineering workstation:

  1. In the Windows search bar, write services.

  2. Select Services.

    Result: The Services window opens.

  3. Select Mosquitto.

  4. Click Start Service.

  5. Ensure that in the line Mosquitto Broker, the status turns to Running.

  6. In the same line, ensure that the Startup Type is set to Automatic.

  7. Close the Services window.

After you verified the state of the services, perform the following Mosquitto settings configuration:

  1. Open the Mosquitto folder, selected during the installation.

  2. Create a subfolder, named certs, and add there the following key file and certificate file:

    • server (type Security Certificate)

    • server.key

    NOTE: If you do not have these files, create them by following the steps described in Secure Mosquitto Communication.

  3. Select the file mosquitto.conf and modify it.

    NOTE: This action requires Windows administrator rights.

  4. In this file, few lines need to be written:

    • listener 8883: This setting modifies the default listener port from 1883 to 8883. Thus, it converts the listener port from non-secure to secured communication.

    • require_certificate false: With this setting, MQTT clients do not need to have a trusted certificate, signed by a CA for communication.

    • keyfile C:\Program Files\mosquitto\certs\server.key: This setting indicates the path to find the key file previously added in the folder certs.

      NOTE: If during the installation, the Windows location has been modified, modify the path in this setting accordingly.

    • certfile C:\Program Files\mosquitto\certs\server.crt: This setting indicates the path to find the certification file previously added in the folder certs.

      NOTE: If during the installation, the Windows location has been modified, modify the path in this setting accordingly.

    • allow_anonymous true: This setting modifies the default value from false to true. Thus, anonymous listeners can be used on the local engineering workstation.

    NOTE: To know more about these parameters, refer to the dedicated Mosquitto page.

  5. Save the configuration file modifications and restart the Mosquitto service for them to be applied.

    NOTE: Restart the Mosquitto service only after the key file and certificate file are added in the folder, as requested in step 2.

Result: The Mosquitto broker is completely configured. You can perform diagnostics on devices.
