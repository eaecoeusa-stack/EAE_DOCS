Watch Pad

The Watch pad displays the variables presently observed, blocked and forced. Refer to the Test and Troubleshooting topic to learn more about Forces, Watches and Block Events. In the main menu, click View > Debug pads > Watch to open the pad.

Right-clicking on a line opens the context menu.

Double-clicking an entry shows the corresponding function block in the system editor (except for column Value). Double-click on an entry in column Value to force this value, and an already forced value can be modified.

A further option for STRING type variables enables you to display the character string as a list of hexadecimal values. The IEC-conform notation $XY is used where X and Y represent the hexadecimal number and accordingly can only be a character from 0 to 9 or A to F. (See description of the context menu.)

Forced values are displayed highlighted yellow. When you no longer need to have the value forced, you can switch back to watch mode using the context menu entry Watch. You can use a multiple selection with Ctrl+click or Shift+click to remove all forced values simultaneously.

NOTE: Forced values have to be explicitly deleted manually. Closing the pad using Logout (see Deploy and Diagnostic Editor) or closing EcoStruxure Automation Expert - Buildtime does not delete forced values.

Toolbar

Click the Only Visible icon to filter the watched elements by displaying only those that are visible in the network view.

Click the Forces icon to see the number of Enabled forces and Disabled forces.

Click the Watches icon to see the number of Enabled watches and Disabled watches.

Click the Block Events icon to see the number of Block Events.

IEC 61131 and Basic Test Container Block Additional Functionality

Controls and elements in the Watch panel include:

Item

Description

Name

In column Name, you can use the <<Add Watch>> entry to add the entire block loaded in the test container or just some individual data points of the block. The current block is proposed in a drop-down list as soon as you start typing something into the field, or you can also select individual data points of the block from the suggested list. An added block can be enfolded via the + symbol.

Value

The Value column displays the current value of the respective data point and is read-only.

Overwrite value

In column Overwrite value the value to be tested can be specified. This 'overwrite value' can be passed to the data point at any given time, using the two buttons Overwrite All Values or Overwrite selected, in order to then continue the test with the overwritten value. Multi-selection of data points by means of Shift-click or Ctrl+click is possible.

Type

Column Type displays the data type of the respective value.

Overwrite All Values

The two buttons Overwrite All Values and Overwrite selected are available as soon as the test container is started, on the one hand before the cyclic test is running and on the other hand during the active test run as soon as stopped in a breakpoint or whenever the test is continued step by step. This allows you to apply the desired values again and again at the appropriate time, as it is useful in the IEC61131-context, without having to type them again. During the cyclic process without breakpoints, the buttons are deactivated.

To do this, set corresponding breakpoints on the appropriate elements at first (see Testcontainer for Basic function blocks) to interrupt the test procedure of the Basic function block.

By use of this button all edited overwrite values are written at once.

Overwrite selected

See description in the line above at Overwrite All Values.

Using this button, only the currently selected overwrite values are written. Multi-selection of data points by means of Shift-click or Ctrl-click is possible.

Context

During an interruption of the test run, the context of the test is displayed here. Example: a function block FB2 is located in a PGM1, then PGM1.FB2 is displayed while debugged into the FB2.

<<Add Watch>>

Used to add new data points, see Description of the Name column.
