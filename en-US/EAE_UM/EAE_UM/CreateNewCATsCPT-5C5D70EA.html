Create New CATs

To create a new CAT in a project, follow this process:

Step Action Result

Access the CAT folder

In EcoStruxure Automation Expert, first open the Solution Explorer pad.

Then in the project, expand the node CAT to see the different CATs available

Two sub-nodes appears:

  • Hardware: Under this category of CAT are gathered the hardware CATs. These CATs are linking the hardware inputs and outputs to EcoStruxure Automation Expert. They are transmitting and receiving the information once added and set up in the Hardware Configuration editor.

  • Application: Under this category of CAT are gathered he application CATs. These CATs are used to create the logic of the application depending on what is added in their interfaces.

Add new item

Right click on the category of CATs wanted (Hardware or Application) and select New Item.

The dialog box Create a New CAT appears.

Choose the type

Click on the drop down list of Type and select one between these types:

  • Normal CAT: This CAT type is the default one, it follows the CAT category selected purpose and also follows the three level mentioned in the topic CAT.

  • Top CAT: This CAT type is a Normal CAT type with one additional particularity. A Top CAT can only be positioned on the top level of the application. Thus, it cannot be positioned inside another CAT, a composite or a subApp.

  • Device CAT: Only available for application CAT, this type is a Normal CAT type with one additional particularity. A Device CAT can only be mapped into one specific device type.

    When a CAT is created with this type, an additional logical Device CAT is created under it. Add this Device CAT in the Logical Devices editor.

  • Procedural CAT: Only available for application CAT, this type is used to manage states created inside a StateManager CAT through sequence diagrams.

  • StateManager CAT: Only available for application CAT, this type is used to create all the states and the transitions that will eventually be used in a Procedural CAT.

    A StateManager CAT is given as example in the library SE.AppStateManagement. This CAT is named StateManager.

NOTE: For more information about these two last types, check the chapter Sequential Control.

The type of the future CAT is updated once the type from the list is selected.

Enter the name

Fill the section Name.

This name will be given to the CAT once created. Once created, it will be possible to modify this name.

Choose how to implement

Click on the drop down list Implemented as and select one between these options:

  • Composite: The CAT created is behaving as a composite for the mapping. The CAT and all its function blocks will be mapped together under the same device.

  • SubApp: Available with the types Normal CAT, Top CAT and StateManager CAT, the CAT created is behaving as a subApp for the mapping. The function blocks of the CAT can be individually mapped to different devices. If so, the CAT remains unmapped.

The mapping behavior of the future CAT is updated once the implementation option from the list is selected.

Select properties

For some CAT types, an addition property is required before completing the CAT creation. For the type:

  • Device CAT: Select the device in which the future Device CAT will only be able to be mapped. This property can be modified once the CAT is created by clicking on the CAT in the Solution Explorer pad and selecting Properties.

  • Procedural CAT: Select the StateManager CAT that will share its state graph and in which will be added the sequence diagrams.

The properties of the future CAT is updated once the property from the list is selected.

Create the CAT

Click on Finish to create the CAT with parameters given to it.

The new CAT is added under the category of CAT selected at the beginning of the process. In this new CAT are available the three levels mentioned in the topic CAT.
