Associating Physical Devices with Logical Devices

All physical devices need to be associated with a device type. Some devices, for example controllers, come pre-configured as a specific, fixed type of device. The device type setting for other devices, for example servers and workstations, is determined by the type of software they host - such as Soft dPAC, Soft HMI, Archive Database or Archive Link.

The maximum number of runtimes that can be assigned to a device (workstation or server) depends on the number of runtime instances installed on that device. Each instance differs from the others according to the network port that instance is configured to listen on.

To create and assign device type instances for a configurable device:

  1. Select the device (not the NIC) in the workspace area. Its properties are displayed in the Properties pad.

  2. In the Properties pad, click the ellipsis (...) to the right of the Runtimes row, to open the Runtimes editor. 

  3. In the Runtimes editor, click Add to add a new item to the Members group.

  4. For some devices, a list appears presenting the types of runtimes available to that device (for example, Soft dPAC, Soft HMI, Archive Database or Archive Link).

  5. If a list appears, select  a type.

  6. A new instance of the selected type (or of the default type) is added to the Members group.

  7. Click OK to close the editor and return to the Properties pad.

  8. For each item added, in the Properties pad, you can configure:

    • Interface/Endpoint: select Any (the default) or specify a NIC interface/endpoint.

    • Logical Port: 51499 is the default.
