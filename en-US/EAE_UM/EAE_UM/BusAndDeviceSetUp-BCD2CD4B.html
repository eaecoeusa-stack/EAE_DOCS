Configuration of a Modbus TCP Communication

The configuration of a Modbus TCP communication requires to add and set up:

  • A bus, here using the protocol Modbus TCP.

  • A generic Modbus device.

  • And Modbus registers.

Add all these elements into the Hardware Configuration editor:

  1. Open the Hardware Configuration editor.

  2. Add a new bus by clicking on the button Add new master HW-CAT:

  3. In the Add Bus dialog box opening, select Standard.IoModbus.MODBUS, name it and click on Add.

    NOTE: Clicking on Add doesn’t close the Add Bus dialog. Thus, several buses can be added in the row. Click on Close to close it.

The bus is now added to the Hardware Configuration editor.

Expand it to see all parameters of the bus. These parameters have default value assigned to them, fine tune them as you wish.

NOTE: For more details about these parameters, a description is available for each parameter in the Comment column.

In addition to these parameters, add a Modbus device to the bus:

  1. Select the bus line.

  2. Once selected a new button next to Add new master HW-CAT appears: Add new HW-CAT. Click on it.

  3. The Add To MODBUS dialog box appears with all the devices available to add to the bus. Select Standard.IoModbus.MODBUSGENTCPS, choose a name and add it.

The Modbus device appears under the bus. Then set up the parameters of the Modbus device (right click on it and select Properties):

  • stationid; it configures the device ID. The Modbus communication will then be done on this device ID. By default, the value is 255.

  • responsetimeout; it defines the response timeout, in millisecond, for the Modbus communication. By default, the value is 500.

  • fallbackValue; it defines which value the Modbus registers keep when a connection error or connection disabled occurs. If set to the default value HOLD, the Modbus registers will keep the latest received value. If set to RESET, it will set Modbus registers value to 0 or False.

  • tcp > ipaddress; it defines the IP address of the Modbus device used by the Modbus client to communicate.

    NOTE: The default ipaddress setting is 127.0.0.1, which causes the device to listen on all available interfaces.

  • tcp > port; it defines the port used by the Modbus device to communicate. By default the port is set to 502.

  • HA > secondaryheartbeat; it is available only for High Availability Soft dPAC devices and with Modbus servers enabling multiple clients connections. If enabled, it creates a passive connection with the secondary controller that directly takes over if primary controller is compromised. To maintain that connection with the secondary controller, a heartbeat is created with one read Modbus register depending on the parameter selected:

    • Enabled-Automatic; it takes automatically the first read Modbus register to perform the heartbeat.

    • The other parameters; it takes the read Modbus register of the chosen category to perform the heartbeat.

    NOTE: This heartbeat is not interfering with the IO value communication.
    NOTE: If you don’t have any read Modbus register, select Enabled-Automatic and the heartbeat will be done on the first write Modbus register.

  • HA > heartbeataddress; it is available only for High Availability Soft dPAC devices and needs to be configured only if other parameters than disabled and Enabled-Automatic is selected in HA > secondaryheartbeat property. It defines what is the address of the read Modbus register chosen to perform the heartbeat.

  • diagnostics; if enabled, it gives diagnostics of the Modbus device and registers. The diagnostics are available with:

    • The device information in the Deploy and Diagnostics editor, see information here.

    • The MIGBET function block, see information in MIGBET topic from the Runtime.Base - Library Guide.

Finally, add Modbus registers to the Modbus TCP device.

To do so, follow one way to add Modbus registers into the Modbus TCP device:

  • Automatic generation

  • Manual development

NOTE: For more information about these ways, refer to the Modbus chapter.

Once this action is completed, the Modbus TCP communication is ready for transmitting information.
