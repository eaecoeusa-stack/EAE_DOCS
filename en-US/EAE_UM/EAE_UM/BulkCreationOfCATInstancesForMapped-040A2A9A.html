7. Bulk Creation of CAT Instances for Mapped Equipment

The final stage of the workflow is:

Step

Action

1

Click Save if some edits were not saved, and, if the System tab is open, close it.

2

Click Next. The Create Control Application tab becomes active.

Result: A hierarchical preview displays on three different standpoints: CAT Instances, Application Layer and Hierarchy (Plant Model).

NOTE:

  • The Hierarchy (Plant Model) tab is void when the data source is DEXPI P&ID.

  • Forbidden characters in the CAT Instances names ($, *, — space) are automatically removed, some are replaced with _.

3

After verifying the previews, click Generate CAT Instances. In the Solution Explorer, this creates under the CAT Instances node, <PlantName> nodes (provided this was paired with the Application during the configuration) and populates the Application node. In the Process View called PlantModel, folders are created.

During bulk creation of CAT Instances:

  • Applications and layers are also created, respectively, based on plant and area information associated with the instances in the selected engineering data source.

  • Bulk created CAT Instances are assigned to the corresponding applications and layers.

  • Applications and layers are created using the same name present in the engineering data source.

The Asset Tags of the created CAT instances are inferred from the name of their engineering instances (but can be changed); those of their SubCATscan be set by the Set Asset Tag Name context menu.
