Create XML File

The first step of the automatic generation of the Modbus registers is to create the Modbus mapping file containing the information and parameters of the Modbus registers. To create this Modbus mapping file, follow these next steps:

  1. Prepare the Excel file.

  2. Add the content in the table.

  3. Export the Excel table to the XML format.

Prepare the Excel file

To help creating the Modbus mapping file, a zipped file containing a pre configured Excel is furnished in the Windows folders of EcoStruxure Automation Expert. The zip file is named CreatingMappingFile and is located in the following folder: C:\Program Files\Schneider Electric\EcoStruxure Automation Expert - Buildtime 25.0\Studio\data\templates\file\ModbusMappingFile.

NOTE: Copy paste this link into the breadcrumps bar of a Windows folder to directly access it.

Once the Windows folder containing the zip file has been opened:

  1. Copy paste the zip file into another folder.

  2. Unblock it by:

    1. Right click on it.

    2. Select Properties.

    3. Tick the Unblock rectangle in the Security section.

    4. Click on OK.

  3. Unzip it.

  4. Open the Excel file.

Add the content in the table

The Excel is composed of one table with seven columns, one for each parameter handled by the Modbus Gateway import:

Name

It is the name of the Modbus register.

Writing rule

The name has to be shorter than 32 characters long. All SPACE characters will be removed during the automatic generation.

Type

It is the type of the variable handled by the Modbus register.

NOTE: For more information, refer to the SE.Modbus Gateway - Library Guide topic List of Type Handled.
Writing rule

The type can be both written in UPPERCASE or lowercase as long as the type is correctly spelled.

NOTE: Even though read Modbus registers handle VTQ value, write here only the value type without VTQ. For more information about VTQ, refer to Transmitting and Interpreting Online Values topic.
Offset

It defines the address number inside the primary table where the variable of the Modbus register will the communicated.

Writing rule

The Offset has to be an integer greater than or equal to 0.

Usage

It defines if the Modbus register will be used to read or write values.

Writing rule

Mind the spelling of this two words.

Comment

It is the comment used to described the variable that will be added to the Modbus register.

Writing rule

No specific rule.

isIO

Used only for the reading Modbus register, it defines if the variable can only read value or if the variable can read and write value. It will define the primary table of the variable communicated:

  • 0x Discrete Input: It is a single bit read only variable. It will be in this state if in EcoStruxure Automation Expert the variable type is a coil and the datatype is set to inonly.

  • 1x Coil: It is a single read and write variable. It will be in this state in EcoStruxure Automation Expert if the variable type is a coil and the datatype is set to input.

  • 3x Input Register: It is a multiple bit read only variable. It will be in this state if in EcoStruxure Automation Expert the variable type is a register (single, double or long) and the datatype is set to inonly.

  • 4x Holding Register: It is a multiple bit read only variable. It will be in this state in EcoStruxure Automation Expert if the variable type is a register (single, double or long) and the datatype is set to input.

Writing rule

To set the datatype to input in EcoStruxure Automation Expert, set the isIO column to TRUE. To set the datatype to inonly in EcoStruxure Automation Expert, set the isIO column to FALSE or keep the cell empty as it is by default set to inonly. For writing Modbus register, it has to be 1x Coil or 4x Holding Register, so no choice can be made in the datatype properties, then keep this cell empty in the Excel.

BigEndian

It defines the reading order of the bits inside the byte, either from the least significant bit (LSB) either from the most significant bit (MSB). By default the reading order is LSB.

Writing rule

To use the LSB way, write FALSE or keep empty to let the MSB cell in the Excel. To use the MSB way, write TRUE into the MSB cell in the Excel. For the coils, keep empty the MSB cell in the Excel as it doesn’t have this MSB parameter.

All these parameters have to be specified for each variable transmitted through Modbus. Eventually, it would have as many lines in the table than Modbus registers be communicated through Modbus with all the column filled (beside the cells that needs to remain empty).

NOTE: Only the lines that are part of the table will eventually be part of the exported Modbusmapping file.

Export the Excel table to the XML format

When the parameters of each Modbus registers have been field, the Modbus mapping file is ready to be exported:

  1. Go to the File ribbon.

  2. Select Save As and click on Browse.

  3. Select where to save the Modbus mapping file and rename it.

  4. In the Save as type line, select XML Data.

  5. Save the file and in the Warning Excel pop up select Continue.

The Modbus mapping file will then be in the folder previously chosen and ready to be imported.
