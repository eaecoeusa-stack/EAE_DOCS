Modbus

Modbus is an application layer messaging protocol. Modbus provides client and server communications between devices connected on different types of buses or networks. Modbus offers many services specified by function codes.

Modbus Parametrization

In EcoStruxure Automation Expert, Modbus is set up in the Hardware Configuration editor of the application needing communication. Modbus parametrization is done through three level of layer (each of them connected through a parent child relation):

1. Bus

It defines the communication type between client and server and also defines how the communication will be done. Implicitly, all the devices added under a bus will have the same communication types. In EcoStruxure Automation Expert three different kinds of bus are available:

Standard.IoModbus.MODBUS

Accessible with the library Standard.IoModbus, this bus is a client. It will get and transmit its information to a server by using the TCP protocol.

Standard.IoModbus.MODBUSSERIAL

Accessible with the library Standard.IoModbus, this bus is a client. It will get and transmit its information to a server by using a serial communication line with the Modbus RTU protocol.

Standard.IoModbusSlave.MODBUS

Accessible with the library Standard.IoModbusSlave, this bus is a server. It will share its information with clients by using the TCP protocol.

Standard.IoModbusSlave.MODBUSSERIAL

Accessible with the library Standard.IoModbusSlave, this bus is a server. It will share its information with clients by using a serial communication line with the Modbus RTU protocol.

2. Modbus Devices

It defines the settings of communication used between itself and the other side bus (IP address, station ID...). Implicitly, all Modbus registers inside a Modbus device will be communicated using the same communications parameters. In EcoStruxure Automation Expert different kinds of Modbus devices are available:

Standard.IoModbus.MODBUSGENTCPS, Standard.IoModbus.MODBUSGENSERIALS, Standard.IoModbusSlave.MODBUSGENBC

Accessible with the library Standard.IoModbus for MODBUSGENTCPS, MODBUSGENSERIALS, and with the library Standard.IoModbusSlave for Standard.IoModbus.MODBUSGENBC, they are generic Modbus devices. All their Modbus registers and their variables have to be added by the user. These Modbus devices are then fully adjustable for the user need.

Custom Field Device

Accessible with the library FieldDevice, these are custom Modbus devices made specifically for communicating with devices from the Schneider Electric catalog. These custom Modbus devices already contain all the parameters used to communicate with the specific devices, the user only need to set them.

3. Modbus Registers

It defines all variables communicated, where and how they are transmitted and what is their types. In EcoStruxure Automation Expert, two different kinds of Modbus registers are available:

The Standard.IoModbus register

Available with the library Standard.IoModbus, these Modbus registers are the basic function blocks used to communicate Coil and Register values. A logic has to be manually setup to enable Modbus registers to read a specific variable type.

The Modbus Gateway register

Available with the library SE.ModbusGateway, these Modbus registers are an improved version of the Standard.IoModbus registers as the conversion logic is already integrated into the Modbus registers. The user will directly get as an output the needed variable type. This type of Modbus register can be added only under the client bus, Standard.IoModbus.MODBUS and Standard.IoModbus.MODBUSSERIAL.

NOTE: The read Modbus registers of both Standard.IoModbus register and Modbus Gateway register support VTQ values. For more details, refer to Transmitting and Interpreting Online Values topic.

In this chapter is detailed the following aspects:

  1. Configuration of a Modbus TCP Communication.

  2. Configuration of a Modbus Serial Communication.

  3. Development of Modbus Registers. There are three different ways to do this action:

    • Automatic Generation: Specific for the Modbus TCP Communication, the Modbus Registers and their parameters are automatically generated based on the pre-filled Modbus mapping file.

    • Manual Development: The Modbus Registers and their parameters are modified manually.

    • Explicit Modbus Communication :The library Standard.IoModbus provides a set of SFB to implement a Modbus Explicit communication from IEC61499 application to Modbus server. Please refer to library documentation for details.

    NOTE: Client registers require either only Standard.IoModbus library or Standard.IoModbus with SE.ModbusGateway libraries. Server registers require Standard.IoModbusSlave library.

  4. Transmitting and interpreting online values.

NOTE: To have more details about the other elements not further detailed in this chapter, like the Modbus Slave library or the standard.IoModbus registers, check their library documentation or check the Modbus protocol.

Online Change with Modbus TCP Client

In EcoStruxure Automation Expert, online change with Modbus TCP client is available when using Soft dPAC devices and hardware CATs from the Standard.IoModbus library.

Check all the online change supported actions in the Online Change topic.

Modbus Limitations for M2●● dPAC Controllers

M2●● dPAC controllers present the following Modbus limitations:

  • Modbus Bus Master (client + server) instances are limited to 2.

  • Process image size is limited to 64 Kb.
