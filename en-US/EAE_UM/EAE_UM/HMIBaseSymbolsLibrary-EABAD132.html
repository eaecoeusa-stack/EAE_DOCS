3. HMI Base Symbols Library

The following describes the elements from the HMIBaseSymbols library, which is accessed by dragging a variable into the graphic editor (dialog Select Generic Symbol):

Element

Description

BarValueHorizontal

This element can be created by dragging a respective CAT_HMI input variable into the graphic editor. It serves to visualize a value graphically in form of a horizontal bar.

BarValueVertical

This element can be created by dragging a respective CAT_HMI input variable into the graphic editor. It serves to visualize a value graphically in form of a vertical bar.

CheckButton

This element can be created by dragging a respective CAT_HMI input- as well as an output variable into the graphic editor. If it is created via input variable, it serves as a graphical display element in form of a button. In this way it cannot be actuated actively but it shows the current state of the variable.

If it is created via output variable, the functionality of this element basically is identical with the element Drawn Button, as it is described at Graphic Editors. However, actuating this button at runtime will directly lead to a change in value of the attached variable.

DatePicker, DateAndTimePicker

This element can be created by dragging a respective CAT_HMI input- as well as an output variable into the graphic editor. If it is created via input variable, it serves as a graphical display element in form of a rectangle containing textual output. In this way it cannot be actuated actively, but it shows the current state of the variable.

If it is created via output variable, the functionality of this element basically is identical with the element Date and Time Picker, as it is described at Graphic Editors. However, selecting a date from the offered date picker field (or overwriting the time) at runtime will directly lead to a change in value of the attached variable.

DropDown

This element can be created by dragging a respective CAT_HMI output variable into the graphic editor. The functionality of this element basically is identical with the element Combo Box, as it is described at Graphic Editors.

To create or modify the individual items, which are offered for selection at runtime, open the DropDownItem Collection Editor by clicking on the ellipsis (...) button at field ((Collection)) at DropDownItems in the Properties pad.

Description of the DropDownItem Collection Editor:

Command

Description

Members:

The available items are listed here and the order of the items defines the order of the list at runtime. Use the arrow buttons on the right edge of the field Members to change the order of the entries.

The properties of the entries are merely listed here. If you want to edit them use the item;0-properties area.

Add

A click on this button adds a further entry. The entry shows the text item and has the value 0 per default.

Remove

A click on this button removes the currently selected entry of field Members.

The arrow buttons serve to change the order of the entries. The currently selected entry will be moved one position into the direction of the arrow of the actuated button.

item;0-properties:

In the right hand area of the editor, at item;0-properties, the properties of the selected entry are listed. Here, in the field on the right hand beside Text, you can modify the text arbitrarily, to equip the entry with a clearly comprehensible definition. This text will be displayed for the respective selection item at runtime.

The entry in the field right hand of Value defines which value will be sent at runtime by selecting this drop down entry.

Shows the properties by category.

Shows the properties in alphabetical order.

OK

Closes the window and saves the modifications.

Cancel

Closes the window without any modifications.

Execute

This element is offered for selection by dragging any CAT_HMI output as well as input variable into the graphic editor.

This creates an element that is invisible at runtime. It can be used via source code by its EventHandler (open them in the properties by a click on the lightning button). In that way it could be used, for example, to make a graphic, contained in the CAT_HMI symbol, visible or invisible, depending on the change in value of the linked variable.

Example:

In the Properties pad, for the Misc > ValueChanged property, select ExecIsManValueChanged.

In the Source Code editor for this graphic (click the Source Code icon on the vertical toolbar on the left), the following logic is created in support of the ValueChanged handler: 

void ExecIsManValueChanged(object sender, ValueChangedEventArgs e)
        {
            imgMan.Visible = (bool)e.Value
        }    

FreeText

This element can be created by dragging a respective CAT_HMI input- as well as an output variable into the graphic editor (except the date and time variables). The element Free Text creates a text element that can be freely positioned. Font, text color, text size, etc. can be defined in the properties (in the same way as element Free Text from the toolbar, which is described at Graphic Editors).

It serves here for variables as a display element, automatically attached to the respective variable. Additionally, in comparison to the element Free Text from the toolbar, it is equipped with some extra properties: for example an editor can be opened by a click on the ellipsis (...) button in the field (Ranges ) at Appearance > Ranges, which allows you to define the text color and an alternative text to be shown for specific values instead of the original value. Examples and a description are located at Define Ranges.

Image

This element can be created by dragging a respective CAT_HMI input variable into the graphic editor. It serves to visualize a value graphically in form of images. A simple example: a control lever, which is shown for different values in different positions. In order to show this graphically, it would be possible to assign the respective values to different images, after opening the editor by a click on the ...-button in the field (Ranges) at Appearance > Ranges in the properties. A description is located at Define Ranges.

 

Label

This element can be created by dragging a respective CAT_HMI input- as well as an output variable into the graphic editor. The element Label creates a text element in the same way as element Label from the toolbar, which is described at Graphic Editors.

It serves here for variables as a display element, which is attached to the respective variable automatically. Additionally, in comparison to the element Label from the toolbar, it is equipped with some extra properties: for example an editor can be opened by a click on the ...-button in the fieldRanges) at Appearance > Ranges, which allows you to change the appearance of the label for specific values: another text color, an alternative text to be shown instead of the original value, an alternative brush and pen. Examples and a description are located at Define Ranges.

Prefix or suffix of label can be provided in the same manner with an attribute as described below at TextBox.

 

Led

This element can be created by dragging a respective CAT_HMI input variable into the graphic editor. It serves for graphical visualization of a value in form of a led. Different colors can be defined for different values or value ranges of the variable. In order to do so, use the property Range of category Appearance from the properties, as described at Define Ranges.

RoundKnob

This element can be created by dragging a respective CAT_HMI input- as well as an output variable into the graphic editor. If it is created via input variable, it serves as a graphical display element and shows the current value of the variable in form of a round knob. It cannot be actuated actively when it is used in this way.

If it is created via output variable, the value of the variable can be set directly by actuating this element.

The appearance, such as coloring and scale graduation as well as scale maximum and minimum can be set as desired. The property SnapDivisions can be used to define the divisions where the round knob will snap in. In that way you can prevent, that a value between the divisions can be set via control knob.

SwitchButton

This element can be created by dragging a respective CAT_HMI input- as well as an output variable into the graphic editor. If it is created via input variable, it serves as a graphical display element and shows the current value of the variable in form of a two state button. It cannot be actuated actively when it is used in this way.

If it is created via output variable, the functionality of this element is basically identical with a Two State Button, as described at Graphic Editors. However, actuating this button at runtime will directly lead to a change in value of the attached variable.

NOTE: When the FalseBrush property is set to NULL, the click event can be executed only on the borders of this element, and not on the button itself.

Note on using the Events ValueChanged and OutputValueChanged for the SwitchButton:

  • ValueChanged : is fired when the variable value of the corresponding output variable changes, without the switch button has been operated (which would be possible, for example, by another Base Symbol that is linked to the same variable).

  • OutputValueChanged : is fired when the variable value is changed by clicking on this exact Base Symbol. (So assuming that there is exactly only one base symbol existing that is associated with this output variable, then OutputValueChanged behaves just in the same way than CheckedChanged at a TwoStateButton.)

 

TextBox, TimeOfDayTextBox

This element can be created by dragging a respective CAT_HMI input- as well as an output variable into the graphic editor. If it is created via input variable, it serves as a graphical display element and shows the current value of the variable in form of a rectangle with textual output. Used in this way, the value cannot be overwritten or modified actively, but it shows the current value in textual form.

If it is created via output variable the functionality of this element is basically identical with a Text Box, as described at Graphic Editors. Overwriting the displayed value at runtime will directly lead to a change in value of the attached variable.

The same rules are true when it is used with variable type TIME or TOD. The format will be adapted accordingly and can be additionally adjusted for TIME in the properties in category Misc.

In the properties of the TextBox there is the possibility to define a text for prefix or suffix. This text can also be created as an attribute. This gives the project creator the option to define its own prefix / suffix text for each CAT instance via attribute, instead of displaying in the HMI only one and the same, fix defined text, coming from the CAT-type. This allows you to provide each TextBox in the HMI with a matching text or a corresponding unit definition when using the same CAT repeatedly. To use this functionality, enable the attribute HMI.Unit at the corresponding ports in the Interface Editor of the CAT_HMI block:

Then insert $attr:HMI.Unit, similar to an alias, in the properties of the TextBox of the CAT Symbol or Faceplate, at prefix or suffix:

Once the CAT has been instantiated in the project, the Attribute can be opened by right-clicking the instance of the block. In tab Exported Attributes in column Value at the corresponding HMI.Unit entry, you can edit the text which you wish to have displayed in the HMI.

TrackerHorizontal

This element can be created by dragging a respective CAT_HMI output variable into the graphic editor. It is basically identical with element Tracker, as described at Graphic Editors. However, dragging the tracker handle at runtime will directly lead to a change in value of the attached variable and this tracker is equipped with some more detailed adjustment options via properties.

TrackerVertical

Same functionality as TrackerHorizontal, but displayed in a vertical position.
