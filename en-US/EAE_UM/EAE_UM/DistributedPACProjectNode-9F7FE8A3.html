Distributed PAC Project Node

The Distributed PAC Project node bears the user-assigned name of the solution. This folder contains the elements System, Canvases, etc., which are described later on.  It is the lowest main node point within the Solution.

Distributed PAC Project Context menu

A right-click on the project node opens a context menu with the following items:

Command

Description

Project Library References

Opens the Project Library References dialog where you can define the libraries referenced in your project.

Add External Reference
Offers the possibility to reference external .net-files used by the HMI, to make them available in the project. In that way, ready-made program parts (e.g. for weather information, to determine the location, etc.) which have been self-made, or are freely available in the internet can be used in the project easily. Specify the path where you have stored the .dll file. With the next compile, it will be integrated in the HMI.
Remove External Reference
Used to remove references to external .net-files that were inserted to be used in the HMI.
Check Integrity

Checks the integrity of the project and provides information of any inconsistencies. If inconsistencies are found, a dialog opens to ask whether they have to be resolved. (See Check Integrity)

    Right click on a line with existing entry opens the context menu entry Delete to delete the whole property.

    Description of the columns of the list:

    • Name: Edit the name of the entry here. To change the name of an existing entry, left-click the name to have it displayed highlighted blue.

    • Type: The type can be adjusted by using the drop down menu which opens after a click in the field and then a click on the button at the right edge of the field.

    • Allowed: Here you can edit a selection of allowed values if you want to have them limited. You can edit the values separated by a ';' directly, or use the dialog which opens when you click on the button . In this dialog, each value (or string) has to be confirmed with Enter before the next value is edited. The use of ';' as a delimiter is not allowed here.

    • Default: Defines the value which is predefined per default. Using the drop down list which opens by a click on the button on the right edge of the field (click into the field at first) the desired value can be selected from the 'Allowed' values that have been edited before.

Named Style

The Named Style command enables the user to customize the visualization of certain editors of the HMI for the current project. For example, it is possible to define the color(s) of an alarm warning or a frequently used label. Each editor is assigned a name and a corresponding definition, making it easy to adapt and reuse.

To open the Named Style tab, follow these steps:

  1. Right-click on a solution.

  2. Select Named style and click Open.

Result: The Named Style tab opens.

When the Named Style command is selected, two additional commands appear above the Open command.

  • Export: Exports colors and fonts of the project in a ZIP file. The storage path can be freely selected.

  • Import: Imports colors and fonts from an archive file (*.DO.zip; *.Colors.zip).

The following editors are displayed in the Named Style tab:

Colors:

Follow these steps to add a color or a blink color in the Named Style tab:

  1. Once the Named Style tab opens, right-click anywhere in the Colors tab.

  2. Select Add Color or Add Blink Color.

  3. The Define new name dialog box opens.

  4. Enter a name and a description for the color or blinking color, then click OK.

Result: The following are displayed:

  • The name of the color or blink color in the Name column

    .

  • The color tag in the Tag column

    .

  • The color description in the Description column.

To rename or remove a color or a blink color:

  • Right-click on the name in the Name column.

  • Select Rename Color or Remove Color.

To change the color, click the arrow in the Tag column.

To update the description, click on the current description in the Description column and edit it.

Brushes:

Follow these steps to add a brush in the Named Style tab:

  1. On the left side of the Named style tab, click Brushes to open the brushes tab.

  2. Right-click anywhere inside the Brushes tab.

  3. Select Add Brush.

  4. The Define new name dialog box opens.

  5. Enter a name and a description for the brush and click OK.

Result: The following are displayed:

  • The name of the brush in the Name column

    .

  • The brush tag with the value Null in the Tag column

  • The brush description in the Description column.

To rename or remove a brush:

  • Right-click on the name in the Name column.

  • Select Rename Brushor Remove Brush.

To update the description of a brush, click on the current description in the Description column to edit it.

Follow these steps to change the color of the brush in the Tag column.

  1. click the ellipsis in the Tag column to open the Brush Editor dialog box.

  2. Select Solid in the Type drop down list.

  3. to define a color for the brush, click the Color drop down list and select the color of your choice.

  4. To create a gradient color, click the ellipsis in the Gradient field .

  5. The Gradient Editor opens. Select a gradient type from the drop down list and click OK.

NOTE: you can either choose a basic color from the Color drop down list within the Brush Editor, or tick the check box Use Color Blend in the Gradient Editor to combine multiple colors.

Pens:

Follow these steps to add a pen in the Named Style tab:

  1. On the left side of the Named style tab, click Pens to open the pens tab .

  2. Right-click anywhere inside the Pens tab.

  3. Select Add Pen.

  4. The Define new name dialog box opens .

  5. Enter a name and a description for the pen, then click OK.

Result: The following are displayed:

  • The name of the pen in the Name column

    .

  • The color tag in the Tag column

    .

  • The color description in the Description column.

To rename or remove the name of a pen:

  • Right-click on the name in the Name column.

  • Select Rename Color or Remove Color.

To update the description of a pen, click on the current description in the Description column to edit it.

Follow these steps to change the properties of a pen:

  • Click the ellipsis in the Tag column to open the Pen Editor dialog box.

  • Define the writing style of your pen by selecting one of the options in the Style drop-down list: Solid, Dashed or Custom.

  • Enter a value in the Width field to define the thickness of the pen’s writing.

  • Select a color for the pen by clicking the Color drop down list.

  • Choose the shape of the pen’s starting point by selecting an option from the StartCap drop down list.

  • Define the size of the pen’s starting point by selecting a value from the StartCapAnchorWidth drop down list.

  • Choose the shape of the pen’s endpoint by selecting an option from the EndCap drop down list.

  • Define the size of the pen’s endpoint by selecting a value from the EndCapAnchorWidth drop down list.

  • Select the type of line join for the pen by clicking the LineJoin drop down list.

Fonts

Follow these steps to add the font of your choice in the Named Style tab:

  2. Right-click anywhere inside the Fonts tab.

The following actions can be performed in the Font dialog box.

  • Pick the font of your choice in the Font column.

  • To underline or cross out a phrase, tick the Strikeout or Underline checkbox in the section Effects and see the result in the Sample Text section.

Theme

This item concerns the possibility of using different themes in a project. Each theme contains the predefined colors, brushes, etc. and those which have been created in Named Styles in the project or those being imported. The theme serves to overrule the adjustments of these elements, while the original definition is preserved. With the creation of individual themes the presentation of the elements used in the HMI can be set individually and you can change to another look fast and easy.

  • Current Theme → here you can select the desired one among the assortment of themes, by the time one theme (theme 'Default') is implemented standardly, but you can create your own themes anytime.

  • New Theme Copied From ... → here you can create new themes by copying an existent theme, opening it and adjust it as desired.

  • Open → opens the selected theme in order to read out or amend the current properties such as colors, brushes, pensand fonts. (see below).

    These properties include a description column, with content restricted to Read Only mode.

  • Remove → this item will be listed only if more than one themes are available in the project. The selected theme will be removed from the project if this item is used.

Default Themes → EcoStruxure Automation Expert - Buildtime offers two default themes: 

  • Default→ Displays solution screen elements in a darker hue. 

  • DefaultLight → Displays solution screen elements in a lighter hue. 

Concerning item Open :

The view Theme:... opens and shows the colors, brushes, pens and fonts defined in the project. As long as the entries are not modified they are read only, that means they are selectable but the entry cannot be modified, although it is possible to click in column Tag in a field and in that way you can read the entire text even if it is too long to be shown in the column completely. In order to adjust the values the entry has to be made editable by a right-click → Overwrite Brush (Overwrite Pen, Overwrite Font). In tab Colors there are two kinds of colors possible: colors and blink colors. Therefore, the right-click context menu contains two commands here - Overwrite As Color, Overwrite As Blink Color. In that way, it is possible to switch a color into a blink color and vice versa.

The red arrow icon turns into an appropriate icon ( = color, brush, pen, font;  = blink color) and the name is displayed bold to highlight the modified entries.

After the entry has been made editable, as described above, and the field in column Tag is selected a ...-button appears on the right edge of the field. A click on this button opens the editor to adjust the color, font, etc.

An already modified entry can be reset to initial value by a right-click → Reset Color (Reset Brush, etc.).

Multilanguage dictionaries

This node comprises three commands for multilanguage text presentation:

  • HMI Translation: For HMI symbols.

  • Device Translation: For device properties accessed from the Logical Devices editor.

  • Theme Translation: Provides translations in all EcoStruxure Automation Expert supported languages for keywords displayed in Named Style .

Image Storage

Use this command to open the Image Storage dialog, which you can use to manage the images which are used in the project. Images stored with this function are saved in the current project path in \HMI\ImageStorage\*\*.jpg. They can be used e.g. for canvases.

A description of the editor can be found at Image Storage.

Adapters...

Opens dialog Adapter Compatibility. This dialog is to admit adapter connections even if the adapters have different namespaces.

Commit...

This entry will appear only if you are working on a project that is already under SVN version control. If you start the commit, although there are still unsaved changes in the project, the Information dialog opens, informing you of this fact. You are asked if you want to save changes before commit so that unsaved changes are not lost by accident. You can select:

  • Yes: any changes are saved and the SVN dialog for the commit is opened.

  • No: any changes are discarded if the dialog opened thereafter is answered with Yes. Then the SVN dialog for the commit is opened.

  • Cancel: the action is terminated, no SVN dialog is called and the changed files remain in the unsaved state.

Revert

This entry will appear only if you are working on a project that is already under SVN version control. Using this item a SVN dialog will open, which shows a list of the files that have been changed. Select those files with undesired modifications to revert them. At the next commit these files will not appear in the list.

NOTE: To clean the solution from unnecessary files, select  Build HMI area  from the Home menu , and select the Clean button. When you use the Revert feature, select the  Delete unversioned objects  button to open the SVN dialog.
Subversion

This entry is only available, if you are working on a project that is already under SVN version control. It opens a sub-context menu containing various functions of SVN, such as Show Log, Merge, etc. For more information about these features please look up the Tortoise SVN help, which can also be opened from this sub-context menu.

Add Solution to Subversion

Adds the opened project to administration by SVN. Only the relevant files for the administration are automatically selected (binary files, objects and documentation files are not included in the administration because they are only generated locally on the computer when required). In any case, it is advisable to clean the project immediately before carrying out this action (in the Home menu Build area select Clean) in order to clean the project from unnecessary files.
