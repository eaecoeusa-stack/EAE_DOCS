Creating and Displaying HMI Alias Names

Preliminaries

Create and add to System > Applications > <App name> one or more application CAT or Sub-CAT objects.

Create the HMI Alias Name

  1. Open the Function Blocks Network editor to System > Applications/Devices.

  2. Right click and select Attributes...

  3. Click the Exported tab to open it.

  4. Refer to the Context and Element Name column values to identify the intended HMI.Alias attribute, then enter an alias name in the Value column.

    NOTE: You can enter localized language text to display the object in that language in the HMI.

  5. Click OK to save the HMI.Alias value.

Display the HMI Alias Name

  1. Create an HMI-Canvas that includes a History Trend control ().

  2. Open the History Trend Configuration Manager:
    In the Solution Explorer pad select <solution name> > Configurations > History Trend.

  3. Either:

    • Select an existing configuration that includes the object with the configured HMI Alias and click OK,
      or

    • Create a new configuration as described below.

  4. Right click on the <solution name> > Canvases > 1280x980 node, then select Start Local HMI. The project builds, then the EcoStruxure Automation Expert - HMI service starts and runs the canvas.

  5. In the running canvas toolbar, click the Configuration icon ( located on the left of the toolbar). The History Trend Configuration Manager opens.

  6. Select the configuration and click OK to apply it to the History Trend control and close the dialog.

  7. The variables assigned to the selected configuration are added to the History Trend control.

  8. Check the name on the bottom of the canvas. It is the HMI Alias name you configured.

Create a New Configuration

  1. In the History Trend Configuration Manager, highlight the Configurations node, right click and select New.

  2. In the History Trend Configuration dialog, in the Series field, click the ellipsis (...). The Trend Series Collection Editor opens

  3. Click Add. The Select Archive Path dialog opens.

  4. Select the path(s) to the HMI input variable values you want to view, then click OK.

  5. Click OK three more times to return to the History Trend Configuration Manager, which now displays the selected variable configuration.

  6. Click Close.
