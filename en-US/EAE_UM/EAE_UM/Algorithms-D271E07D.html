Algorithms

The editor lists the existing algorithms in the Name column. Select the algorithm to execute or provide a new name for a new algorithm. Then select the set of events to raise after the algorithm execution. The list of events provided includes all outputs events from the CAT interface (if the expected event is not present ,check the CAT output interface).

The system creates a basic block called {procedural cat type name}_actionExecutor. In this basic, you can fill the ST code to execute. The complete procedural CAT interface is accessible from this type, so that the algorithm can write outputs and read inputs from the CAT interface.

NOTE:

  1. The interface of the Basic and the ECC parts are completely generated. Any modification here will be erased when triggering the next verify

  2. Renaming an algorithm requires to reselect this algorithm name in the Step Editor.

  3. Deleting an algorithm requires to remove the reference to this in the Step Editor.

  4. Adding a new algorithm is allowed without error. It will be proposed in the algorithm list at next open of the Step Editor.

  5. When performing a copy-paste operation of a Step, the algorithms are copied without their internal content (ST code).Adding a new algorithm is allowed without error. It will be proposed in the algorithm list at next open of the Step Editor.
