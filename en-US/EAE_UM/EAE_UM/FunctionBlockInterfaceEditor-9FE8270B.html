Interface Editor

Use the Interface editor to create the interfaces of a function block and to set its general adjustment, that is, which event interface will start which data processing. The editor presents the following functions for function block interfaces:

  • Generate new interfaces

  • Delete or rename interfaces

  • Generate associations between interfaces

  • Parameterize interface attributes for supported object types

The Interface editor can be used with Service, Basic, Composite, and Adapter function blocks, sub-applications, and CATs.

To open the Interface editor:

  • Open the Service, Basic, Composite, or Adapter function block editor, or the Sub-Application editor and click the Interface icon in the vertical menu on the left.

  • Double-click on a Service, Basic, Composite, or Adapter function block, Sub-Application, or CAT in the Solution Explorer pad

  • Right-click on a function block in the Function Blocks Network editor of the System editor and click Go to definition.

Generate New Interfaces

Step

Description

1

In the Function Block Interface editor, left-click on <new interface>, <new variable>, or<new adapter>, according to the requirements) .The row turns highlighted blue.

2

Select the text by left-clicking again - or press F2 -, so that only the text is highlighted blue.

3

Type in the new name.

4

Confirm with Return or click on another row.

5

Adjust the parameter if it is a variable (see below).

Delete or Rename an Interface

Step

Description

1

Left-click once on the interface to be deleted or renamed.

2

When the whole row is highlighted blue, delete with Del key or click once again on the name - or press F2 - and type in a new name.

alternatively

1

Right-click on the interface to be deleted or renamed.

2

Click on the respective context menu entry Rename or Remove and, in case of Rename, type in a new name.

Generate Associations Between Interfaces

Step

Description

1

With the left mouse key held down, drag the input variable to the corresponding input event (likewise for output).

2

Release mouse key on the event.

alternatively

In the With column in the event area: click and then hold down the ellipsis (...)  button and follow the dialog With Type Editor.

NOTE: You can delete associations using the reverse of the same procedure as for generating them. 

Interface Parameter Attributes

Use these attributes to define Interface variables and items:

Column

Description

Name

The name of the interface is defined here.

Type

The variable type can be adjusted here out of a drop-down list (BOOL, INT, REAL, STRING, etc.).

At Sockets → Plugs: the Adapter which has to be used can be chosen here. (Only available if some Adapter does already exist.)

NOTE: When using the data type STRING with Structured text-based blocks, a variable with a maximum length of 15 characters is generated per default. If this variable is to process longer strings, the length of the variable can be defined by adding the value in square brackets: e.g. STRING[60]

However, this behavior only affects Structured text-based blocks such as Basic function blocks. For IEC 61499 RT-based blocks (Composite, Service, Adapter) a length definition of string variables is not necessary and also not intended. The variables automatically allocate the required memory area according to the contents of the string variable. Example: The input via the HMI can be of any length without having to define the variable accordingly.

For STRING variables: There is no limit on the possible length of the variable. Only the size of the controller memory limits the size.

Array size

An array of the adjusted variable type can be defined here by inserting the array size. For this reason only the relevant number has to be inserted into the field.

Initial value

A default-value can be pre-defined here. This value will be active, if no other value has arrived at this interface. On first start, the variable will be initialized with this value, when defined.

NOTE: For CAT_HMI, if you want to use a variable as a persistent variable, check out Offline Parametrization Editor topic.

For IEC61131 user defined datatypes Structure and Array, right click in the cell and select Reset to restore the initial value.

With

In this column the links between EventIO and VariableIO are listed and can be edited. The dialog With Type Editor opens by a click on the ellipsis (...) button within the column With (select the field to have the button visible) in one of the event rows. This dialog serves to edit the 'with construction' between event and data interfaces.

Attr

Attributes can be inserted here with the assistance of a dialog. A click on the ...-button within the column Attr will open the dialog Attribute Editor to edit the attributes.

Comment

In this column explanatory comments to the relevant interface can be inserted.

Object Types

Description of the top level navigation tree nodes of the Interface editor:

Column

Description

EventInputs

The Event Inputs are located on the left side of the function block head. They serve to trigger the processing of specific logic processes within the function block. The state or value of the input variables connected with the event are read out by triggering the event.

EventOutputs

The Event Outputs are located on the right side of the function block head. They serve to pass on the event when the function block has finished its processing, to cause the next function block in the chain to start processing its logic.

InputVars

The Data Inputs are located on the left side of the function block body. They have to be connected with the relevant EventInputs, providing the necessary data for the processing. The name of a data input variable is displayed italic when no 'With-connection' with an event exists.

OutputVars

The Data Outputs are located on the right side of the function block body. They have to be connected with the relevant EventOutputs. When the internal logic of the function block has been processed and an output event is fired, the data of the connected output variables will be provided at the data outputs for further processing. The name of a data output variable is displayed italic when no 'With-connection' with an event exists.

Sockets

Sockets represent the Adapter inputs of the function block. They can only be used together with existing adapters. To use the new adapter menu an Adapter Function block has to be created first. Sockets are available in Composite, CAT, and Service function blocks.

Plugs

Plugs represent the Adapter outputs of the function block. They can only be used together with existing adapters. To use the <new adapter> menu, an Adapter Function block has to be created first. Plugs are available in Composite, CAT, and Service function blocks.

InternalVars

InternalVars does not create a visible port for the function block. It serves to define variables which are needed for the logic of the function block, but not required as an in- or output. These variables are available for the entire function block. If you need a variable just for a special part of the code, it is fully sufficient to define it in this part of the code.

AdapterInputs

Basically identical to Plugs, but an adapter port, which is created as AdapterInput, can be used also within the function block only in its collectivity. Individual ports of the adapter cannot be accessed. AdapterInputs are available in Composite, CAT and subapplication function blocks.

AdapterOutputs

Basically identical to Sockets, but an Adapter port, which is created as AdapterOutput, can be used also within the function block only in its collectivity. Individual ports of the adapter cannot be accessed. AdapterOutputs are available in Composite, CAT and subapplication function blocks.
