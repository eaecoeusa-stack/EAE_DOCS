Secure the Devices

To finalize the commissioning, the configured devices need to be secured and trusted. Afterwards, EcoStruxure Automation Expert - Buildtime can communicate with the discovered device.

The first step is to click the Start Monitoring icon in the physical view. The security status of the device is indicated by an icon at the upper-right corner of the device.

Example: If the device supports security but has not yet been secured, an icon in the form of an empty red shield ( ) appears.

NOTE: If no icon appears, the discovered device does not exist, or the device does not support security settings, or you may need to assign the correct device settings to the device in the Discovery view.

The rest of the process depends on the Authentication Mode of the device. This mode defines the level of rights required for setting the security. Accessible in the Security section of the Properties tab of the device, Authentication Mode offers the selection of two modes:

  • Custom User: This is the standard and recommended device mode. The actions performed on the device request user credentials that enforce device security.

  • Anonymous: In this mode, a user is considered as a guest with administrator rights on the device. This mode allows a user to bypass user credentials when setting the security. This mode helps the user during the commissioning process but is not recommended for normal operations as security can be removed or reset by any guest, because no user credentials are requested.

As a consequence, the process to set the security and trust the device depend on the selected mode.

With Custom User Mode Selected

  1. Right-click the device and select Configure security.

  2. If the Login dialog opens, enter valid User and Password credentials for the device, then click OK.

    NOTE: Valid user and password settings are determined as follows:

    • If this is a device with a pre-existing configuration, enter user and password settings previously created in the User Management dialog.

    • If this is the first configuration of a new device (out-of-the-box), or of a device with its configuration reset to the factory default settings, enter the default user name and password provided by the device manufacturer. After deployment of the device configuration, which includes the collection of valid users for the solution, use only the user name and password created in the User Management dialog.

    NOTE: After you first enter user credentials during a session, these credentials are stored in memory and automatically reused when a login is required. This avoids the need to repeatedly enter credentials during a session.

  3. The Certificate Information dialog opens.

  4. Review the certificate information in the General and Details tabs to verify the device can be trusted.

  5. If the device certificate can be trusted, click Trust.

    NOTE: A trusted certificate remains in effect until it expires or changes.

  6. The security icon on the device changes to a green shield with checkmark ( ) indicating the device is trusted.

NOTE: If the security status is not as mentioned in the step 6, refer to the topic Security Status for more information.

At this point, the device is ready for operation in Custom User mode. It is ready to communicate with a logical device to eventually deploy an application on this device.

With Anonymous Mode Selected

  1. Right-click on the device and select Configure security.

  2. A message pops up to inform you that the Anonymous Security mode is not recommended.

  3. If you decide to proceed, click Yes.

  4. The out-of-the-box user device credentials are requested to secure the device.

The device displays the anonymous security icon, which is the same as the Custom User mode security icon plus the anonymous icon on the top right ().

NOTE: To learn more about the different anonymous and authentication status, check the Monitoring the Physical Architecture chapter.

At this point, the device is commissioned and secured. It is ready to communicate with a logical device to eventually deploy application on this device.

NOTE: After the commissioning is complete, it is recommended to switch to the Custom User mode to restore the security settings of the device.
