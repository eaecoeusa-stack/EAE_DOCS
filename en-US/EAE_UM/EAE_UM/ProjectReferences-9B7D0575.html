Project Library Reference

This topic defines the Project Library Reference dialog box. This dialog box is used to modify, remove or add library references in the project.

NOTE: To have more information on how to do these actions, refer to the topic Define Project Library References.

  1. The search bar allows to research libraries in the Project Referenced Libraries section and in the

    Installed/ Source Libraries section simultaneously.

    When text is added to the search bar, only libraries with this specific text in their name remain displayed in the sections.

  2. The Project Referenced Libraries

    section lists the libraries already referenced in the project as well as all the changes made with this dialog box.

  3. When the check box is ticked, the chosen library is referenced in the project. If the check box is unticked, the library is removed from the referenced library list of the project and the library line disappears from the section.

    NOTE: Find description of this action in the topic Define Project Library References.

  4. The Current Version column, shows the current version of the libraries referenced in the project.

    NOTE: This column is empty when the user has just added a new referenced library in the project as the library is not officially referenced to the project until the modification got applied.

  5. The

    Selected Version column, shows all modifications made to libraries referenced with this dialog box. If the column is empty, no libraries have been added nor modified.
  6. The columns in both sections indicate the type of libraries. There are 3 possible origins :

    • Source: Is a library created by the user that is directly integrated it in the current solution.

    • Installed: Is a library that is installed in the engineering workstation and can be referenced in the project.

    • Base: Is a base library that is installed in the engineering workstation and can be referenced in the project.

      NOTE: To have more information about the different kind of libraries and how to create them, refer to the topic Solution Node (CPT).

  7. The Installed/Source Libraries section lists all the libraries installed and available on the current engineering workstation.

    NOTE: To install new libraries on the engineering workstation, check the topic Installed Libraries.
    NOTE: If a library is referencing a runtime base library that is not referenced in the project, then that library will not appear in the Project Library References dialog box

  8. There is three different types of check box:

    • : Indicates that the library is not referenced in the project.

    • : Indicates that the latest version of this library is referenced.

    • : Indicates that this specific library is referenced, but not the latest version. Click on the + to see which version of this library is referenced.

  9. The + sign displays all the versions of this specific installed library.

  10. The Version column within the Installed/Source Libraries section gives the version of the library present on this line.

  11. This help button opens the online help directly on this page.

  12. The Select Latest Library(s) button selects the latest version of each referenced libraries in the project.

  13. The Apply button validates and executes all the changes made in the box dialog. When clicking on the Apply button all the modifications are applied in the project.

  14. The Cancel button closes the dialog box and discards the changes.
