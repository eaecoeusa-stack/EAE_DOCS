11. Testing AVEVA OMI Locally

In the Deploy and Diagnostic editor, you can deploy the AVEVA server to the HMI runtime. This pushes all the created objects to the runtime HMI.

To do this:

  1. Set (Local Test) as the Active Network Profile.

  2. Verify that you have compiled the solution and synchronized instances to the AVEVA system platform.

  3. Select the AVEVA server device, then select Action > Test AVEVA OMI on local computer.

This action deploys the galaxy objects and automatically creates the OPCUA connection node (OPCUA_<DeviceName>) and OPCUA group connection node (DeviceGroup_<DeviceName>) for each EcoRT device under the Operations Integration Server Manager > <GalaxyName> > <PlatformName> > [Admin] Operations Integration Supervisory Servers > Standards-Gateway > OI.Gateway.3 > Configuration in the Operations Control Management Console (OCMC).

OPCUA server details are updated automatically based on the EcoRT device properties. Security policy is set as None and in User credentials Anonymous User is selected. In OPCUA Namespace Alias section, the alias details are filled as:

  • Index: 2

  • Alias: EcostruxureSystemExpert

  • Namespace URI: http://se.com/ EcostruxureSystemExpert

The OI gateway will auto—start after every update to the configuration.

NOTE: The OPC UA connection node is created under the OI gateway server instance configured for the AVEVA server device in the property (OPCUA Gateway > Gateway). The configured server instance needs to be created manually in the Operations Control Management Console (OCMC) before performing the action Test AVEVA OMI on local computer.

The AVEVA operation management interface (OMI) is launched automatically, after the AVEVA server device has been deployed locally.

Once theAVEVA OMI loads, you can view the ASP runtime HMI. Use the AVEVA OMI to:

  • View alarms

  • Open and inspect faceplates

  • Issue operating commands

NOTE: On un-deploying a Galaxy, the AVEVA OI gateway configuration completed earlier is not removed. You can remove the OPC UA connection and device group connection nodes manually.
