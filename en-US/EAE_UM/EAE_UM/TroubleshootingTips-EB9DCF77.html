When you click on Sync to AVEVA System Platform or Test AVEVA OMI on local computer. With a pre-condition that Galaxy Repository service was restarted while any system platform operation were in progress.

Sync to AVEVA System Platform and Test AVEVA OMI on local computer operations are performed by the Windows service SE ASPSupervisoryFacetServices EcoStruxure Automation Expert 24.0.

This service is installed as part of the Asset Link for AVEVA OMI feature of EcoStruxure Automation Expert - Buildtime installer.

When the Galaxy Repository service is stopped in the middle of a System Platform operation, the operation may not complete.

To overcome this problem, restart the service SE ASPSupervisoryFacetServices EcoStruxure Automation Expert 24.0.

Refer to Restarting AVEVA Server Device Services

Required services are not running to start Build/Deploy. Refer to troubleshooting tips in user guide to start required services.

Build and Deploy operations are performed by the WindowsTM service SE ASPSupervisoryFacetServices EcoStruxure Automation Expert 24.0.

This service is installed as part of the  Asset Link for AVEVA OMI feature of the EcoStruxure Automation Expert - Buildtime installer. When this service is not running, you will see this message in the Output window. 

To start these services, refer to procedure: Starting AVEVA Server Device Services
