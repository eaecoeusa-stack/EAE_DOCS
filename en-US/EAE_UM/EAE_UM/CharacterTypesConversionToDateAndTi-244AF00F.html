5. Character Types Conversion to Date and Time Types

All functions described in this topic are conforming to IEC 61131-3 third edition.

STRING_TO_DATE, STRING_TO_LDATE

Synopsis 

TO_DATE(IN: STRING): DATE
STRING_TO_DATE(IN: STRING): DATE
 
TO_LDATE(IN: STRING): LDATE
STRING_TO_LDATE(IN: STRING): LDATE

Description

Converts parameter IN of type STRING to a value of DATE or LDATE. Base input format is ‘y-m-d’ with any of the valid date prefixes (D#, LD#, DATE# or LDATE#) being optional.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the value in parameter IN cannot be converted to a valid date value.

ERR_RT_TYPERANGE: A type range error is reported if the input string syntax is invalid or any if any of the components is out of range.

Example 

IF TO_DATE(STRING#'LDATE#2021-12-2') = D#2021-12-2 THEN
  (* correct *) ;
END_IF;

STRING_TO_DT, STRING_TO_LDT

Synopsis 

TO_DT(IN: STRING): DT
STRING_TO_DT(IN: STRING): DT
 
TO_LDT(IN: STRING): LDT
STRING_TO_LDT(IN: STRING): LDT

Description

Converts parameter IN of type STRING to a value of DATE_AND_TIME or LDATE_AND_TIME. Base input format is ‘y-m-d-h-m-s.s’ with any of the valid date prefixes (DT#, LDT#, DATE_AND_TIME#, LDATE_AND_TIME#, DT_UTC#, LDT_UTC#, DATE_AND_TIME_UTC# or LDATE_AND_TIME_UTC#) being optional.

By default, the input date is considered to be local time except if any of the UTC type prefixes is used. In all other cases a type prefix has no significance in the conversion process.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the value in parameter IN cannot be converted to a valid date value.

ERR_RT_TYPERANGE: A type range error is reported if the input string syntax is invalid or any if any of the components is out of range.

Example 

IF TO_DATE_AND_TIME('2021-12-3-12:0:0') = DT#2021-12-3-12-12:00:00 AND TO_DT('DT_UTC#2021-12-3-12:0:0') = DT_UTC#2021-12-3-12-12:00:00 THEN
  (* correct *) ;
END_IF;

STRING_TO_TIME, STRING_TO_LTIME

Synopsis 

TO_TIME(IN: STRING): TIME
STRING_TO_TIME(IN: STRING): TIME
 
TO_LTIME(IN: STRING): LTIME
STRING_TO_LTIME(IN: STRING): LTIME

Description

Converts parameter IN of type STRING to a value of TIME or LTIME. Base input format is ‘y-m-d-h:m:s.s’. All components are unsigned integer numbers. Valid date prefixes (T#, LT#, TIME# or LTIME#) are optional and have no significance in the conversion process.

Errors

ERR_RT_CONVERSION: A conversion error is reported if the value in parameter IN cannot be converted to a valid time value.

ERR_RT_OVERFLOW: An overflow error is reported if the value in parameter IN defines a time span which is larger that the allowed value range for data types TIME or LTIME.

ERR_RT_TYPERANGE: A type range error is reported if the input string syntax is invalid or any if any of the components is out of range.

Example 

IF TO_TIME(IN := '-48h') = TIME#-2d THEN
  (* correct *) ;
END_IF;
(* The input follows the same rules as time literals! The following is therefore not correct and produces a conversion error: *) ;
IF TO_TIME('-1d48h') <> TIME#-3d THEN
  (* definitely incorrect *) ;
END_IF;

STRING_TO_TOD, STRING_TO_LTOD

Synopsis 

TO_TOD(IN: STRING): TOD
STRING_TO_TOD(IN: STRING): TOD
TO_TIME_OF_DAY(IN: STRING): TOD
STRING_TO_TIME_OF_DAY(IN: STRING): TOD
 
TO_LTOD(IN: STRING): LTOD
STRING_TO_LTOD(IN: STRING): LTOD
TO_LTIME_OF_DAY(IN: STRING): LTOD
STRING_TO_LTIME_OF_DAY(IN: STRING): LTOD

Description

Converts parameter IN of type STRING to a value of type TIME_OF_DAY or L TIME_OF_DAY. Base input format is ‘h:m:s.s’. All components are unsigned integer numbers. Valid date prefixes (TOD#, LTOD#, TIME_OF_DAY# or LTIME_OF_DAY#) are optional and have no significance in the conversion process.

Errors

ERR_RT_CONVERSION: A conversion error is reported if any of the components of the time of day input string are out of range.

ERR_RT_TYPERANGE: A type range error is reported if the input string syntax is invalid or any if any of the components is out of range.

Example 

IF STRING_TO_TIME_IME_OF_DAY('LTOD#12:00:00.234567') = TIME_OF_DAY#12:00:00.234 THEN
  (* correct *) ;
END_IF;
