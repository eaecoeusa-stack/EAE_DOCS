2. INF Error Code List

INF_EMPTY_TYPETABLE

No types available in type table of a library

The mentioned library does not contain any library elements. The complier then gives hint that:

  • The library mentioned might be from an old format explaining why expected types are missing.

  • The library mentioned might not be correctly set up as it does not contain any information useful to the complier.

NOTE: Interpret this message with an understanding of project libraries, their content, and purpose.

TIPS

Remove this library from the project and build.

If a type that you are expecting from this library is missing then request a new library from your library supplier to replace that one.

INF_INCREMENTAL_UPDATE

${1},${2} ${eCls} ${eSym} : Incremental update of type '${3}' from file '${4}'

This message is given if parameter /incremental was given on the command line and the symbol table for the type given in ${3} has been successfully extracted from the library or binary given in ${4}. If the operation does not succeed, the compiler quietly continues in build mode.

INF_INST_MEM_SIZE

(${1},${2}) ${eCls} ${eSym}: Layout optimizer reduced memory size of FB '${3}' by ${4}/${6} bytes (${5}%/${7}%)

When compiler feature FBMemSizeMinimizer is enabled, which causes function block member variables to be sorted in descending order by their alignment value, this message gives an idea of how much memory was actually saved. By sorting the variables, allocation gaps due to alignment are avoided. Parameter ${3} gives the name of the function block being optimized, detected error message parameter ${4} and ${6} are assigned the amount of memory that has previously been occupied by gaps and has been saved in a 32-bit and a 64-bit context respectively, and parameter ${5} and ${7} gives the relative amount saved in percent which is calculated by dividing the number of bytes saved by the original memory size.

NOTE: The FBMemSizeMinimizer option should not be used on FBs which require variables to be arranged in some well defined order. Examples are:

  • FBs containing symbolically linked or user managed process variables.

  • FBs compiled for EcoRT (event counter variables). In this case, the compiler option FBMemSizeMinimizer is not available.

INF_KEYWORD

(${1},${2}) ${eCls} ${eSym}: Identifier '${3}' looks like a keyword

This detected error is similar to detected error ERR_KEYWORD. For a description of keyword handling in the compiler use the documentation there.

INF_MODULE_LOADED

0,0 (0,0) ${eCls} ${eSym}: Module '${1}' has been loaded successfully

This is an informational message only to be able to make sure the correct and intended library or binary file has been loaded.

INF_NO_SUCH_TYPE

(${1},${2}) ${eCls} ${eSym}: Type '${3}' is undefined

This informational message is given during checking constant values attached to the input variables of function block instances if the function block interface declaration has not been found in any of the loaded libraries. Without knowing a function block type, the compiler will not be able to find out the input variables type and therefore cannot check the correctness of the constant. In this case the parameter will be passed to the runtime unchecked which generally works for simple constants but it will not work for any UDT value, for named values and for constant expressions. This message is the only hint that input parameters for a function block type are not checked, there are no individual messages for every unchecked input variable so it may not be wise to suppress this message.

INF_NO_TYPETABLE

0,0 (0,0) ${eCls} ${eSym}: No type table found in library '${1}'

This informational message is given if a library is presented to the compiler which does not have a type table and compiler is checking constant values attached to the input variables of function block instances. The type table is a dictionary listing all the types defined in the library and the name of the file implementing the type or defines the types interface. The message can be considered harmless if the library is not used anyway. Otherwise a follow up detected error message will be given naming some undefined type.

INF_RT_LIBLOADED

(${1},${2}) ${eCls} ${eSym}: library element '${3}' loaded at position #${4}

This information is usually presented if a library element did not load because of an unresolved symbol (detected error ERR_SDB_UNRESOLVED). The list of loaded libraries can help to identify the source of the detected error. The unresolved library element must exist in any of the loaded libraries. If none of the loaded libraries contains the required symbol an error is detected in the deploy operation, otherwise the symbol lookup did not succeed. Submit the event to the software supplier.

INF_RT_STATUS

(${1},${2}) ${eCls} ${eSym}: ${3}

This is an informative status message which is generally presented in the test container to visualize internal operations.

INF_TIME_LITERAL

(${1},${2}) ${eCls} ${eSym}: Time literal type t# is obsolete. Please use T# instead

All elementary data types are considered keywords and have to be coded in uppercase letters only (also see ERR_KEYWORD). The same is true for typed constants and requires that data type prefixes are written in uppercase letters. Since parameters at function block input variables are also checked by the compiler all constants in a project have to follow strict ST coding rules. For historical reasons the short time prefix has always be coded in small letters. This is no longer possible since it is checked by the development environment but existing code still exists which should continue to work. At some time, this informational message may be changed to a warning and later to a detected error.

INF_XML_VAR_WITHOUT_IEVENT

${1},${2} ${eCls} ${eSym} : Variable '${3}' is not mapped to any input event

The variable named in parameter ${3} is not mapped to any input event indicating that the variable can be initialized with a constant or may be left unassigned. Unmapped input variables will be initialized during function block startup and no further assignment will happen.
