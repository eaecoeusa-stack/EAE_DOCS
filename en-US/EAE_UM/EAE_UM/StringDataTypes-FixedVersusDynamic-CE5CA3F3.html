STRING Data Types - Fixed versus Dynamic

EcoStruxure Automation Expert - Buildtime supports two STRING data types: fixed and dynamic.

Fixed STRING Data Type

A variable of the fixed string data type is declared using the STRING[N] format, where N is the maximum number of string characters/bytes. The maximum size of N is 65535. If N is not specified, the fixed string declaration automatically defaults to STRING[15]. Both the Basic function block and user defined data types (UDTs) - for example arrays and structures - can include only the fixed version of a string data type.

Dynamic STRING Data Type

In EcoStruxure Automation Expert, except for Basic function blocks or UDTs, every string type variable where [N] is not specified is a dynamic string. A default size of 32 characters/bytes is automatically assigned in memory. If a dynamic string data type variable size exceeds 32 characters, the variable size is increased in increments of 32 characters up to a maximum size of 65535 characters/bytes.

STRING Data Type Size Limits

Both a fixed string and a dynamic string variable is truncated after 65535 characters/bytes, with the additional characters discarded. The truncation event is recorded in the cyclic log file .

STRING Data Type Impact on Performance

Passing a dynamic string variable over a data connection between function blocks is slower than passing a fixed size string during event execution.

For high availability projects, the Logical Devices editor includes a runtime configuration property, found at Runtime Configuration > HA > MemPool > MaxDynPools, that limits the total amount of memory available to host dynamic string variables in the entire device. Increasing this number allows you to have more and larger dynamic strings in you application, but may cause slower performance.

Programming Design Impact on Performance

The maximum string size of 65535 characters/bytes is a technical limit that cannot be easily overcome. It is often also an indication of inefficient application design, where the controller is asked to deal with string variables of excessive size. Generally, good application design indicates a developer should use fixed size strings whenever possible, and use dynamic strings only when the maximum size is uncertain during design.
