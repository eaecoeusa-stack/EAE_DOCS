2. Migrating a Galaxy from an old ASP version to the newest one

If you are using the newest version of AVEVA System Platform (ASP), you can create a new Galaxy two ways:

  • Using the CFLBlankGalaxy cab file.

  • In EcoStruxure Automation Expert

Using the CFLBlankGalaxy Cab File

Using the CFLBlankGalaxy cab file, which is created using the old ASP version (that includes the new OPC client ASP template and the script library required for auto-configuration of the AVEVA OI Gateway), follow below steps:

Step

Action

1

Copy the CFLBlankGalaxy cab file from its current location (see the following NOTE) to the path “C:\Program Files (x86)\ArchestrA\Framework\Bin\BackupGalaxies”.

NOTE: The specific location of the CFLBlankGalaxy cab file might vary, depending on the EcoStruxure Automation Expert - Buildtime version you are using. For example:

  • For v24.0, the path is: "<EAE installation path>\ASP Libraries\Common Process Library\Galaxy Backup".

  • For v24.1, the path is: "<EAE installation path>\ASP Libraries\Common Process Library V2\Galaxy Backup".

2

Launch ASP Integrated Development Environment (IDE), then:

  • Select New.

  • Select the CFLBlankGalaxy template.

  • Enter the Galaxy name.

  • Click Create.

3

Connect to the galaxy. In the Connect To Galaxy migration dialog, click Yes.

4

Configure this galaxy as an existing galaxy, as described in the topic Configuring Galaxy Details in AVEVA SP Configuration to connect with ArchestrA Galaxy.

Using EcoStruxure Automation Expert

In EcoStruxure Automation Expert, follow these steps:

Step

Action

1

Open the solution.

2

In Solution Explorer, right click on the Solution node and select Configurations > AVEVA SP configuration.

3

Select New Galaxy, enter the Galaxy name and click Create.

NOTE: The Galaxy will be created but configuring security will not succeed.

4

Close the configuration dialog.

5

Launch ASP IDE and connect to the Galaxy created in step 3.

6

In the Connect To Galaxy migration dialog, click Yes

Configure this galaxy as an existing galaxy, as described in the topic Configuring Galaxy Details in AVEVA SP Configuration to connect with ArchestrA Galaxy.
