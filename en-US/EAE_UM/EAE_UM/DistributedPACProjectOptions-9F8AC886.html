Distributed PAC Project Options

The command Options from the Distributed PAC node context menu includes three selections:

  • General

  • IEC61499

  • HMI

General

General information about company address, author and web site of the project holder can be entered.

IEC61499

Compiling > Basic FBs and Components > Warning Levels

Command

Description

Warnings

If this check mark is set, warning messages generated during compilation are displayed in the output window.

Messages

If this check mark is set, messages produced by the compiler are displayed in the output window. Selecting Messages  automatically selects Warnings.

Compiling > Basic FBs and Components > Extensions

Command

Description

Allow operators +,-,*,/,MOD,<,>,... to be applied on bit string values and variables

This option concerns the possibilities of bit string variables within the algorithm. When this option is set, calculations with BOOL, BYTE, WORD, DWORD and LWORD are allowed without first converting them to a calculable variable. When this option is not set, the operation is not allowed (standardized) and a detected error message is displayed.

Allow general boolean expressions in transition conditions

This option concerns state transition conditions (see Transitions Editor). When this option is set, Boolean expressions are allowed as guard expressions within the ECC logic. If this option is not set, only events are allowed as guard expressions.

Allow names of variables, types etc. for being a 61131 or 61499 keyword

Contrary to the IEC standards, the function block type compiler requires keywords to be all uppercase. For example, in IEC 61131, "foR" is a valid keyword. When this option is not set, all versions of keywords written in any combination of lower/uppercase letters are marked as errors for reasons of compatibility. When this option is set, keywords containing a mix of uppercase and lowercase letters can be used, but are marked with a warning.

Report type cast problems in composite FB networks as detected error

When this option is set, the compiler enforces similar conversion rules for connections between IEC 61499 function block instances as in ST (only value-safe conversions are allowed). When this option is not set, the compiler also allows conversion where otherwise an explicit conversion operator (ANY2ANY) is allowed. Invalid conversions are always marked as errors.

Allow the use of an extended set of built-in Library functions

When this option is not set, only standard IEC 61131 functions are allowed. In this case, CURRENT_DT, UTC-related functions, UMPV-related functions, MOD with real arguments, and certain other functions are disabled and cannot be used.

Compiling > System

Command

Description

Show warning messages if there are obsolete components

When this option is set, warnings are shown if the project contains components from obsolete libraries.

Communication

Command

Description

Enable Quality of Service (QoS) configuration for VTQ variables

Select this option to configure Quality of Service (QoS) — using value, time, quality (VTQ) data — on communications between controllers. Then select one of the following options:

  • Do not retry if data is lost (default). Data is exchanged cyclically and some amount of lost data is acceptable. A connection loss is reported, but event losses are ignored.

  • Retry if data is lost. Data is considered to be unique and/or critical and transmission is retried if a value is reported as lost. Both connection loss and event losses are reported.

Max Latency

This parameter is used when Quality of Service (QoS) is configured on the connection between two controllers.

Specify the period of time that the sending controller buffers events so they are available if retransmission is required.​

Events that are older than the value of this parameter cannot be recovered.​

The maximum latency is the value of this parameter plus the trip time.

Range: 10 ms ... 60 s

Packet rate

This parameter is used when Quality of Service (QoS) is configured on the connection between two controllers.

The value of this parameter specifies the maximum elapsed time between two sent data packets​. The loss of an event cannot be reliably identified before this period of time has elapsed​.

Range: 1 ms ...10 s

Deploy > Deploy Options

Command

Description

Verbose on simple deploy

When this option is enabled, detailed information is shown when deploying an application.

Connection time out [s]

Defines the maximum time span for connecting.

Answer time out [s]

Defines the maximum time span for an answer to arrive after an initiated action or request.

Pre-connect time (tunneling) [s]

Defines the maximum time span for build up of tunneling.

Archive link service timeout [s]

Defines the maximum time span which may pass between the first request to the data server and the reply from data server that all data has arrived completely. In case that the data transfer takes a longer time the value can be adjusted here accordingly.

Deploy changes only

When this option is set, only changes within the project are deployed.

Node Watermark

Command

Description

Load Image

Enter file name and path of the respective image and confirm. The function blocks will be shown with this image as watermark. If another image is selected, the first one is overwritten.

Remove Image

Removes the selected image; the function blocks are displayed in the original style.

Transparency

The transparency of the selected image can be adjusted by a slider.

Settings

Command

Description

OPCUA Client

Enable OPCUA Client configuration: Enables the OPC UA Client Editor, where you can configure and view configuration settings for OPC UA service function blocks.

OPCUA Server

Select the format of OPC UA server node identifiers:

  • Guid (default).

    Use a globally unique identifier (GUID).

  • String (as Function block path). Use a string. The string generated is the path of the function block instantiated within the device, for example DEV1.RES0.FB1.Var1.

NOTE: After a change of format, OPC UA clients are no longer accessible and must be reconfigured to use the new format.

Refer to OPC UA Server for details.

HMI

Opens the HMI Libraries tab, which you can use to select HMI Libraries from among the collection of installed libraries. Refer to the topic Symbol Editor for help on how to make these selections.

Command

Description

HMIBaseSymbols

Use the HMI base symbol library with re-mastered HMI Symbols (among others DrawButton, Two State Button, Combo Box, Tracker, Round Knob, etc. graphical re-mastered and partly provided with new setting options). Use this library for every new project.

HMISymbols

Use the HMI symbols library; this is the older version, it is necessary for older projects which have this library already in use. Deactivate it for new projects, in order to keep the dialog Select Generic Symbol clear.
