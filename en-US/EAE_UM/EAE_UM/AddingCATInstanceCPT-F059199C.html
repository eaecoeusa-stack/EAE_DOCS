Handling Application Instances

In a hierarchy view, all the application instances (CAT instances, SubApp instances and SubCAT instances) available in the Solution Explorer pad can be:

  • Assigned to one or several hierarchy views (Process view and AVEVA plant model view).

  • Unassigned to the current hierarchy view.

  • Moved within the current hierarchy view.

All these actions can be done without impact on the application instances as the hierarchy views are independent of the application instances. Thus actions in the Hierarchy View pad don’t affect the application instances in the Solution Explorer pad.

Here are all the actions possible related to the application instances:

  • Assigning existing CAT instance(s) to the current hierarchy view through a dialog

  • Assigning an existing CAT instance to the current hierarchy view through dragging-and-dropping.

  • Creating a new CAT instance directly into the Hierarchy View pad.

  • Moving elements inside a hierarchy view.

  • Assigning an instance of a hierarchy view to an application.

  • Unassigning application instances from a hierarchy view.

  • Finding references of an application instance.

Whenever an instance is assigned to the current hierarchical view, a ● mark is prepended to the instance name in the solution tree-view.

NOTE: All these actions (except Find references of an application instance) are disabled within an AVEVA hierarchy view when a Galaxy is deployed: all the elements inside the view are greyed out and in italics.

Assigning existing CAT instance(s) to the current hierarchy view through a dialog

  1. Open the Hierarchy View pad and select the wanted hierarchy view in the drop-down.

  2. Right-click on the root node of the hierarchy view or on the folder which the CAT instance will be assigned to.

  3. Select Assign CAT Instance; this opens the Assign CAT Instance dialog box.

  4. This dialog contains all the CAT instances and SubApp instances not already assigned to the current view (whether they are assigned or not to an application). Search for the wanted instance by scrolling into the list or by typing the instance name(s) into the search bar.

    NOTE: This dialog box doesn’t allow to assign SubCAT instances.

  5. Select the CAT instance and click Finish.

NOTE: The SubCAT instances are included or not depending on the check box File > General > Generic Settings > Retain CAT hierarchy while assigning CAT to Hierarchy View is ticked or not (this is reflected by the presence or absence of the icon above the tree-view);
NOTE: A • mark is added before the instance name in the Solution Explorer.
NOTE: With an AVEVA plant model view, you can only use the next method..

Assigning an existing CAT instance to the current hierarchy view through dragging-and-dropping

  1. Open and detach the Hierarchy View pad.

  2. Select the wanted hierarchy view in the drop-down and expand the CAT Instance section in the Solution Explorer pad.

  3. Search and select the wanted application instance.

  4. Drag-and-drop it into the hierarchy view, onto the root node of the hierarchy view or a folder. The application instance gets assigned to it.

NOTE: Only one application instance can be dragged and dropped at one time. Thus repeat this action as many times as there are CATs, SubApp or SubCAT instances to assign. No duplicates are allowed wheresoever they may be.
NOTE: A • mark is added before the instance name in the Solution Explorer.
NOTE: The SubCAT instances can be dropped under any CAT instance provided the check box File > General > Generic Settings > Retain CAT hierarchy while assigning CAT to Hierarchy View is NOT ticked (this is reflected by the absence of the icon above the tree-view);

Creating a new CAT instance directly in the hierarchy view

  1. Open the Hierarchy View pad and select the wanted hierarchy view in the drop-down.

  2. Right-click on the root node of the hierarchy view or on the folder which the CAT instance will be assigned to.

  3. Select Create New Instance and navigate until finding the wanted CAT.

  4. Click the wanted CAT. This opens the Create Instance dialog box.

  5. Enter the name of the new CAT instance that starts with a letter and excludes *, $, space.

  6. Select an application an a layer.

  7. Click OK.

  8. If the System tab is active on instantiation (whatever editor is open), choose:

    • Save: if you want to create the instance in the Solution Explorer and have it in the hierarchy view AND already in the Function Block Network of the selected application and layer.

    • Discard: if you want ONLY to create the instance in the Solution Explorer and have it in the hierarchy view.

    NOTE: If no application is selected in the Define new name dialog, the choice does not matter.
NOTE:

  • Only one CAT instance can be created at one time.

  • You can still assign it later to an application and layer: refer to Adding instances into an application.

To assign an Asset tag, right-click the CAT instance and select Set Asset Tag Name.

Moving elements inside a hierarchy view

All the elements assigned or created inside a hierarchy view (application instance and folders) can be moved from a place to another inside the hierarchy view. If the moved element is a folder, all elements contained in it will also be moved.

There are two different ways to do this:

  • Drag and drop the element. The element has to be dropped onto either the hierarchy view backbone or onto a folder.

  • Use the context menu:

    1. Open the Hierarchy View pad and select the wanted hierarchy view.

    2. Right-click the element name to move.

    3. Select Move To or Move To Folder to open the Move To dialog box.

    4. Select the folder where the element will be moved to. It can be:

      • A new folder created through the Add new folder button in the dialog box.

      • An existing folder found by searching for its name in the search bar.

      • An existing folder found by drilling down with the > icon

    5. Select the folder and click Finish.

      NOTE: The Finish button will be available only if the folder is closed.
NOTE: The SubCAT instances can be moved under any CAT instance provided the check box File > General > Generic Settings > Retain CAT hierarchy while assigning CAT to Hierarchy View is NOT ticked (this is reflected by the absence of the icon above the tree-view).

Assigning an instance of a hierarchy view to an application

All the CAT or SubApp instances available in a hierarchy view can be added to an application:

  1. In the Function Blocks Network editor, open the application where the application instance needs to be added.

  2. In the hierarchy view, select the application instance that needs to be added.

  3. Drag and drop it into the application.

If the instance is already assigned to an application, a pop up informs you about this.

Unassigning application instance(s) from a hierarchy view

To unassign CAT, SubApp or SubCAT instance(s), select it/them and:

  • either select the Unassign CAT Instance in the context menu

  • or press the Delete key.

    NOTE: This action does NOT delete the application instances from the solution, but only unassign them from the hierarchy view. Application instances cannot be deleted from the Hierarchy View pad.

The selected application instances are removed from the hierarchy view and the ● mark before the application instance name(s) in the Solution Explorer pad is/are removed.

NOTE: The SubCAT instances can be unassigned separately of the parent provided the check box File > General > Generic Settings > Retain CAT hierarchy while assigning CAT to Hierarchy View is NOT ticked (this is reflected by the absence of the icon above the tree-view);

Finding references of an application instance

To find all the references of a CAT or a SubApp instance, right-click it and select Find All References.

All the references found are listed in the Search Results pad. Double-click one of the results to access the instance into the Function Blocks Network editor.
