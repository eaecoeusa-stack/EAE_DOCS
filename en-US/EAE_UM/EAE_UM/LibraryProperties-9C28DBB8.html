Library Properties

The Library properties dialog opens while creating an Installation Package of a self made library. It serves to define which compiler target(s) can be used with the previously compiled library.

To open the dialog, right-click on an open source library (i.e., a user-created custom library in the solution) in the Solution Explorer, right click, then select Create Installation Package from the context menu.

NOTE: Concerning the compiler target, the selection of the compiler depends on two criteria:

  1. The compiler target(s) has to smoothly support the functions used with the function blocks. Thus the functions used in the function blocks are necessary for the selection of the compiler target.

  2. The device, which is used to run a project referencing this library, also has to support the compiler target. Thus the compiler target has to be selected in a way that takes into consideration the devices, and their runtime systems, which are to be used.

For that reason, it is possible to select a specific compiler version to run the function block test while programming the library. In this way, the function block test is executed with the desired compiler already selected (see Basic Function Block Editor).

Configuring Library Properties

The Library properties dialog includes the following settings:

Setting

Description

Public

If this check box is marked, the library will be created as a public version. That means the function blocks of this library can be opened, when the library is installed, and the content (e.g. ECC-scheme and algorithm  of Basic function blocks and Composite view of CAT or Composite function blocks etc.) is visible.

If the check mark is not set the function blocks of the installed library can only be viewed from the outside. A double-click on a function block merely will open the Interface Editor in a read only version.  ECC-scheme,  algorithm, Composite view, etc. will not be shown.

Library version:

Increase the version number of the library to be created, if another version of the same library does already exist, by using the +button or typing in the desired number.

Install in:

Shows the symbolic path, that means the group of libraries, where the readily generated library will be installed to. Licence depending there may be some additional options. (With this the installation location of a library can be selected, for example, at the time of the installation. Refer to the topic Installed Libraries regarding Install.

In installation include compile targets:

Available compiler targets are listed, one is marked per default, others can be marked additionally if necessary, to have the library using those compiler targets. The library will be created in such a way that the chosen compiler targets are supported. Multiple selection is possible.

Allow Compile on the Fly

Parameter is used for future implementation, internally used

OK

Confirms the current settings and starts creating of the library.

Cancel

Cancels dialog and shuts the window.
