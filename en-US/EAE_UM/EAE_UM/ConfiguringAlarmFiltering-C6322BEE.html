Configuring Alarm Filtering

  1. In a function block canvas – for example, in the HMI node of an Application CAT – add an Alarm Grid control to the canvas.

  2. In the Properties window for the grid, under Misc configure the following settings:

    • ShowToolbar: Select True to activate the grid toolbar. This will display and enable the filtering tools on the alarm grid during runtime.

    • ShowOnlyLocalAlarms: Select either:

      • True: to display alarms only for the local CAT or SubCAT.

      • False: to display all alarms.

  3. AlarmType

    : Select either:

    • Online: to display only active alarms.

    • Archive: to display only alarms stored in the PostgreSQL archive server.

  4. AlarmFilter: Click the ellipsis. A message informs you that filtering alarms can lead to possible hidden alarms:

     WARNING
    UNINTENDED EQUIPMENT OPERATION
    If you choose to filter some alarms, the operator will not be able to see some alarms.
    This can result in unnoticed alarms coming from the process
    Failure to follow these instructions can result in death, serious injury, or equipment damage.

  5. To proceed with alarm filtering, press the I want to filter alarms anyway button. As the button tool tip indicates:

    By pressing this button, you confirm that some alarms will be hidden based on the applied filter. You should also check that remaining alarms are displayed as expected by running EcoStruxure Automation Expert HMI.

  6. In the Alarm Filter dialog, configure the following settings:

    • Filter from searched text: Enter filtering text. You can use the asterisk (*) as a wildcard to represent any combination of characters.

    • Interval Type: For filtering archived alarms only, you can select:

      • A defined interval (e.g. All, ThisWeek, and so forth)

      • FromTime to specify a start time (From) and a subsequent time period (Range)

      • FromTimeToTime to specify a date range via From and To dates.

  7. After alarm filtering is configured, save your edits, then compile and deploy your solution.
