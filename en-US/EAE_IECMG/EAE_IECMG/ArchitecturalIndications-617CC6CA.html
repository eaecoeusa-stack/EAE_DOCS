Architectural Indications

EcoStruxure Automation Expert is built upon the IEC61499 standard, the foundation for modeling and designing distributed industrial automation systems.

In the IEC61499 approach, the process begins with defining a high-level view of the system and breaks it down into smaller, more manageable components.

Thus, developing an application begins with identifying clear functional requirements and system objectives. Next, establish the essential framework of the application by defining the system's key goals: performance, scalability, and behavior.

Following CBP standard, you can break down the system into sub-systems or functional blocks. Each sub-systems, or function block, represents a distinct functional component unit within the system. In the context of EcoStruxure Automation Expert, these components are known as CAT (Composite Automation Types).

Next, focus on the high-level structure, aligning each function block with a corresponding system component, such as a Process cell, Unit, Equipment module, etc.

To do so, the development of a singular CAT object encompasses all the underlying objects responsible for constructing specific sections of the plant.

NOTE: The decision on which type of highest object needs to be developed depends on the complexity of the given use case.

Following ISA88 standard, particularly its plant-structuring recommendations, the top CAT object can be modeled as an area, process cell, unit, or equipment model.

Upon identifying the high-level object, you can proceed by disassembling the code within the top CAT into smaller code fractions, which will now be referred to as sub-systems in the rest of this document.
