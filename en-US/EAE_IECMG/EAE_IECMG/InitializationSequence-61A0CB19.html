Initialization Sequence

As the IEC61499 system is inherently passive, it necessitates the firing of an event to initiate any actions within the application.

Then, at the start of the application, the first action to do is the initialization process. This process configures all internal parameters and initializes essential services within the system (e.g. I/O management, HMI, various communication protocols...).

NOTE: For more information about the initialization process, refer to the Initialization CAT (Frame-0) topic.

To perform the initialization, let’s use the CAT defined in the Initialization CAT (Frame-0) topic. This CAT will initialize all object components using its IAdapterInit adapter. Thus, all components requiring initialization will then add this adapter type in their interface which creates a straightforward daisy-chaining connection.

Creating this initialization process is important as each component is responsible for its own initialization process. Which implies that all associated child components, constituting the sub-system of the object, also need to implement their own initialization proces.
