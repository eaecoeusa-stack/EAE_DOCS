About the Document

Document Scope

This document presents an overview of EcoStruxure Automation Expert application development by exploring the use of component-based programming (CBP) as a methodology and by providing detailed guidelines for developing and structuring EcoStruxure Automation Expert applications.

NOTE: For more information about component-based programming, refer to Component-Based Programming Overview topic.

The decision to adopt the CBP methodology is rooted in its alignment with the fundamental principles of the IEC61499 modeling methodology, emphasizing modularity, reusability, and scalability in system design. Both IEC61499 and component-based programming share the objective of enhancing the development, reusability, and scalability of complex systems by breaking them into smaller independent units.

NOTE: For more information about the fundamental principles of the IEC61499 modeling methodology, refer to the IEC61499 Standard Overview topic.

Additionally, in this document the ISA88 standard is introduced and it explains how its recommendation can be applied and incorporated to the EcoStruxure Automation Expert application.

NOTE: For more information about the ISA88 standard, refer to the ISA88 Overview topic.

Finally, this document details the process of developing an IEC61499 real use case application on EcoStruxure Automation Expert following CBP.

Validity Note

This document is valid for EcoStruxure Automation Expert version 25.0 or any subsequent supporting version(s).
