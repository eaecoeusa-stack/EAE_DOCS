Modicon Edge IO

EcoStruxure Automation Expert will allow user to integrate Edge I/O offer as distributed IO node over EtherNet/IP in control system architecture. This is Schneider Electric new cost-effective offer with advance functionalities:

  • User can instantiate Edge I/O avatar from Physical View and configure network settings.

  • User can configure and parameterize Edge I/O head and modules in offline mode.

  • User can read and write Discrete & Analog data over EtherNet/IP.

  • User can control and monitor Discrete & Analog module in EcoStruxure Automation Expert - HMI.

Platform supported:

  • Non-HA SoftdPAC & M262dPAC over EtherNet/IP

  • Non-HA & HA SoftdPAC over Modbus TCP/IP – (With Limitation)

Limitations:

  • Edge I/O integration over Modbus TCP/IP is not seamless. User can use Generic Modbus TCP device hardware CAT to engineer Edge I/O modules and can exchange data over Modbus TCP/IP.

  • 40k OPC-UA Server Tags – HA/Non-HA Soft dPAC

    In your EcoStruxure Automation Expert application, the OPC UA Server supports a default maximum of 20,000 OPC UA variables.

    If your application requires more variables, this limit can be extended to 40,000 variables by using a specific version of the SE.DPAC library, which is available upon request from Schneider Electric Support.

Changes in Module and Symbol Names

Introduction of the new Modicon Edge I/O NTS I/O sub-system that provides a performant lower cost solution that fits in between the TM3 and x80 I/O families while providing the flexibility of using standard protocols (Modbus TCP, EthernetIP) for connectivity.​

  • Selected modules supported from 1.1,1.11 and 1.2 release​ (for more details go to Appendix section)

  • Including HART and Counter Modules​

In version 25.0, the NTSDDI1602X and NTSDDO0802X modules available in version 24.1 have been renamed to NTSDDI1602 and NTSDDO0802 to ensure naming consistency across all modules. As a result, EdgeIO users upgrading from version 24.1 to 25.0 must reconfigure their setup using the Edge I/O Configurator.

Additionally, the default module symbol name sDefault has been replaced by sSingleSlot and sDoubleSlot in version 25.0. Users are required to delete the existing symbol instances from the canvas and instantiate the new symbols accordingly.

Offline Edge I/O Deployment

User cannot do online configuration and deployment to Edge I/O modules using Integrated Edge I/O editor. However, user can do offline configuration and export configuration file. This file then can be imported in Edge I/O web editor and deployed to Edge I/O modules.
