EcoStruxure Automation Expert - Archive

Overview

EcoStruxure Automation Expert - Archive provides storage for alarms history and data values. PostgreSQL 64 bits edition is used for this storage.

EcoStruxure Automation Expert - Archive (Microsoft Windows) Installation

When you launch the EcoStruxure Automation Expert - Archive installer, it verifies first if any compatible PostgreSQL version is installed on your computer. If no compatible PostgreSQL version is detected, the installation is canceled. 

Install the required PostgreSQL version and its components, then launch the EcoStruxure Automation Expert - Archive installer again. Instructions on how to install these components are provided in the EcoStruxure Automation Expert - Archive User Manual. Open the Documentation.htm file in the Documentation folder of the EcoStruxure Automation Expert installation package to find this document or open this document from the EcoStruxure Automation Expert help ribbon bar.

During the installation of EcoStruxure Automation Expert - Archive, the following information is displayed and requested: 

  • A PostgreSQL version must be selected from the detected compatible PostgreSQL versions available on this computer. The selected version is used for your Archive database called searchive. 

  • The data folder of the selected PostgreSQL version, where your Archive database is stored, is displayed. 

  • The password of the database superuser postgres may be requested to verify if your Archive database is created properly. This is the password entered during the installation of the selected PostgreSQL version. 

    NOTE: If you forget the password or intend to change a complex password, it can be reset without deleting the content of the database by following the instructions from this web site: https://neon.com/postgresql/postgresql-administration/postgresql-reset-password .

    • The port number configured for the PostgreSQL database server of the selected PostgreSQL version is displayed.

      NOTE: This port number is stored into the postgresql.conf file, in the data folder of the selected PostgreSQL version. The same port number must be configured also inside the EcoStruxure Automation Expert solution.

      Passwords for the required user roles searchive and seservice are requested for the selected PostgreSQL version.

      NOTE: Starting with EcoStruxure Automation Expert - Archive V24.0 there are new password requirements introduced and the passwords are encrypted with scram-sha-256 algorithm to fulfill standard security requirements. In earlier versions, default passwords were used (identical to the user names) and encrypted with unsecure MD5 algorithm. The password encryption is configured in the pg_hba.conf file.

      Password will not be prompted during installation if searchive and seservice users in PostgreSQL already has scram-sha-256 encrypted passwords.

EcoStruxure Automation Expert - Archive (Microsoft Windows) Database Migration

Starting with EcoStruxure Automation Expert - Archive V23.0, the installer lists all detected compatible PostgreSQL versions available on your computer. If more than one PostgreSQL version is listed, a PostgreSQL database may already exist on this computer, created with the installation of an earlier major version of PostgreSQL. You can migrate this database to a later major version of PostgreSQL. 

Instructions on how to manually migrate your existing Archive database to a later major version of PostgreSQL are provided in the EcoStruxure Automation Expert - Archive User Manual. Open the Documentation.htm file in the Documentation folder of the EcoStruxure Automation Expert installation package to find this document or open this document from the EcoStruxure Automation Expert help ribbon bar.

EcoStruxure Automation Expert - Archive (Microsoft Windows) Database Backup

It is recommended to make a backup of the archive database before updating the EcoStruxure Automation Expert - Archive (Windows) version or before doing an update of you​r machine from Windows 10 to Windows 11 or before installing a later PostgreSQL version. In some cases during or after such version updates, it may happen that the Windows service named postgresql-x64-XX (XX is the PostgreSQL major version) is not running and fails also when trying to start it manually. If you face this issue, delete the Data directory of your archive database, reinstall the required PostgreSQL version and check that the service is running. Afterwards restore your backup of the archive database and if needed install again EcoStruxure Automation Expert - Archive (Windows). If you need help for solving of this issue, contact your local Schneider Electric support.

EcoStruxure Automation Expert - Archive (Ubuntu) Installation

During installation, passwords for postgres superuser, the required user roles searchive and seservice are requested.

NOTE: Starting with EcoStruxure Automation Expert - Archive V24.1 there are new password requirements introduced and the passwords are encrypted with scram-sha-256 algorithm to fulfill standard security requirements. In earlier versions, default passwords were used (identical to the user names) and encrypted with unsecure MD5 algorithm. The password encryption is configured in the pg_hba.conf file.

Access to About Dialog in Archive Manager

To improve usability and align with HMI Manager and Soft dPAC Manager, a new entry point to the About dialog has been added directly in the Archive Management window.

Users can now access the version number, copyright, and licensing information from this dialog without using the tray icon.
