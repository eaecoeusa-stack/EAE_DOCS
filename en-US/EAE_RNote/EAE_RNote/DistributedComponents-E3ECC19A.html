Create multiple Scan groups

This release introduces a new feature aimed at improving the operational effectiveness of scan group distribution during the Aseetlink for OMI synchronization process.

Previously, scan groups were automatically created using the solution name, with a default rule of one scan group per approximately 5000 OPC I/O attributes. This rigid configuration limited flexibility and could lead to performance bottlenecks in systems with varying attribute loads.

A new ASP Attribute Mapping Editor has been added at the system level, featuring a configurable Scan Group column.  

  • Default Value: default

  • User Options: Slow, Medium, Fast

Provides users with enhanced flexibility to manually assign scan groups based on specific operational requirements.

Optimizes scan group distribution to significantly improve system performance and scalability.

Smarter Template Sync

A new user interface now simplifies and optimizes the synchronization of templates from AVEVA System Platform to EcoStruxure Automation Expert.

Key Features

Selective Template Synchronization

Users can now select specific templates to synchronize from ASP to EcoStruxure Automation Expert, without requiring the full Galaxy data to be synced. This allows for more targeted and efficient synchronization.

Enhanced User Interface

The updated interface includes:

  • Sorting options to organize templates by name or synchronization status

  • Filtering tools to quickly identify relevant templates

  • Visual indicators displaying synchronization status and progress

This update provides greater flexibility and improves the overall synchronization experience.

Buildtime Performance Improvements

Editor Loading User interface hang issues have been resolved, and performance has been improved through dynamic loading for the following editors: Offline Parameterization, OPC UA Server, ASP Attribute Mapping, and Asset Mapping editors.
