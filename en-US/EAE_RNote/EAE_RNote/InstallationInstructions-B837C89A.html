Installation Instructions

EcoStruxure Automation Expert Installation

If you have already installed EcoStruxure Automation Expert Platform, make sure there is no remaining active reservations. For more details, see Multi-User Environment - User Guide

Launch the EcoStruxure Automation Expert - Buildtime installer and follow the instructions.

The installation requires administration rights.

Local Windows Defender firewall rules are created. If you are using another firewall software, you must configure the rules described in chapter Computer Firewall. Installing EcoStruxure Automation Expert also installs the following components:

  • Schneider Electric Software Update

  • EcoStruxure Automation Expert EtherNetIP Master DTM library

  • FDTEnv32

  • MS SQL Server 2022 LocalDB (V16.0.1000.6)

  • Microsoft Windows Desktop Runtime 8.0.16 (x64) (V8.0.16.34817)

  • MS ASP.NET Core - 8.0.16 (x64) (V8.0.16.34817)

  • MS Edge WebView 2 Runtime (x64) (V136.0.3240.76)

  • Microsoft Visual C++ 2015-2022 Redistributable (x64) (V14.44.35112.1)

  • Microsoft Visual C++ 2015-2022 Redistributable (x86) (V14.44.35112.1)

NOTE: Any modification to select or unselect the options Asset Link for Bulk Engineering, Asset Link for AVEVA OMI or AVEVA Industrial Graphics Editor requires to launch the EcoStruxure Automation Expert - Buildtime installer again, and then to select Modify.
NOTE: In case of simultaneous installation of AVEVA System Platform 2020 R2 SP1, AVEVA options and EcoStruxure Automation Expert - Buildtime on the same computer, you must also install the Hot Fix - IMS1755821 on top of ASP 2020 R2 SP1.
NOTE: Install EcoStruxure Automation Expert - Buildtime to the default installation path. If you choose a custom installation path, ensure that the total path length does not exceed 85 characters. The installation will fail if the selected path exceeds this limit.

Installation of EcoStruxure Automation Expert Platform via EcoStruxure Automation Expert Buildtime installer

EcoStruxure Automation Expert Platform can be installed with the EcoStruxure Automation Expert - Buildtime installer of V25.0

If you want to install the software platform on your machine, select the checkbox in the "EcoStruxure Automation Expert Platform Installation" dialog that is shown after the "Custom Setup" dialog.

The software platform consumes 1.6 GB disk space on your hard drive, in addition to the required disk space for EcoStruxure Automation Expert - Buildtime which is shown in the "Custom Setup" dialog.

The "EcoStruxure Automation Expert Platform Installation" dialog is not shown if the same or a previous version of the software platform is already installed.

If the software platform is selected for installation, a restart of the system needs to be confirmed at the end of the installation.

NOTE: If you have installed the software platform via the modify installation of the EcoStruxure Automation Expert - Buildtime installer and you selected or unselected also any feature in the "Custom Setup" dialog, you must run the modify installation of EcoStruxure Automation Expert - Buildtime again after the system restart and you need to redo the changes in the "Customer Setup" dialog.

The EcoStruxure Automation Expert Platform is not uninstalled automatically during uninstallation of EcoStruxure Automation Expert - Buildtime as this software platform may also be used by other application. It is possible to manually uninstall EcoStruxure Automation Expert Platform using the standard Microsoft Windows procedure.

Windows Defender Firewall Message

When launching first time the EcoStruxure Automation Expert Platform or launching EcoStruxure Automation Expert from the EcoStruxure Automation Expert Platform in private network, you may encounter a Windows Defender Firewall pop-up. To enable proper functionality and security, please follow these steps when the Windows Defender Firewall pop-up appears:

Network Selection:

  • Unselect the Public Network option.

  • Select the Private Network option only.

Confirmation:

  • Click the Allow Access button to confirm your selection.

By following these steps, you will ensure that the application has the necessary permissions to operate securely within your private network.

Additional Firewall Configuration Instructions

However, the firewall rule for otelcol.exe is not created by the EcoStruxure Automation Expert installer. Users must manually create this rule in "Windows Defender Firewall with Advanced Security to allow incoming network connection for profile Domain and Private.

EcoStruxure Automation Expert - HMI (Windows), EcoStruxure Automation Expert - Archive (Windows) and
EcoStruxure Automation Expert - Soft dPAC (Windows) Installation

EcoStruxure Automation Expert - HMI (Windows), EcoStruxure Automation Expert - Archive (Windows) and EcoStruxure Automation Expert - Soft dPAC (Windows) can be installed side by side on the same computer, or on different computers.

Launch the EcoStruxure Automation Expert - Runtime installer and follow the instructions.

The installation requires administration rights.

Local Windows Defender firewall rules are created. If you are using another firewall software, you must configure the rules described in chapter Computer Firewall (HMI), Computer Firewall (Archive) or Computer Firewall (Soft dPAC – Windows).

Installing EcoStruxure Automation Expert - Runtime, also installs Microsoft Visual C++ 2015-2022 Redistributable (x86) (V14.44.35112.1).

Installing EcoStruxure Automation Expert - HMI (Windows), also installs Microsoft Windows Desktop Runtime - 8.0.16 (x64) (V8.0.16.34817).

Installing EcoStruxure Automation Expert - Archive (Windows), also installs Microsoft Visual C++ 2015-2022 Redistributable (x64) (V14.44.35112.1).

NOTE: Any modification to select or clear the options EcoStruxure Automation Expert - HMI (Windows), EcoStruxure Automation Expert - Archive (Windows), or EcoStruxure Automation Expert - Soft dPAC (Windows) requires to launch the EcoStruxure Automation Expert - Runtime installer again, and then to select Modify.

Documentation

The technical documentation of EcoStruxure Automation Expert - HMI (Windows), EcoStruxure Automation Expert - Archive (Windows) and EcoStruxure Automation Expert - Soft dPAC (Windows) is installed as part of the technical documentation of EcoStruxure Automation Expert.


Microsoft Windows Power Options

EcoStruxure Automation Expert - Archive (Windows) and EcoStruxure Automation Expert - Soft dPAC (Windows) require the following power options in Microsoft Windows:

  • Hard disk > Turn off hard disk after > On battery, configured to Never

  • Hard disk > Turn off hard disk after > Plugged in, configured to Never

  • Sleep > Sleep after > On battery, configured to Never

  • Sleep > Sleep after > Plugged in, configured to Never

  • Sleep > Allow hybrid sleep > On battery, configured to Off

  • Sleep > Allow hybrid sleep > Plugged in, configured to Off

  • Sleep > Hibernate after > On battery, configured to Never

  • Sleep > Hibernate after > Plugged in, configured to Never

 WARNING
INTERRUPTION OF SERVICES
Always deactivate power options that could interrupt services execution on Microsoft Windows PC when running EcoStruxure Automation Expert - Archive (Microsoft Windows) or EcoStruxure Automation Expert - Soft dPAC (Microsoft Windows).
Failure to follow these instructions can result in death, serious injury, or equipment damage.

EcoStruxure Automation Expert Uninstallation

Use the standard Microsoft Windows procedure to remove the EcoStruxure Automation Expert software from your computer.

Before uninstalling EcoStruxure Automation Expert - Buildtime, ensure that the EcoStruxure Automation Expert application is closed. If the application remains open during the uninstallation process, the uninstallation will proceed, but the application will stay open. Consequently, an exception dialog will appear when any operation is attempted within the application.

The following components are not uninstalled during the uninstallation of EcoStruxure Automation Expert software and require a manual uninstallation using the standard Microsoft Windows procedure:

  • Schneider Electric Software Update

  • EcoStruxure Automation Expert EtherNetIP Master DTM library
    NOTE: Uninstallation is only possible by using Windows standard uninstallation procedure.

  • FDTEnv32

  • MS SQL Server 2022 LocalDB (V16.0.1000.6)

  • Microsoft Windows Desktop Runtime 8.0.16 (x64) (V8.0.16.34817)

  • MS ASP.NET Core - 8.0.16 (x64) (V8.0.16.34817)

  • MS Edge WebView 2 Runtime (x64) (V136.0.3240.76)

  • Microsoft Visual C++ 2015-2022 Redistributable (x64) (V14.44.35112.1)

  • Microsoft Visual C++ 2015-2022 Redistributable (x86) (V14.44.35112.1)

NOTE: If you cancel the uninstallation process of EcoStruxure Automation Expert - Buildtime and EcoStruxure Automation Expert - Runtime, it can result in an incomplete installation. To resolve this issue, you should uninstall the software again without interruption and then reinstall EcoStruxure Automation Expert to ensure a complete and functional installation.

EcoStruxure Automation Expert - HMI (Windows), EcoStruxure Automation Expert - Archive (Windows), and
EcoStruxure Automation Expert - Soft dPAC (Windows) Uninstallation

During the uninstallation of EcoStruxure Automation Expert - Runtime, Microsoft Visual C++ 2015-2022 Redistributable (x86) (V14.44.35112.1) is not uninstalled and requires a manual uninstallation using the standard Microsoft Windows procedure.

During the uninstallation of EcoStruxure Automation Expert - Archive software, Microsoft Visual C++ 2015-2022 Redistributable (x64) (V14.44.35112.1) is not uninstalled and requires a manual uninstallation using the standard Microsoft Windows procedure.

NOTE: The installed Microsoft Windows Defender Firewall rule SE-PostgreSQL-Server-XX-EcoStruxure-Automation-Expert-Runtime is no longer removed while uninstalling EcoStruxure Automation Expert - Archive. If you want to remove your archived data after the uninstallation of EcoStruxure Automation Expert - Archive, you can also uninstall the corresponding PostgreSQL version, the pgAdmin 4 and the pgAgent, used for your Archive database. Then you can delete the Archive database and remove also the corresponding PostgreSQL firewall rule in Microsoft Windows Defender Firewall.
