The IP address of the device are synced in the Deploy and Diagnostic. No need of compilation for the address to be updated. In case of address or port conflict , an info icon is highlighted in the Deploy and Diagnostic.

Open Adapter Overview

Open Adapter is a flexible integration framework designed to simplify the connection and data exchange between diverse solutions. It enables seamless interoperability through straightforward configuration, significantly reducing the complexity of system integration.

Users can create their own adapters tailored to specific devices or systems. This promotes reusability and consistency, especially for frequently connected devices.

Leveraging previously validated adapters increases reliability and minimizes integration errors.

Simplified configuration and reusable components help lower the risk and cost associated with system integration.

Streamlined integration processes lead to faster development cycles and improved productivity.

Ideal for medium to large-scale projects, Open Adapter supports scalable architectures that grow with project needs.

OPCUA Client CONNECT Service Function Block

Migration notes for users of Function BlockOPCUA_CONNECTin versions prior to EcoStruxure Automation Expert 25.0, please refer to the F1 Help in version 25.0 for further information.

EcoStruxure Automation Expert 25.0 block enhancements are:

  • Function Block event inputs and outputs for CONNECT/DISCONNECT and CONNECTO/DISCONNECTO. Prior to version 25, a connection was activated and connected through the INIT event.  The user must now convert existing applications to use the INIT and CONNECT events.

  • Configurable connection and reconnection parameters in the OPCUA Client Configurator

  • Connection textual status that expands on the ERROR code: NetworkState, ProtocolState

Additional Events:

There are two additional input events: CONNECT and DISCONNECT and output events CONNECTO and DISCONNECTO.

CONNECT/CONNECTO:

Prior to version 25.0, an OPCUA Connection was initiated through the INIT event. INIT will perform initialization of the Function Block and the user must trigger the CONNECT event to begin activation and connection on a successfully initialized block. The CONNECTO is a confirmation event of CONNECT.

DISCONNECT/DISCONNECTO:

The user may trigger the DISCONNECT event on the block to disconnect an existing connection. Triggering this event will begin disconnection and fire the DISCONNECTO event.

Additional output parameters:

The following new output parameters will contain textual status of the ERROR code of the connection. 

NetworkState – textual information related to an error code of the network

ProtocolState – textual information related to an error code of the protocol

OPCUA Client configuration

The OPCUA Client configurator provides a new section for Connection parameters. These parameters allow the user to specify connection timeout, connection retry time, and connection retry count.  Prior to version 25.0, the connection timeout and connection retry time were a non-configurable 5 seconds and infinite reconnection was always enabled. The user can disable reconnection attempts by setting retry count to 0 and configure it for infinite retries by setting the retry count to 65535, the default. Connection timeout defaults to 10 seconds and Retry time defaults to 15 seconds.   

Advanced System Management - High Availability Disruption Avoidance (HADA)

Building on the capabilities of the System Management app, the Advanced System Management app introduces advanced features designed to automate and optimize maintenance operations like self-healing mechanisms for high availability system.

The High Availability Disruption Avoidance is the ability to maintain, at all time, the Edge Controller High Availability, by adding several spares parts controllers to allow self-heal from a controller failure .

When a failure is detected on the High-Availability pair, we minimize call-outs by selecting dynamically the right spare part and then restore the High Availability almost immediately.

This function requires the System Management Add-on available through EcoStruxure Automation Expert Platform.

The application adaptor allows integration of multiple EcoStruxure Automation Expert solutions enabling users to scale their plant applications. E.g. Users can combine as many solutions as needed to support the total Plant I/O requirement (individual controller & solution limits shall still be respected).

Value proposition & Key features:

  • Engage multiple system integrators to work in parallel to meet schedule objectives

Modbus TCP Client – Online Change

Online change allows engineers to update a running PLC program without stopping the system. This enables real-time logic modifications and dynamic management of Modbus TCP networks.

Key Features

  • Modify logic and parameters live.

  • Apply only the changes—no full program download needed.

  • Add, delete, or update Modbus TCP networks and slave devices on the fly.

Main Benefits

  1. System Availability – No downtime during updates.

  2. Minimal Interruption – Processes continue without disruption.

  3. Scalability – Easily expand or reconfigure networks.

  4. Efficient Maintenance – Real-time fixes and optimizations reduce planned downtime.

40k OPC-UA Server Tags – HA/Non-HA Soft dPAC

In your EcoStruxure Automation Expert application, the OPC UA Server supports a default maximum of 20,000 OPC UA variables.

If your application requires more variables, this limit can be extended to 40,000 variables by using a specific version of the SE.DPAC library, which is available upon request from Schneider Electric Support.
