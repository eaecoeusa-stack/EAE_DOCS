DPAC_FULLINIT Function Block

The DPAC_FULLINIT function block is used to initialized hardware and application function blocks.

To do so, it is transferring the initialization triggers from the E_RESTART function block of the resource to all the function blocks mapped to this resource containing a plcStart function block inside their logic.

NOTE: For information regarding DPAC_FULLINIT function block, refer to the SE.DPAC library documentation.

plcStart Function Block

As mentioned above, the plcStart function block is working along with a DPAC_FULLINIT function block.

This function block should be added inside the logic of a sub-system to transmit all the initialization triggers of the resource to the sub-system.

NOTE: Many function blocks in the Schneider Electric libraries do not provide an explicit initialize input event. It is because they use internally the plcStart function blocks to transmit to the logic the initialization triggers of the resource.

When adding a plcStart function block into a logic, check which priority to put so the initialization of that function block and the ones using it is done is to correct order.

NOTE: For information regarding plcStart function block, refer to the SE.AppBase library documentation.
