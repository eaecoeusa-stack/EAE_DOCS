Optimize Controller Selection

Select a controller that will maximize the events processing rate (per second). This results in an improved Application Response Time, because the overall queued events are processed faster.

Considerations

When targeting jitter reduction and Application Response Time improvement, the determining criteria for selecting a controller are, in order of importance:

  1. Single-core performance, which is driven by core frequency and the amount of cache(s). Usually the highest controller runtime load results from processing event threads at the resource level. A thread can be migrated by an OS scheduler between cores, but can execute only on a single core at a time. This means that the faster an event can be processed by a core, the faster the queue of events will be emptied.

  2. The number of cores, which helps by executing in parallel multiple resources (and related Bus Masters in the Hardware Configuration) as well as other runtime threads (for example, cross-communication, OPCUA, and so forth).

Be on the lookout for CPU over-heating. This can cause throttling to be applied at the hardware level (usually consisting in decreasing core frequency and/or disabling cores), which can add jitter in an unpredictable manner.

Select a controller so that in the normal execution of the EcoStruxure Automation Expertapplication, the CPU Load never exceeds 60% at core level(over a period of 1 second).
