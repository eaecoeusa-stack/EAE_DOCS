Check Log Files

Log files contain protocol information about the solution and its devices.

The log files identifies specific errors within the solution and the devices. Thus, use them to troubleshoot the issue faced.

In each device two log files are stored. Alternatively, each of these two log files is overwriting the other one to help prevent data loss. In addition, another file cyclically maintains record of detected errors.

Runtime Device and Cyclic Logs

To open the EcoRT log file:

  1. Open the Deploy and Diagnostic editor.

  2. Open the list of actions of the wanted device.

  3. Select Log file > Fetch Log Files.

The Device Log Viewer dialog box opens.

To switch to the cyclic log files:

  1. In the Device Log Viewer dialog box, select File.

  2. In the drop down list, select Load Log that opens the Open Log File dialog box.

  3. Click on Log Files and in the drop down list select Cyclic Log Files.

  4. Select the file available in the page. This text file finishes with .cyclic.

  5. Click on Open to finalize the switch to the cyclic log files.

EcoStruxure Automation Expert - Buildtime Logs

Find the EcoStruxure Automation Expert - Buildtime logs in the following pads:

  • Log View pad.

  • Message Log pad.

  • Output pad.

    NOTE: From the drop down list located at the top left of this pad, change between Build, Debug and the devices to get different log files.
