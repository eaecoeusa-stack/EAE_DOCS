User Data Type

Here are the naming conventions for the different user Data Type available:

  • Structure: Use the PascalCase naming convention with the lowercase 'str' at the beginning of the name as shown below:

    • strMotorData

    • strRecipeParms

  • Alias: Use the PascalCase naming convention with a lowercase 'a' at the beginning of the name as shown below:

    • aFrame

    • aSymbol

  • Enum: Use the PascalCase naming convention with a lowercase 'e' at the beginning of the name as shown below:

    • eProductType

    • eSelectAction

  • Array: Use the PascalCase naming convention with the lowercase 'arr' at the beginning of the name as shown below:

    • arrRecipeBuffer

    • arrSlot

  • Subrange: Use the PascalCase naming convention with the lowercase 'sr' at the beginning of the name as shown below:

    • srAnalogValue

    • srInputLimit
