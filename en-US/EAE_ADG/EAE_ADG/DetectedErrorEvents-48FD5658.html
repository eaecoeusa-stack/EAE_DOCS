Detected Error Events

Events Sent by Device Firmware

If a system error is detected, the firmware moves to Firmware Suspended (FW Suspended) state. The log file is updated and reboot CPU occurs:

Device Firmware Detected Error

Effect

A Task suspended

Log & reboot in Clean state

Processor exception

Log & reboot in Clean state

HW watch Dog

Reboot & log in Clean state

Events Sent by EcoRT

These detected errors are not system errors: they don’t stop the EcoRT process or send the firmware to FW Suspended state.

NOTE:

  • Only detected errors that halt the runtime stop the EcoRT process, e.g. segmentation fault or array out of bounds.

  • You can override process (and not halt the runtime) for the device by setting Runtime Configuration > Resource > C2FBException > StrictHandling to False.

EcoRT Detected Errors Include:

Effect

<LimitTime> ms without IDLE task.

If event execution does not become idle for <LimitTime> , EcoRT pauses for <PauseTime>.

If after pausing, the execution is still not idle for <HaltTime>, the device moves to ErrorHalt state.

Default <LimitTime> is zero, meaning the throttling pause is disabled by default. Default <PauseTime> is 20ms. This allows lower priority tasks a chance to be executed. The application simply makes a log entry here.

Default <HaltTime> is 2 seconds. The first 10s (default, configurable) after cold-start into RUNNING state is exempted.

Pause on event processing 20 ms.

Information EcoRT logs ErrorHalt state.

Event queue full.

EcoRT logs a message and drops the incoming event. EcoRT does not enter ErrorHalt state.
