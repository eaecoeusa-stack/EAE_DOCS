Impact of I/O Changes on CPU Load and Event Management

One aspect that impacts the CPU load is the I/O pattern, which is the percentage of I/Os changing in a specific period of time.

Thus, an increase in the I/O pattern leads to an higher impact on the CPU load than increasing the number of I/O itself. Thus, both I/O number and I/O pattern should be taken into consideration.

NOTE: Ignoring this can lead to Errorhalt on the CPU.

I/O Pattern Iteration Impact on the Operating System Selection for PC Based Solution

The I/O Pattern is key for an event driven system. If I/O pattern is:

  • 100% of I/O changing < 10 ms: It is recommended to use Soft dPAC Linux with PREEMPT_RT patch.

  • 100% of I/O changing < 50 ms: It is recommended to use Soft dPAC Linux.

  • 100% of I/O changing > 50 ms: Both Soft dPAC Windows or Linux can be used.

Impact of Varying Cross Communication Volume on CPU Load

Biggest impact on CPU load comes from variable change rate, followed by the number of variables.

Number of connections has low impact, but it increases slowly when the number of variables increases.

Impact of Values Exchanged with OPC UA Servers on CPU Load

Increasing the tag quantity may have high impact on CPU load,

CPU load increases between changing and idle is low, especially because of the low changing rate (500ms).

Impact of Varying Numbers of Connected Modbus TCP Servers on CPU Load

BusCycleTime and server quantity both have high impact on CPU load.

Variable quantity for fixed server quantity has linear impact on CPU load

Ethernet IP impact on CPU Load

It is better to estimate performance by total number of variables than number of servers.

The biggest impact on CPU load comes from RPI (exponential increase), followed by number of variables (linear increase).
