Check Integrity

When opening a solution, EcoStruxure Automation Expert checks the integrity of that solution.

If there are integrity errors, a pop up window opens asking if you want to resolve them or not. If selecting:

  • Yes: The integrity errors are resolved and the solution opens.

  • No: The integrity errors are ignored.

NOTE: This check integrity action can be initiated any time by right clicking on the project node of the Solution Explorer pad and selecting Check Integrity.

Here is a list of integrity errors that could be found during the integrity check:

  • Unused CAT Instances: A CAT instance is considered unused if it is not assigned to any system application. Thus, the check integrity lists all the unused CAT instances into the Solution Integrity dialog. Decide which one to keep and which one to delete.

  • Missing Files: A dialog box opens listing the missing files. When right clicking on an item, two options are listed:

    Options Description
    Copy File Path

    Copies the entire file path to the clipboard. Paste it in Windows folder to open the relevant folder and then you can copy the missing file(s) into that folder.

    Remove from Project

    Removes any references of the missing files from the project. Use this option if you do not need the missing files anymore.

  • Instance Conflict: A detected error message appears describing the instances having conflict. Select the respective instances and click on Repair to correct the detected error.

  • Missing Instances: If an instance is referenced in a project but is not found, a dialog box opens to select between creating the missing instances referenced or deleting the references of those instances.

  • Non-referenced Library Instances: EcoStruxure Automation Expert checks for CAT instances coming from non-referenced libraries. If non-referenced library references are found, a message displays listing the non-referenced instances. Click OK to close the message and then either remove the references or add the appropriate library. Then the solution cannot be compiled or deployed until this is completed.

  • Non-referenced CAT Instances: It is possible to delete non-referenced CAT instances if you do not want to use them. To do this, move the unused CAT instances from the Found section to the Delete section, then click Delete.

  • Deleted CAT Instances: On saving, if you delete some CAT Instances from networks (CAT, Application, Resource, and so on) coming from a non-referenced or missing installed library. Another dialog box then asks if you also want to remove corresponding configurations from the system database (System.cfg file).
