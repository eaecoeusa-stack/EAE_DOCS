Application Deployment

An application can be fully deployed (sent to devices) or it can be partially deployed (to deploy application changes that affect only a given resource).

Before a full deployment, each device must be cleaned first. This can be commanded explicitly each time or made automatic through the Clean before deploy property of the device (default is setting is FALSE).

NOTE:

  • The deployment on the devices is not necessarily successful for each device.

  • The deployment can be finalized after a different amount of time for each device.

  • There is no native time synchronization for the actual start of the application on the different devices.

Consequently, any requirement for time synchronization between any parts of the application execution must be managed within the application itself.

 WARNING
PARTIAL EXECUTION OF THE APPLICATION
Ensure that the application is designed to safely handle situations where a subset of devices is not operational.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

The deployment of an application doesn’t make it persistent after a device power cycle (it would be lost). Only an application stored in the device as a boot project is persistent and could run after a power cycle. If there is no stored boot project, the device will restart in Clean state.

 WARNING
AUTOMATIC APPLICATION START AFTER DEVICE REBOOT
Ensure that the application can safely start after a voluntary or involuntary device reboot.
Failure to follow these instructions can result in death, serious injury, or equipment damage.
