Availability Monitoring and Execution Synchronization

EcoStruxure Automation Expert allows users to create and manage IEC61499 based distributed systems and automatically generates the necessary communication links between remote (distributed) function blocks. But there is no native synchronization principle governing function blocks’ execution, though this may be required to ensure the control of the overall system.

The application execution can be impacted when:

  • Some controllers are inoperative (ErrorHalt state, communication issue, power off…), and unable to execute their part of the distributed application.

  • Some distributed parts of the application are not ready for execution in due time (especially after an application deployment or a device power cycle).

Special application design patterns can be implemented at the application level so that the global process only starts when all participants are up and ready, and stops, or goes into a specific state, when one or more controllers become inoperative.

 WARNING
PARTIAL EXECUTION OF THE APPLICATION
Ensure that the application is designed to safely handle situations where a subset of devices is not operational.
