Error Halt Causes and Inspection Methods

Error halt can be caused due to boot Project

If the boot project causes an execution error, the running device goes to ErrorHalt state after each attempt to execute the endless loop.

This could happened if:

  • One of the following command is triggered: Load Boot Project, Restart Runtime, Reboot Device

  • And the parameters trigger the boot project after these commands execution.

To avoid such loop, perform these actions in the Deploy and Diagnostics editor:

  1. Boot Project > Delete Boot Project (the boot project causing the error loop).

  2. Deploy > Clean.

  3. Deploy > Deploy.

Segmentation Fault and arithmetic errors

The following examples will stop the EcoRT process: Segmentation fault, array or string out of bounds or division by zero.

See these errors on the cyclic log file. It displays the name of the function block that triggered the error. To help prevent such event, all variables must have permission to access memory by assigning the right size to each variable.

Out of Memory and High CPU Load

The device also moves to ErrorHalt state if:

  • Maximum allowed memory is exceeded: When the data being accessed or processed does not align with the allocated device memory.

    To manage this issue, redistribute the application to different devices or optimize the logic.

  • High CPU load: It potentially causes the system to become unresponsive.

    To investigate this issue, monitor the system resource usage. Then Identify the processes consuming excessive CPU resources and check in the log files, with Fetch Log Files command in the Deploy and Diagnostics editor, if there is no event chain with infinite loop, by checking which function blocks have been fired which event. Finally, optimize the logic or processes contributing to the high load.

    NOTE: To identify if the instance of the Soft dPAC is having high CPU load, open the task manager of the engineering station
NOTE: For watch and forcing variables, refer to the topic Observing Event Flows and Event Counts.
