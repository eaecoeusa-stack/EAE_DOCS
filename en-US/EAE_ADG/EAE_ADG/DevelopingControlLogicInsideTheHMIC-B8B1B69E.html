Developing Control Logic Inside the HMI

The HMI Runtime utilizes the .NET Framework based on C#, it is primarily designed for displaying and supervising processes.

This .NET Framework based on C# can be used to develop control within the HMI, canvas or symbol. This logic would then be executed only when the concerned HMI, canvas or symbol is open on the HMI device.

For seamless control functionalities, it is recommended to separate the control logic from the HMI and to put the minimal control logic inside the HMI, canvas or symbol.

By doing so, the control logic is implemented at EcoStruxure Automation Expert - Runtime level and supervised through the HMI, which keeps the operation consistent and continuous.
