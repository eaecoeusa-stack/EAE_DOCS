EVENTCHAINHEAD and EVENTCHAIN Function Blocks

The EVENTCHAINHEAD function block can be used in a given resource initialization function block to trigger all EVENTCHAIN function blocks in the CATs or Composites of that resource that have the same string set in their NAME input data and the same boolean set in their INVERT input data.

If all resources of the system own an instance of EVENTCHAINHEAD, then the CAT instances can be mapped and moved anywhere without the need to re-link them.

Function Blocks with INIT and INITO Pins

The INIT input event and INITO output event, available in function blocks and in many CATs and Composites, can be used to chain the initialization of all logical function blocks.

The initialization chain is created by linking the INITO output event of the upstream function block to the INIT input event of the downstream function block.

This is the simplest way to initialize the logic in the resources, but it has the following drawbacks:

  • It requires function blocks having an INIT input event.

  • It can result in a long initialization chain.

  • It requires the function blocks to be already mapped to a resource.

  • Any re-mapping requires re-linking the function blocks.

NOTE: If you use a link between FB1.INITO and FB2.INIT, verify that FB1 and FB2 are part of the same resource.
