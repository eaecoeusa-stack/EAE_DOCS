ErrorHalt Prompts and Mitigation

When the ErrorHalt state occurs, the controller is idle. The controller does not execute the application and report the detected error.

From the ErrorHalt state, three actions can be executed:

  • Clean

  • Restart Runtime

  • Reboot Device

Behavior on Reboot set to Run or Stop:

After the command:

  • Restart Runtime: The state of the device is determined by the value set to the logical device parameter Runtime Configuration > Deploy Settings > Auto start after deploy: True or False.

  • Reboot Device: The state of the device is determined by the value set to the logical device parameter Runtime Configuration > Boot > Behavior on reboot parameter setting: Run, Stop, LastState or LastStateIgnoreErr.

    NOTE: To prevent reproducing the error that moved the state to ErrorHalt, it is recommended to set the parameter to Stop.
NOTE: Find these two parameters in the Properties pad after having click on the wanted device in the Logical Devices editor.
