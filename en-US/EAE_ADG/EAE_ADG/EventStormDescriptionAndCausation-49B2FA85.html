Event Storm Description and Causation

Event storms are characterized by an excessive count of events stacking-up in the external or internal Event Queue while the CPU load is too high for the controller to process them in a timely manner. Typical consequences include the delay of tasks execution, including cyclic communication with I/O, which can lead to perturbations in the driven process.

 WARNING
DELAY IN TASKS EXECUTION
Test the application under operating stress conditions to ensure the system is capable of managing peak number of events (‘event storms’) without performance degradation.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Typical symptoms of event storms:

  • Apparent delay in process execution.

  • Log message showing event queue overwhelming.

  • Glitches on I/O going to fallback.

  • Loss of cross communication or delays.

  • Loss of events.

  • Device going to Error Halt state (too many events).

Typical Causes of Event Storms

The main causes for an excessive count of generated events usually arise from the way the application is written. Therefore, careful examination, understanding and design of the application should help prevent or solve such issues.

Application design flaws often result in unnecessary events ‘multiplication’: a first set of events triggers a cascading effect of creating more events, quickly resulting in an event storm that temporarily saturates the system processing capabilities.
