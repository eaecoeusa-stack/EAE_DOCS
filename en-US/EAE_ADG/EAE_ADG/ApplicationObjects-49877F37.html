Application Objects

An EcoStruxure Automation Expert application is made of several object types, including:

  • CAT instances

  • Composite instances

  • Basic function block instances

  • Service function block instances

  • Adapters (connections between function blocks)

There are two kinds of CAT objects:

  • Application CAT: Object containing embedded objects with all their facets. The instance is an atomic object which cannot be split between resources; the entire object logic and all the embedded object’s logic will run in the same resource.

  • Hardware CAT: same as application CAT but dedicated to the hardware or communication handling (configuration, I/Os). These kinds of CATs are linked to the configuration of your hardware and as such are not movable from one device to another device.

CATs can be implemented as:

  • Composites: standard composition of logic blocks and their graphic representation.

  • SubApp: Not so different from a composite CAT but can be split so that embedded instances may be deployed on different resources.

    NOTE: This SubApp configuration is only available for application CATs.
