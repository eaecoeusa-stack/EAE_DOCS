HMI Design Recommendation

In EcoStruxure Automation Expert, different types of HMI graphics are available:

  1. Symbol: It is the graphic representation of a CAT. Once added on a canvas and displayed online, the symbols dynamically inform about the CAT and control it as the CAT I/Os are connected to the symbol. In addition, faceplate can be associated to symbol.

  2. Faceplate: It is a dialog box linked to a symbol enabling to access to additional adjustment or functionalities of the CAT. Faceplates may also contain additional graphical elements for controlling and operating.

  3. Canvas: It is the pages that are displayed on the HMI runtime. Create a canvas under the Canvases node in the Solution Explorer pad. It is used to visualize dynamically the CAT symbols as the symbols can react to their CAT I/O values. In addition, some static tools allows to personalize the page (lines, rectangles...).

  4. Graphic: It is reusable static graphic. Create a graphic under the Graphics node in the Solution Explorer pad. Three different types of graphic is available:

    1. Graphic: Used and reused on canvases.

    2. Dialog: Used as a faceplate.

    3. Customer Class: Is a user defined graphical object written in C# and permitting continuous code execution inside HMI runtime. However, they cannot have direct access to object data.

NOTE: It is recommended to develop the HMI graphics of applications inside their CAT symbols instead of developing them directly in the canvas. This way, the developed HMI views generate instance once added in the canvas that can be sized, reused and exported.
