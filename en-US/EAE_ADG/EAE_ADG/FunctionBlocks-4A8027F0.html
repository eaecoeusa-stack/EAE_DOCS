Function Blocks

The following function blocks can be used within the application to monitor the load on system resources, detect momentary peaks and identify their causes:

MIBGET

This function block can be used to get detailed information about EcoRT execution that can help identifying event storm related issues and their causes.

Topic

Name

Description

Event chain

Resource.Resources.EVENTCHAINMaxLength

Maximum length of the event chain since the start of the resource.

Resource.Resources.EVENTCHAINMaxTime

Maximum time (in microseconds) of the event chain in the last minute.

External queue

Resource.Resources.EventQueueMaxUsed

Maximum number of the events in the queue since the start of the resource.

Total number of the events lost since the start of the resource.

Resource.Resources.ThrottleCountTotal

Total number the event processing was throttled due to the overload (too many events without an idle period).

System.PhysMemUsed

Used physical memory in KB.

CPU

CPU Load in tenths of percent.

CPUTICK

Connected at the beginning and end of the event chain, it gives the duration of the processing of the chain of events.

For example: if you want to measure the time between the request to write a value in setVal and the notification of the change in getVal you can use a CPUTICK function block:

  1. Connect the tick.START event with the setval.CNF (write done).

  2. Connect the tick.REQ with the getval.CNF (received).

  3. Trigger the setval.REQ and it automatically returns the time spent.

NOTE: For more detail about these two function blocks, refer to their specific description into their library documentation.
