Online Change

After an Online Change, the application may generate too many events, thereby degrading overall performance, if improper initialization links are established between function blocks.

This is especially true with the E_RESTART function block:

  • Avoid adding other E_RESTART function blocks to the one automatically instantiated in each resource, because they can generate an excessive number of events during startup and online change phases.

  • The ONLINECHANGE output event of E_RESTART function block:

    • Must only be used to trigger simple logic or events flow that can be executed quickly.

    • Do not use it to trigger any function block requiring a long initialization sequence.

    • Do not use it to trigger a DPAC_FULLINIT.INIT event, because this would re-initialize the entire system after online change, restart all I/O systems, and restart the application.

 WARNING
RISK OF UNINTENDED OPERATION
Do not connect the E_RESTART ONLINECHANGE output to a DPAC_FULLINIT.INIT event.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Instead, in order to initialize and automatically re-start the newly added application code after online change, consider:

  • Adding a new plcStart function block connected to the INIT of the newly added code.

  • Using the DPAC_FULLINIT function block, together with application libraries that natively contain logic to manage operating modes during startup and online change.

Symlink Online Change

Use the enableSymlinkOC parameter (in the Hardware Configuration editor) to enable and disable online change, as follows:

  • True enables online change of the mapping between hardware Symlinks and application Symlinks for the selected bus master.

  • False disables online change of this mapping.

Disabling Symlink online change reduces the chance of rough or uneven online change, which is more likely to occur as the number of a bus master's child Hardware CATs increases.
