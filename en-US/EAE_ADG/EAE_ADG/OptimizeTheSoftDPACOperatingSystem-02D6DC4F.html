Optimize the Soft dPAC Operating System

To improve OS responsiveness, and limit runtime disturbances:

  • Use a low-latency kernel is for applications that require a high degree of determinism.

  • Execute a bash script during the installation procedure to tune the OS configuration (the OS that hosts Docker daemon and runs the Docker images). Design the script to tune multiple settings that impact OS behavior, including:

    • Disable swap: if the amount of host RAM is small, enabling swap can lead to an unresponsive or hung system.

    • Tune kernel throttling to limit jitter on real-time tasks under high CPU load: a faulty real-time task with high-priority could affect system operations by preventing non-real time tasks (like a SSH server) from properly executing.

    • Set the CPU governor to performance for each core. This will lead to a more deterministic response time as the core frequency does not vary.

    • Increase the execution priority of more critical kernel threads. Execution of higher priority tasks will preempt other tasks with lower priority.

    • Disable irqbalance to improve network determinism.

      NOTE: This may impact performance in the event of an IRQ storm (i.e., a CPU burst at core level).

    • Disable automated OS updates. The system can still be updated manually.

    • Disable non-vital services and CRONs for controllers and for accessing the network. This will limit disturbances on field bus communications but can affect ordinary computer usage.
