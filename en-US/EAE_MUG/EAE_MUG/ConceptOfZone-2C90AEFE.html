Multi-User Zone Feature

The Multi-User zone is a feature of EcoStruxure Automation Expert. A zone is a part of a Distributed Control Application, that is dependent enough to let someone make edits without impacting significantly the other colleagues.

A zone can be any of these elements:

  • The entire Distributed Control Application

  • The System

  • An application or one of its layers

  • An application HMI property (Named Styles, Themes, Multilanguage dictionaries or General HMI Properties)

  • A device or one of its resources

  • An application CAT or a hardware CAT, its HMI CAT, one of its faceplates or one of its symbols

  • A function block (a SubApp, a composite, a basic or an adapter)

  • A canvases

  • A graphic

  • A data type

  • A function

By default all the zones are in read-only for all the users*. You then must reserve a specific zone to do edit in it. That zone then becomes reserved by you, which means that you can edit it and it remains in read-only for all the other users.

NOTE: To know how to reserve a zone, refer to the Reserve a Zone step of the recommended Multi-User workflow.

(*) Except the layers and resources that can be edited without reservation. The reservation is done automatically when starting to edit.

Additional zone behaviors:

  • A zone can remain reserved as long as you want. You must release it by yourself to finish the reservation.

    NOTE: To know how to release a zone, refer to the Release the Zone step of the recommended Multi-User workflow.

  • All users reservations are visible in the Reservation Overview dialog box.

  • A message appears after each reservation request to inform you about the success or the failure reason.

  • When reserving a zone, all sub-zones are also reserved.

    NOTE: If a sub-zones is already reserved by another user, the current-zone cannot be reserved.

  • You can reserve as many zone as you want, however it might impacts other users.

    NOTE: If you reserve a lot of zones, you might consider instead to reserve the parent-zone to include all of them.

  • The User Management button in the View tab is available only if the entire Distributed Control Application is reserved.

  • Users must be connected to the server hosting EcoStruxure Automation Expert Platform to perform a reservation. Without a confirmation from the server, the zone will not be reserved. If the connection is lost, the Distributed Control Application closes and the reservations are retained for the reserved zones.

    To recover the editing capacity, follow the next steps:

    1. Close EcoStruxure Automation Expert - Buildtime.

    2. Establish anew network connection.

    3. Open again EcoStruxure Automation Expert - Buildtime from EcoStruxure Automation Expert Platform.

This zone feature then helps to avoid risks of conflict in a Multi-User environment as only one user can edit a zone at the time. If another user wants to perform edit, they must request the user with the current reservation to release the zone and its dependencies to perform edit.

NOTE: The user must perform this request on its own as there is no dedicated functionality available. The users might communicate together via any preferred method of communication.
