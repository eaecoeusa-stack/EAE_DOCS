Explicit messaging with external EtherNet/IP controller, using the parameters' CIP Path

Overview

  • The ATV dPAC hosts a dedicated EtherNet/IP explicit communication server, enabling access to Altivar drive parameters. While this server does not allow direct access to IEC 61499 application variables, communication is possible through ATV Exchange parameters.

  • Network connectivity: direct connection to ATV dPAC module.

  • The external EtherNet/IP controller connects directly to the Ethernet port of the ATV dPAC.

  • Through this connection, the controller can access the following parameters:

    • Mailbox exchange parameters: [DEP•] and [DSP•] (• from 0 to J) via their CIP Path.

    • Altivar drive parameters via their CIP Path.

Setup Steps

  1. Connect the external EtherNet/IP controller to the Ethernet port of the ATV dPAC.

  2. The external EtherNet/IP client can explicitly read and write:

    • Mailbox exchange parameters: [DEP•] and [DSP•] (• from 0 to J)

      .

    • Any Altivar drive parameter.

    Refer to the CIP Path information in the:

    • ATV340 Communication parameters addresses.

    • ATV600 Communication parameters addresses.

    • ATV900 Communication Parameters Addresses.

  3. The ATV dPAC application can also read and write these parameters internally using:

    • ATVCyclicScanner (Hardware CAT)

    • ATV_DD_IN and ATV_DD_OUT function blocks

  4. This setup allows the ATV dPAC application-specific data to be exposed via the EtherNet/IP explicit communication server

NOTE:

Refer to Drive Parameters Mailbox, for additional information on:

  • Non-Persistent Exchange Parameters Management [DEP•] (• from 0 to J).

  • Persistent Exchange Parameters Management [DSP•] (• from 0 to J).
