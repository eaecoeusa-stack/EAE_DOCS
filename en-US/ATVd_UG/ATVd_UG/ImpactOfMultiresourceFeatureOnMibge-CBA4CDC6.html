Impact of Multiresource Feature on Mibget Output Format

When using the Multiresource feature on a device, the output format of the Mibget block changes for paths starting with "Resource.Resources.". This change can break existing implementations that assume a single-value output.

What changes?

  • Before Multiresource: The output is a single value string, such as "154".

  • With Multiresource enabled: The output becomes a comma-separated string representing values for each resource, such as "142,10,2" for Res0, Res1, and Res2.

Risk of Runtime Errors

If your solution attempts to directly convert the output string to an integer (e.g., using string → int conversion), it will fail at runtime when the output contains commas. This can lead to a runtime error halt in the dPAC.

Condition for the Change

  • Only when multiple resources are configured on the device.

  • Only for paths under "Resource.Resources.".
