Implicit messaging with external EtherNet/IP controller, using 100/101 customized assembly

Overview

  • The ATV dPAC does not host an EtherNet/IP implicit communication server. The drive's embedded EtherNet/IP adapter enables access to Altivar drive parameters. While this server does not allow direct access to IEC 61499 application variables, communication is possible through ATV Exchange parameters.

  • Network connectivity: Connection to the embedded Ethernet port of the drive. No direct connection with the ATV dPAC module.

  • The ATV dPAC module does not natively support EtherNet/IP assemblies.

  • when the ATV dPAC is hosted by ATV900 drives or ATV340•••••E drives (≥ 30 kW). Through this connection, the controller can access:

    ◦ Mailbox exchange parameters: [DEP•] and [DSP•] (•from 0 to J), when configured in the drive's 100/101 CIP assembly.

Setup Steps

  1. Connect the external EtherNet/IP controller to the embedded Ethernet port of the drive (not the dPAC).

  2. Customize the 100/101 assembly in the drive’s DTM (via SoMove software) to include mailbox parameters [DEP•] (• from 1 to J).
    NOTE: The drives require a firmware version 4.3 or later for ATV900 or ATV340•••••E drives.

  3. Configure the EtherNet/IP scanner in the controller to match the customized 100/101 assemblies.

  4. The ATV dPAC application can access these parameters internally using:

    • ATVCyclicScanner (Hardware CAT)

    • ATV_DD_IN and ATV_DD_OUT function blocks

Result: This setup allows the ATV dPAC’s application data to be shared through the drive’s implicit EtherNet/IP adapter.
