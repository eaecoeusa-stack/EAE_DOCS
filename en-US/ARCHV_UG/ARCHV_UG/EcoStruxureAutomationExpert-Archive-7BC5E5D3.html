EcoStruxure Automation Expert - Archive Installation

The EcoStruxure Automation Expert - Archive installation requires administration rights.

Hibernation mode must be turned off on the PC before launching the installation.

During the installation of EcoStruxure Automation Expert - Archive, the following information is displayed and requested:

  • A PostgreSQL version must be selected from the detected compatible PostgreSQL versions available on the PC. The selected version is used for the creation of the archive database template, named searchive.

  • The data directory of the selected PostgreSQL version, where your archive database is stored, is displayed.

  • The password of the database superuser (postgres) is requested to verify if your archive database is created properly. This is the password set during the installation of the selected PostgreSQL version.

    NOTE: If you forget the password, it can be reset without deleting the content of the database by following the instructions on Reset Forgotten Password for Super User (postgres).

  • The port number, configured for the PostgreSQL database server of the selected PostgreSQL version, is displayed.

    NOTE: This port number is stored in the postgresql.conf file, available in the data directory of the selected PostgreSQL version. The same port number must be configured also inside the EcoStruxure Automation Expert solution.

  • Passwords for the required user roles searchive and the seservice are requested for the selected PostgreSQL version.

    NOTE: Starting with EcoStruxure Automation Expert - Archive V24.0 there are new password requirements introduced and the passwords are encrypted with scram-sha-256 algorithm to fulfill standard security requirements. In earlier versions, default passwords were used (identical to the user names) and encrypted with unsecure MD5 algorithm. The password encryption is configured in the pg_hba.conf file.

  • The checkbox on the Archive Database Partition Table Backup dialog is selected and the path to the backup folder is displayed.

    NOTE: By default, the data of the Data tables from the postgres is deleted by pgagent to save space. If you select the checkbox, backups are created before deleting the tables. The selection is applied to databases, created after the setting is configured, and not to existing databases. The value of the backup setting is visible in the Properties table of the searchive database in pgAdmin 4, under the TablePartitioningExportBeforeDrop property. If the value is 1, then backup is activated, and if the value is 0, the backup is turned off.
  1. Launch the EcoStruxure Automation Expert - Runtime installer and click Next to start the installation.
  2. If you install EcoStruxure Automation Expert - Archive for the first time, click Next. Otherwise, in the Program Maintenance dialog, select Modify and click Next.
  3. In the list of features in the Custom Setup dialog, select EcoStruxure Automation Expert - Archive and click Next.
  4. In the PostgreSQL Database dialog, select the version of PostgreSQL that you want to use for your archive databases and click Next.
    NOTE: If no compatible PostgreSQL version is detected, the installation is cancelled.
  5. In the Archive Database Information dialog, enter the password of the database superuser (postgres), set during the installation of PostgreSQL, and click Next.
    NOTE: If the configured PostgreSQL password encryption is still the unsecure MD5 algorithm there is shown a question if it should be updated to scram-sha-256 algorithm. Click Yes to update the configuration in the pg_hba.conf file or No if you want to abort the installation to e.g. check this configuration first and make a backup of this configuration file.
  6. In the Archive Database User Passwords dialog enter the passwords of the required user roles searchive and seservice and click Next.
    NOTE: Remember this password as they are required also at configuration of the Archive in your EcoStruxure Automation Expert solution.
  7. Optionally, in the Archive Database Partition Table Backup dialog, select the checkbox if you want pgAgent to backup the properties tables of your databases before deleting, then click Next.
  8. Click Install, then click Finish when the installation is completed.
Result:

  • The Archive template database named searchive and users named searchive and seservice are created, if not available from a previous installation.

  • The password encryption in the pg_hba.conf file is replaced by scram-sha-256 if it was encrypted before with MD5 algorithm.

  • Local Windows Defender firewall rules are created.

    NOTE: If you are using another firewall software, you must configure the rules described in Computer Firewall of the system requirements.

  • The following software is installed on your PC:

    • Microsoft Visual C++ 2015-2022 Redistributable 14.44.35112 (x64)

    • Archive Manager V2x.x, a software tool that lets you:

      • View the status of the archive service.

      • View and manage archive link connections.

      NOTE: When you launch the Archive Manager tool with permissions of standard users via the Windows Start menu, the User Access Control (UAC) dialog pop up appears. An administrator needs to enter the credentials to launch the tool. If you are logged in with administrator privileges, you don’t see this UAC message. If Archive Manager is launched and running in the background, it can be accessed from the Windows tray area.
