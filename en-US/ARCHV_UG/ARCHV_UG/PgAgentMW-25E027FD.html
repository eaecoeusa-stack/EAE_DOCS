pgAgent Configuration

  1. In the pgpass.conf file, located in C:\Users\(username)\AppData\Roaming\postgresql, add the following line, containing a wildcard character (*), if the database name is not available:
    hostname:port:database:username:password
    Example: localhost:5432:*:postgres:YourPassword
    NOTE:

    • This step is needed so that later you can execute the pgAgent jobs for your databases.

    • The C:\Users\(username) is the name given to the Windows system user in step 8 of the pgAgent installation.

    For more information refer to https://www.postgresql.org/docs/current/libpq-pgpass.html

    To test or troubleshoot the pgAgent configuration. the following steps can also be executed.

  2. Open pgAdmin 4 and enter the master password, created during its configuration.
    NOTE: This step may be optional, depending on the specific pgAdmin configuration.
  3. In the left panel, click Servers > PostgreSQL XX > Databases(1).
    NOTE: XX is the installed PostgreSQL major version, used for your Archive database.
  4. Right-click one of the pgAgent Jobs created for any Archive database, then click Run now.
  5. Click the pgAgent job, then click the Statistics tab.
    Result:
    The Status is S: the pgpass.conf file is configured correctly and the pgAgent job is executed successfully.
    The Status is F: the configuration of the pgpass.conf file or the job definition must be verified.
  6. Click the used PostgreSQL version in the left panel, then click the Properties tab to display the port number used to connect or the path to the data directory.
