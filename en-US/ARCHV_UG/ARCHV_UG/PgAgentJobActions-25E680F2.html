Configuring Backup for Disk Space Management

For each Archive database created in PostgreSQL, a pgAgent job is created to:

  • Backup tables data before delete.

  • Delete unused tables.

The pgAgent jobs are configured to run at 4:00 am every day and to perform their tasks, based on the configuration of the Properties table.

You can modify the values of the properties in the Properties table to customize the job actions.

If you modify the Properties table of the searchive database, the modification is also applied to any database created afterwards.

If you modify the Properties table of one of your other databases, the modification is only applied to the modified database.


To modify the properties of the Properties table:

  1. Launch pgAdmin 4 and enter the password created during its configuration.
  2. In the left panel, click Servers > PostgreSQL XX > Databases.
    NOTE: For PostgreSQL XX, XX is the installed PostgreSQL major version, used for your Archive database.
  3. In the left panel, click the database to be modified (for example, searchive) and in the database hierarchy, click Schemas > Public > Tables > Properties.
  4. Right-click the Properties table, select View/Edit data and click All rows.
    Or
    Run the following command in the query tool:
    SELECT * FROM public."Properties"
  5. Click the Data Output tab, then double-click the property value to be modified.
  6. Type the new property value and click OK.
  7. Save the modifications.

Property

Description

Supported Values

TablePartitioningDropOlderInterval

After every interval specified by the TablePartitioningInterval property, the pgAgent job creates a new partition table. Sometimes, a big number of tables is created. The TablePartitioningDropOlderInterval property can then be used to configure the interval to delete the tables that are no longer in use. The unused partition tables are deleted, based on the interval specified in this property. The latest partition tables are not deleted until the creation of new partition tables, even if they are older than the specified interval.

For example, if you want to delete the tables that are created 10 days earlier, set the property value of TablePartitioningDropOlderInterval to 10 days.

  • N days

  • N weeks

  • N months

  • N years

TablePartitioningInterval

New partition table is created, based on the interval specified by the TablePartitioningInterval property and the TablePartitioningIntervalCount property together.

For example, if you want tables to be created every month, set the TablePartitioningInterval value to month and the TablePartitioningIntervalCount value to 1.

For TablePartitioningInterval:

  • day

  • week

  • month

  • year

TablePartitioningIntervalCount

For TablePartitioningIntervalCount:

  • Unsigned integers from 1 to 2147483647.

TablePartitioningExportBeforeDrop

This property specifies if the tables are to be exported or not for a backup before being deleted to save space. This property can be set during the EcoStruxure Automation Expert - Archive installation, or in pgAdmin 4.

If the value of this property is set to 1, the TablePartitioningExportPath property must also be set in order to export. The archived data is exported as a .bak.bin file.

When the property is set during the EcoStruxure Automation Expert - Archive installation, the value and the path to the backup folder are filed automatically in pgAdmin 4.

  • 0 for false — no backup is created.

  • 1 for true — backup is created.

TablePartitioningExportPath

This property specifies the full path to the folder where the table data backup is exported (for example, C:/Users/Public/Archive/Backup/ for Windows, or /archive/backup/ for Ubuntu and Debian).

This folder must be accessible for non-administrator users.

If you want to import and restore the exported archived data, complete the following steps:

  1. Launch pgAdmin 4 and enter the password created during its configuration.
    NOTE: This step may be optional, depending on the specific pgAdmin configuration.

  2. In the left panel, click Servers > PostgreSQL XX > Databases.

    NOTE: For PostgreSQL XX, XX is the installed PostgreSQL major version, used for your Archive database.

  3. In the left panel, click the database to be modified.

  4. In the database hierarchy, click Public > Schema > Tables, then right-click the Properties table and click Import/Export Data.

  5. Click Import next to Import/Export.

  6. Enter or select the path to the exported file.

  7. Set the Format to binary.

  8. Click OK.
