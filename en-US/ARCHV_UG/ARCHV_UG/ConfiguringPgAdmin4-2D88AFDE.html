pgAdmin 4 Configuration

  1. Install the latest available version of pgAdmin 4 from https://www.pgadmin.org/download/pgadmin-4-apt/, following the instructions provided on the webpage.
  2. Open pgAdmin 4 and if required by the tool, set a password to be used when accessing pgAdmin 4.
  3. In the left panel, click Servers > PostgreSQL XX > Databases(1), and enter the password of the database superuser (postgres), set during the installation of PostgreSQL.
    NOTE: XX is the installed PostgreSQL major version, used for your Archive database.
    NOTE: If the server node is not available under Servers:

    • Right-click Servers, select Register and click Server.

    • In the General tab, enter the name of the server (for example, PostgreSQL 14).

    • In the Connection tab, enter your IP address in Host name/address (for example, 127.0.0.1), and enter the password of the database superuser (postgres), set during the installation of PostgreSQL.

    • Click Save > Close.

  4. In the left panel, right-click the postgres database and click Query Tool.
  5. Copy and run the following command in the query tool, to verify the pgAgent schema version:
    SELECT pgagent.pgagent_schema_version()
  6. If the value of the schema version is 3, copy and run the following command in the query tool:
    CREATE OR REPLACE FUNCTION pgagent.pgagent_schema_version()
    RETURNS int2 LANGUAGE 'plpgsql'
    VOLATILE AS $BODY$
    BEGIN
    RETURN 4;
    END;
    $BODY$
    NOTE: Create a backup of the PostgreSQL database before you modify the schema version. For instructions on how to backup and restore a PostgreSQL database, refer to https://www.postgresql.org/docs/14/backup.html.
