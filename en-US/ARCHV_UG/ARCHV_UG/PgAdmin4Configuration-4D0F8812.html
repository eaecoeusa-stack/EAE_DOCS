pgAdmin 4 Configuration

  1. Open pgAdmin 4 and, if required by the tool, set a master password to be used when accessing pgAdmin 4.
    NOTE: This step may be optional, depending on the specific pgAdmin configuration.
  2. In the left panel, click Servers > PostgreSQL XX > Databases, and enter the password of the database superuser (postgres), set during the installation of PostgreSQL.
    NOTE: XX is the installed PostgreSQL major version, used for your Archive database.
  3. In the left panel, right-click the postgres database and click Query Tool.
  4. Copy and run the following command in the query tool, to verify the pgAgent schema version:
    SELECT pgagent.pgagent_schema_version()
  5. If the value of the schema version is 3, copy and run the following command in the query tool:
    CREATE OR REPLACE FUNCTION pgagent.pgagent_schema_version()
    RETURNS int2 LANGUAGE 'plpgsql'
    VOLATILE AS $BODY$
    BEGIN
    RETURN 4;
    END;
    $BODY$
    NOTE: Create a backup of the PostgreSQL database before you modify the schema version. For instructions on how to backup and restore a PostgreSQL database, refer to https://www.postgresql.org/docs/15/backup.html.
