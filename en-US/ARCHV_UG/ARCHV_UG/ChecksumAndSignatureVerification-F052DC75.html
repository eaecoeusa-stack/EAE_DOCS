Signature Verification

Verify Signature

The Archive installation file (.deb) is delivered with a digital signature file, for example:

  • eae-archive-v24.0.24192.04_amd.deb (Archive installation (.deb) file)

  • eae-archive-v24.0.24192.04_amd.deb.sig (Archive signature (.sig) file)

  • eae-archive-v24.0.24192.04_arm.deb (Archive installation (.deb) file)

  • eae-archive-v24.0.24192.04_arm.deb.sig (Archive signature (.sig) file)

The signature file lets you verify the authenticity and integrity of the installation image file.

NOTE: When downloading SE_GPG.asc key file from Schneider Electric website, the host system is required to connect to the Internet. Exposing a product to the Internet is a risk. It is recommended to connect the host system to the Internet via a secured network only while the key file is being downloaded.

To verify the signature:

  1. Download the Schneider Electric public key (SE_GPG.asc) from the Schneider Electric web site:
    https://www.se.com/ww/en/work/support/cybersecurity/pki-for-firmware.jsp

  2. Open a Linux terminal console window and navigate to the directory where the SE_GPG.asc file is located.

  3. Use the gpg tool to import the key using the following command:
    $ gpg --import SE_GPG.asc

    NOTE:

    • The gpg tool is included by default in most Linux distributions. If it is not included by default, you will need to install it before performing step 2.

    • To verify the key is imported, execute the command:
      $ gpg --list-keys

  4. Verify the Archive installation file:

    1. Navigate to the directory where the Archive .deb package and signature file are located.

    2. Execute the command:

      $ gpg --verify [archive signature file] [archive installation deb file]

      For example:

      $ gpg –verify eae-archive_24.0.24192.04_amd64.deb.sig eae-archive_24.0.24192.04_amd64.deb

    The output of the above command should read: “Good Signature from Schneider Electric GPG Code Signing”.

    NOTE: If there is no gpg web of trust or public keys on the host machine, the following message appears: “This key is not certified with a trusted signature!”.
