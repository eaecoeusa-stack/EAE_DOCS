Creating and Configuring CAT_HMI Input Variables for Archiving

Only values of the input variables of the iThis (CAT_HMI) function block of the CAT nodes can be archived. When configuring a variable for archiving, you can determine which variable values should be archived. To do this, specify a minimum time between samples and a minimum difference between the last archived value and the next value to be archived.

You can also control the level where the decision to archive the values of a variable is made — the variable type definition level, or the variable instance level.

  1. In the Solution Explorer pad, select the project node, then click CAT > Application.
  2. Expand a CAT node of your choice to configure for archiving, and double-click the IThis : X_HMI node.
    Result: The X_HMI editor opens.
  3. In the Interface editor of the X_HMI, under InputVars, click <new variable>, then enter a Name (for example, Input_Archive) and select a variable type (for example, INT).
    Result: A new input variable is created.
  4. Click the Attr cell of the new variable, then click the ellipsis (...) to open the Attributes dialog.
  5. Select the TrendingAndArchive attribute and click the Value column, then click the ellipsis (...) to open the Historization settings dialog.
  6. In the Historization settings dialog, configure the variable sampling and click OK. For more information on how to configure these settings, refer to Inserting Trend into a CAT_HMI in the EcoStruxure Automation Expert User Manual.
  7. After saving the newly configured input variable, associate it with an input event.
    NOTE: For an example of how to associate a variable with an event, refer to Creating a Connection Between an OPC UA Server and an OPC UA Client in the EcoStruxure Automation Expert User Manual.
  8. Double-click the CAT node of your choice, and open the Function Blocks Network editor, then in the Solution Explorer navigation tree, select the IThis : X_HMI node and drag and drop it into the CAT function block editor. Add any other function blocks that will be used in your project and complete the necessary connections among the inputs and outputs of each function block.
