EcoStruxure Automation Expert - Archive Database Migration

Starting with EcoStruxure Automation Expert - Archive V23.0 the installation wizard lists all detected compatible PostgreSQL versions available on your computer. If more than one PostgreSQL version is listed, a PostgreSQL database may already exist on this computer, created with the installation of an earlier major version of PostgreSQL. You can migrate this database to a later major version of PostgreSQL.

  1. Install the latest major supported version of PostgreSQL (including pgAdmin) to which the existing Archive database is to be migrated.
    NOTE: Your existing Archive database, created with an earlier major version of PostgreSQL remains unchanged, and a new Archive database is created with the new major version of PostgreSQL.
  2. Install EcoStruxure Automation Expert - Archive and select the latest major version of PostgreSQL when requested by the setup wizard.
  3. Open a Windows command prompt and navigate to the bin folder of the PostgreSQL version used for the existing Archive database, for example: cd “C:\Program Files\PostgreSQL\14\bin”.
    NOTE: Open pgAdmin 4 from the Windows Start menu to find out to which PostgreSQL version your existing Archive database is related.
  4. Dump the Archive data from your existing Archive database into an SQL file, for example: pg_dumpall.exe > %temp%\ArchiveDB_V14.sql -U postgres.
  5. Navigate to the bin folder of the new major PostgreSQL version installed in step 1, for example: cd “C:\Program Files\PostgreSQL\15\bin”
  6. Import your data into your new Archive database, for example: psql.exe -U postgres -p 5432 -f "%temp%\ArchiveDB_V14.sql”
    NOTE: Command line parameter -p must be your configured port number for the PostgreSQL version installed in step 1 of this procedure. The default port number is 5432.
  7. Verify that the port number, used in step 7 of this procedure, is also configured inside your EcoStruxure Automation Expert solution for the Archive device.
  8. Verify that the newly installed Archive is working correctly, and that your data history is present.
  9. Optionally, uninstall the previous PostgreSQL version from your computer if not required by any software, then delete the old Archive database from your computer.
    NOTE: After uninstallation of the previous PostgreSQL version, a message appears, displaying the path to the location where your old Archive database is installed.
