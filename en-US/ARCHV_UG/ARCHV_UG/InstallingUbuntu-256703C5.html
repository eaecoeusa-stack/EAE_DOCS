EcoStruxure Automation Expert - Archive and PostgreSQL Installation

An internet connection is required on the PC for the download and the installation of EcoStruxure Automation Expert - Archive (Ubuntu and Debian).
PostgreSQL V14.0 is installed during the installation of EcoStruxure Automation Expert - Archive.
NOTE: Starting with EcoStruxure Automation Expert - Archive V24.1 there are new password requirements introduced and the passwords are encrypted with scram-sha-256 algorithm to fulfill standard security requirements. In earlier versions, default passwords were used (identical to the usernames) and encrypted with unsecure MD5 algorithm. The password encryption is configured in the pg_hba.conf file.
  1. Copy the .deb installation file from the Archive/Linux subfolder of the EcoStruxure Automation Expert - Runtime installation package to the target PC, using winscp or a USB, into a folder of your choice (for example: /home/<username>).
  2. Open the folder where the file is copied, then right-click and select the Terminal application.
    Result: The Terminal application opens.
  3. Run the command sudo apt update to update the list of installation packages.
  4. Depend on processor architecture, run the command $ sudo apt install ./eae-archive_2x.y_amd64.deb or $ sudo apt install ./eae-archive_2x.y_arm64.deb, then:
    NOTE: The 2x.y is the version of EcoStruxure Automation Expert - Archive that you install.
    1. Set a port number or keep the default port number (5432).
    2. When a dialog is displayed, set a password for the database superuser (postgres) for PostgreSQL.
      NOTE:

      • Remember this password as it is required for later use.

      • Use a password that follows the standard security requirements that is displayed in the dialogue during the installation.

  5. Enter the password of the database superuser (postgres), set during the installation of PostgreSQL, and confirm.
    Passwords for the required user roles searchive and the seservice are requested. Enter the password that follows the standard password security requirements displayed in the dialogue during installation.
  6. When the Configuring eae-archive dialog is displayed:
    • Click Yes to backup the data, or
    • Click No to not create backup.
    NOTE: By default, the data of the Properties tables of your databases is deleted by pgAgent to save space. If you click Yes, backups are created before deleting the tables. The selection is applied to databases, created after the setting is configured, and not to existing databases. The value of the backup setting is visible in the Properties table of the searchive database in pgAdmin 4, under the TablePartitioningExportBeforeDrop property. If the value is 1, then backup is activated, and if the value is 0, the backup is turned off. If you want to modify the configuration of this setting, run the command $ sudo dpkg-reconfigure eae-archive and follow the installation steps.
    Result: The installation is completed.
  7. Logout and login to start the services.
