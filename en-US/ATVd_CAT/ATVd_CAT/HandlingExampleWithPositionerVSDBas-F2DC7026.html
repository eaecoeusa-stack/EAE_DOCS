Handling Example with PositionerVSDBase and EncoderDigitalModuleVW3A3420

Perquisites:
Connect the following devices to your PC, and then set their IPv4 addresses to be on the same network

  • ATV340 or ATV9••

  • ATV dPAC

  • Encoder

The procedure explains how to use an EncoderDigitalModuleVW3A3420 and the PositionerVSDBase with ATV dPAC to create a positioning application.

  1. Open Physical Devices
    1. Click on the System tab icon .
    2. Select the Physical Devices icon .
    The physical device window opens.
  2. Drag and drop your altivar device to the Physical Devices window.
    In this example, we choose an ATV9••. Moreover, you can also choose an ATV340 device, since both of them are compatible with ATV dPAC and EncoderDigitalModuleVW3A3420 modules.
  3. Add an ATV dPAC module to your device.
    1. Select the added device (ATV9•• in this example) to display its properties.
    2. Select ATV dPAC as a Communication Card Reference.
  4. Configure the IPv4 Address of your ATV dPAC.
    1. Select the added ATV dPAC to display its properties.
    2. Type the IPv4 Address of your ATV dPAC module.
  5. Add an ATV dPAC device.
    1. Click on the Logical Devices icon .
    2. Click on Add Device icon .
    3. Type the device name.
    4. Select the device type.
    5. Click OK.
    The ATV dPAC device is added in the Logical Devices window.
  6. Link the ATV dPAC device added in Physical Device to the ATV dPAC device added in Logical Device.
  7. Add a DPAC_FULLINIT and do the necessary connections.
    1. Click on the Function Blocks Network icon .
    2. Add DPAC_FULLINIT composite function block to the Function Blocks Network editor.
    3. Link the DPAC_FULLINIT to the START function block.
  8. Add PositionerVSDBase application CAT and AtvPosControlInterface composite function block, then link them together.
    1. Add PositionerVSDBase application CAT to the Function Blocks Network editor.
      Develop the logic based on your process,to provide values in positioner inputs (such as ProgOwner, Acc, Dec, Velocity, etc.).
      NOTE: To run, the block must have the following features set to TRUE: Enable, QuickStop, LimitSwitchPos, LimitSwitchNeg, and EncReady.
      In this example, all the inputs will be set as constants, but you have the option to link them to your logic. This includes inputs from a PushButton, limit switch, emergency stop, input from HMI, and more.
    2. Add AtvPosControlInterface composite function block to the Function Blocks Network editor.
    3. Define the following data inputs of the AtvPosControlInterface composite function block as follow:

      • Set a value to ApplyBrakeWhenMoveDone depending on your application (If TRUE: The drive goes to RDY state after a movement and the brake closes, it is advised for vertical movement).

      • Set the default value to EncPulseInitDelay.

    4. Link the added application CAT and composite function block as follow:
  9. Add the hardware CAT.
    1. Click on the Hardware Configuration icon .
    2. Right-click in the Hardware Configuration editor, select Add, select SE.IoATVdPACxVW3A3530D, then click Add.
      The ATVdPACxVW3A3530D bus master is added to the Hardware Configuration editor.
    3. Type 10 ms as the busCycleTime value.
    4. Right-click on the ATVdPACxVW3A3530D bus master, select SE.IoATV.AtvPosControlLight, then click Add.
      NOTE: You can also use SE.IoATVSpeedControlLight, but it is advisable to use SE.IoATVPosControlLight.

      If you decide to use SE.IoATVSpeedControlLight, make sure that AtvUseRpmSpeed is set to TRUE.

  10. Link the Symbolic Links to the hardware CAT.
  11. Configure the properties of ATVPosControlLight hardware CAT.
    1. Right-click ATVPosControlLight hardware CAT, select Properties Editor.
      A new window appears to select the encoder type.
    2. Select your encoder module based on your application, then click OK.
      In this example, we choose Digital Encoder VW3A3420
      A new window appears, showing the properties of ATVPosControlLight.
    3. Click Command & Reference tab, then configure the parameters depending on your application.

    4. Click Ramp tab, then configure the parameters depending on your application.
      In this example, we choose the following configuration:
    5. Click Encoder Warning/Error tab, then configure the parameters depending on your application.
      In this example, we choose the following configuration:
    6. Click SSI Encoder tab, then configure the parameters depending on your application, then click OK.
      In this example, we choose the following configuration:
      NOTE: Restart the drive after changing the SSI encoder parameters.
  12. Click on the Build icon .
  13. Add the sPosition generic symbol to the Canvas.
    1. Create a new Canvas.
    2. Drag and drop the PositionerVSDBase CAT instance to the Canvas.
    3. Select sPosition generic symbol.
    4. Click OK.
    The sPosition generic symbol is added to the Canvas.
  14. Deploy your solution.
    1. Click the Deploy and Diagnostic icon .
      The Deploy and Diagnostic opens.
    2. Click Action > Compile.
    3. Click Action > Login.
      Use the credentials of your ATV dPAC module to connect to your device.
    4. Click Action > Deploy > Clean.
    5. Click Action > Deploy > Deploy.
    6. Click Action > Device actions > Run.
    Result:
  15. Right-click on the resolution, then click Start Local HMI.
    Use your solution credentials to start the local HMI.
  16. Open the positioner settings faceplate.
    1. Click the sPosition generic symbol to open the associated faceplate.
    2. Click the icon to open the parameter data faceplate.
  17. Configure the Motor tab.
    Configure the Quick Deceleration so that the motor can stop rapidly enough to prevent any physical collisions. Additionally, set the Velocity Nominal to the value indicated on motor nameplate.
  18. Configure the Control tab.
    1. Set up the Profile to match the acceleration profile you desire for your application.
    2. Set up the SpVelocity Output to -/+ rpm (mandatory with ATVPosControl).
    3. Set the following parameters, to ensure precise movement for your application:

      • Max Pos Deviation

      • Position Window

      • In Pos Time

      • Max duration

    4. Set up the Cycle Time to the cycle time already set in the ATV dPAC in Hardware Configuration (It is advised to set the cycle time to 10 ms).
    NOTE: If you are using an ATVSpeedControlLight hardware CAT, select Abs rpm for Sp Velocity Output.
  19. Configure the Encoder tab.
    1. Fill informations: Pulse/Revolution, Encoder Type and the Resolution according to your encoder type.
    2. You can enable Position Incr. Save to be able to save the position of the axis after restarting your drive even if you have an incremental encoder.
    3. Configure the Numerator and Denominator to manage the mechanical cinematic of your system, according to the following formula:

      The Numerator and Denominator parameters are used to scale position values. The calculated scaling factor defines the relationship between drive revolutions and corresponding distance in user units.

      The following examples show the relation: If a 10 mm axis movement corresponds to 7 motor revolutions, set the Numerator to 7 and the Denominator to 10. The corresponding user unit would be 1 mm. In this example, if the denominator is set to 100, the user unit is 0.1 mm.

      Similarly, parameters in modulo can be configured in angular units. When one machine axis revolution (Gearbox output shaft) equals 15.5 motor revolutions and the expected resolution is 0.1°, the Numerator is 155 and the Denominator is 36000.

      Numerator and Denominator that are not relatively prime can be divided by their greatest common divisor (In this example 5):

      Setting of these parameters is only possible in off state (Positioner Disabled) . After changing these parameters homing with method 35 is performed automatically.

  20. Configure the Axis tab.

    Configure Software Limit Switch to quick stop the positioner when the axis position exceeds the positive limit (overshoots) or falls below the negative limit (undershoots).

    Define the behavior of the axis after triggering a physical limit switch (positive or negative), allowing you to choose between a quick stop or a halt.

    Specify the axis type (linear or modulo) and its range (single or multi-range).

    NOTE: After a quick stop from the application CAT, the Drive state remains in the RUN mode to preserve the current position.
  21. Verify the settings of your application.
    1. Set Owner to Operator.
    2. Set Positioner to Enabled.
    3. For Operating mode, select Homing.
    4. Select the Homing tab.
    5. For Homing Method, select 35, Position setting.
    6. Type your Homing Reference.
    7. Click Execute.
    NOTE:

    CurrentPosition represents the actual position influenced by the encoder and controller parameters, while PGSpPoition is the calculated position setpoint.

    In the case where Gain Pos. Control is higher than the Gain Vel. FeedFwd, PGSpPoition will be higher than CurrentPosition. Conversely, when the Gain Pos. Control is lower than the Gain Vel. FeedFwd, PGSpPoition will be lower than CurrentPosition.

    In summary, depending on the settings of the Gain Pos. Control and Gain Vel. FeedFwd, PGSpPoition will either lead or lag behind the CurrentPosition.
