ATVDisplay Integration

  1. Configure your physical device:
    1. Click on the System icon to open the System tab.
    2. Click on the Physical Devices icon to open the Physical Devices editor.
    3. Drag and drop your altivar device to the Physical Devices editor.
    4. Click on your altivar device to display its properties, then select ATV dPAC for the Communication Card Reference.
    5. Click on the ATV dPAC module to display its properties, then type the IPV4 Address of your ATV dPAC
  2. Configure your logical device:
    1. Click on the Logical Devices icon to open the Logical Devices editor.
    2. Click Add Device icon to add a new logical device.
    3. Select SE.DPAC:ATV_dPAC as a Type, then click OK.
    4. For the Physical Device column, select your ATV dPAC module (Already created in the Physical Devices editor).
  3. Configure your hardware configuration
    1. Click on the Hardware Configuration icon to open the Hardware Configuration editor.
    2. Right-click in Hardware Configuration editor, then select Add.
    3. Select the bus element type SE.IoATV.ATVdPACxVW3A3530D, then click Add.
    4. Right click on the added bus ATVdPACxVW3A3530D, select the configuration module SE.IoATV.ATVSpeedControl, then click Add.
    5. Right click on the added bus ATVdPACxVW3A3530D, select the configuration module SE.IoATV.ATVDisplay, then click Add.

  4. configure the Function Blocks Network.
    1. Click on the Function Blocks Network icon .
    2. Add the following logic to the Function Blocks Network editor and connect them as shown in the figure.

      • Add a SYMLINKMULTIVARDST and assign the inputs as follows:

        • TRUE

        • '$${PATH}ATVDisp_Run'

        • '$${PATH}ATVDisp_Stop'

        • '$${PATH}ATVDisp_ComStatus'

      • Add a NOT function block to invert the VALUE3

      • Add 3 E_PERMIT function block

        • Run: Triggers an event when VALUE1 = True (when you click on RUN on the graphic display terminal).

        • Stop: Triggers an event when VALUE2 = True (when you click on STOP on the graphic display terminal).

        • Connection: Triggers an event when VALUE3 = True (when the graphic display terminal is disconnected, the run command becomes false and the QuickStop becomes true).

      • Add an E_SR function blockn to trigger an event when the drive starts running.

  5. Connect the Symbolic Links
    1. Click on the Hardware Configuration icon to open the Hardware Configuration editor.
    2. Connect the Symbolic Links as shown in the following figure.
  6. Deploy your solution
    1. Open the deploy and diagnostic editor .
    2. Compile, Login, deploy and run device actions.
