Setting up a positioning application using ATV dPAC

Setup Overview

You’ll need to connect the following devices to your PC and ensure they are on the same IPv4 network:

  • ATV340 or ATV9••

  • ATV dPAC

  • Encoder

The following procedure is step-by-step guide for setting up a positioning application using ATV dPAC PositionerVSDBase and EncoderDigitalModule VW3A3420.

Step-by-Step Configuration

Step

Action

1

Open the Physical Devices

  • Go to System tab and open Physical Devices .

  • Drag and drop your altivar device to the Physical Devices window.

NOTE: In this example, we choose an ATV9••. Moreover, you can also choose an ATV340 device, since both of them are compatible with ATV dPAC and EncoderDigitalModuleVW3A3420 modules.

2

Add an ATV dPAC module to your device.

  1. Select the added device (ATV9•• in this example) to display its properties.

  2. Select ATV dPAC as a Communication Card Reference.

4

Configure the IPv4 Address of your ATV dPAC.

  1. Select the added ATV dPAC to display its properties.

  2. Type the IPv4 Address of your ATV dPAC module.

5

Add an ATV dPAC device.

  1. Click on the Logical Devices icon

  2. Change the type of your device.

6

Link the ATV dPAC device added in Physical Device to the ATV dPAC device added in Logical Device.

7

Add a DPAC_FULLINIT and do the necessary connections.

  1. Click on the Function Blocks Network icon .

  2. Add DPAC_FULLINIT composite function block to the Function Blocks Network editor.

  3. Link the DPAC_FULLINIT to the START function block.

8

Add PositionerVSDBaseExt composite function block and AtvPosInterface2 composite function block, then link them together.

Develop the logic based on your process,to provide values in positioner inputs (such as ProgOwner, Acc, Dec, Velocity, etc.).

9

Create an application CAT that has 3 adapters to develop your logic

  1. Add IPositioner to your logic to control the positioner by retrieving the values (you can develop the logic based on your process,to provide values in positioner inputs such as ProgOwner, Acc, Dec, Velocity, etc.).

    NOTE: To run, the block must have the following features set to TRUE: Enable, QuickStop, LimitSwitchPos, LimitSwitchNeg, and EncReady.
    In this example, all the inputs will be set as constants, but you have the option to link them to your logic. This includes inputs from a PushButton, limit switch, emergency stop, input from HMI, and more.

  2. Add The encoder Value to your logic to set the value of the encoder. Here the value is force to TRUE but you can create your own logic to manage it conforming to your application.

  3. Add The encoder ready to your logic to control the ready status of the encoder. The encoder ready needs to be true to make the positioner working.

10

Add the created application CAT to the Function Blocks Network and link it with PositionerVSDBaseExt composite function block

11

Add the hardware CAT.

  1. Click on the Hardware Configuration icon .

  2. Right-click in the Hardware Configuration editor, select Add, select SE.IoATVdPACxVW3A3530D, then click Add.

  3. Type 10 ms as the busCycleTime value.

  4. Right-click on the ATVdPACxVW3A3530D bus master, select SE.IoATV.AtvPosControlLight, then click Add.

    NOTE: You can also use SE.IoATVSpeedControlLight, but it is advisable to use SE.IoATVPosControlLight.

    If you decide to use SE.IoATVSpeedControlLight, make sure that AtvUseRpmSpeed is set to TRUE.

12

Link the Symbolic Links to the hardware CAT.

13

Configure the properties of ATVPosControlLight hardware CAT.

  1. Right-click ATVPosControlLight hardware CAT, select Properties Editor.

  4. Click Encoder Warning/Error tab, then configure the parameters depending on your application. In this example, we choose the following configuration:

  5. Click SSI Encoder tab, then configure the parameters depending on your application, then click OK. In this example, we choose the following configuration:

NOTE: Restart the drive after changing the SSI encoder parameters.

14

Click on the Build icon .

15

Create your canvas, add the sPosition generic symbol, Deploy your solution to configure your position solution.
