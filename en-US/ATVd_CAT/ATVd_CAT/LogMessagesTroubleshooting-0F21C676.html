Log messages troubleshooting

Abstract

This section explains the log messages that will be displayed each time you compile.

The log messages are 3 types:

  • Errors

  • Warnings

  • Messages

Description

Errors:

Message log

Root Cause

Solution

E01

You should add ATVSpeedControl or ATVSpeedControlLight Hardware CAT for this device in the Hardware Configuration IO_Embedded needs ATVSpeedControl or ATVSpeedControlLight Hardware CAT

E02

You cannot add EncoderEmbeddedATV340x075to22kW before ATVSpeedControl, ATVSpeedControlLight, or IO_Embedded Hardware CATs in the Hardware Configuration

ATVSpeedControl, ATVSpeedControlLight, or IO_Embedded is not added first under the Bus Master ATVdPACxVW3A3530D, it will be offline and it will not be taken into account

Define ATVSpeedControl or ATVSpeedControlLight before EncoderEmbeddedATV340x075to22kW or IO_Embedded

E03

You cannot add more than 1 ATVSpeedControl, ATVSpeedControlLight, or both for this device in the Hardware Configuration The ATVdPACxVW3A3530D Bus Master supports only 1 ATVSpeedControl, ATVSpeedControlLight Hardware CAT Add only 1 ATVSpeedControl, or 1 ATVSpeedControlLight for ATVdPACxVW3A3530D Bus Master

E04

The primary (IO_Embedded) needs to be added before the secondary (IO_Extended) Add IO_Embedded modules before IO_Extended modules

Solution

W01

You could add ATVSpeedControl before embedded or optional modules for Test Without Device function to work

In Hardware configuration, when adding embedded or optional modules before ATVSpeedControl, Test Without Device function will not work

Add ATVSpeedControl before embedded or optional modules

W02

You could add ATVSpeedControl in the first or the second slot

Open the Properties Editor of IoEmbeddedATV340x075to22kW and assign all its I/Os to IEC 61499

Message log

Root Cause

Solution

M01

You may add ATVSpeedControl for Test Without Device function to work

Only ATVSpeedControl Hardware CAT has the functionality of Test Without Device

Add an ATVSpeedControl

M02

ATV Device type is not defined. You may rename the ATVdPACx3A3530D to ATV9xx, ATV6xx, or ATV340 matching with your physical device

The compiler needs to know the used device to generate the adequate errors/warnings

Rename the CPU (ATVdPACx3A3530D) to ATV9xx, ATV6xx, or ATV340, matching with the physical device, in the Hardware configuration

M03

Ensure that the parameters of the device Scanner Profile are correctly set in SoMove. Refer to Field Device > Configuration section in Online Help Document for further reference

Make sure that the Scanner I/O Profile is set on EAE dPAC in SoMove/DTM. Refer to the Field Device Library User Manual for further information

The I/Os scanner Profile need to be set correctly to not have a mismatching parameters with the scanner parameters from EcoStruxure Automation Expert

Open SoMove and set correctly the device Scanner Profile.

Refer to the field device library user guide for more information about this step
