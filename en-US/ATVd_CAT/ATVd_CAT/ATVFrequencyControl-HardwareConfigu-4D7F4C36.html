ATVFrequencyControl - Hardware Configuration

Abstract

The ATVFrequencyControl Hardware CAT allows the IEC 61499 application to control and monitor an Altivar Variable Speed Drive with an ATV dPAC module.

The ATVFrequencyControl Hardware CAT supports frequency command and it can be used to control the drive at low frequency.

The associated HMI also allows you to observe the key Drive parameters values, Drive status, and take manual control on the device.

Description

Hardware Configuration

  1. In EcoStruxure Automation Expert, open the System tab → Logical Devices → Add Device that should have SE.DPAC.ATV_dPAC as a type → Click OK.

  2. Right click on the device, then open the HW Configurator.

  3. Right click in the Hardware Configuration window, then add ATVdPACxVW3A3530D, then right click in the added ATVdPACxVW3A3530D, then add ATVFrequencyControl.

    NOTICE

    busCycleTime is the period during which data will be exchanged between EcoStruxure Automation Expert application and the hardware CAT in the hardware configuration, including the drive hosting the ATV dPAC (motor and inputs/outputs), Modbus TCP client, and server.

    By default, busCycleTime for ATV dPAC is set to 250 ms, You can change this value to fit the application requirements.

    The range of values supported by the ATV dPAC is from 5 ms to several tens of seconds.

Hardware Properties Editor

The Hardware Properties Editor window allows you to edit and configure the Hardware parameter settings, which are transferred to the device upon the application start-up.

NOTE:

  • These parameters will override the existing parameter settings stored in the Altivar device

  • If these parameters are modified during run-time by a source external to EcoStruxure Automation Expert (e.g. the Altivar device’s Webserver, SoMove, Modbus parameter write), then the IEC 61499 application might be impacted.

  1. Right click on the ATVFrequencyControl, Then select Properties Editor.

  2. Select the device type, then click OK.

The following figure illustrates the Properties Editor window for the Command & Reference tab:

According to each selected device type (ATV340, ATV6••, ATV9••), [CHCF] Control Mode Configuration, [FR1] Configuration Reference Frequency 1 and [CD1] Command Channel 1 Assign fields are editable.

NOTE:
The following safety message appears if you want to change the value of the [CLL] parameter:
 WARNING
LOSS OF CONTROL
If this Parameter is set to No, Modbus Communication monitoring is disabled.
  • Only use this setting after a through risk assessment in compliance with all regulations and standards that apply to the device and to the application.
  • Only use this setting for tests during commissioning.
  • Verify that communication monitoring has been re-enabled before completing the commissioning and performing the final commissioning test.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

  • If [[Speed Maint Behavior] RLS = 0, then the speed setpoint is 0 Hz.

  • If [Speed Maint Behavior] RLS = LSP, then the speed setpoint is the value of the [Low Speed] (LSP) parameter.

NOTE: Display in SoMove

Values set by the IEC61499 application won't be displayed in SoMove unless saved in non-volatile memory using the SaveToATVconf function.

Saving Parameters to Non-Volatile Memory

To save parameters in the device's non-volatile memory:

  1. In the Hardware Configuration, right-click on ATVdPACxVW3A3530D Bus Master.

  2. Select Properties.

  3. Set SaveToATVconf to Yes.

  4. Click OK.
