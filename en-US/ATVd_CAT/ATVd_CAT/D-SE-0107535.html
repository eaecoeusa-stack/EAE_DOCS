8. Steps to Program the ATV dPAC with EcoStruxure Automation Expert Build time

Before you begin:

  1. Connect the Ethernet cable between the ATV dPAC port and the computer where EcoStruxure Automation Expert is running.

  2. Put your computer IP address in the same range as the IP address of your ATV dPAC module.

  3. Disable the firewall if it doesn’t allow the ATV dPAC to connect your computer.

Procedure:

To program the ATV dPAC application for motor control using EcoStruxure Automation Expert we will follow 5 steps:

  1. Create a new solution and reference the libraries

  2. Add and configure the ATV dPAC

  3. Configure the security

  4. Add the Modules to the Hardware Configuration

  5. Create the canvases

10. Procedure to test without device

The Test without device function enables the EcoStruxure Automation Expert to simulate a logic using dPAC module. Test without device allows you to check a logical operation without using the actual device.

Before you begin:

  • In the HW Configuration, add an ATVSpeedControl under an ATV dPAC Module.

  • In Function Blocks Network Add a DPAC_FULLINIT and do the necessary connection.

  • Build the solution, create a new Canvas and add the sDrive faceplate of the ATVSpeedControl CAT instance

  • Compile, Start runtime simulation, Login, Clean, Deploy and Run Device actions in Simulation mode or with an actual device, using Action option from Deploy and Diagnostic tab.

NOTE: Refer to the section number 8 for more details.

Procedure:

  1. Start Local HMI

  2. Click on the Command icon on the sDrive faceplate to open the Drive Control.

  3. Set Test without Device to ON.
  4. Enable FORCE mode → accept the warning message.

  5. Set the Speed Reference value.

  6. Give Run Command.

    NOTE: The drive state will change from RDY to RUN.
    Actual Speed reaches the set speed during the acceleration time.

  7. Click the STOP command to stop the Motor. The drive decelerates and reaches 0 rpm during the deceleration time.

13. Perform a Motor [Autotuning]

Overview

The purpose of the Auto-tuning function is to command the drive to measure the motor characteristics. Auto-tuning allows the device to measure the motor characteristics and improve the motor control performance. Auto-tuning command is communicated to the connected device with EcoStruxure Automation Expert to measure and update the value of several parameters. The EcoStruxure Automation Expert indicates the status of auto-tuning during progress and also when it has finished.

Pre-condition

Before you start the autotuning process, make sure that:

  • Drive is in ready (RDY) state

  • Motor is stopped

  • Auto-tuning is not in progress

  • Ensure that [Nom Motor Voltage] ((UnS)), [Nominal Motor Freq] ((FrS)), [Nom Motor Current] ((nCr)), [Nominal Motor Speed] ((nSP)), and [Nominal motor power] ((nPr)) or [Motor 1 Cousinus Phil] ((COS)) parameters are correctly defined.

 WARNING
UNEXPECTED MOVEMENT
Auto-tuning moves the motor in order to tune the control loops.
  • Only start the system if there are no persons or obstructions in the zone of operation.
Failure to follow these instructions can result in death, serious injury, or equipment damage.
NOTE: Autotune is not mandatory. It is not recommended for applications or where a load is connected.

Steps to follow

  1. Connect the drive to your PC with an Ethernet cable

  2. As shown in the 8th section.

    • Create a solution

    • Add the ATVSpeedControl Hardware CAT to the Hardware Configuration

    • create a canvas, add the ATVSpeedControl CAT Instance to it

      and choose sDrive.
  3. Start Local HMI, click on the Command icon on the sDrive faceplate to open the Drive Control faceplate.
  4. Enable the Force mode → accept the warning message.
  5. APPLY the Auto-tuning.
    NOTE: During Auto-tuning, the motor makes a small movement. Noise development and oscillations of the system are normal.

    Erase Auto-tuning

    This function is used to reset the motor parameters values measured during Auto-tuning.

 WARNING
LOSS OF CONTROL
  • Fully read and understand the manual of the connected motor.
  • Verify that all motor parameters are correctly set by referring to the nameplate and the manual of the connected motor.

Memory limitation due to archiving

Abstract

Hardware CATs can be configured to transmit information to an archiving server. This allows the recording of various data, such as the performance of motors and the history of warnings. However, it's important to note that excessive data accumulation can overload the memory, resulting in an ErrorHalt situation.

Description

If an ErrorHalt appears due to archiving, it is important to consider the hardware limitations of ATV dPAC on EcoStruxure Automation Expert. Here are the key points to keep in mind:

  • The default server configuration can only handle a maximum of 2000 variables per second for data collection.

  • If you are archiving variables periodically, you need to adjust the E_CYCLE frequency accordingly. For example, if you have 500 variables, the minimum value for E_CYCLE should be set to 500 ms.

  • It is recommended to avoid using a frequency below 50 ms for any archiving operation.

  • Archiving data with a frequency smaller than 1 second may lead to data saturation on the archiving server. Therefore, it is advisable to set a frequency of 1 second or more, depending on your specific application.
