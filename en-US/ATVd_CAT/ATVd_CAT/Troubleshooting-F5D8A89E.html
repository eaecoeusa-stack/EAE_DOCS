Troubleshooting

If the requested position is not reached, follow these steps:

  • Ensure that all the event inputs of the positioning block have been triggered at least once.
    1. Right-click on the PositionerVSDBase application CAT, then select Add watch.
    2. Right click on the REQ event, then select Trigger Event.
    3. Right click on the REQ_ENCODER event, then select Trigger Event.
    4. Right click on the REQ_SWITCH event, then select Trigger Event.
    5. Right click on the REQ_OPEROWNER event, then select Trigger Event.
    6. Right click on the REQ_PROGSEL event, then select Trigger Event.
    7. Right click on the MOVE_PREPARED event, then select Trigger Event.
  • Execute a positioning request and check if the target is reached.
    1. In the local HMI, open the faceplate of your positioner.
    2. Set Owner to Operator.
    3. Type the value of your Setpoint Position.
    4. Type the value of your Positioning velocity.
    5. Select the Method.
    6. Click on Execute, then check if the target is reached.

Error halt troubleshooting

  • If the ATV dPAC leads to error halts in the Positioner Application, ensure that the output events of the PositionerVSDBase application CAT are not linked to the input events.
    NOTE: The output events of PositionerVSDBase should not be linked back to the input Events, particularly the CNF output event.
