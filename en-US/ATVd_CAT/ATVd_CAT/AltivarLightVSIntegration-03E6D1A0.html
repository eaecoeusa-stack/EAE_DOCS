NOTE: The AltivarVsLight2D application CAT is no longer applicable and obsolete. If you use this application CAT in your solution, a warning message is displayed in the Message log window.

If you integrate a solution from EcoStruxure Automation Expert 22.1 that is using the obsolete application CAT AltivarVsLight2D to a newer version, you might need to follow these actions:

  • Reconfigure the symbolic links of all the application CATs.

  • Reconnect the events and the data of the functions blocks, that are no longer connected in the Function Blocks Network.

NOTE:

If an error reset and a run order happen simultaneously (with the same event), the default requested speed is ignored by the AltivarVsLight Application CAT, but the error is reset. The drive then needs another run event to start running. To address this, follow these steps:

  1. Wait for initialization (which happens automatically inside the AltivarVsLight Application CAT) and avoid giving immediate request events (like REQ_CMD and REQ_SPEED) after deploying.

  2. Reset the drive error using REQ_RESET (the drive changes from FLT to RDY/NST, and the Active Error Status will be NOF No Error).

  3. Send REQ_SPEED, and REQ_CMD.
