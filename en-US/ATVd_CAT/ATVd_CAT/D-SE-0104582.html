ATVSpeedControl - Hardware Configuration

Abstract

The ATVSpeedControl Hardware CAT allows the IEC 61499 application to control and monitor an Altivar Variable Speed Drive with an ATV dPAC module.

The associated HMI also allows you to observe the key Drive parameters values, Drive status, and take manual control on the device.

Description

Hardware Configuration

  1. In EcoStruxure Automation Expert, open the System tab → Logical Devices → Add Device that should have SE.DPAC.ATV_dPAC as a type → Click OK.

  2. Right click on the device, then open the HW Configurator.

  3. Right click in the Hardware Configuration window, then add ATVdPACxVW3A3530D, then right click in the added ATVdPACxVW3A3530D, then add ATVSpeedControl.

    NOTICE

    busCycleTime is the period during which data will be exchanged between EcoStruxure Automation Expert application and the hardware CAT in the hardware configuration, including the drive hosting the ATV dPAC (motor and inputs/outputs), Modbus TCP client, and server.

    By default, busCycleTime for ATV dPAC is set to 250 ms, You can change this value to fit the application requirements.

    The range of values supported by the ATV dPAC is from 5 ms to several tens of seconds.

Hardware Properties Editor

The Hardware Properties Editor window allows you to edit and configure the Hardware parameter settings, which are transferred to the device upon the application start-up.

NOTE:

  • These parameters will override the existing parameter settings stored in the Altivar device

  • If these parameters are modified during run-time by a source external to EcoStruxure Automation Expert (e.g. the Altivar device’s Webserver, SoMove, Modbus parameter write), then the IEC 61499 application might be impacted.

  1. Right click on the ATVSpeedControl, then select Properties Editor

  2. Select the device type, then click OK.

According to each selected device type (ATV340, ATV6••, ATV9••), [CHCF] Control Mode Configuration, [FR1] Configuration Reference Frequency 1 and [CD1] Command Channel 1 Assign fields are editable.

NOTE:
  • Safety message before changing the value of the [CHCF], [FR1] or [CD1] parameters:
     WARNING
    LOSS OF CONTROL
    Avoid modifying the CHCF, FR1 or CD1 parameters externally (outside the Ecostruxure Automation Expert properties editor).
    Failure to follow these instructions can result in death, serious injury, or equipment damage.
  • Safety message before changing the value of the [CLL] parameter:
     WARNING
    LOSS OF CONTROL
    If this Parameter is set to No, Modbus Communication monitoring is disabled.
    • Only use this setting after a through risk assessment in compliance with all regulations and standards that apply to the device and to the application.
    • Only use this setting for tests during commissioning.
    • Verify that communication monitoring has been re-enabled before completing the commissioning and performing the final commissioning test.
    Failure to follow these instructions can result in death, serious injury, or equipment damage.

Overwriting the parameters during the deployment

Abstract

To avoid overwriting all the parameters during the deployment. you are provided with a check box for each parameter to decide whether to overwrite a parameter or not.

Description

The following table describes the steps to choose whether to overwrite a parameter or not:

Step

Action

1

Under an ATVdPACxVW3A330D Module in the hardware configuration Add:

  • An ATVSpeedControl

  • or an ATVSpeedControlLight

2

  • Right click on the added device to open the “Properties Editor”

  • Select the device Type

3

  • Check the box of the parameters that you want to be deployed to the device

  • Click the OK button

Ramp Parameters Configuration

  • Default State: Ramp parameters are disabled by default.

  • Safety Message: When you check any ramp parameter for the first time, a safety message appears. Accept the message to proceed, once you accept the safety message, you can check or uncheck any parameter.

The values of the selected parameters will be automatically applied to the device, during the IEC61499 application start-up phase. They will also be saved in the boot project. They will override the values which may already be stored in the device.

 WARNING
UNANTICIPATED EQUIPMENT OPERATION
  • Only allow this behavior after a thorough risk assessment in compliance with all regulations and standards that apply to the device and to the application.
  • Only override the values already stored in the device if you can positively exclude that this can result in unsafe conditions.
  • Perform a comprehensive commissioning test to verify correct operation of the application.
Failure to follow these instructions can result in death, serious injury, or equipment damage.
NOTE: Display in SoMove

Values set by the IEC61499 application won't be displayed in SoMove unless saved in non-volatile memory using the SaveToATVconf function.

Saving Parameters to Non-Volatile Memory

To save parameters in the device's non-volatile memory:

  1. In the Hardware Configuration, right-click on ATVdPACxVW3A3530D Bus Master.

  2. Select Properties.

  3. Set SaveToATVconf to Yes.

  4. Click OK.

Managing live and configuration data

Abstract

By default, the drive parameter values set by the application are not saved in the device’s configuration file (EEPROM).

The drive parameters settings can be written from the property editor of Altivar hardware CAT.

Description

The following table describes how to save the properties editor values in the drive after the deployment of the application:

Step

Action

1

Under an ATVdPACxVW3A330D Module in the hardware configuration Add:

  • An ATVSpeedControl

  • or an ATVSpeedControlLight

2

Right click on ATVdPACxVW3A330D and select Properties

3

Select Yes for SaveToATVConfig

NOTE: A DPAC_FULLINITfunction block should be:

  • Added to the function blocks network

  • Connected to the E_RESTART function block

These values will be then displayed in SoMove Parameters List view.

It may be different from the live unsaved values, visible on the drive’s Graphic Display Terminal (VW3A1111), or in EcoStruxure Automation Expert (Properties Editor, Symbolic Links, HMI, ATV_DD_OUT, or watches).

[BFR] Basic Frequency and [TFR] Max Frequency for ATVSpeedControl

  1. Right click on the ATVSpeedControl from the Hardware Configuration and choose Properties Editor

  2. A new window will pop out to select the device type, which could be either ATV6xx, or ATV9xx, or ATV340

  3. After selecting the device type, a new window gathering the properties of ATVSpeedControl will pop out, select the ramp tab in order to type the value of the max frequency

  4. We will have two cases

    1. Case 1: if the selected device is ATV9xx or ATV340

      • If [BFR] = 50 Hz : [TFR] can be set up to 500 Hz.

      • If [BFR] = 60 Hz : [TFR] can be set up to 599 Hz.

    2. Case 2: if the selected device is ATV6xx

      Regardless of the [BFR] value, the [TFR] value will always be below 500 Hz.
