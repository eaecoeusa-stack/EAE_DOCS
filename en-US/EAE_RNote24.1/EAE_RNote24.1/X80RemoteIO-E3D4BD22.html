EcoStruxure Automation Expert — BMECRD0100 X80 Remote I/O

Overview

A BMECRD0100 module can be used as an I/O adapter in each pair of high availability remote I/Os. The BMECRD0100 components (that is, the associated controller and I/Os) are configurable in the Physical Devices editor.

The BMECRD0100 module includes three external Ethernet ports with Ethernet backplane. Two external ports serve as integrated EtherNet/IP connections to the device network and the remaining Ethernet port is the service port. Input data is collected and published to the remote controller and output modules are updated with the data received from the controller. The two network ports support RSTP and can be combined with Modbus TCP/IP and EtherNet/IP in a network topology.

The SE.IoX80 library provides an HMI CAT for the BMECRD0100 module.

Cybersecurity Rotary Switch

The BMECRD0100 cybersecurity rotary switch can be used to reset the module to factory default settings. The rotary switch is located on the back of the BMECRD0100 module. 

Use the screwdriver to select a switch position that meets your cybersecurity requirements: 

Icon 

Setting 

Description 

Reset 

Factory reset or out-of-the-box configuration is set. 

The BMECRD0100 module resets to factory settings. The factory settings delete the boot project, the device configuration, and the persistent data. Then, the factory reset implements its out-of-box cybersecurity configuration. 

Advanced 

Advanced mode is ON. 

The BMECRD0100 module is now in secured cybersecurity mode. 

Not Used 

– 

This rotary position is not used. 

Limitations

The BMECRD0100 deployment and I/O communication do not provide cryptographically protected data integrity or confidentially. 

BM●A●●●●●● modules cannot adjust input alignment, filter, range threshold, and output fallback while running. This can be done during buildtime in the Hardware Configuration editor. 

NEW FEATURES

Cyber Security Syslog Support

EcoStruxure Automation Expert is able to capture and archive an audit trail of cyber security related events. BMECRD0100 module supports Syslogs, they can be accessed by a third-party Syslog server. 

The log of events and event details cannot be modified on the BMECRD0100 module. 

  • The CS Syslog message format conforms to industry accepted standard format (provides specific messaging format). 

  • The CS Syslog messages are time stamped with a timestamp that is synced across EAE subsystems. 

  • The audit record storage capacity can be allocated, enlarged or reduced by configuration.
