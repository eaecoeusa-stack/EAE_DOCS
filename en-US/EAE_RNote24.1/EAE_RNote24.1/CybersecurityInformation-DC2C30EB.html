Cybersecurity Information

Binary Files Integrity

Schneider Electric provides EcoStruxure Automation Expert components through signed installer packages, and ensures they have not been tampered with, from the moment they were built to the point they are executed.

Microsoft provides the ability to view the certificate by right-clicking the file in question and viewing the certificate details to validate any component that may be in question.

Communication Hardening

Communications between the following devices and components are secured thanks to certificate management:

  • EcoStruxure Automation Expert - Buildtime

  • Modicon M580 Distributed PAC

  • Altivar dPAC Module VW3A3530D

  • Modicon M251 Distributed PAC

  • Modicon M262 Distributed PAC

  • EcoStruxure Automation Expert - Soft dPAC

  • OPC UA Server

  • EcoStruxure Automation Expert - HMI

Users must be authenticated in EcoStruxure Automation Expert - Buildtime.

DPACs must be enrolled first before any application can be deployed.

Altivar dPAC Module VW3A3530D, Modicon M251 Distributed PAC, Modicon M262 Distributed PAC and Modicon M580 Distributed PAC 21.1 and later, are delivered with security turned on by default.

EcoStruxure Automation Expert - Buildtime manages the security settings of such communications.

Compliance

Our system conforms to the California SB-327 Information Privacy: Connected Devices bill by requiring the user to generate a new means of authentication before granting initial access. This includes the Modicon M580 Distributed PAC, Altivar dPAC Module VW3A3530D, Modicon M251 Distributed PAC, and Modicon M262 Distributed PAC. Industry vulnerabilities are regularly identified and mitigated in order to minimize cybersecurity risks.

Denial-of-Service (DoS) Protection

Denial-of-Service (DoS) protection is configured by default to support a maximum rate of incoming messages. In case this limit is exceeded, a dPAC might encounter out-of-memory situations. Please configure the protection to match the current situation.

For Modicon M251 Distributed PAC:

  • On Ethernet 2 interface: as soon as the incoming messages rate is greater than 800 frames during 10 ms, the Ethernet 2 interface is turned off for eight seconds.

  • On Ethernet 1 interface: as soon as the Ethernet job queue is full (capacity is 100 frames during 10 ms), the Ethernet 1 interface is turned off for eight seconds.

  • If Modicon M251 Distributed PAC is overloaded by communication tasks, both Ethernet 1 and Ethernet 2 interfaces are turned off for eight seconds.

Hardware Agnostic

End-users are responsible for applying security patches on their machines when using Windows and Linux operating system distributions. This especially applies for the EcoStruxure Automation Expert - Soft dPAC (Linux) due to the flexibility to use any Linux distribution that supports docker.

Improved Security Robustness

This release of EcoStruxure Automation Expert includes several cybersecurity fixes addressing vulnerabilities and improving security robustness.

Best practices

We strongly recommend the following industry cybersecurity best practices:

  • Locate control and safety system networks and remote devices behind firewalls and isolate them from the business network.

  • Install physical controls so no unauthorized personnel can access your industrial control and safety systems, components, peripheral equipment, and networks.

  • Place all controllers in locked cabinets and never leave them in the Program mode.

  • Never connect programming software to any network other than the network for the devices that it is intended for.

  • Scan all methods of mobile data exchange with the isolated network such as emailed files, USB drives, etc. before use in the workstations or any node connected to these networks.

  • Never allow mobile devices that have connected to any other network besides the intended network to connect to the safety or control networks without proper sanitation.

  • Minimize network exposure for all control system devices and systems and ensure that they are not accessible from the Internet.

  • When remote access is required, use secure methods, such as Virtual Private Networks (VPNs). Recognize that VPNs may have vulnerabilities and should be updated to the most current version available. Also, understand that VPNs are only as secure as the connected devices.

For more information refer to the Schneider Electric Recommended Cybersecurity Best Practices document and the EcoStruxure Automation Expert Tested and Validated Architecture document.

For details and assistance on protecting your installation, contact your local Schneider Electric Industrial Cybersecurity Services organization or see Cybersecurity Services on the Schneider Electric website.
