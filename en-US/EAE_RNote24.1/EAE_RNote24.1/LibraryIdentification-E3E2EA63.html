Known Limitations

Revert the WEBSOCKET_SERVER to match the original design

The Websocket Server function block has the following modifications. Please refer to the below notes to make any adjustments to this block in existing applications. Additionally, refer to the F1 Help of the Websocket Server block.

NOTE: STATUS and statusId are only updated as a result of user-initiated actions on the Websocket server block.  This specifically means any INIT or REQ event that fires an INITO or CNF will update the STATUS and statusId. Client connect/disconnect and messages received from clients do not update STATUS and statusId.  Note this behavior as it may be confusing if a message fails to send to a client and the STATUS reflects 'Error'; it may still accept client connections and messages from currently connected clients.

Changes in STATUS/statusId:

  • Removed: CONNECTED, DISCONNECTED, RECEIVED and  the corresponding statusIds 1, 2, 4.

  • Added: INACTIVE statusID6.

  • Changed: ONERROR is now ERROR.

    NEW LIST:

    • STATUS: status id

    • RUNNING: 0

    • ERROR: 3

    • SENT: 5

    • INACTIVE: 6

Client Connection events (connEvent).

  • Removed: SERVER_OPEN (this value will no longer be reported in connEvent (2).

    • Now: CONNECTION_ESTABLISHED, CONNECTION_CLOSED (these are values in connEvent 0, 1)

Change in behavior:

  • No status messages will be reported in STATUS or statusId for clients connecting, disconnecting, or messages received through client connections.

  • Any client action is reported through CONN_IND and IND affecting only the outputs:  connIdRcvd and/or msgRcvd.

Standard.IoModbus Library

Address Modbus CS Threats & Vulnerabilities

The configuration of Function Code 43(Encapsulated Interface Transport) with MEI/Subcode 14 in Modbus Server has been modified in order to meet Cyber Secirity requirement of the product. The support of FC 43/Subcode 14  is now configurable. User can enable or disable the FC43 support through Modbus server parameter in the HW configuration  namely (enableReadDeviceIdentification) .  By default, the parameter is set to false meaning the FC 43 support in the Modbus Server is disabled. 

MODBUSGENTCPS and MODBUSGENBC

"connStatus" input provided with Modbus Buscouplers devices,  MODBUSGENTCPS and MODBUSGENBC, can be used to halt and resume bus communication from  user application. The input is typically used in the event any configured server is undergoing maintenance. 

[NEED MB CLIENT] MB - Explicit Read/Write MB TCP data

A new feature called Modbus Explicit introduces a set of Modbus TCP Client function blocks used to access Modbus TCP server registers. This is an additional way of accessing the Modbus fieldbus devices, through Function Block Network editor instead of through Modbus Hardware Configuration editor.

The new function blocks are part of Standard.IoModbus library. 

With new function blocks, it is possible to access Modbus TCP IOs using event based approach. It provides a flexibility of dynamically changing some properties like device address and well as register addresses.

Modbus client (TCP & RTU) VTQ

When reading a Modbus coil or register using a VTQ, we used to receive a value, a timestamp, and a quality with only 3 possible quality values: Good, Bad, and Uncertain. However, we have now enhanced our support for quality coming from the Modbus server device to reflect Modbus errors thus helping during commissioning. 

Autocalculation fails when there is a mix of ModbusRReg and ModbusWReg

Automatic register address calculation is enabled when the addressautocalculation property is set to enable and the address property is set to 0. The auto-calculation function assigns address values to registers in the order in which they appear in the Hardware Configuration editor, based on Register Type. 

Register Type can be determined by the combination of Device Type and Data Type and can be:

  • Coil bits (register range 1 - 9999), Discrete input bits (register range 10001 - 19999), Input (register range 30001 - 39999), or

  • Holding (register range 40001 - 49999).
