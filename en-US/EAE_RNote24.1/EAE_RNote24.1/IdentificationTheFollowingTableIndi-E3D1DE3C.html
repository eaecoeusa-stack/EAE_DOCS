New Features

Cyber Security Syslog Support

EcoStruxure Automation Expert is able to capture and archive an audit trail of cyber security related events.Modicon M580 Distributed PAC supports Syslogs, they can be accessed by a third-party Syslog server. 

The log of events and event details cannot be modified on the Modicon M580 Distributed PAC . 

  • The CS Syslog message format conforms to industry accepted standard format (provides specific messaging format). 

  • The CS Syslog messages are time stamped with a timestamp that is synced across EAE subsystems. 

  • The audit record storage capacity can be allocated, enlarged or reduced by configuration. 

Known Limitations

Bus Cycle Monitoring

EcoStruxure Automation Expert V22.0 introduced a compatibility break in the management of bus cycle for Modicon M580 Distributed PAC that could result in unexpected I/O glitches despite configuration settings. 

 WARNING
UNEXPECTED TRANSITION OF I/O VALUES TO INITIALIZATION VALUES
When using the option busCycleActionWhenMissed value Error halt in bus cycle monitoring configuration, make sure to use V22.0 version or greater for both Modicon M580 Dustributed PAC and SE.loX80 library.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Make sure to use V22.0 version or later for the SE.IoX80 library and the Modicon M580 Distributed PAC and to check the BMXBUS settings.

If you use the Modicon M580 Distributed PAC with a firmware V22.0 or later, upgrade the used SE.IoX80 library to V22.0 or later, otherwise the Modicon X80 bus is not supported. 

If you use the SE.IoX80 library V22.0 or later, and a Modicon M580 Distributed PAC with a firmware earlier than V22.0, the option busCycleActionWhenMissed value Error halt in bus cycle monitoring configuration has no effect.

Unexpected I/O Fallback

Promptness for Outputs

If unexpected I/O fallback occurs during maintenance operation (such as module hotswap, rack power cycle or any deploy or diagnostic operation to device from (EcoStruxure Automation Expert) update promptness parameters from each active module from Auto to 256ms. If I/O fallback still occurs, update promptness parameters from each active module from 256ms to 1s. 

 WARNING
UNPEXPECTED I/O FALLBACK
Make sure that the promptness value for outputs is set to at least the recommended value of 256 ms. Test the system under heavy network communication worload and check that inputs and outputs do not go to fallback mode unexpectedly.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Fallback Mode for Inputs and Outputs

Modicon M580 Distributed PAC supports the maintaining of I/O values (when Fallbackmode is configured to Maintain), but does not support the restart of the system with previously maintained I/O values. As a consequence, any restart temporarily sets I/O values to the initialization value (zero), until the system computes and sets the I/Os values according to the application logic. Several situations can lead to a restart, depending on the system configuration (ex: Behavior After Reboot setting set to Run) including: 

  • Device Power Cycle 

  • Reboot Device 

  • Restart Runtime 

  • Error detection putting the device in ErrorHalt state 

Refer to the Operating States chapter in the online help for more information. 

 WARNING
UNEXPECTED TRANSITION OF I/O VALUES TO INITIALIZATION VALUES
Before restarting devices, which I/Os are configured to the Maintain Fallback mode, make sure that the temporary transition of I/O values to their initialization value has no adverse effect on operations.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Update all Digital Outputs at Once with Symbolic Links

It is not possible anymore to update all digital outputs at once by using one symbolic link. You must modify the application to use explicitly one symbolic link per digital output.
