Editor for open adapter connections

This feature provides a user interface to configure an open adapter to create a solution-to-solution interface.

A new icon has been added to the main vertical navigation bar to launch the Open Adapter Editor from the System tab. This new editor is used to create cross-solution connections via an open adapter function block. This editor displays the list of plugs and sockets of open adapter instances added across device resources.

The import/export facility is provided to export the open adapter configuration from one solution and import it into the other. solution to create the necessary connection to enable a solution-to-solution interface.

MQTT_CONNECTION block enhancement

Migration notes for users of Function Block MQTT_CONNECTION in versions prior to EcoStruxure Automation Expert Version 24.1, please refer to the F1 Help for further information. Please pay particular attention to the Note sections for necessary modifications to existing applications.

EcoStruxure Automation ExpertVersion 24.1 MQTT_CONNECTION block enhancements are:

  • Security

  • Reconnection

  • Connection health/state

NOTE: EcoStruxure Automation Expert Version 24.1 provides a Runtime Configuration->Security device property Insecure Application which defaults to False.  If you are currently using insecure connections, your connections may be faulted – see the block’s F1 Help section on Insecure Configurations for further information.

Additional Events:

There are two additional output events: DISCONNECTO and CONN_STATE_IND.  These outputs require no migration from your existing blocks, though please note:

  • The DISCONNECT event no longer fires the CONNECTO event.

  • The CONN_STATE_IND event will fire for synchronous and asynchronous connection state changes.

Additional input parameters:

CertificatePassword – Allows use of an encrypted CertificateKey by providing its password.

ValidateCert – Validate the server certificate including the hostname.Three options are available with Server certificate and hostname being the default:  

  • Server certificate and hostname.

  • Server certificate.

  • No validation.

NOTE: The default is maximum security and migrated configurations will most likely not meet this security level. This, combined with device property attribute for Insecure Application may cause existing configurations to fail.

The below additional input parameters allow for reconnection.

ConnectionTimeout – The waiting time during an attempt to resolve a connection endpoint.

ConnectionRetryCount – How many retries should be attempted. Use 0 for no reconnection attempts.

ConnectionRetryTime – Wait time between connection attempts.

Additional output parameters:

The following new output parameters will contain textual status of the state of the connection.  This expands the user’s visibility into health of the connection:

  • NetworkState – messages related to the network state or health.

  • SecurityState – messages related to the security layer of the connection.

  • ProtocolState – messages related to the MQTT protocol.

NOTE: STATUS and ReturnCode  report additional values.

Make CAT/SubApp instance navigable from solution treeview to AVEVA OMI graphics

EcoStruxure Automation Expert now supporting navigation of SubApp instances to locate its usage in different editors.

While mouse hovers on the SubApp instance, it enables three icons to be searched in different ways and right-clicking will also provide context menu, as given here:

  • Locate Instance in FB network editor:

    • The System - Function Blocks Network is shown.

    • The context is changed to the application into which the SubApp instances of the mapped CAT is present.

    • The focus is put on that instance and is highlighted.

    NOTE: If no instance is found, a message such as Usage of instance name in system (0 Occurrence's in 0 files) is displayed in the search result pad.

  • Locate Instance in OMI Graphics:

    • Clicking on Locate Instance in OMI Graphics icon returns a search result.

    • Double-clicking on this result will locate the CAT instance usage in the OMI graph.

  • Locate Instance in System:

    • Displays a search result for a system with a hierarchy, where you can double-click to navigate to the location where it was used

Quality status returns a bad value when communication between Master and Slave is lost

This problem occurs ONLY with this configuration:

  • High Availability Soft dPAC

  • BMECRD0100 or BMECRD0100C

  • One BMEAHI0812∕BMEAHI0412 module with corresponding BME_AHI_0812∕BME_AH0_0412 HWCATs

In this particular situation, Watch inside the HWCONF on BME_AHI_0812∕BME_AH0_0412 HWCATs under BC_RIO_EIP node show a quality=good even if communication between BMECRD0100 or BMECRD0100C and Soft dPAC is lost.

But the quality has the expected value on BME_AHI_0812∕BME_AH0_0412 HWCATs under BC_RIO_XBUS.
