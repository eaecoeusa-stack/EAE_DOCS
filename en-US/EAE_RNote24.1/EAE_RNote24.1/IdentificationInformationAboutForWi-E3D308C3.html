EcoStruxure Automation Expert - Soft dPAC

Identification

Information about EcoStruxure Automation Expert - Soft dPAC for Windows and EcoStruxure Automation Expert - Soft dPAC for Linux is available in the EcoStruxure Automation Expert - Soft dPAC installation package.

Known Limitations

Limitations observed for MQTT Client on SoftdPAC

When running only EcoStruxure Automation Expert and the broker:

  • For HP6 Celeron and Essential Edge SoftdPAC devices with a cloud-based broker, the recommended maximum is 1,000 topics at a 1-second interval rate for QoS 2.

  • For Windows SoftdPAC devices with a standalone broker, the recommended maximum is also 1,000 topics at a 1-second interval rate for QoS 2.

To prevent broker disconnection, monitor CPU usage according to your application’s requirements.

Keep in mind that QoS 1 and QoS 2 consume significantly more resources than QoS 0, both on EcoStruxure Automation Expert and the broker. If the transferred data is updated frequently, QoS 0 is usually sufficient. Use QoS 1 or QoS 2 only for critical data where no updates must be lost.

Cybersecurity Limitation

All Linux EcoStruxure Automation Expert - Soft dPAC instances, instantiated from the Physical View in EcoStruxure Automation Expert - Buildtime, are secured by default, using secure communications, certificate management and device level authentication. 

If you use the legacy workflow for creating a Linux. EcoStruxure Automation Expert - Soft dPAC, there are currently no options to provide security features for the device. 

Installation Side by Side with EcoStruxure Automation Expert

By default, Local Test (simulation) mode and EcoStruxure Automation Expert - Soft dPAC are using the same port number to communicate with EcoStruxure Automation Expert. 

If both, Local Test (simulation) mode and EcoStruxure Automation Expert - Soft dPAC are running at the same time on the same computer, it is not possible to log in and deploy an application to the Local Test (simulation mode). 

To be able to log in and deploy to simulation controller, change the EcoStruxure Automation Expert - Soft dPAC Manager port number to an unused port. The default value is 51499. 

For support on installation, contact your local administrator. 

System Functions of EcoStruxure Automation Expert - Soft dPAC for Windows

EcoStruxure Automation Expert - Soft dPAC for Windows does not support the following services: 

  • Discovery from the Physical devices editor 

  • Set remotely the IPv4 address from the Physical devices editor 

  • Locate remotely 

  • Diagnostics from the Physical devices editor 

For more information on how to commission EcoStruxure Automation Expert - Soft dPAC for Windows, refer to EcoStruxure Automation Expert User Manual.
