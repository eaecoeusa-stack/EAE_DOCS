Multi-user capability

You can now collaborate with up to five colleagues simultaneously on the same project, using EcoStruxure Automation Expert Platform

Two configurations are supported:

Configuration N°1:

  • One engineering workstation hosting at the same time three EcoStruxure Automation Expert Platform and EcoStruxure Automation Expert.

  • Up to 4 engineering workstation hosting EcoStruxure Automation Expert.

    Configuration N°2:

    • One server hosting EcoStruxure Automation Expert Platform.

    • Up to 5 engineering workstation hosting EcoStruxure Automation Expert.

This initial phase of the multi-user capability helps prevent merge conflicts, particularly when design outcomes must be shared with colleagues. To achieve this, a reservation mechanism is implemented based on a zone concept: when a user reserves a zone, it becomes read-only for others.

Zones are hierarchical:

  • A zone can contain children zones.

  • A zone can have dependencies.

In such case, When a user reserves a zone, all its dependent zones are automatically reserved as well.

Devices themselves function as zones. If a user is interacting with a device (logging in, deploying, monitoring, etc.), others cannot access it simultaneously.

The reservation lock is managed by EcoStruxure Automation Expert Platform, requiring a live connection from each engineering workstation to EcoStruxure Automation Expert Platform.
