Modicon M262 Distributed PAC

Overview

Modicon M262 Distributed PAC is designed for performance-demanding machines, IIoT-ready and combine logic, motion and safety control applications.

The controller embeds two Ethernet-based networks (EtherNet/IP, Modbus/TCP) and one serial port (Modbus, ASCII). It is powered with a non-isolated 24 Vdc power supply, has a built-in overload protection, embeds a dual-core processor, reaches a 5ns/instruction performance. The controller has one communication bus (Ethernet), one I/O bus, one I/O port.

The controller is designed for machinery requiring integrated IIoT connectivity or encrypted communication.

Modicon M262 Distributed PAC installs on 35mm DIN rail with Clip-on locks. The controller is the IIoT-ready controller for performance machines.

New Features

Reliable cross communication

Modicon M262 Distributed PAC now supports Reliable Cross Communication (HASRT). 

For more information on Reliable cross communication, refer to the EcoStruxure Automation Expert User Manual. 

TM3 Bus robustness improvement

Modicon M262 Distributed PAC will not go to error halt state due to a bus error detected on TM3 IO.. 

IP routing through Ethernet ports

ETH ports on Modicon M262 Distributed PAC can now be used for IP routing. 

Known Limitations

Ethernet Interfaces

Explicit messages can be sent/received through both Ethernet interfaces. 

The EIP_EXPLICIT function block can be used 30 seconds after DPAC_FULLINIT execution.

Memory Consumption

The Modicon M262 Distributed PAC enters in ErrorHalt state if it does not have enough free memory (less than 4MB). 

Online Change increases memory consumption and requires at least 5MB of free memory to complete successfully. While performing Online Change, if the free memory becomes less than 5MB, the Online Change is refused. You can monitor the memory consumption information through the MIBGET function block with the entry System.PhysMemFree. 

Perform the following steps to allow the Online change again or to allow the Modicon M262 Distributed PAC to enter in RUNNING state: 

Reduce required memory by: 

  1. Disabling both OPC UA client and server if you don't use it (allows you to save about 11MB memory). 

  2. Mapping only on useful EtherNet/IP variables.  

    Do not exceed: 

  1. 500 mapped EtherNet/IP variables if OPC UA is enabled. 

  2. 800 mapped variables if OPC UA is disabled. 

For EtherNet/IP remote I/O devices, prioritize mapping on the whole word instead of mapping on the bits of the word. The function block BITMAN can be used inside the application to extract the bit of the word. 

  • Deploy the application to the Modicon M262 Distributed PAC.

Modbus Serial RTU Polarization

The polarization on the Modbus Serial RTU is not effective.
