New Features

ATV dPAC and Graphic Display Terminal

The ATV dPAC Graphic Display Terminal supports extended data types, allowing you to display and set process application data efficiently.

  • Real-time clock: Equipped with a 10-year backup battery to monitor time even when the power is off.

  • Pixel resolution: 240 x 160, capable of displaying 8 lines of messages.

  • Protection: IP65 rating offers durability and resistance to external elements.

The ATV dPAC can display EcoStruxure Automation Expert application variables on the Graphic Display Terminal of its host Altivar drive, by customizing the following UI elements:

  • Monitoring and Setting screens, based on user-selected CAT instance variables.

  • Information screens.

  • Menus and sub-menus.

The Design Engineer can create the ATV HMI tree view in EcoStruxure Automation Expert. After deployment, it is displayed on the ATV Graphic Display Terminal, and can be used by the Operator.

ATV Mailbox Parameters for Exchanging ATV dPAC Data

The Drive Parameters Mailbox facilitates the exchange of specific parameter values between the ATV dPAC and the drive. There are two types of parameters:

  1. Non-Persistent Exchange Parameters [DEP•] (• from 0 to J):

    • 20 non-persistent exchange parameters.

    • Not saved by the drive over a restart (when it is turned off and on again).

    • These parameters can be read or written by the ATV dPAC and the drive:

      • On-event, using the ATV_DD_IN and ATV_DD_OUT function blocks.

      • Cyclically, using the ATVCyclicScanner hardware CAT, if their value changes frequently.

    • Note: Manual parameter entry in ATVCyclicScanner is required. These parameters should be selected carefully, as the ATV dPAC IO scanner input and output tables are limited to 32 parameters of 16 bits each, including parameters from other hardware CATs.

  2. Persistent Exchange Parameters [DSP•] (• from 0 to J):

    • 20 persistent exchange parameters.

    • Saved by the drive and retained over a drive restart (when the device is turned off and on again), for example, in case of power loss.

    • These parameters can be read or written by the ATV dPAC and the drive:

      • On-event, using the ATVPersistEvent function block, if their values do not change frequently (e.g., actual system state).

      • Cyclically, using the ATVPersistCyclic function block, if their values change frequently.

ATV EtherNet/IP Explicit Messaging Server for Drive and Mailbox Parameters Access

The ATV EtherNet/IP Explicit Messaging Server enables access to drive and mailbox parameters over an Ethernet network. It facilitates communication between the network and the drive, allowing for the retrieval and modification of various settings and parameters.

To access a drive or a mailbox parameter using EtherNet/IP messaging server, use the CIP path value defined in folder 6. Altivar Communication parameters, within the SEDP_ATVDP delivery archive folder.

Fast Device Replacement (FDR)

The Fast Device Replacement (FDR) service helps quickly set up new drives and controllers on an Ethernet network. When a device needs replacing, FDR automatically configures the new one using settings from a server. This speeds up the process and reduces the need for manual setup, allowing everything to run smoothly with minimal downtime. Additionally, FDR simplifies the restoration of the ATV dPAC application, making it easy to reapply all configurations and settings..

MQTT Server for IoT Integration

The MQTT server facilitates IoT integration by enabling access to the ATV Distributed PAC application from standard MQTT brokers..

Known Limitations

EtherNet/IP Device Management

The Altivar dPAC Module VW3A3530D does not support the EtherNet/IP Bus Master, and cannot control EtherNet/IP devices.

The Altivar drives EtherNet/IP hardware CATs can be controlled by other dPAC.

PositionerVSDBase Application CAT with Altivar dPAC Module VW3A3530D

There are certain limitations associated with using the PositionerVSDBase application CAT in conjunction with ATV dPAC. These limitations can impact the functionality and performance of the system. Be aware of these limitations when configuring and using the PositionerVSDBase application CAT with an Altivar dPAC Module VW3A3530D. 

The AtvPosControl Hardware CAT and AtvPosInterface function block must be used to interface with the Positioner library.

Time Management

In Altivar drives, the battery used to save the time is located inside the Graphic Display Terminal. In order to keep the clock correct in conditions of loss of power, the Graphic Display Terminal must be connected before the drive is powered off.

In Altivar Machine ATV340 range, the Graphic Display Terminal must be purchased separately. The product reference is VW3A1111.

NOTE: The Graphic Display Terminal VWA3A1111 cannot be directly mounted on ATV340 drives up to 22 kW.

ATV dPAC Displays

When you rename or make changes to a screen in the ATV dPAC display, these changes might not update Deploy and Diagnostic view for online change. Additionally, you can't verify if the latest changes were deployed correctly because the deploy and diagnostics view incorrectly shows that the ATV dPAC is up to date. To ensure the latest changes are deployed correctly, you may need to perform a normal deploy.

NOTE:

In the 24.1 release,, if you try to display a value that is the 33rd linked to the same event, it will not be updated on the Graphic Display Terminal. Therefore, we are limited to 32 values per event on the IThis component.

This limitation only affects the ATV dPAC Display feature and does not impact any other features. If you do not use this feature, you can disregard this limitation.

In the 24.0 release, this limitation was even stricter, allowing only 16 values per event.

Additional Delay in ComStatus Parameter of MBTCP Hardware CATs

When there is a communication interruption, the ComStatus variable will change from True to False, but this change will occur after a delay. This delay is influenced by the value you set for the ComStatusTimeout parameter, plus an additional delay of 3 to 4 seconds.

Store to Device Functionality limitation

The function Store to Device on SoMove is not working properly for ATV9•• and ATV6•• .

To resolve the issue, you need to change the firmware version in the DTM topology settings from V4.2IEXX to V4.3IEXX.
