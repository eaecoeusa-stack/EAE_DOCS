Distributed Components New Features

Multiple resources for jitter reduction

On DPAC M26d and Soft dPAC docker Linux non-HA we have now the ability to create 3 IEC61499 resources with different level of execution priority (Normal, High, Highest). We are also providing a guideline to lower jitter on IO and avoid event storm on online change by for example configuring Bus Master on resource with very high execution priority.

EtherNet/IP

NOTICE
unexpected behavior
Adding more than one EtherNet/IP Scanner per dPAC leads to unexpected behavior while communicating with the EtherNet/IP Scanner devices. Ensure to have declared only one EtherNet/IP Scanner per dPAC, and remove any extra EtherNet/IP Scanner. 
Failure to follow these instructions can result in equipment damage.

EtherNet/IP Scanner

The EtherNet/IP Scanner is supported by the following platforms: 

  • EcoStruxure Automation Expert - Soft dPAC (Ubuntu) 

  • Modicon M580 Distributed PAC 

  • Modicon M251 Distributed PAC 

  • Modicon M262 Distributed PAC 

The EtherNet/IP Scanner is not supported by the Local Test Runtime (Simulation) of EcoStruxure Automation Expert. Deployment to Local Test Runtime (Simulation) is not possible.

System Performances Related to Events Management

Incorrect application design often results in unnecessary events multiplication: a first set of events triggers a cascading effect of more events creation, quickly resulting in an event storm that temporarily saturates the system processing capabilities. Typical symptoms of event storms include: 

  • Apparent delay in process execution 

  • Log warning messages showing event queue overwhelming 

  • Glitches on I/Os going to fallback 

  • Loss of cross communication or delays 

Therefore careful examination, understanding and design of the application is required to prevent or solve such issues. Refer to the EcoStruxure Automation Expert Applications Design Guidelines Reference Manual, available in the online help.
