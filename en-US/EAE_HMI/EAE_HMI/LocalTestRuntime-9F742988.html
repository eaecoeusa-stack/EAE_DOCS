Simulation Runtime

The simulation runtime is a component of the package EcoStruxure Automation Expert. Runtime Simulation is a distributed runtime system according to standard IEC 61499 that you can use to run your solution in simulation mode.

 WARNING
NON-DETERMINISTIC APPLICATION BEHAVIOR
Do not use Simulation with Microsoft Windows™ operating system for real-time operations.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

Starting Runtime Simulation

This feature – also known as simulation mode – is configured and operated using the engineering software EcoStruxure Automation Expert - Buildtime.

Configure simulation mode by setting the Network Profile field to (Local Test) in the Logical Devices editor.

Operate devices in simulation mode after setting the Active Network Profile field to (Local Test) in the  Deploy and Diagnostic editor, then using the commands in the Action menu.

When the simulation is started local addresses (127.0.0.1) are automatically used for dPAC simulated runtimes. But the configured addresses for other devices (for Modbus TCP, OPC UA...) are used as is and running the application may affect these physical devices also in Local Test (simulation) mode.

 WARNING
UNINTENDED EQUIPMENT OPERATION
When using the Local Test mode (simulation), make sure that the potential activation of communications with physical devices, addressed through the Hardware Configuration or anywhere in the application logic (i.e.: Modbus TCP, NETIO, OPC UA, etc...), will not cause unwanted effects to the operations.
Failure to follow these instructions can result in death, serious injury, or equipment damage.

For a description of the available commands, and their sub-menu commands, refer to the EcoStruxure Automation Expert User Manual topic for the Deploy and Diagnostic editor.

NOTE: The default port for the simulation runtime for a single device is TCP or UDP Port 51499. If more than one device is used, the default port of the next device is 51500, and so forth. If you receive a detected error message such as the following while starting:

2013-05-06-11:28:50.387 E: SrcID: 253 SubID: 16: TCP-Socket bind() failed: 10013 or T#notime: ERROR: SrcID: 253 SubID: 16, :: UDP Bind failed: 10013

This indicates that  the ports are already used by another software application. In this case close this application or change the port configuration in project and start simulation runtime anew. (Choose the device in the Logical Devices editor, and adjust the port settings in the properties.)

Some hardware (e.g. printer) allocate several ports during installation without actively using them. These ports are not visible with netstat and other tools as long as they are only allocated but no process is running. That means if you receive the detected error 10013 you need to verify whether the specified ports are in use or free.

If you receive the following detected error when loading a project onto the runtime simulator:

DEV0:ID=18, Action=START: INVALID_STATE, Detail: 532 Stored persistent data do not match this project, delete it and start over

take the following action:

  1. In the Deploy and Diagnostic editor Action column, select Persistence Data > Delete Persistent Data at the affected device.

  2. Then deploy anew.
