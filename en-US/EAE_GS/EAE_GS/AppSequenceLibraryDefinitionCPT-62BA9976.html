SE.AppSequence Library Definition

As described in the topic Filling Sequence Definition, the filling of the tank TR1 goes through several steps. Each of them follow the same pattern:

  1. Activation.

  2. Action.

  3. Wait for the transition condition to be true.

  4. Deactivation.

As the sequence is clearly defined and has for each step the same pattern, the SE.AppSequence library can be used to develop it.

The SE.AppSequence is composed of several function blocks.

Here are the ones that will be used for the sequence:

  • SeqHead: Sequence Head

  • SeqStep: Sequence Step

  • SeqTerminate: Sequence Terminate

This SE.AppSequence library and these function blocks will allow the user to:

  • Develop step-by-step applications with, for each of their steps, actions and transition conditions.

  • Control the sequence by providing sequence stages as Running, Stopped, Held or Completed.

NOTE:

To get further information about the SE.AppSequence library, go to its documentation by pressing F1 on SE.AppSequence in the Solution Explorer pad.
