Stop Sequence Steps

As explained in the topic Stop Filling Sequence, the two steps of the Stop sequence are indentical to the fifth and sixth steps of the Filling sequence. Which means that most of the function blocks used in the Filling sequence for the actions and transitions can be reused.

What cannot be reused is the SeqStep function blocks. Each sequence needs its own one.

Then add two SeqStep function blocks, one per step, name them StopStep1 and StopStep2 and link their ISeqChain adapter to create a chain.

Step 1: Closing the valves

As defined in the topic Stop Filling Sequence, in this step the reception and the inlet valves have to be closed even though the level may not be sufficient in the tank TR1 since the Filling sequence is required to stop.

To do so, connect StopStep1.STEP_ENTER to the DS_SELECTX function block already used in the Filling sequence for this action, which is inletValveCommand.REQ2.

The actions of closing the inlet and reception valves will be executed the same way as the fifth step of the Filling sequence is doing. So the action of this first step is done. To complete the step, the transition condition has to be set.

As defined in the topic Stop Filling Sequence, the transition condition of this first step is: both inlet and reception valves are closed.

This condition is already checked with the function block InletReceptionCloseCheck. Then to set the transition condition to StopStep1:

  1. Link InletReceptionCloseCheck.CNF to StopStep1.TRANS_REQ.

  2. Link InletReceptionCloseCheck.OUT to StopStep1.Exceeded.

  3. Add the constant 2 to StopStep1.TransStepId.

Step 2 : Turning OFF the agitator

As defined in the topic Stop Filling Sequence, in this step the agitator is turned OFF.

To do so, connect StopStep2.STEP_ENTER to the DS_SELECTX function block already used in the Filling sequence for this action, which is agitatorCommand.REQ2.

The actions of turning OFF the agitator will be executed the same way as the sixth step of the Filling sequence is doing. So the action of this second step is done. To complete the step, the transition condition has to be set.

As defined in the topic Stop Filling Sequence, the transition condition of this second step is: the agitator is turned OFF. To check this, a SMOOTH function block will be used the same way as agitatorOffCheck.

  1. Add a SMOOTH function block and name it StopM01OffCheck.

  2. Link agitatorOffCheck.INITO to StopM01OffCheck.INIT.

  3. Link agitatorOffCheck.CNF to StopM01OffCheck.REQ.

  4. Link agitatorStatusDecode.Passive to StopM01OffCheck.IN1.

  5. Link StopM01OffCheck.CNF to StopStep2.TRANS_REQ.

  6. Link StopM01OffCheck.OUT1 to StopStep2.Exceeded.

  7. Add the constant 3 to StopStep2.TransStepId.

The development of the two steps of the Stop sequence is now completed. Cross reference the links between these new function blocks to the already existing ones.

Step Terminate

To complete the development of the sequence, add a SeqTerminate function block, name it StopTerminate and link it to StopStep2.

The Stop sequence is now fully completed. It will allow the operator to stop correctly the Filling sequence whenever needed.
