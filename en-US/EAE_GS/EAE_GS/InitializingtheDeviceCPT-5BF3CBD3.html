Initializing the Device

An additional function block needs to be added directly in the resource.

NOTE:

This function block will perform the correct initialization of the applications and function blocks assigned to it.

This function block is DPAC_FULLINIT.

This function block is used to initialize all the application and hardware function blocks by sending them the initialization trigger.

 
NOTE:

If this initialization trigger is not sent, then the function blocks won’t operate online.

On the application side, the initialization trigger is received by the plcStart function block.

This function block is then used in all the instruments’ function blocks to initialize them and set them ready to be used online.

 

Add to the resource layer from the Function Blocks Network editor the DPAC_FULLINIT function block and named INITIALIZE.

 

As visible in the image above, the layer already contains an other function block named START from the E_RESTART function block.

This function block indicate by events all the different restarts of the device. It triggers COLD if the device is doing a cold restart, WARM if a warm restart and so on.

This function block then must be linked to:

  • INITIALIZE to initialize the function blocks every time the device is restarting.

Then, link:

  • START.COLD and START.WARM to INITIALIZE.INIT for initialization.

  • START.ONLINECHANGE to INITIALIZE.OC_RETRIGGER for the online changes.
