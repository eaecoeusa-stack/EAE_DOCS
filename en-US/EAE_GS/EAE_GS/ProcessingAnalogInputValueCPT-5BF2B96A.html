Processing the Analog Input Value

As required in the topic Purpose of the Connections, the TR1LT01 analog input process value must be compared to a constant value.

NOTE:

If the event of the process value is greater than or equal to the constant, the result of the comparison will be ‘TRUE’ otherwise ‘FALSE’.

To make this comparison, use a COMPARE function block.

 

This function block gets two values as inputs through IN${CNT}:

  • IN1 for the 1st input

  • IN2 for the 2nd input

NOTE:

These two input variables get updated thanks to the REQ event.

Whenever REQ triggers, the first input value is compared to the second one.

The result is given for all the possible comparison:

  • LT : ‘TRUE’ if IN1 is Lower than IN2.

  • LE : ‘TRUE’ if IN1 is Lower than or Equal to IN2.

  • EQ : ‘TRUE’ if IN1 is Equal to IN2.

  • NE : ‘TRUE’ if IN1 is Not Equal to IN2.

  • GE : ‘TRUE’ if IN1 is Greater or Equal than IN2.

  • GT : ‘TRUE’ if IN1 is Greater than IN2.

NOTE:

In our example, the GE output will be used since the process value has to be greater than or equal to the constant value.

Now, add a COMPARE function block and name it LevelCompare.

 
NOTE:

The inputs are currently indicating a detected error. It is because their types have to be configured.

To configure types of input:

  1. Go to the Interface Editor of LevelCompare.

     
    NOTE:

    In the Interface Editor, you see that the two inputs currently have the ANY type. That is the reason why the inputs both raise error.

     

  2. As the process value of TR1LT01 is given the VTQREAL type, the two input types will be set to REAL.

     
    NOTE:

    The VTQ types (as VTQBOOL, VTQREAL…) transmit a value and additional information. As only the REAL value matters for the comparison, the REAL type has been selected instead of VTQREAL.

    The REAL value of the VTQREAL PV will then be transmitted while the additional ones will be lost.

  3. To change the type of the inputs in the Interface Editor, click on the cell containing the value ANY.

    It becomes editable.

     

  4. Then, either:

    • Type REAL in the cell.

    or

    • Click on the downward arrow to show a drop-down list.

      Then, look for the REAL type and select it.

     

  5. Press OK to validate.

The detected errors on the inputs disappear.

Now to send the process value to LevelCompare:

  1. Link TR1LT01.Pv to LevelCompare.IN1 and TR1LT01.CNF to LevelCompare.REQ.

     

  2. Set a constant to IN2.

    A blank box linked to IN2 appears.

  3. Type the desired value and validate by pressing Enter (or by clicking anywhere in the layer).

     
    NOTE:

    Sometimes, the constant value does not appear after pressing Enter. To show it, unselect the function block by clicking anywhere in the layer.

The process value of the analog input IN1 is now compared to the constant 82.5 every time the REQ event is triggered.
NOTE:

The constant value can be replaced anytime by any other real value. The change will be updated the next time the REQ event is triggered.
