Opening the Third Valve

As previously described in the topic Purpose of the Connections, the valve TR1V03 gets opened when:

  • The valve TR1V01 and TR1V02 are opened and when the agitator TR1M01 is ON.

NOTE:

The agitator is turned ON manually by the operator through the HMI. The valves TR1V01 and TR1V02 have been already set in the previous topics.

The information needed to command the opening of the valve TR1V03 comes from:

  • The open feedback from TR1V01 and TR1V02

  • The running feedback from the agitator TR1M01.

 

To command the opening of TR1V03, the feedback of these three actuators need to be activated at the same time: Agitator ON and the two valves opened.

To check this condition, a logic function block ‘AND’ will be used:

  1. Add it to the layer and name it AgitatorOnValvesOpen.

     

    The three feedbacks will then be linked to this function block and its output will be the answer to this logic ‘AND’ condition.

    However, AgitatorOnValvesOpen currently has only two inputs available, instead of the three needed.

    An additional one must be added.

    NOTE:

    One variable input can only be linked to one output variable.

    The opposite is not true though, an output variable can be linked to several input variables. The input and output event can also be linked to several events.

  2. Add an input variable to AgitatorOnValvesOpen using one of the two following methods:

    • Method 1

      1. Go into the Interface Editorof the function block.

         

      2. Then, change the count of input from 2 to 3.

      3. Validate the changes.

         

    • Method 2

      Drag down the bottom right icon of the function block until having the right number of variables.

       

Once done with one of these two methods, it should look like the image below:

 

Connect each of the input variables of AgitatorOnValvesOpen to the needed feedback of the three actuators.

For the events, each ‘CNF’ of the actuators has to be linked to AgitatorOnValvesOpen.REQ.

So, each of them can update the input variables of the ‘AND’ function block and can trigger the inner logic of the function blocks.

NOTE:

Even though one trigger from any actuator could update the inputs and trigger the inner logic of the function block, it is still important that each actuators ‘CNF’ event is linked.

Example

If only the event of the agitator output event is linked:

  • The values of the valves may change but they will not be considered until the TR1M01 value changes too.

  • CNF is triggered. But maybe the valves’ value would have changed several times before the agitator got triggered and the AND function blocks would not have been able to notice those changes.

To avoid that potential issue, link TR1M01.CNF, TR1V01.CNF and TR1V02.CNF to AgitatorOnValvesOpen.REQ.

At this stage, AgitatorOnValvesOpen has been correctly configured. Hence, when all the three feedbacks turn ‘ON’, then AgitatorOnValvesOpen’s output will also turn ‘ON’.

To transmit AgitatorOnValvesOpen output value to TR1V03:

  • A transmission composite needs to be added and linked between the ‘AND’ function block and TR1V03 (like it has been done for the previous valve).

To do so, proceed the following steps:

  1. Add a Transmission composite and rename it TransVal3.

  2. Link AgitatorOnValvesOpen.OUT to TransVal3.Value.

  3. Link AgitatorOnValvesOpen.CNF to TransVal3.REQ.

  4. Link TransVal3.Isc to TR1V03.ISc.
