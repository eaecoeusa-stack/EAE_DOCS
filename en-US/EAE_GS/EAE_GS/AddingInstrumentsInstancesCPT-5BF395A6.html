Adding Instruments Instances

All the instruments have to be represented on the canvas.

NOTE:

As the instruments state changes as the simulation runs, their symbol also need to be changed to match the state. That is the reason why the canvases are considered as dynamic.

The dynamic representations to the canvas are located in the CAT Instances node, under the project GettingStarted in the Solution Explorer pad.

 
NOTE:

If the CAT instances of the instruments don’t appear then you must save all. It will update all the elements that have changed since the last save (elements added, renamed or deleted).

To do so, either press CTRL + SHIFT + S or click on the Save All icon.

 
The editing canvas menu is now opened.

To add a CAT instance to the canvas, drag-and-drop the needed one from the list into the canvas. Add this way TR1M01.

 
The Select generic symbol dialog box opens. It allows you to choose the symbol needed, between the ones available.
 
NOTE:

For one type of instruments, various uses may be available.

E.g. This cyclic motor can be represented as a motor, pump and agitator. As these instruments are quite different, it is then normal to get different aspects available to differentiate them.

When various uses are available for one type of instruments:

  1. Select sAgitator and validate.

    It is then added to the canvas.

     

  2. Then, select the following aspects for the next instruments:

    • FinalValidation: only one aspect is available

    • TR1LSH01: sIndicator

    • TR1LSL01: sIndicator

    • TR1LT01: sDefault

    • TR1V01: sLflValve

    • TR1V02: sLflValve

    • TR1V03: sLflValve

  3. Now, let’s order the instruments so it follows the same graphical aspect as on the layer.

NOTE:

If you have forgotten which instance belongs to which instrument: Check the Properties pad. Its name is shown.
