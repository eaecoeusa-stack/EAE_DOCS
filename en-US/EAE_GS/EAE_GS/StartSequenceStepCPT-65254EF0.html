Start Sequence Step

As detailed in the topic Filling Sequence Definition, this starting step is controlling the initial conditions allowing or not the sequence to begin.

NOTE:

To check the initial conditions, a SeqHead function block will be used.

Drag and drop a SeqHead function block and rename it StartHead.

 

As mentioned above, StartHead is checking the initial conditions. These conditions are:

  • Not Level high

AND

  • Not TR1V03 open

Level High Condition

To check the level of the tank, the information needs to be imported from outside the CAT. To import something from outside the CAT, inputs need to be added, as detailed below:

  1. Go to the Interface editor.

  2. Add an event input and name it REQ_LvlHigh.

  3. Add a variable input and name it LevelHigh.

  4. Change the type of LevelHigh from INT to BOOL since the value is a boolean: the high level sensor in the tank can only be TRUE or FALSE.

 
NOTE:

The LevelHigh input variable is currently in italics because it is not connected with any event. If it is let like this, it means that it won’t be able to be updated along the sequence.

LevelHigh has to be connected with an input event.

To do so, there are two methods:

  • Method 1:

    1. Click in the cell.

    2. Open the With Type Editor dialog box of the event to connect with LevelHigh, which is REQ_LvlHigh.

    3. Tick the LevelHigh checkbox to connect with REQ_LvlHigh .

    4. Click OK.
       

  • Method 2:

    From the Interface editor, drag LevelHigh and drop into the REQ_LvlHigh With line.

     
    NOTE:

    During the drag-and-drop, when the mouse hovers the With line, the cursor appearance will change to indicate the dropping is available.

LevelHigh is not in italics anymore: The input is ready to be used for the initial conditions.

Valve TR1V03 condition

You have to check the status of the TR1V03 valve. The dialog between the TR1V03 valve and the CAT is done through the IPasteurizerValve adapter.

It means that this adapter is transmitting information with the other IDevice adapter located in the function block of TR1V03.

NOTE:

The status of TR1V03 is received through the Status output variable of IPasteurizerValve. The status is given in a VTQINT type.

 

To be used in a condition, the Status has to be decoded into a boolean:

  1. Add a decodeState function block and name it pasteurizerValveStatusDecode.

  2. Link IPasteurizerValve.CNF_STATUS to pasteurizerValveStatusDecode.REQ.

  3. Link IPasteurizerValve.Status to pasteurizerValveStatusDecode.State.

 

Initial Condition Links

Now all the inputs of the conditions have been set, then they need to be linked to StartHead with the right logic.

As the valve state needs to NOT be opened and the level state NOT high, their signals have both to be inverted:

  1. Add a NOT function block and name it LvlNotHighV03NotOpen.

  2. Add a second input to LvlNotHighV03NotOpen using either method detailed in the topic Opening the Third Valve.

  3. Link the input REQ_LvlHigh event to LvlNotHighV03NotOpen.REQ and the LevelHigh input variable to LvlNotHighV03NotOpen.IN1.

  4. Link pasteurizerValveStatusDecode.CNF to LvlNotHighV03NotOpen.REQ and pasteurizerValveStatusDecode.Active1 to LvlNotHighV03NotOpen.IN2.

Now that the inputs have been inverted, they need to be controlled with an AND function block since both need to be FALSE at the same time to allow the sequence to start:

  1. Add an AND function block and name it LvlHighV03Check.

  2. Link LvlNotHighV03NotOpen.CNF to LvlHighV03Check.REQ.

  3. Link LvlNotHighV03NotOpen.OUT1 to LvlHighV03Check.IN1.

  4. Link LvlNotHighV03NotOpen.OUT2 to LvlHighV03Check.IN2.

  5. Link LvlHighV03Check.CNF to StartHead.REQ_PERM.

  6. Link LvlHighV03Check.OUT to StartHead.PermStart.

The initial conditions are now correctly checked. When StartHead.PermStart becomes TRUE, which means that both TR1V03 and Level high are FALSE, then the sequence is allowed to start.

Simplifying the Network

Before moving to the next step, what has been developed so far shall be organized to simplify the network.

NOTE:

This action might feel useless due to the few amount of function blocks in the network, but doing this regularly will keep the network readable even when all the steps have been added.

  1. Hide the unused ports of pasteurizerValveStatusDecode.

    Since no other ports will be used during the development, unused port can be hidden.

     

  2. Links between pasteurizerValveStatusDecode and LvlNotHighV03NotOpen shall be cross referenced.

    NOTE:

    This action hides the link between the function blocks. Instead is appearing a square on both of the ports to indicate a cross reference is on these ports.

    Here are some actions available with the cross reference: (the list of action is related to the graphic below)

    1. Simple click on a cross reference box: Highlights the cross reference linked to the selected one.

    2. Double click on a cross reference box: Brings to the cross reference box linked to the selected one or if the port has more than one cross references then shows the list of them.

    3. Right click on a cross reference box: Opens the dialog box of the cross reference which allow to:

      1 | Bring to the cross reference box linked to the selected one.

      2 | Enable / Disable the cross reference

      3 | Delete the link.

      4 | Link the current port to another port through cross reference.

     

Increase Space of the Network

To increase the size of a layer or a network horizontally or vertically and then having more space to put function blocks and organize them, click on the grey arrow box on a side of the network window. It will increase the size on the direction arrow clicked.
