Get the Filling Sequence Started

Now that the Watch, Trigger Event and Force Value actions are known and that TankFillingSequence has been initialized, let’s get the Filling sequence ready to be started.

The first action is to enter inside the CAT. To do so, double click on it.

The CAT opens and all the function blocks previously developed are visible.
NOTE:

This function block is a child of the class TankFilling, so it is composed of all its elements. Then no modification can be done in TankFillingSequence directly. If something has to be modified, it has to be done in TankFilling directly. Hence, this will modify TankFillingSequence and all the other CATs with this class in the project.

To start the sequence, its permissive input shall first be approved. To do so, add to watch StartHead and the function blocks related to its permissive.

From the graphic above, something odd is noticeable. Even though both LevelHigh and TR1V03 Open inputs are FALSE, as wanted in the definition of the sequence, the permissive input of StartHead is still FALSE.

It is because the NOT and AND function blocks need to be triggered once to get rid of their default values and be correctly updated.

To change that, come back on the layer level through the Function Block Network breadcrumb or using the navigation button of the System editor.

Trigger REQ_LvlHigh then come into the CAT to notice the update.

Triggering the input has lead to trigger StartHead, NOT and AND function blocks. Their REQ and CNF events are set to 1.

They are now updated to the correct value and the permissive input of StartHead is set to TRUE. The Filling sequence is ready to be started.
