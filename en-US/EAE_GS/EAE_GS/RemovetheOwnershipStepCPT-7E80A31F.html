Remove the Ownership Step

To start the development of the last step, add a new SeqStep function block, name it Step7 and link Step6.ISeqChainOut to Step7.ISeqChainIn.

NOTE:

As defined in the topic Filling Sequence Definition, in this step the ownership of instruments used during the sequence is removed.

As done in the previous topics, the DS_SELECTX function blocks used to take the ownership of the instruments will be reused to remove the ownership.

  1. Add a second input to agitatorProgLocCommand, inletValveProgLocCommand and receptionValveProgLocCommand.

  2. Give them the FALSE constant value.

  3. To create the chain of actions, link Step7.STEP_ENTER to agitatorProgLocCommand.REQ2, then agitatorProgLocCommand.CNF2 to inletValveProgLocCommand.REQ2 and finally inletValveProgLocCommand.CNF2 to receptionValveProgLocCommand.REQ2.

At this point, the commands of removing the ownership of the instruments are sent. To complete this last step, the transition condition has to be developed.

As defined in the topic Filling Sequence Definition, the transition condition of this seventh step is to wait 2 seconds after the last command is sent like in the step Take the Ownership.

  1. Add a E_DELAY function block and name it TransStepTerminateDelay.

  2. Link receptionValveProgLocCommand.CNF2 to E_DELAY.START to start the timer when the command of removing the last instrument is done.

  3. Add the constant of 2 seconds to the timer.

  4. Link TransStepTerminateDelay.EO to Step7.TRANS_REQ to trigger the step when the delay is exceeded.

  5. Link Step7.STEP_LEAVE to TransStepTerminateDelay.STOP to stop the timer once the step is leaved.

  6. Finally, add the constants 8 to Step7.TransStepId and TRUE to Step7.Exceeded.

The development of the last step is now completed.

Cross reference the links between Step7 and agitatorProgLocCommand and between receptionValveProgLocCommand and TransStepTerminateDelay.
