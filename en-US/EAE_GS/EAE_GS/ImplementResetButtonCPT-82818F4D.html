Implementing Reset Button

Now, the TR1 layer has its Filling process completed with the sequence, the instruments, the simulation and the initialization function block. All these function blocks are correctly linked together, so it is fully completed.

However, one thing is missing. The process currently is only filling the tank and there is no way to empty it from the HMI.

To fulfill this need, a reset button will be develop to bring back the tank level to 0.
NOTE:

This solution is acceptable since it is only a simulation. For a real application a Discharge sequence would have been also required.

This choice of reset button has been taken since the Getting Started is already quite consequent and since a Discharge sequence would have been quite similar to the Filling one, hence it would have been repetitive.

To create this reset button, the same process as in the part Setting-Up the Instruments to open the valves will be used:

  1. Add a DigitalInput function block and name it ResetButton.

    This digital input will be used to receive the reset request done on the HMI from the user.

  2. Add a E_PERMIT function block and name it PermitReset.

    This function block will get the value from ResetButton and transfer the reset trigger only if the value is TRUE.

  3. Link ResetButton.CNF to PermitReset.EI, ResetButton.Pv to PermitReset.PERMIT and PermitReset.EO to TR1Sim.RESET_LEVEL.

  4. Link plcStart.FIRST_INIT to ResetButton.REQ_SUPPRESS_ALM and add the constant TRUE to SuppressAlarmSignal.

    This way, the alarm of the ResetButton is suppressed during the initialization. Hence, the digital input won’t raise any alarm when it is turned ON to reset the tank level.

The reset button is now fully done. It is possible to reset the tank level just by turning ON ResetButton on the HMI. Cross reference the links between plcStart and ResetButton then move to the next topic.
