Setting-Up Filling Sequence CAT

As described in the chapter SE.AppSequence Library Definition, the sequence is composed of several function blocks:

  • SeqHead

  • SeqSteps

  • SeqTerminate

  • And other function blocks related to the actions and the transition conditions.

All these function blocks could be added directly to the TR1 layer of the Whole_Milk_Tank application.

However, as seen in the topic Process Overview, the creamery is composed of several tanks (TR1, TR2, TS1, and so on). Each of them is composed of the same types and same amount of instruments with the same filling process order. These tanks are identical.

There are two options available to program all these tanks:

  • Option 1: Developing the sequence of each tank directly in their application layer.

  • Option 2: Creating a Composite Automation Type (CAT) and developing the sequence inside it.

To standardize the development of the tank sequences, the option 2 is chosen.

So, the odds to make a mistake with the option 2 are reduced since it is developed only once.

NOTE:

As the Getting Started is only focusing on the tank TR1 it could have been acceptable to choose the option 1. However, the Getting Started also tends to share the good practices for a full and realistic application. That is the reason why the second option is chosen in the development of the full creamery application. The CAT would have definitely being reused.
