Finalizing all Function Blocks

In this topic, as mentioned in Finalize the System Editor, three main actions need to be done to the function blocks:

  1. Removing all the unused function blocks from the TR1 layer.

  2. Linking the function blocks left to pass their initialization and their communication.

  3. Modifying some Offline Parametrization.

Removing the Unused Function Blocks

All the elements related to the dummy application done in the part Setting-Up the Instruments need to be removed as they don’t have usefulness anymore in the Filling sequence.

These elements are:

  • All the Transmission composites.

  • All the logic function blocks (AND and COMPARE).

  • All the links between function blocks done during this dummy application.

  • The FinalValidation digital output.

The only function blocks left are: the seven instrument function blocks, the Filling sequence CAT and the simulation composite.

Linking the Function Blocks

In this section, all the function blocks will be linked so that the communications between the Tank Filling sequence, the instruments and the simulation composite correctly be done.

Link each instrument to the Filling sequence function block as follows:

Link from to
TankFillingSequence.IAgitator TR1M01.ISc
TankFillingSequence.IPasteurizerValve TR1V03.ISc
TankFillingSequence.IReceptionValve TR1V02.ISc
TankFillingSequence.IInletValve TR1V01.ISc
TankFillingSequence.ITankLevel TR1LT01.IPv
NOTE:

Now the sequence will be able to dialog with each actuator. The IDevice adapter included inside each CAT directly communicate with the adapter in TankFillingSequence they are linked to.

TankFillingSequence has been linked to all the instruments. Some links related to the simulation of the tank level now has to be added.
NOTE:

The simulation of the tank level is the only simulation that requires links. The feedback simulations of instruments don’t require any physical link since Symlink communication is linking them. To know more about this other way of linking variables, check the topic Symbolic Links in the User Manual.

To simulate the tank level, link TR1LT01.Pv to TR1Sim.LevelVal and TR1LT01.CNF to TR1Sim.REQ.

Linked like this, the value of the analog input is transmitted to the simulation function block so the level of whole milk in the tank can be simulated.

In addition, TR1Sim is comparing the simulated value of the tank level to the min and max values. If the simulated value is lower or higher than these, then TR1Sim will simulate alarms.

NOTE:

TR1Sim is comparing the level to the high and low level. If the level becomes lower or higher than these limits then TR1Sim will do two actions:

  • Set TR1Sim.LevelHigh to TRUE and fires TR1Sim.LVL_HIGH if the value is higher than the max limit.

  • Activate TR1LSL01 or TR1LSH01 (depending on which limit is crossed) by using Symlink again.

To be able to simulate the alarms:

  1. Add the constants 10 to TR1Sim.MinLimit and 90 to TR1Sim.MaxLimit. These two values are the max and min limits.

  2. Link TR1Sim.LVL_HIGH to TankFillingSequence.REQ_LvlHigh and link TR1Sim.LevelHigh to TankFillingSequence.LevelHigh to transmit to the sequence the max limit information.

  3. Add the constant 90 to TankFillingSequence.LevelSp to inform the sequence about the level of whole milk needed in the tank.

NOTE:

The new links added may not be full lines but in dashes. It means that the linked function blocks are not in the same mapping state, one mapped and the other no.

All the elements related to the simulation have been added to the layer. Cross reference all the links that have been added in this part.

Finally, the last links to add between these function blocks are related to their initialization. TankFillingSequence and TR1Sim need to receive a trigger in their INIT input event to initialize some of their function blocks.

  1. Add a plcStart function block and name it plcStart.

    NOTE:

    This function block is connected to through Symlink to the DPAC_FULLINIT function block added to the resource RES0. So when INITIALIZE is triggered by the START function block in the resource, then this trigger will be transmitted to plcStart. Once the trigger is received, plcStart will fire FIRST_INIT and then initialize all the function blocks of the layer once linked.

  2. Link plcStart.FIRST_INIT to TankFillingSequence.INIT.

  3. Link TankFillingSequence.INITO to TR1Sim.INIT to create a chain of initialization.

  4. Finally, link TR1Sim.INITO to plcStart.ACK_FIRST to acknowledge that all the function blocks have been correctly initialized.

The initialization of all the function blocks of the application is now completed. Cross reference all the links that have been added in this part.

Offline Parametrization

Some parameters need to have their initial value changed from the default one.

To change them, go to the Offline Parametrization editor. There are in this editor all the parameters of each function block. Change the following parameters:

Parameter Default Value New Value Comment
TR1LT01.IThis.Pt1FTime 1 5.0

This parameter defines the duration of update of the analog input in seconds.

TR1LT01.highHigh.IThis.LimitSp 0 95.0

This parameter defines the limit of the high high alarm in percent.

TR1LT01.high.IThis.LimitSp 0 90.0

This parameter defines the limit of the high alarm in percent.

TR1LT01.low.IThis.LimitSp 0 10.0

This parameter defines the limit of the low alarm in percent.

With the change of these parameters, the Offline Parametrization is then completed and all the function blocks have been fully set.
