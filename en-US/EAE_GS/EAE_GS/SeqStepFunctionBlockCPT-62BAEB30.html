SeqStep Function Block

The SeqStep function block is the one used to handle individual step. Hence, there are in the CAT as many SeqStep as there are steps in the sequence.

Each SeqStep used in the sequence is linked in chain with the SeqHead and all the other SeqStep function blocks through its adapters ISeqChainIn and ISeqChainOut. The first SeqStep is connected to the SeqHead and to the second SeqStep. The second one is connected to the first and third, and so on.

With its ISeqChain adapters, a SeqStep is creating communication with the SeqHead and the other SeqStep function blocks.

The adapter links allow the SeqStep function blocks to transfer the following state to the SeqHead:

  • Started

  • Active

  • Left

  • Reset

  • Held

  • Resumed

The adapter links also allow the SeqHead to fire activation to the SeqStep. When activated then the SeqStep fires its ISeqActionChain adapter, triggering its actions.

The SeqStep remains active until the TRANS_REQ is triggered with the transition condition true. The SeqStep is then left and the next SeqStep is activated.
 
NOTE:

To know more about this function block, look at the SeqStep topic into SE.AppSequence library. To access it, search SeqStep in the Solution Explorer pad and press F1 to the result.
