SeqHead Function Block

The SeqHead function block is the leader of the sequence.

It leads the sequence and indicates which step is active through its ports:

  • START, RESET, HOLD, RESUME input events: Used to start, reset, hold and resume the sequence.

  • REQ_PERM event: Used to inform the SeqHead on the initial condition. When this event is triggered, variable PermStart set to TRUE, then the sequence is ready to start.

  • SeqStopped, SeqRunning, SeqHeld, SeqComplete variables data: These ports give the current state of the sequence. Only one of them is TRUE at one time.

  • ISeqChain adapter: Used to communicate with the SeqSteps function blocks of the sequence. It transfers them the state of the sequence and which step is active.

NOTE:

An adapter is a group of ports summarized in only one port. This notion will be developed in the topic Introduction to Adapters.

 
NOTE:

To know more about this function block, look at the SeqHead chapter into SE.AppSequence library. To access it, search SeqHead in the Solution Explorer pad and press F1 to the result.
