EtherCAT

Overview

EtherCAT designed for high-speed industrial automation, is a true real-time Ethernet protocol that pushes the limits of precision, speed, and reliability.

With EtherCAT library in simplex Soft dPAC Linux, you will be able to:

  • Configure Main device & EtherCAT network (ENI) , Class A & Class B

  • Autodetecing for the Subdevices network

  • Full PDOs selections by control in the data mapping and Autogenerated HWCAT

This functionnality requires an Application Addons license . Refer to the License chapter.

New Features

Here the main features of an EtherCAT Master.

High-Performance Real-Time Communication

  • Extremely fast cycle times (typically in the microsecond range)

  • Very low jitter for precise timing

  • Ideal for motor drivers & I/O

  • Deterministic frame handling due to “on-the-fly” processing in each slave

Multi-Vendor Interoperability

  • Supports a wide range of EtherCAT-compatible slave devices

  • Mix I/O modules, servo drives, safety modules, sensors, and gateways from different manufacturers

  • Integration through vendor-provided ESI (EtherCAT Slave Information) files

  • No vendor lock-in

Automatic or Manual Configuration

Automatic configuration:

  • Auto-detection of slave devices

  • Automatic PDO and process-image generation

  • Simplified mapping of inputs and outputs

Manual / expert configuration:

  • Custom PDO mapping

  • SyncManager adjustments

  • Manual topology definition

  • Distributed I/O stations at long distances

Distributed Clock Synchronization

  •  microsecond-level synchronization across slaves

  • deterministic and synchronized sampling of I/O

Advanced Protocol Support

CoE (CANopen over EtherCAT) – parameter access, SDO/PDO

Comprehensive Diagnostics

  • Slave state monitoring (INIT, PRE-OP, SAFE-OP, OP)

  • Error detection: link loss, frame errors, watchdogs

  • Real-time performance monitoring

  • Topology detection and validation

Unmatched Speed and Precision

  • Cycle times in µs

  • Synchronization accuracy µs

  • High data throughput with minimal overhead

Known Limitations

Devices Compatibility
