New Features

New Edge Controllers compatibles: AP310 and M590 dPAC

Two edge controllers are now compatible and pre-installed with HA Soft dPAC:

  • AP310

  • M590 dPAC

High Availability Disruptance Avoidance (HADA)

Self-healing is now within reach!

You are now able to maintain at any time your Edge Controller High Availability , by self-heal from a controller failure.

Thanks to the new System Management app of the EcoStruxure Automation Expert Platform , combined with the power of an Intel Tiber orchestrator, it is now possible to define  in advance for each of your High Availability Soft dPAC peers where you would like to automatically relocate them in case of failure on their initial host. 

Once in production and a failure occuring, the maintenance engineer is notified through this new portal and is able to replace the defective device without compromising the required high availability.

For more information about how to obtain this functionality, refer to Advanced System Management .

Time Synchronisation Robustness Improvements

High Availability partners time synchronization robustness has been improved in isolated mode. In Isolated Mode, if the primary node fails and the secondary node takes over, your system remains resilient. Even if the nodes fall out of sync, they can now be resynchronised seamlessly without interrupting the active node or requiring a reboot.

40K OPC-UA Tags

In your EcoStruxure Automation Expert application, the OPC UA Server supports a default maximum of 20,000 OPC UA variables. If your application requires more variables, this limit can be extended to 40,000 variables by using a specific version of the SE.DPAC library, which is available upon request from Schneider Electric Support.

Known Limitations

NOTE: To learn more about High Availability Soft dPAC limitations introduced in a previous release version and still active today , please refer to the following link :HA Soft dPAC limitations. For new limitations, please see the following sections.

HADA Controller Authentication

In the physical topology, when working with HADA controllers, the user must authenticate these controllers to enable proper monitoring. This requirement applies exclusively to HADA controllers.

Syslog Server Configuration

To have access to the Syslog server functionality, you must configure Syslog server with the following port:

  • Unsecure port: 601

  • Secure port: 6514

This limitation will be fixed in the next EcoStruxure Automation Expert release.

Docker Compatibility

HA Soft dPAC supports only Docker versions prior to 28. Installation will fail if an incompatible version is detected. Version 27.x is required, with 27.5 recommended.
